# 09 · 跑酷拾字（Hanzi Runner）PRD

> 一句话：横版无尽跑酷。地图持续右滑，路径上有正确字 / 错误字 / 障碍；按目标句序拾字。

---

## 1. 目标与价值

| 项 | 内容 |
|---|---|
| 目标用户 | S5-S12 短句 / 中句 / 长句 |
| 解决问题 | 词序 (Q8)、选词填空 (Q7)、阅读流畅性 |
| 成功指标 | 句完成率 ≥ 75%；单局 90s 沉浸感强 |

---

## 2. 核心玩法

### 2.1 场地

- 横版 16:9，2D 跑酷。
- 角色（小忍者 emoji 或几何小人）固定在屏幕左 1/3，世界右滑过来。
- 3 条平行赛道（上 / 中 / 下），可上下切换。

### 2.2 题面与目标

- 屏幕顶部固定显示当前目标句（带空），如：
  - "我 ___ 喝 ___ 杯 ___ 咖啡"
  - 玩家需按顺序拾起 [要 / 一 / 热] 三字。
- 路径上的字以"漂浮气泡"形式从右进入，分布于 3 条赛道。
- 拾字方式：跳跃 / 滑铲 / 切换赛道靠近气泡 → 自动收集。

### 2.3 错字与障碍

| 物体 | 处理 |
|---|---|
| 正确字（按当前应填序号） | 收集 → 句中空位填上 + 1 节连击 |
| 错字 | 收集 → 扣容错 + combo 清零 |
| 障碍（围栏 / 大石） | 撞到 = 扣容错 + 速度短暂归零 |
| 加速带 | 踩到 = 速度 +50% × 3s（更难拾字，得分翻倍） |
| 金币 | 收集纯加分 |

### 2.4 一回合（Round）= 完成 5-10 个句子或体力归零

填完 1 句 → 屏幕短暂庆祝 + 顶部换下一句 → 不停跑。

---

## 3. 交互

- **手机**：上滑跳；下滑滑铲；左 / 右滑切换赛道。
- **桌面**：W 跳；S 滑铲；A/D 切赛道。

## 4. UI

```
┌──────────────────────────────────────┐
│ 我 [要] 喝 [   ] 杯 [   ] 咖啡   2/8 │
│ ❤❤❤  SCORE 2,140  COMBO ×7         │
├──────────────────────────────────────┤
│ ─────── 上 ──────  要               │
│ ─────── 中 ── 🥷 ── 一 ── 障 ── 热 │
│ ─────── 下 ──────────  错 ──── 金 │
└──────────────────────────────────────┘
```

视觉：
- 角色 = 几何 + emoji + 跑步循环（4 帧 SVG 形变）。
- 字气泡 = 圆框 + 描边色：正确按序 = 金；错 = 红描边；待定 = 灰。
  > 注意：正确气泡颜色提示**仅 easy 难度**给，normal 之上不区分颜色。
- 背景 = 程序生成色块视差 3 层 + 简单几何剪影（山 / 楼 / 树）。

## 5. 难度

| 项 | easy | normal | hard |
|---|:--:|:--:|:--:|
| 基础滚动速度 | ×0.7 | ×1.0 | ×1.5（持续递增）|
| 干扰错字密度 | 20% | 35% | 50% |
| 障碍密度 | 1/8s | 1/5s | 1/3s |
| 容错 (❤) | 5 | 3 | 1 |
| 颜色提示正确字 | ✅ | ❌ | ❌ |

---

## 6. 评分

- 拾对 1 字：base × 难度
- 完成 1 句：+200
- 全句无错：×1.5 ratio
- 跑酷距离每 100m：+50

---

## 7. 验收

- [ ] AC-01 句子按 (track, stage) 取，长度匹配 stage
- [ ] AC-02 路径生成器保证可达（每个正确字至少 1 条无障碍可达路径）
- [ ] AC-03 中端机 60fps；视差 3 层
- [ ] AC-04 死亡后断点续句（仍当前句继续）

## 8. 风险

| 风险 | 应对 |
|---|---|
| 单调 | 每 30s 切换背景主题 + BGM |
| 难拾字 | 角色拾取范围 1.2x，自动磁吸正确字 0.5x 半径 |
| 长句过长 | 句子 ≥ 12 字时分段填空，每段 4-6 字 |
