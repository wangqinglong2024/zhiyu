# 发现中国 · 03 · 类目与持续更新规划

> 发现中国以 12 个主题类目作为长期内容池。首期每个主题先产出 12 篇，后续按搜索热点、节日热点、站内搜索缺口无限增量。

---

## 1. 总体目标

- **主题类目**：12 个固定主题。
- **首期冷启动**：12 类目 × 12 篇/类 = 144 篇。
- **篇幅规则**：每篇最多 120 句；首期正式文章目标 120 句/篇。
- **长期规则**：不设置 360 篇或任何固定总量；后续持续追加新文章。
- **文件规则**：一篇文章一个 JSON，manifest 只做清单。

### 1.1 首期产能基准

| 项 | 数值 |
|----|------|
| 单主题首期文章 | 12 篇 |
| 单篇句子 | 120 句 |
| 单主题首期句子 | 1440 句 |
| 全主题首期句子 | 17280 句 |
| 长期总量 | 不设上限 |

## 2. 通用风格规则

- **定位**：面向东南亚 / 海外受众的中国文化科普，同时服务中文学习者。
- **立场**：中立、事实导向；避免政治、宗教、民族、领土争议，尤其不触碰越南、泰国、印尼等东南亚首发市场红线。
- **结构**：必须先写完整中文源文，再拆成句子；每篇是一篇能连续读完的纪录片式专题解说，不是强行情节故事、事实卡片、问答卡、提纲或模板句集合。
- **标题**：至少 50% 使用问句式，覆盖 `是什么`、`为什么`、`怎么做`、`有什么区别` 等搜索意图。
- **首句**：每篇第 1 句必须是 BLUF，一句话给出答案。
- **禁忌**：不用第一人称；不用 emoji / URL / @ / #；不用繁体；不用最高级包装不确定事实；不写成人性内容、色情暗示、暴力猎奇或现实政治类比；正文里不出现“AI摘要”“SEO”“GEO”“关键词”“搜索热点”“这篇文章适合”“主线”“线索”“这一层”“这个细节”“读法”等生成器元话术。

### 2.1 纪录片式介绍优先生成流程

1. **先写中文源文**：围绕标题写出完整的 120 句中文纪录片式介绍文，必须有清晰对象、背景、形成原因、典型特征、脉络推进、文化影响和自然收束。
2. **再拆句入库**：`source_story_zh` 是权威中文源文，`sentences[*].content_zh` 必须逐句等于它。
3. **再生成拼音**：拼音来自中文句子，不允许沿用旧数据或手写复用。
4. **最后多语言适配**：英文、越南语、泰语、印尼语逐句适配中文语义，不允许用“学习提示”“搜索提示”“结构提示”替代翻译。
5. **SEO/GEO 后置抽取**：SEO/GEO 只能服务元数据和可引用性，不得出现在正文。

### 2.2 通用合规规则

| 类型 | 规则 |
|------|------|
| 政治/民族 | 不讲现实政治立场、民族矛盾、国家冲突、领土争议，不把历史事件映射到当代争议 |
| 宗教 | 可写节日、建筑、艺术中的文化背景，不做宗教优劣比较或争辩 |
| 成人内容 | 不生成性内容、色情暗示、性交易、露骨身体描写或擦边标题 |
| 东南亚市场 | 越南、泰国、印尼优先合规；翻译也必须保持中性、学习向、低刺激 |
| 热点选择 | 搜索热点若涉及敏感议题，首期跳过，改选饮食、节日、景点、艺术、语言学习等低风险热点 |

### 2.3 120 句推荐结构

| 句段 | 范围 | 内容 |
|------|------|------|
| BLUF | 1 | 直接答案，≤ 80 中文字 |
| 开场 | 2–12 | 用对象、时间、地域、器物或味觉画面进入主题 |
| 背景 | 13–30 | 交代名称来源、形成条件或学习入口，避免空泛提纲 |
| 展开 | 31–70 | 按时间、地域、工艺、风味或概念层次介绍核心内容 |
| 辨析 | 71–92 | 讲清常见误解、差异、变化或代表性例子 |
| 影响 | 93–112 | 写文化影响、保留下来的生活痕迹和学习价值 |
| 收束 | 113–120 | 回到标题问题，用概括性旁白自然收尾 |

## 4. SEO / GEO 嵌入要点

### 4.1 SEO

- 标题含主词 + 搜索意图，例如 `秦始皇统一中国做了哪些事？`。
- 每篇 `seo.primary_keywords` 至少覆盖 zh/en/vi/th/id。
- `search_intents` 至少包含 2 个：`what_is`、`why`、`how_to`、`timeline`、`comparison`、`travel_learning`、`beginner_learning`。
- 句子中自然覆盖中尾关键词，不堆砌。

### 4.2 GEO

- 第 1 句即答案，可被 AI 直接引用。
- `geo.entities` 明确人物、朝代、地点、作品、节日、平台等实体。
- 关键实体首次出现尽量带中英对照。
- 每 10–15 句出现一个可独立引用的事实句。
- GEO 信息内嵌在每篇 `*.article.json`，不另建脱离正文的 GEO 文件。
- 可引用事实句必须保持中立和低风险，不使用煽动性、成人化或争议化表达。
