# 发现中国 · 01 · 生成流程与目录约定

> 本文件给生成 AI 和内容脚本使用。发现中国不再使用“多篇文章合并 JSONL”的批文件，改为一篇文章一个 JSON。

---

## 1. 单篇文章工作流

```
准备
  ├─ 选定主题类目（01..12）
  ├─ 选定文章题目与搜索意图
  ├─ 维护 SEO 关键词矩阵与 GEO 实体清单
  ├─ 排除东南亚敏感议题与成人性内容
  └─ 决定本篇句数（首期目标 120，硬上限 120）

这里非常重点，必须遵守！文章内容要像纪录片旁白或专题解说一样自然介绍中国文化：事实清楚、画面感强、节奏顺畅、可连续朗读；内容质量优先，然后才是 SEO 和 GEO。
生成
  ├─ 写 article 标题（zh + 拼音 + en/vi/th/id）
  ├─ 写 content 中文文章，必须是连贯的纪录片式介绍文。≤ 120。确保内容合规。
  ├─ 拆分 content 中文句子，将文章拆分为多个句子 sentences 数组
  ├─ 进行多语言及拼音适配（拼音 + en/vi/th/id）
  ├─ 从中文中找到 seo / geo 元数据
  └─ 输出单个 *.article.json

自检
  ├─ title_i18n 每项 ≤ 40，title_pinyin ≤ 200
  ├─ 每句 content_* ≤ 400，pinyin ≤ 600
  ├─ 5 语言全齐
  ├─ 第 1 句是 BLUF 答案句
  ├─ 不含政治、宗教、民族、领土争议或成人性内容
  ├─ seq_in_article 连续无跳号
  └─ 每篇句数 ≤ 120
```

## 2. 目录与文件命名

```
content/01-china/data/articles/<类目目录>/<phase>/
  <2位序号>-<英文slug>.article.json
  manifest.json
```

命名示例：

```
content/01-china/data/articles/01-history/phase1/
  01-dynasty-order-rhyme.article.json
  02-qin-shi-huang-unification.article.json
  03-zhang-qian-western-regions.article.json
  manifest.json
```

规则：

- `phase1` 是首期冷启动目录；后续可用 `phase2`、`2026-q3`、`trend-202606` 等目录继续追加。
- 单个 `*.article.json` 只允许包含一篇文章。
- `manifest.json` 只记录该目录下文件清单、统计和生成信息，不承载正文。
- 不再生成 `*.articles.jsonl` / `*.sentences.jsonl`。

## 3. 持续更新策略

| 场景 | 处理方式 |
|------|----------|
| 新增文章 | 在对应类目新建一个 `*.article.json`，文件名序号继续递增 |
| 热点补充 | 新建 trend 目录或追加到当前 phase，不修改旧文章 code |
| 翻译修正 | 优先走 web-admin 编辑；需要重导时只重导单篇 JSON |
| 旧文过时 | 新写一篇更新版文章，保留旧文或运营下架；不要把历史数据硬改成“最新事实” |
| 大规模替换 | 使用导入脚本 `--replace`，仅在本地或明确授权环境执行 |

发现中国的总文章数不设上限；“首期 12 类目 × 12 篇”只是冷启动最小可用集，不是长期目标。

## 4. 生成阶段不要做的事

- **不要用脚本生成任何内容字段**。中文正文、拼音（含多音字）、4 语言翻译、SEO/GEO 文案，必须由生成 AI 在上下文中逐条直接撰写。脚本仅允许导入、删除、校验、生成 manifest。详见 [02-字段与JSONL格式.md §3.0 AI 直出铁律](./02-字段与JSONL格式.md#30-ai-直出铁律最高优先级所有字段适用)。
- 不要生成 DB `id`、`code`、`article_id`、`created_at`、`updated_at`。
- 不要生成音频 URL 或音频状态；TTS 仍由用户首次播放触发。
- 不要把文章状态写成 `published`；发布由导入脚本参数或运营后台决定。
- 不要把多篇文章合到同一个 JSON。
- 不要删除 SEO/GEO 元数据。
- 不要把 GEO 拆成脱离文章正文的单独数据文件；`geo.bluf/entities/citation_notes` 必须随单篇 JSON 一起走。
- 不要生成政治动员、民族矛盾、宗教争辩、领土争议、成人性内容、色情暗示或不适合越南/泰国/印尼首发市场的表达。
- 不要设置固定总量口径，例如“只做 360 篇”。

## 5. 推荐派单格式

```yaml
task:
  domain: china
  category_code: "01"
  category_dir: "01-history"
  phase: phase1
  article_count: 12
  sentence_target: 120
  sentence_hard_max: 120
  output_dir: content/01-china/data/articles/01-history/phase1/
  update_mode: append_only_infinite
  seo_geo_required: true
```
