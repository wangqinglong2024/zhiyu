# 课程 · 06 · 导入与校验

> 给课程域"导入运维"看的操作手册。课程导入比 china 复杂，分**骨架批次**与**内容批次**两类。

> ⚠️ **AI 直出铁律提醒**：导入脚本只做导入、校验、回滚，不允许生成任何内容字段。所有可读文本（标题、拼音、翻译、解释、例句、题面）必须由 AI 在上下文中直接撰写并对每一个多音字、每一句翻译负责。详见 [01-生成流程与目录约定.md §0 AI 直出铁律](./01-生成流程与目录约定.md#0-ai-直出铁律最高优先级所有内容字段适用)。

---

## 1. 导入脚本（计划路径）

```
system/scripts/db/import_course_skeleton.ts   ← 跑一次：tracks/stages/chapters/lessons
system/scripts/db/import_course_content.ts    ← 每节一次：kp + lesson_kp + question
system/scripts/db/import_course_exam.ts       ← 每卷一次：exam blueprint
```

## 2. 骨架导入

### 2.1 用法

```bash
pnpm tsx system/scripts/db/import_course_skeleton.ts \
  --manifest content/02-course/data/_skeleton/skeleton.manifest.json \
  --operator 00000000-0000-0000-0000-000000000001
```

### 2.2 流程

```ts
// 1. 解析 manifest，校验 sha256
// 2. 按依赖顺序入库：
//    a. tracks  → upsert by code
//    b. stages  → upsert by (track_code, stage_no)
//    c. chapters→ upsert by (stage_id, chapter_no)；按 stage_track + stage_no 反查 stage_id
//    d. lessons → upsert by code
// 3. 全部 is_published=false
// 4. zod 校验全部 5 语言键 / code 正则
```

### 2.3 zod schema

```ts
const TRACK = z.object({
  code: z.enum(['share','ec','fc','hsk','dl']),
  name_zh: z.string().min(1).max(20),
  name_i18n: I18N5,
  sort_order: z.number().int().min(0),
  is_enabled: z.boolean(),
}).strict();

const STAGE = z.object({
  track_code: z.enum(['share','ec','fc','hsk','dl']),
  stage_no: z.number().int().min(0).max(6),
  title_zh: z.string().min(1).max(40),
  title_i18n: I18N5,
  desc_i18n: z.record(z.string().max(200)).optional().nullable(),
  hsk_mapping: z.string().regex(/^HSK[1-6]$/).nullable().optional(),
  unlock_rule: z.object({
    prev_stage_pass: z.boolean(),
    require_shared_stage_done: z.boolean().optional(),
    min_score: z.number().min(0).max(100).optional(),
  }),
  vocab_increment: z.number().int().min(0).max(5000).nullable().optional(),
  sort_order: z.number().int().min(0),
  is_published: z.literal(false),
}).strict().refine(
  v => !(v.stage_no === 0) || v.track_code === 'share',
  'stage_no=0 only allowed when track_code=share'
);

const CHAPTER = z.object({
  stage_track: z.enum(['share','ec','fc','hsk','dl']),
  stage_no: z.number().int().min(0).max(6),
  chapter_no: z.number().int().min(1).max(20),
  title_zh: z.string().min(1).max(40),
  title_i18n: I18N5,
  cover_url: z.string().nullable().optional(),
  sort_order: z.number().int().min(0),
  is_published: z.literal(false),
}).strict();

const LESSON = z.object({
  chapter_track: z.enum(['share','ec','fc','hsk','dl']),
  chapter_stage_no: z.number().int().min(0).max(6),
  chapter_no: z.number().int().min(1).max(20),
  lesson_no: z.number().int().min(1).max(30),
  code: z.string().regex(/^(share|ec|fc|hsk|dl)-\d{1,2}-\d{1,2}-\d{1,2}$/),
  title_zh: z.string().min(1).max(40),
  title_i18n: I18N5,
  goal_i18n: z.record(z.string().max(80)).nullable().optional(),
  est_minutes: z.number().int().min(5).max(30),
  sort_order: z.number().int().min(0),
  has_quiz: z.boolean(),
  is_published: z.literal(false),
}).strict();
```

`I18N5 = z.object({zh,en,vi,th,id}).strict()`，每值 `z.string().min(1)`。

## 3. 内容导入

### 3.1 用法

```bash
pnpm tsx system/scripts/db/import_course_content.ts \
  --manifest content/02-course/data/ec/stage2/chapter03/lesson01/batch_20260520_001.manifest.json \
  --operator 00000000-0000-0000-0000-000000000001
```

### 3.2 内部流程（伪代码）

```ts
const m = readJSON(args.manifest);
const lesson = await sql`select id from zhiyu.course_lessons where code=${m.scope.lesson_code}`;
if (!lesson) throw 'COURSE_LESSON_NOT_FOUND';

// Step 1: 写入 import_batch
const batch = await sql`
  insert into zhiyu.course_import_batches (batch_code, batch_type, status, ...)
  values (${m.batch_code}, 'kp+question', 'validating', ...)
  returning id
`;

// Step 2: KP
const kpRows = readJsonl('batch_xxx.kp.jsonl');
const localToKp: Record<string, {id:string, kp_code:string}> = {};
for (const k of kpRows) {
  // jsonb 用 raw client（drizzle 双重编码 bug，见 user memory）
  const row = await rawClient`
    insert into zhiyu.course_knowledge_points
      (kp_code, kp_type, primary_track, title_zh, pinyin, difficulty, hsk_level,
       content, translations, tags, source_batch_id)
    values (
      zhiyu.fn_gen_kp_code(${k.primary_track}, ${k.kp_type}),
      ${k.kp_type}, ${k.primary_track}, ${k.title_zh}, ${k.pinyin ?? null},
      ${k.difficulty}, ${k.hsk_level ?? null},
      ${rawClient.json(k.content as Record<string,unknown> as never)},
      ${k.translations ? rawClient.json(k.translations as never) : null},
      ${k.tags},
      ${batch.id}
    )
    on conflict do nothing
    returning id, kp_code
  `;
  localToKp[k.__kp_local_id] = row;
}

// Step 3: lesson_kp
const lkRows = readJsonl('batch_xxx.lesson_kp.jsonl');
for (const lk of lkRows) {
  const kpRef = lk.__kp_local_id ? localToKp[lk.__kp_local_id]
              : await sql`select id, kp_code from zhiyu.course_knowledge_points where kp_code=${lk.kp_code}`;
  if (!kpRef) { errors.push({...}); continue; }
  await sql`
    insert into zhiyu.course_lesson_kp (lesson_id, kp_id, position, is_new_in_lesson)
    values (${lesson.id}, ${kpRef.id}, ${lk.position}, ${lk.is_new_in_lesson})
    on conflict (lesson_id, kp_id) do update set position=excluded.position
  `;
}

// Step 4: questions
const qRows = readJsonl('batch_xxx.questions.jsonl');
for (const q of qRows) {
  const kpRef = q.__kp_local_id ? localToKp[q.__kp_local_id]
              : await sql`select id from zhiyu.course_knowledge_points where kp_code=${q.kp_code}`;
  if (!kpRef) { errors.push({...}); continue; }
  await rawClient`
    insert into zhiyu.course_questions
      (q_code, kp_id, q_type, exam_scope, difficulty, payload, payload_hash,
       audio_url, image_url, source_batch_id)
    values (
      zhiyu.fn_gen_q_code(${(await sql`select primary_track from zhiyu.course_knowledge_points where id=${kpRef.id}`).primary_track}),
      ${kpRef.id}, ${q.q_type}, ${q.exam_scope}, ${q.difficulty},
      ${rawClient.json(q.payload as Record<string,unknown> as never)},
      encode(sha256(${rawClient.json(q.payload as never)}::text::bytea), 'hex'),
      ${q.audio_url ?? null}, ${q.image_url ?? null},
      ${batch.id}
    )
    on conflict (kp_id, q_type, payload_hash) where deleted_at is null do nothing
  `;
}

// Step 5: 更新 batch 状态
await sql`update zhiyu.course_import_batches set status='imported', stats=${rawClient.json(stats)} where id=${batch.id}`;
```

> ⚠️ 写 jsonb 时务必走 `rawClient.json(...)`，否则 drizzle-orm 0.30 会双重编码。

## 4. 试卷导入

```bash
pnpm tsx system/scripts/db/import_course_exam.ts \
  --file content/02-course/data/_exams/lesson_quiz/ec-2-3-1.exam.json \
  --operator <uuid>
```

```ts
const e = readJSON(args.file);
// 反查 scope_ref_id
let scopeRefId: string;
if (e.scope_type === 'lesson_quiz') {
  scopeRefId = (await sql`select id from zhiyu.course_lessons where code=${e.scope_ref.lesson_code}`).id;
} else if (e.scope_type === 'chapter_test') {
  // chapter_code = "ec-2-3" → 反查
  const [tk, sn, cn] = e.scope_ref.chapter_code.split('-');
  scopeRefId = (await sql`
    select c.id from zhiyu.course_chapters c
    join zhiyu.course_stages s on s.id=c.stage_id
    where s.track_code=${tk} and s.stage_no=${+sn} and c.chapter_no=${+cn}
  `).id;
} else if (e.scope_type === 'stage_exam' || e.scope_type === 'hsk_mock') {
  const [tk, sn] = e.scope_ref.stage_code.split('-');
  scopeRefId = (await sql`
    select id from zhiyu.course_stages where track_code=${tk} and stage_no=${+sn}
  `).id;
}

// blueprint 内部 kp_filter 也把 lesson_code/chapter_code/stage_code 转 uuid
const blueprint = transformBlueprint(e.blueprint);

await rawClient`
  insert into zhiyu.course_exams
    (code, scope_type, scope_ref_id, title_zh, title_i18n,
     pass_score, total_score, time_limit_minutes, blueprint, created_by)
  values (
    zhiyu.fn_gen_exam_code(${scopeAbbrev[e.scope_type]}),
    ${e.scope_type}, ${scopeRefId}, ${e.title_zh},
    ${e.title_i18n ? rawClient.json(e.title_i18n as never) : null},
    ${e.pass_score}, ${e.total_score}, ${e.time_limit_minutes ?? null},
    ${rawClient.json(blueprint as never)},
    ${args.operator}
  )
  on conflict (scope_type, scope_ref_id) where deleted_at is null and scope_type in ('lesson_quiz','chapter_test','stage_exam')
  do update set blueprint=excluded.blueprint, title_zh=excluded.title_zh, updated_at=now()
`;
```

## 5. 校验 zod schema（内容批次）

### 5.1 KP

```ts
const KP = z.object({
  __kp_local_id: z.string().min(1).max(32),
  kp_type: z.enum(['pinyin','hanzi','word','phrase','grammar','sentence','dialog']),
  primary_track: z.enum(['share','ec','fc','hsk','dl']),
  title_zh: z.string().min(1).max(60),
  pinyin: z.string().max(200).nullable().optional(),
  difficulty: z.number().int().min(1).max(6),
  hsk_level: z.number().int().min(1).max(6).nullable().optional(),
  content: z.record(z.unknown()),                       // 7 类细分另做
  translations: I18N5_OPTIONAL.nullable().optional(),
  tags: z.array(z.string().max(20)),
}).strict();
```

按 kp_type 分支再校验 content 子 schema（pinyin/hanzi/word/phrase/grammar/sentence/dialog 7 个 strict object）。

### 5.2 Question

```ts
const QUESTION = z.object({
  __kp_local_id: z.string().min(1).max(32).optional(),
  kp_code: z.string().regex(/^kp_(share|ec|fc|hsk|dl)_(p|h|w|ph|g|s|d)_\d{6}$/).optional(),
  q_type: z.enum([
    'mcq_meaning','mcq_zh','listen_pick','listen_pinyin','tone_pick',
    'match_pairs','sort_words','fill_blank_choice','type_pinyin',
    'type_zh_ime','image_pick','dialog_cloze'
  ]),
  exam_scope: z.array(z.enum(['practice','lesson_quiz','chapter_test','stage_exam','hsk_mock'])).min(1),
  difficulty: z.number().int().min(1).max(6),
  audio_url: z.string().nullable().optional(),
  image_url: z.string().nullable().optional(),
  payload: z.record(z.unknown()),
}).strict()
.refine(d => !!(d.__kp_local_id || d.kp_code), 'either __kp_local_id or kp_code')
.refine(d => {
  const ex = d.exam_scope;
  const exclusives = ex.filter(s => s === 'chapter_test' || s === 'stage_exam' || s === 'hsk_mock');
  return exclusives.length <= 1;
}, 'chapter_test/stage_exam/hsk_mock are mutually exclusive');
```

## 6. errors.jsonl 与幂等

| 失败 | 处理 |
|------|------|
| schema 校验失败 | 写 errors.jsonl，跳过该行，其它继续 |
| `__kp_local_id` 在 kp.jsonl 中不存在 | 跳过对应 lesson_kp / question |
| KP 已存在（title_zh + primary_track + kp_type 重复） | 跳过新建，但仍记录映射，让后续 lesson_kp/question 用旧 id |
| Question payload_hash 已存在 | 跳过 |
| lesson_code 不存在 | 整批失败：骨架未导入 |

幂等：同 batch_code 重跑 → 已 imported 直接 skip。

## 7. 发布流程

1. 默认 KP/Question/Exam 全部 `is_published=false`
2. 节内容齐全后，运营在 web-admin **节详情页**点"发布节"，调用：

```sql
update zhiyu.course_lessons set is_published=true, published_at=now(), published_by=$op
 where code=$lesson_code and is_published=false;
update zhiyu.course_knowledge_points set is_published=true, published_at=now(), published_by=$op
 where id in (select kp_id from zhiyu.course_lesson_kp where lesson_id=$lesson_id);
update zhiyu.course_questions set is_published=true, published_at=now(), published_by=$op
 where kp_id in (select kp_id from zhiyu.course_lesson_kp where lesson_id=$lesson_id)
   and exam_scope && array['practice','lesson_quiz']::text[];
```

3. 章/阶段考的 KP 题目可在章/阶段发布时补发布
4. `auto_publish: true` 在 manifest 里**生产环境严禁**

## 8. 故障速查

| 现象 | 排查 |
|------|------|
| `COURSE_LESSON_NOT_FOUND` | 骨架未导入；先跑 import_course_skeleton |
| `KP_TYPE_NOT_ALLOWED_IN_TRACK` | 主题层用了 pinyin / Stage 0 用了 word；改 kp_type 或转用复用 |
| `EXAM_SCOPE_EXCLUSIVE_VIOLATED` | exam_scope 同时含 chapter_test / stage_exam / hsk_mock |
| `EXAM_INSUFFICIENT_QUESTIONS` | 试卷 blueprint.count > 题库实际数；回去补题 |
| jsonb 写入但 select 出来是字符串 | drizzle 双重编码；改用 rawClient.json() |
| `payload_hash duplicate` | 同 KP 同题型同 payload 已存在；正常去重，不报错 |

## 9. 监控

| 指标 | 查询 |
|------|------|
| 节内容完整度 | `select l.code, count(distinct lk.kp_id) kp_cnt, count(distinct q.id) q_cnt from course_lessons l left join course_lesson_kp lk on lk.lesson_id=l.id left join course_questions q on q.kp_id=lk.kp_id group by l.code having count(distinct lk.kp_id) < 12` |
| 题型分布 | `select q_type, count(*) from course_questions where is_published=true group by q_type` |
| exam_scope 分布 | `select unnest(exam_scope) s, count(*) from course_questions group by s` |
| 试卷题源充足度 | 见 [05 §6](./05-考试与组卷规则.md) |
