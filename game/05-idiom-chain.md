## 二、核心玩法

### 2.1 基础规则

1. 系统给出一个起始成语（如"一马当先"）
2. 玩家需用**最后一个字**（或同音字）接出新成语
3. 输入方式：**打字输入** 或 从**提示卡牌**中选择
4. 每回合倒计时 **15 秒**
5. 接出的成语不能重复
6. 接不出 → 可使用提示（从 3 个候选中选 1 个）→ 提示用完则失败
7. 每局 3 回合制（Bo3），或无尽模式

### 2.2 接龙规则细节

| 规则 | 说明 |
|------|------|
| 尾字接头 | "一马当**先**" → "**先**发制人" |
| 同音接龙 | "一马当**先**" → "**鲜**为人知"（xiān = xiān） |
| 禁止重复 | 同一局内成语不可重复使用 |
| 提示卡 | 每局 3 张免费提示卡，用完需等 |
| 判定 | 服务端验证成语合法性（存在于词典中） |

### 2.3 成语展示

每次接龙成功后，短暂显示该成语的：
- 📖 释义
- 📜 出处/故事（一句话版）
- 💡 例句
- 这既是游戏也是学习

## 三、游戏模式

### 3.1 单人·无尽

| 维度 | 说明 |
|------|------|
| 规则 | vs AI 无限接龙 |
| AI 难度 | 可选：简单/中等/困难/大师 |
| 目标 | 最长接龙链（个人纪录） |
| 结束 | 自己接不出（包括用完提示后） |

### 3.2 单人·限时

| 维度 | 说明 |
|------|------|
| 规则 | 60 秒内接出最多成语 |
| 每接一个 | 10 秒 → 自动起始下一个 |
| 目标 | 最多接龙数 |

### 3.3 1v1 PK

| 维度 | 说明 |
|------|------|
| 规则 | 真人轮流接龙，15 秒/回合 |
| 胜负 | 接不出者输（Bo3 赛制） |
| 星数 | 胜 +1🌟，败 -1🌟 |
| 特殊 | 用生僻成语接龙额外加分（鼓励学新成语） |

### 3.4 多人车轮战

| 维度 | 说明 |
|------|------|
| 人数 | 3-6 人 |
| 规则 | 轮流接龙，接不出淘汰 |
| 胜负 | 最后留下的人获胜 |
| 节奏 | 每人 10 秒 |

## 四、视觉与交互设计

### 4.1 画面风格

| 元素 | 设计 |
|------|------|
| 场景 | 中国象棋风格对弈桌 + 古风背景 |
| 成语卡牌 | 竹简/卷轴造型，毛笔字体 |

## 五、上瘾机制

### 5.1 成语图鉴

- 收集所有在游戏中用过的成语 → 填充"成语图鉴"
- 图鉴分类：动物、数字、历史、战争、品德、自然…
- 集齐某个分类 → 解锁对应成就和皮肤

### 5.2 段位系统

- 专属段位名：成语小白 → 成语新秀 → 成语达人 → 成语大师 → 成语之王 → 成语圣手

### 5.3 成就

| 成就 | 条件 |
|------|------|
| 初学接龙 | 首次成功接 5 个 |
| 百词斩 | 图鉴收集 100 个成语 |
| 冷门专家 | 使用 10 个冷门成语 |
| 无提示通关 | 单局不使用提示接 10+ |
| 车轮战王 | 多人战中最后存活 |

### 5.4 每日挑战

- **主题接龙**：只接含"动物"的成语
- **速度挑战**：每个只给 5 秒
- **长链挑战**：尽可能接最长链

## 七、技术要点

### 7.1 场景结构

```
IdiomChainGame
  ├── BootScene
  ├── MenuScene
  ├── GameScene
  │     ├── ChainDisplay    # 接龙链展示
  │     ├── InputHandler    # 输入框/卡牌选择
  │     ├── Timer           # 倒计时管理
  │     ├── HintSystem      # 提示卡系统
  │     ├── IdiomValidator  # 成语验证（服务端）
  │     └── AlmanacTracker  # 图鉴收集追踪
  ├── ResultScene
  └── MultiScene           # 多人车轮战
```

### 7.2 关键技术

| 技术点 | 方案 |
|--------|------|
| 成语库 | 服务端维护完整成语词典（5000+），按难度分级 |
| 接龙验证 | 服务端验证：成语存在 + 首字匹配尾字 + 未重复 |
| 提示生成 | 服务端从合法接龙中随机选 3 个 |
| AI 对手 | 简单模式用常见成语，大师模式用冷门成语 |
| 实时对战 | WebSocket 轮流发送答案 |
