## 二、核心玩法

### 2.1 基础规则

1. 屏幕底部不断飞出汉字卡片（每张标注拼音或图片提示）
2. 屏幕上方显示当前目标（如"māo"的拼音、或一张猫的图片）
3. 玩家**滑动手指/鼠标**切割飞过的汉字卡片
4. **切对**（切中"猫"）→ 得分 + Combo 计数
5. **切错**（切中其他汉字）→ 扣一条命（共 3 条）
6. **漏切**（目标汉字飞出屏幕）→ 不扣命但断 Combo
7. 特殊卡片：炸弹（切到扣命）、金色卡片（双倍分）

### 2.2 难度递进

| 阶段 | 出题速度 | 同屏汉字数 | 干扰项 | 触发条件 |
|------|---------|-----------|--------|---------|
| 初始 | 慢（1.5秒/个） | 2-3 | 无 | 开局 |
| 加速 1 | 中（1秒/个） | 3-4 | 形近字 | 得分 > 100 |
| 加速 2 | 快（0.7秒/个） | 4-5 | 形近字+同音字 | 得分 > 300 |
| 极速 | 极快（0.4秒/个） | 5-6 | 大量干扰 | 得分 > 600 |
| 无尽 | 持续加速 | 6+ | 全部 | 得分 > 1000 |

### 2.3 目标出题逻辑

```
题库：L1 课程 300 个基础汉字

每轮出题：
  1. 从题库随机选 1 个目标汉字
  2. 显示目标提示（拼音/图片/释义，随难度切换）
  3. 发射目标汉字 + 2-5 个干扰汉字
  4. 干扰汉字优先选形近字/同音字（增加难度和学习效果）
  5. 玩家切中目标 → 进入下一轮
  6. 自适应：频繁切错的汉字会更多出现（SRS 联动）
```

## 三、游戏模式

### 3.1 单人·经典（Solo Classic）

| 维度 | 说明 |
|------|------|
| 规则 | 3 条命，无时间限制，无尽模式 |
| 目标 | 尽可能得高分，挑战个人纪录 |
| 结束条件 | 3 条命耗尽 |
| 评分 | 基础分 + Combo 加成 + 连击奖励 |
| 记录 | 个人最高分、最长 Combo、本局答对率 |

### 3.2 单人·限时（Solo Timed）

| 维度 | 说明 |
|------|------|
| 规则 | 60 秒倒计时，不扣命 |
| 目标 | 限时内切对最多汉字 |
| 结束条件 | 时间耗尽 |
| 评分 | 正确数 × 基础分 + 准确率加成 |
| 特殊 | 切错不扣命但断 Combo |

### 3.3 1v1 PK（实时对战）

| 维度 | 说明 |
|------|------|
| 匹配 | 段位匹配，等待 < 10 秒 |
| 规则 | 双方同时看到相同目标汉字，分屏显示 |
| 计分 | 谁先切对得分，对方不得分；同时切对各得一半 |
| 时长 | 2 分钟 / 20 题（先到先得） |
| 结果 | 总分高者胜，平局加赛 3 题 |
| 星数 | 胜 +1🌟，败 -1🌟（最低 1🌟） |

### 3.4 多人混战（Multiplayer）

| 维度 | 说明 |
|------|------|
| 人数 | 2-4 人 |
| 匹配 | 快速匹配或好友房间 |
| 规则 | 所有人同题，各自切割，实时比分 |
| 时长 | 3 分钟 |
| 结果 | 积分排名 1-4 名，前 2 名获胜 |

## 四、视觉与交互设计

### 4.1 画面风格

| 元素 | 设计 |
|------|------|
| 背景 | 中国风画卷轴（横屏展开） |
| 汉字卡片 | 毛笔字风格方形卡片，带轻微 3D 旋转 |
| 切割特效 | 默认：金色刀光 + 光效飞溅；皮肤可替换 |
| 目标提示 | 屏幕顶部居中大字显示拼音/图片 |
| 计分板 | 右上角实时分数 + Combo 计数 |
| 生命值 | 左上角 3 颗红心 |

### 4.2 横屏适配

| 场景 | 处理 |
|------|------|
| **移动端 H5** | 进入游戏检测方向 → 竖屏时显示"请旋转设备"提示 → CSS `orientation: landscape` 锁定 |
| **Web 桌面** | 默认横屏，宽高比 16:9，居中显示 |
| **触控 vs 鼠标** | 移动端：手指滑动切割；桌面：鼠标拖拽切割 |
| **安全区域** | 适配 iPhone 刘海屏和 Android 异形屏 |

### 4.3 音效设计

| 事件 | 音效 |
|------|------|
| 切对 | 清脆"刷"声 + 汉字读音（TTS） |
| 切错 | 低沉"咚"声 + 屏幕震动 |
| Combo 3连 | 叮叮叮 + "不错！" |
| Combo 5连 | 更华丽音效 + "太棒了！" |
| Combo 10连 | 最华丽特效音 + "无敌！" |
| 金色卡片 | 金币叮当声 |
| 炸弹 | 爆炸声 |
| Game Over | 晕散音效 |

## 五、上瘾机制

### 5.1 即时反馈

- 每次切对播放汉字读音 + 视觉特效（学习+爽感双重）
- Combo 连击数字放大显示（3→5→10→20→50）
- 高分时屏幕边缘发光

### 5.2 成就感

| 成就 | 条件 | 奖励 |
|------|------|------|
| 初出茅庐 | 首次达到 100 分 | 头像框 |
| 百发百中 | 单局准确率 100% | 称号 |
| Combo 大师 | 单局最长 Combo ≥ 20 | 皮肤碎片 |
| 汉字达人 | 累计切对 1000 个汉字 | 积分 ×500 |
| 切切王者 | 段位达到王者 | 限定刀光特效 |

### 5.3 每日挑战

每天刷新一个特殊模式：
- **形近字专场**：干扰项全是形近字（如 大/太/犬）
- **同声调专场**：所有汉字同一声调
- **极速模式**：初始就是极速，挑战极限反应
- **反向模式**：切掉**不是**目标的汉字

### 5.4 段位追求

专属段位名：
- 书童 → 识字者 → 切字匠 → 汉字猎手 → 切字王者 → 汉字之神

## 七、技术要点

### 7.1 Phaser 3 场景结构

```
HanziSlashGame
  ├── BootScene         # 资源预加载
  ├── MenuScene         # 游戏主菜单（模式选择）
  ├── GameScene         # 核心游戏场景
  │     ├── HanziSpawner   # 汉字卡片生成器
  │     ├── SlashDetector  # 切割手势/鼠标检测
  │     ├── ScoreManager   # 计分+Combo 管理
  │     ├── LifeManager    # 生命值管理
  │     └── EffectManager  # 粒子特效+音效
  ├── ResultScene       # 结算页面
  └── PKScene           # 对战专用场景（分屏）
```

### 7.2 关键技术

| 技术点 | 方案 |
|--------|------|
| 汉字卡片物理 | Phaser Arcade Physics，设置速度/重力/旋转 |
| 滑动切割检测 | Pointer drag 事件 → 线段与卡片碰撞检测 |
| 切割动画 | Sprite 一分为二 + 粒子飞散 |
| 横屏锁定 | Screen Orientation API + CSS media query |
| 音频管理 | Phaser Sound Manager + TTS 预缓存 |
| PK 同步 | WebSocket 双向同步出题+答案+分数 |

### 7.3 性能目标

| 指标 | 目标 |
|------|------|
| 帧率 | 稳定 60 FPS |
| 首次加载 | < 3 秒（资源 < 2MB） |
| 内存占用 | < 100MB |
| 输入延迟 | < 16ms |

## 八、题库与课程联动

### 8.1 题库来源

```
L1 课程 300 个基础汉字
  ├── 每个汉字包含：
  │     ├── 字符（猫）
  │     ├── 拼音（māo）
  │     ├── 释义图片 URL
  │     ├── 形近字列表（描、瞄、喵）
  │     ├── 同音字列表（毛、矛、茅）
  │     └── 难度等级（1-5）
  └── 题库由服务端下发，客户端不预存答案
```

### 8.2 自适应出题

- 用户频繁切错的汉字 → 增加出现频率
- 用户已掌握的汉字 → 减少出现，提升新字比例
- 与 SRS 系统联动：游戏中切错 → 自动加入 SRS 复习队列
- 游戏中切对的汉字 → 标记为"已巩固"

### 8.3 学习报告

每局结束展示：
- 本局切对/切错/漏切统计
- 最容易切错的 3 个汉字（高亮学习）
- 累计掌握率：X / 300 个 L1 汉字
