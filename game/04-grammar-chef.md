## 二、核心玩法

### 2.1 基础规则

1. **中国风餐厅**场景，顾客排队点菜
2. 每个顾客的"菜单"是一个**目标句意**（如"小明因为下雨所以没去学校"）
3. 屏幕下方传送带上滚动着各种**词语食材**
4. 玩家从传送带上**抓取正确词语** → 按**正确语序**放入锅中
5. 语序正确 → 句子"烹饪完成" → 端给顾客 → 得分
6. 语序错误 → 菜品失败 → 重做（扣时间）
7. 超时未完成 → 顾客离开 → 扣分
8. 顾客源源不断，速度递增

### 2.2 食材类型

| 食材类型 | 视觉 | 说明 |
|---------|------|------|
| 普通词语 | 白色方块 | 名词、动词、形容词 |
| 关联词 | 🌶️ 红色辣椒 | "因为""所以""虽然""但是" 等 |
| 标点符号 | 🧂 调味料 | 逗号、句号、问号 |
| 干扰词 | 🥬 烂菜叶 | 不属于目标句的词语（选了扣分） |

### 2.3 难度递进

| 阶段 | 句子复杂度 | 传送带速度 | 干扰词 |
|------|-----------|-----------|--------|
| 初始 | 简单句（主谓宾） | 慢 | 无 |
| 中等 | 复合句（1 个关联词） | 中 | 1-2 个 |
| 困难 | 复杂句（2+ 关联词） | 快 | 3-4 个 |
| 极难 | 长难句 + 倒装 | 极快 | 5+ 个 |

## 三、游戏模式

### 3.1 单人·经营

| 维度 | 说明 |
|------|------|
| 规则 | 无尽顾客，3 个顾客离开则 Game Over |
| 目标 | 服务最多顾客 |
| 奖励 | 累计积分装修餐厅（永久可视化进度） |

### 3.2 单人·限时

| 维度 | 说明 |
|------|------|
| 规则 | 90 秒内完成最多订单 |
| 目标 | 最多完成数 |
| 不扣命 | 失败订单跳过，不结束游戏 |

### 3.3 1v1 PK

| 维度 | 说明 |
|------|------|
| 规则 | 各自厨房，同时接单（题目相同） |
| 胜负 | 2 分钟内完成更多订单者胜 |
| 策略 | 速度+准确率的平衡 |

### 3.4 多人协作（2 人合作）

| 维度 | 说明 |
|------|------|
| 规则 | 2 人共享厨房，一人负责选词，一人负责排序 |
| 沟通 | 快捷消息（"选这个！""顺序不对！"） |
| 目标 | 合作完成最多订单 |

## 四、视觉与交互设计

### 4.1 画面风格

| 元素 | 设计 |
|------|------|
| 场景 | 中国风餐厅厨房（横屏全景） |
| 顾客 | 卡通风格中国人物，不同表情（等待→不耐烦→离开） |
| 传送带 | 屏幕下方横向滚动 |
| 锅/灶台 | 屏幕中央，拖入词语的目标区域 |
| 词语食材 | 不同颜色的食材造型方块 |

### 4.2 横屏布局

```
┌───────────────────────────────────────────────┐
│  [顾客队列：💬菜单1  💬菜单2  💬菜单3]        │
│                                               │
│          ┌─────────────────┐                  │
│  [分数]   │   🍳 灶台/锅    │   [Combo]        │
│  [时间]   │  [已放入的词语]  │   [进度]         │
│          └─────────────────┘                  │
│                                               │
│  ←← [词1] [词2] [词3] [词4] [词5] [词6] ←←   │
│           传送带（横向滚动）                     │
└───────────────────────────────────────────────┘
```

### 4.3 交互

| 操控 | 方式 |
|------|------|
| 移动端 | 点击传送带上的词语 → 自动飞入锅中（按点击顺序排列） |
| 桌面 | 点击/拖拽词语到锅中 |
| 撤销 | 点击锅中已放入的词语可撤回到传送带 |
| 确认 | 点击"上菜"按钮提交 |

## 五、上瘾机制

### 5.1 餐厅装修系统

累计积分可解锁餐厅装修：
- 基础厨房 → 小饭馆 → 中式餐厅 → 宫廷御膳房 → 满汉全席殿堂
- 每个阶段视觉效果完全不同，激励持续玩

### 5.2 连续 Combo

- 连续正确完成 3 单 → Combo ×1.5
- 连续 5 单 → Combo ×2.0 + 特殊顾客出现
- VIP 顾客 → 双倍积分

### 5.3 成就

| 成就 | 条件 |
|------|------|
| 学徒厨师 | 首次正确完成 10 单 |
| 语法大师 | 累计正确使用 50 个关联词 |
| 五星餐厅 | 餐厅装修达到最高级 |
| 无差错厨神 | 单局 0 失误完成 20 单 |

## 七、技术要点

### 7.1 场景结构

```
GrammarChefGame
  ├── BootScene
  ├── MenuScene
  ├── GameScene
  │     ├── CustomerQueue     # 顾客队列管理
  │     ├── ConveyorBelt      # 传送带词语滚动
  │     ├── CookingPot        # 灶台/锅（拖入区域）
  │     ├── SentenceValidator # 语序正确性验证（服务端）
  │     ├── RestaurantDecor   # 餐厅装修状态
  │     └── EffectManager
  ├── ResultScene
  └── CoopScene              # 合作模式场景
```

### 7.2 语序验证

- 服务端下发：目标句意 + 正确词语列表 + 正确顺序
- 客户端只做视觉展示和拖拽
- 提交时发送词语排列到服务端验证
- 关联词的位置容错：如"因为"可在句首或分句首

## 八、题库与课程联动

### 8.1 题库来源

- L4 课程句型结构 50+
- 关联词配对 30+ 组（因为-所以、虽然-但是、不但-而且 等）
- 复句类型：因果、转折、并列、递进、条件、假设
- 每题随课程进度解锁新句型

### 8.2 学习报告

每局结束：
- 本局组装的句型分布（因果 X 个、转折 Y 个…）
- 最常出错的关联词搭配
- 语序准确率
