## 二、核心玩法

### 2.1 基础规则

1. 8×8 棋盘上每个格子是一个**词语方块**
2. 玩家交换相邻两个方块
3. 如果交换后形成**语义配对**则消除得分：
   - **近义词配对**：如"高兴-开心"（相邻 2 个即消）
   - **反义词配对**：如"大-小"（相邻 2 个即消）
   - **搭配词组**：如"认真-学习"（相邻 2 个即消）
4. 消除后上方方块下落，顶部补充新词语
5. 无法形成任何配对时 → 棋盘自动重排
6. 无尽模式：棋盘永远可玩；限步模式：30 步内尽量高分

### 2.2 消除规则与特殊方块

| 消除类型 | 条件 | 产生 |
|---------|------|------|
| 普通消除 | 2 个语义配对 | 基础得分 |
| 3 连消 | 3 个同类词相邻 | 💣 炸弹方块（消周围 8 格） |
| 4 连消 | 4 个同类词一行/列 | 🔥 横扫方块（清一行或一列） |
| 5 连消 | 5 个同类词 | 🌈 彩虹方块（消所有同类） |
| L/T 消 | L 或 T 形排列 | ⭐ 十字方块（清十字范围） |

### 2.3 词语分类显示

棋盘上词语方块带**颜色标记**表示词性：
- 🔵 名词 | 🟢 动词 | 🟠 形容词 | 🔴 副词 | ⚪ 其他

这帮助玩家直觉判断哪些词可能是搭配。

## 三、游戏模式

### 3.1 单人·无尽

| 维度 | 说明 |
|------|------|
| 规则 | 无限消除，计分无上限 |
| 目标 | 最高分 |
| 结束 | 玩家主动退出 |
| 特殊 | 每消除 50 对，难度提升（加入更相近的词对） |

### 3.2 单人·限步

| 维度 | 说明 |
|------|------|
| 规则 | 限 30 步，每交换一次算 1 步 |
| 目标 | 最高分 + 最高效利用步数 |
| 评价 | ⭐-⭐⭐⭐ 按分数评星 |

### 3.3 1v1 PK

| 维度 | 说明 |
|------|------|
| 规则 | 同棋盘布局，2 分钟限时，比谁分高 |
| 策略 | 看到同样的棋盘，考验谁找配对更快 |
| 实时 | 能看到对手分数实时变化 |

### 3.4 多人对战

| 维度 | 说明 |
|------|------|
| 人数 | 4 人同棋盘 |
| 规则 | 2 分钟限时竞速 |
| 排名 | 分数排名 1-4 |

## 五、上瘾机制

### 5.1 cascade 连锁爽感

消除后上方掉落的词语如果恰好形成新配对 → 自动连消 → 连锁反应可能持续多次。
- 每次 cascade 分数 ×1.5 倍递增
- cascade ≥ 3 次触发全屏"Fantastic！"特效

### 5.2 学习反馈

- 每次消除显示语义关系（"高兴 ≈ 开心"/"大 ↔ 小"）
- 局结显示"本局你学到了 X 组近义词、Y 组反义词"
- 错误交换时短暂显示"这两个词没有语义关系哦"

### 5.3 成就

| 成就 | 条件 |
|------|------|
| 词汇入门 | 累计消除 100 对 |
| 连消达人 | 单局 cascade ≥ 5 次 |
| 反义词专家 | 累计消除 200 对反义词 |
| 搭配高手 | 累计消除 200 对搭配词组 |
| 消消乐王者 | 段位达到王者 |

## 七、技术要点

### 7.1 场景结构

```
WordMatchGame
  ├── BootScene
  ├── MenuScene
  ├── GameScene
  │     ├── BoardManager     # 8×8 棋盘状态管理
  │     ├── MatchDetector    # 语义配对检测（查表）
  │     ├── SwapHandler      # 交换动画+验证
  │     ├── CascadeEngine    # 连消引擎（递归检测）
  │     ├── FillManager      # 补充新方块
  │     └── EffectManager    # 特效
  ├── ResultScene
  └── PKScene
```

### 7.2 配对数据

```json
{
  "semantic_pairs": [
    { "type": "synonym", "words": ["高兴", "开心"] },
    { "type": "antonym", "words": ["大", "小"] },
    { "type": "collocation", "words": ["认真", "学习"] }
  ]
}
```
- 配对表由服务端根据用户当前 L3 进度下发
- 棋盘生成时保证至少存在 3 对可消除配对
