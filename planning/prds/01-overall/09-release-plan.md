## 四、回滚策略

### 4.1 紧急回滚
- 严重 bug → 1h 内回滚到上一稳定版本
- DB 迁移失败 → 回退 + 数据修复
- 支付故障 → Feature flag 切到备援

### 4.2 灰度回滚
- 5% → 25% → 50% → 100% 灰度
- 任一阶段 KPI 异常 → 暂停 / 回退

### 4.3 数据回滚
- 数据库每日全备
- WAL 流式备份（PITR）
- RPO < 1h / RTO < 4h

## 七、发布流程

### 7.1 Code Freeze
- 发布前 48h 停止合并新功能
- 仅接受 P0 bug fix

### 7.2 RC（Release Candidate）
- 部署到 stg
- 全员 smoke test
- QA 跑回归

### 7.3 发布
- 周二 03:00 UTC+7 维护窗
- 数据库迁移先行
- 灰度 5% → 监控 1h → 100%
- 发布后 4h 内 on-call

### 7.4 复盘
- 发布后 24h 内复盘
- 记录问题 + 改进项
