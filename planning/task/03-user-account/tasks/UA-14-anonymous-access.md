## 需求落实

- 页面：DC 类目页、DC 文章页、NV 章节页、课程学习页、游戏词包选择页、注册引导弹窗。
- 组件：AnonymousLimitGate、RegisterPromptModal、CourseAccessGate、GamePackPermissionSelector。
- API：匿名 token 签发、访问权限检查 API 或中间件、游戏词包权限 API、事件上报 API。
- 数据表：`events`、可选匿名访问计数表；`security_events` 记录异常。
- 状态逻辑：DC 按类目开放白名单判断；NV 匿名仅首章；登录后 DC/NV/GM 全开；CR 按每主题 Stage 1-3 免费 + 购买阶段 + 会员/人工授权判断；GM 根据 CR 权限生成可选 track/stage。

## 技术假设

- 匿名 JWT 由 SC 模块提供，UA 负责产品访问状态汇总。
- DC/NV/CR/GM 模块调用统一 access gate。
- 课程免费试学范围按“每个主题 Stage 1-3 全部章节”落库；UA/CR/GM 权限任务必须保持一致。

## 不明确 / 风险

- 处理：按 `content/china/00-index.md` 与本任务裁决为“前 3 类目”。
- 风险：课程旧文档中“Stage 1 前 3 章免费”和本次裁决“前 3 阶段全部章节免费”冲突。
- 处理：按本次裁决更新 CR/GM 权限任务，旧“Stage 1 前 3 章免费”不得再作为实现依据。
