# ADC-10 · 实现前后台闭环验收

## PRD 原文引用

- `planning/rules.md`：“列表 → 详情 → 关键交互（学习/游戏/解锁/购买/阅读）一条主链路。”
- `AD-AC-007`：“4 模块内容 CRUD + 发布。”
- `DC-AC-001~014` 覆盖 DC 前台验收。

## 需求落实

- 页面：后台 DC 编辑页、前台 `/discover` 链路。
- 组件：无新增。
- API：后台 CRUD + 前台查询。
- 数据表：DC 内容表、admin_audit_logs、events。
- 状态逻辑：后台发布后前台可见；撤回后前台不可见；权限门禁始终生效。
- 覆盖：必须串起类目、文章、句子、审校、发布、缓存、搜索、SEO、分享、阅读进度、收藏、笔记、评分。
- 数据：同时跑开发 seed 36 篇校验与 W0 600 篇内容包校验（正式内容可由外部导入包提供）。

## 不明确 / 风险

- 风险：只测后台不测前台会漏内容状态。
- 处理：验收必须同时跑后台和前台链路。

## 技术假设

- E2E 使用 MCP Puppeteer 直连 3100/4100。

## 最终验收清单

- [ ] 后台创建文章并发布。
- [ ] 前台登录用户可读新文章。
- [ ] 未登录访问受限类目新文章被拦截。
- [ ] 撤回后前台不可读。
- [ ] 未发布预览链接匿名不可访问，过期 token 返回 401。
- [ ] 搜索、sitemap、缓存、相关推荐、分享卡片均不能泄露受限或撤回正文。
- [ ] 导入/编辑/发布链路后 JSONB 类型正确，审计日志完整。
- [ ] 限流、预览 token、音频 proxy、红线规则引擎、缓存撤回竞态均有 E2E 或集成验证记录。
- [ ] MCP Puppeteer 直连 3100/4100 完成前后台主链路截图或记录。
