# ADC-09 · 实现 DC 访问模型配置可视化

## PRD 原文引用

- `content/china/00-index.md`：“未登录访客 | 只能浏览前 3 个类目（中国历史、中国美食、名胜风光）。”
- `content/china/00-index.md`：“登录用户 | 可浏览全部 12 个类目的全部内容（免费）。”

## 需求落实

- 页面：类目管理页、发布预览页。
- 组件：AccessModelSummary、AnonymousVisibleBadge。
- API：随 category 管理 API 返回访问模型。
- 数据表：`content_categories`。
- 状态逻辑：后台明确显示哪些类目匿名开放；编辑文章时提示前台匿名可见性。
- 一致性：后台展示必须与 DC-13 中央守卫同源，不能由前端硬编码另一份规则。

## 不明确 / 风险

- 风险：运营误以为可按篇设置免费。
- 处理：v1 后台不提供按篇免费开关，只展示按类目规则。

## 技术假设

- 如果后续要更改匿名开放类目，必须先改内容规则/PRD。

## 最终验收清单

- [ ] 前 3 类目显示“匿名开放”。
- [ ] 第 4-12 类目显示“需登录”。
- [ ] 文章编辑器显示该文章发布后的访问模型。
- [ ] 无按篇扣减/按篇免费配置。
- [ ] 搜索、SEO、缓存、分享和预览页面展示相同访问模型提示。
