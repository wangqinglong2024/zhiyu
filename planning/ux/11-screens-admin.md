# 11 · 后台屏幕规格

## 一、后台原则

后台是运营和内容团队的工作台：密度优先、审计优先、批量操作优先。保留松烟雅瓷 token，但不做沉浸式装饰。

## 二、登录

- 邮箱、密码、TOTP 二步验证。
- 失败 5 次锁定 15 分钟。
- 页面使用 paper 实底表单，避免透明背景影响安全输入。

## 三、Dashboard

KPI：DAU、新增用户、收入、转化率、留存、活跃订阅、内容产出、客服未读、待审稿件、系统告警。

图表使用自托管前端库，事件来自 `events` 与业务表，不接外部分析 SaaS。

## 四、内容管理

通用列表能力：筛选、搜索、状态、批量操作、新建、导入、导出、分页、软删。

模块：
- 发现中国文章：12 类目、句子级拼音/翻译/音频、红线检测、预览、发布。
- 课程：4 轨道、12 阶段、12 章、12 节、步骤编辑器。
- 小说：12 类目、小说、章节、角色字典。
- 游戏词包：汉字、拼音、翻译、音频、关联课程权限。

## 五、内容工厂

v1 只提供占位说明、CSV/YAML 导入入口、手动/脚本化写库入口。自动化 LangGraph/双 LLM/TTS 工厂属于 v1.5，不在 v1 后台作为可执行功能。

## 六、审校工作台

- 队列：待审、我的、今日完成、超时。
- 主面板：原文、拼音、4 语翻译、音频试听、红线结果、批注。
- 操作：通过、打回、改写、保存草稿。
- 所有动作写 admin audit。

## 七、用户管理

列表支持按注册日期、国家、等级、会员、活跃度筛选。详情含基本资料、学习数据、订阅/订单、知语币流水、分销关系、客服历史、操作日志。高危操作必须二次确认并审计。

## 八、订单与订阅

通过 PaymentAdapter 展示订单、订阅、退款状态；fake provider 在 Docker 验证中可生成完整订单流。页面不得写死外部支付 provider。

## 九、知语币与分销

- 知语币：余额、流水、奖励、兑换、调整。
- 分销：邀请、转化、二级返佣、pending/confirmed/issued/reversed，单位 ZC。
- v1 无现金提现审批。

## 十、客服工作台

三栏：会话列表、当前会话、用户档案。支持标签、转接、内部备注、FAQ 插入、工单升级。客服按语言路由。

## 十一、系统设置

Feature Flags、公告、邮件模板、服务密钥引用、系统状态、操作审计、导出任务。密钥只展示引用/状态，不展示明文。

## 十二、验收

- [ ] 所有列表支持筛选、搜索、分页、批量操作。
- [ ] 所有写操作有 RBAC 与审计。
- [ ] 内容编辑支持自动保存、历史版本、预览。
- [ ] 后台无外部分析/支付 provider 硬编码。
- [ ] 移动端基础可用，桌面端高密度高效率。
