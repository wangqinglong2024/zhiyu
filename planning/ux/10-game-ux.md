# 10 · 游戏 UX（Game UX · Landscape Forced）

## 一、强制横屏机制

### 1.1 流程
1. 用户点击"开始游戏" CTA
2. 检测当前方向：
   - **landscape** → 直接进入画布
   - **portrait** → 显示 `OrientationMask` 引导用户旋转设备
3. 监听 `orientationchange` / `screen.orientation`
4. 横屏后自动消失 mask，进入游戏

### 1.2 OrientationMask 视觉
```
┌──────────────────────────────────┐
│                                  │
│        [📱 旋转动画]              │
│                                  │
│      请旋转设备                   │
│      游戏需要横屏体验               │
│                                  │
│   [取消，回到游戏列表]             │
└──────────────────────────────────┘
```
- 全屏毛玻璃 `.glass-elevated`
- 中央设备旋转动效（loop 2s）
- 提示文字 + 取消按钮

### 1.3 锁定 API
优先使用 Screen Orientation API：
```ts
try {
  await screen.orientation.lock('landscape');
} catch {
  // 不支持锁定 → 仅监听不强制
}
```
- iOS Safari 不支持 → fallback CSS rotate（不推荐，仅 mask 引导）
- Android Chrome 支持

### 1.4 PWA Manifest
```json
{
  "orientation": "any",
  "display": "standalone"
}
```
游戏内动态请求 landscape，其他页面 default。

### 1.5 退出游戏
- 退出 `/games/:game/play` → 解锁方向 `screen.orientation.unlock()`
- 自动恢复竖屏

## 二、横屏画布布局

### 2.1 安全区
```
┌──────────────────────────────────────────────────┐
│ [safe-left]                          [safe-right] │
│   ┌──────────────────────────────────────────┐   │
│   │  [HUD Top: 分数 / 时间 / 暂停]            │   │
│   ├──────────────────────────────────────────┤   │
│   │                                          │   │
│   │            游戏画布 (PixiJS)               │   │
│   │                                          │   │
│   ├──────────────────────────────────────────┤   │
│   │  [HUD Bottom: 命数 / 道具 / 控制按钮]      │   │
│   └──────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```
- 左右 padding = max(16, safe-area-inset-left/right)
- 上下 padding = 8px
- 画布居中

### 2.2 分辨率
- 设计基准：1280×720
- 适配：等比缩放，多余区域显示 MeshGradient 背景
- 最小：568×320（小屏 iPhone 横屏）
- 最大：无上限（桌面浏览器全屏）

### 2.3 PixiJS 配置
```ts
const app = new Application({
  width: 1280,
  height: 720,
  backgroundAlpha: 0,
  resolution: window.devicePixelRatio,
  autoDensity: true,
  antialias: true,
});
```

## 四、桌面端交互（键鼠）

### 4.1 通用键位
| 键 | 行为 |
|---|---|
| `←→↑↓` 或 `WASD` | 方向移动 |
| `Space` | 主操作（射击 / 跳跃 / 选择） |
| `Esc` | 暂停 |
| `Enter` | 确认 |
| `R` | 重新开始 |
| `M` | 静音 |
| `1-9` | 快捷选项（如多选答案） |

### 4.2 鼠标
- 点击 → 主操作
- 悬停 → 高亮目标
- 拖拽 → 拖动元素
- 滚轮 → 缩放 / 切换

### 4.3 游戏类型映射
| 游戏 | 主键位 | 鼠标 |
|---|---|---|
| 拼音射手 | 字母键打字 | - |
| 汉字忍者 | 鼠标拖拽切割 | 主 |
| 声调泡泡 | 1-4 选声调 | 点击泡泡 |
| 俄罗斯方块 | ←→↓ 旋转空格 | - |
| 打地鼠 | 1-9 数字键 | 点击 |
| 三消 | - | 拖拽交换 |
| 贪吃蛇 | ←→↑↓ 转向 | - |
| 节奏 | 拼音字母键 | - |
| 跑酷 | Space 跳，↓滑铲 | - |
| 拼音塔防 | 1-9 选塔，鼠标放置 | 主 |
| 记忆翻牌 | 鼠标点击翻牌 | 主 |
| 弹弓 | 鼠标拖拽瞄准 | 主 |

## 五、移动端交互（触屏）

### 5.1 通用手势
- 单击 → 主操作
- 长按 → 蓄力 / 菜单
- 滑动 → 移动 / 切换
- 双指捏合 → 缩放（仅特定游戏）

### 5.2 虚拟控件
| 控件 | 用途 | 显示游戏 |
|---|---|---|
| 虚拟方向键 | 8 方向 | 贪吃蛇 / 跑酷 |
| 虚拟摇杆 | 360° | 方块旋转 |
| 大按钮 | 跳跃 / 射击 | 跑酷 / 射击 |
| 拼音键盘 | 拼音输入 | 拼音射手 / 节奏 |
| 数字按钮 1-4 | 声调选 | 声调泡泡 |
| 选项按钮 | 多选 | 多种 |

### 5.3 拼音键盘组件
```
┌──────────────────────────────────────┐
│ q w e r t y u i o p                  │
│  a s d f g h j k l                   │
│   z x c v b n m                      │
│ [←]  [space]   [⏎]                   │
└──────────────────────────────────────┘
```
- 半透明覆盖在画布底部 1/3
- 触觉反馈
- 自动隐藏（关卡间）
- 键宽 ≥ 32dp

### 5.4 触觉反馈
| 事件 | 强度 |
|---|---|
| 答对 | success |
| 答错 | error |
| 倒计时最后 5s 每秒 | light |
| 切方向 | light |
| 失败 | medium |
| 通关 | heavy + 烟花 |

## 六、游戏全局功能

### 6.1 暂停 / 恢复
- 自动暂停：失去焦点 / 接电话 / 切换 App
- 手动暂停：⏸ 按钮 / Esc 键
- 恢复：3-2-1 倒计时

### 6.2 退出
- 退出按钮 → 二次确认（"当前进度将丢失"）
- 强行退出 → 自动保存进度（部分游戏支持）

### 6.3 设置面板
- 音乐音量 0-100%
- 音效音量 0-100%
- 振动 ON/OFF
- 配色模式（亮 / 暗）
- 难度（v1.5：自适应；v2：手动）

### 6.4 词包切换
- 游戏内按钮可切换词包（中断当前回合）
- 切换后下一关生效

## 七、游戏教程

### 7.1 首玩教学
- 强制小教学（30s 内）
- 高亮交互区 + 引导文字
- "知道了"按钮跳过

### 7.2 重玩跳过
- 已通关后自动跳过教学
- 可手动重看

## 八、关卡 / 进度系统

### 8.1 关卡结构
- 每个游戏 N 关卡（v1: 10-20 关）
- 解锁制：完成上一关
- 三星评级（速度 / 准确 / 完成）

### 8.2 难度曲线
| 关卡区间 | 难度 |
|---|---|
| 1-5 | Easy（教学） |
| 6-10 | Medium |
| 11-15 | Hard |
| 16-20 | Expert |

### 8.3 自适应难度（v1.5+）
根据玩家最近 5 局表现调整：
- 准确 > 95% → 升级
- 准确 < 60% → 降级

## 十二、可访问性

### 12.1 必备
- 色弱模式（替代色 + 形状）
- 字幕（语音指令）
- 慢速模式（×0.5 ×0.75）
- 关闭闪烁动效

### 12.2 不强制
- 屏幕阅读器：游戏画布不强制（标注降级版）
- 键盘游戏：桌面端必须
