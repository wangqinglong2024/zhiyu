# 01 · 设计原则

## 一、体验目标

知语的视觉目标是“现代大气 + 全组件毛玻璃 + 东方内容气质”。用户打开后应感到这是一个成熟、清晰、昂贵感强的学习产品，而不是普通后台模板，也不是节庆装饰页。

## 二、五条原则

| 原则 | 说明 | 实现要求 |
|---|---|---|
| shadcn 统一 | 所有基础组件从 shadcn/ui 生成或封装 | 禁止业务组件绕开 `@zhiyu/ui` 自写 Button/Card/Input |
| 全组件玻璃 | 按钮、卡片、表格、输入、弹窗、导航、菜单、Toast 全部有玻璃材质 | 通过 `zy-glass-subtle/default/strong/ink` 统一控制透明度与 blur |
| 大气但高效 | 页面有纵深和呼吸感，工作台仍能高密度扫描 | 后台不做大 Hero，不做卡片套卡片 |
| 内容压住装饰 | 文化感来自真实内容、版式和材质 | 不用龙、灯笼、中国结、红金大字堆砌 |
| 本地可实现 | 视觉资源和验证路径必须 Docker 内闭环 | 不依赖外部设计云、监控云、测试云 |

## 三、视觉气质

- 背景：明亮模式使用暖白到雾青的低对比玻璃背景；暗色模式使用深墨、石青、冷白高光。
- 主色：朱砂只服务主动作、危险和重点状态，小面积但有力量。
- 辅色：青玉、瓷蓝、雾银用于导航、选中、进度、信息状态。
- 文字：松烟黑/瓷白为主体，表格和学习内容必须清晰。
- 图形：使用真实封面、抽象线条、细边框、轻质纹理，不使用离散彩色 blob。

## 四、Do / Don't

Do：
- 用宣纸纹理、细边框、青瓷玻璃、朱砂印记建立识别度。
- 用 shadcn/ui 的 Button、Card、Input、Dialog、Sheet、Dropdown、Tabs、Table、Toast 等作为唯一基础组件。
- 所有组件状态都要有 glass hover、press、focus、disabled 和 loading 版本。
- 用真实内容封面承载发现中国的主题差异。
- 在按钮、卡片、Tab、Toast 等组件中统一 token。
- 在后台优先展示表格、筛选、状态、审计与批量操作。

Don't：
- 不做大面积红金配色，不用龙、灯笼、中国结、扇子堆砌。
- 不用彩色 blob、粒子、霓虹渐变当主要背景。
- 不把所有页面做成漂浮卡片集合；页面区块可以是全宽玻璃带或工作台面板，不能卡片套卡片。
- 不在功能界面写“本页使用毛玻璃/古风设计”等说明性文案。
- 不引用外部 SaaS、CDN、云测试作为 v1 必需路径。

## 五、模块气质

| 模块 | 视觉重点 |
|---|---|
| 发现中国 | 内容封面 + 类目纹样 + 书页式阅读 |
| 课程 | 主题选择、清晰路径、学习进度、低干扰练习卡；UI 显示“主题”不用“轨道” |
| 游戏 | 横屏高对比 HUD，古韵只保留在边框/纹理/奖励反馈中 |
| 小说 | 沉浸阅读、纸感背景、章节节奏 |
| 我的 | 账户、进度、权益、设置清晰可扫 |
| 后台 | 专业、密集、低装饰，保留品牌 token |

## 六、质量判定

页面若只替换颜色仍像通用模板，判定失败。页面若只有少数卡片有毛玻璃、按钮/输入/表格仍是普通样式，判定失败。页面若装饰影响阅读、输入或表格扫描，也判定失败。合格页面必须同时具备 shadcn 组件一致性、全组件玻璃反馈、清晰信息结构和可辨认的知语气质。
