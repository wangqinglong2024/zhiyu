# 01 · 设计原则

## 一、体验目标

知语的视觉目标是“现代产品效率 + 古典文化气韵”。用户打开后应感到清雅、可信、愿意学习，而不是进入一个普通毛玻璃模板。

## 二、五条原则

| 原则 | 说明 | 实现要求 |
|---|---|---|
| 古韵先于特效 | 文化感来自材质、留白、排版、内容 | 不用装饰替代信息架构 |
| 清透但有骨 | 毛玻璃要有层级和边界 | 玻璃层最多 3 层可见，重点内容用实底保护可读性 |
| 留白有章法 | 参考书页、卷轴、碑帖的节奏 | 重要内容有呼吸感，后台仍保持密度 |
| 学习优先 | 拼音、翻译、音频、题目不能被装饰干扰 | 阅读/学习页降低背景纹理与动效 |
| 本地可实现 | 视觉资源和验证路径必须 Docker 内闭环 | 不依赖外部设计云、监控云、测试云 |

## 三、视觉气质

- 背景：宣纸暖白或夜墨深色，上覆极淡水墨雾带。
- 主色：朱砂红小面积点睛，不能铺满页面。
- 辅色：青瓷、玉色、雾蓝用于导航、次级按钮和信息状态。
- 文字：松烟黑为主体，正文清晰，标题可略有书卷气但不能仿古难读。
- 图形：窗棂线、印章小块、瓷片弧线、山水留白可作为抽象纹样。

## 四、Do / Don't

Do：
- 用宣纸纹理、细边框、青瓷玻璃、朱砂印记建立识别度。
- 用真实内容封面承载发现中国的主题差异。
- 在按钮、卡片、Tab、Toast 等组件中统一 token。
- 在后台优先展示表格、筛选、状态、审计与批量操作。

Don't：
- 不做大面积红金配色，不用龙、灯笼、中国结、扇子堆砌。
- 不用彩色 blob、粒子、霓虹渐变当主要背景。
- 不把所有页面做成漂浮卡片集合。
- 不在功能界面写“本页使用毛玻璃/古风设计”等说明性文案。
- 不引用外部 SaaS、CDN、云测试作为 v1 必需路径。

## 五、模块气质

| 模块 | 视觉重点 |
|---|---|
| 发现中国 | 内容封面 + 类目纹样 + 书页式阅读 |
| 课程 | 清晰路径、学习进度、低干扰练习卡 |
| 游戏 | 横屏高对比 HUD，古韵只保留在边框/纹理/奖励反馈中 |
| 小说 | 沉浸阅读、纸感背景、章节节奏 |
| 我的 | 账户、进度、权益、设置清晰可扫 |
| 后台 | 专业、密集、低装饰，保留品牌 token |

## 六、质量判定

页面若只替换颜色仍像通用模板，判定失败。页面若古风装饰影响阅读、输入或表格扫描，也判定失败。合格页面必须同时具备清晰的信息结构和可辨认的知语气质。
