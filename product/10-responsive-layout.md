## 一、总体布局策略

### 1.1 核心原则

- **Mobile First**：以 H5 移动端（375px 宽）为基准设计
- **PC 渐进增强**：大屏设备通过容器限宽 + 布局调整适配
- **产品形态**：MVP 为 PWA（Progressive Web App），非原生 App
- **不做平板优化**：平板自动使用 PC 布局（≥ 768px）

### 1.2 两种核心场景

| 场景 | 说明 | 布局方式 |
|------|------|---------|
| **页面场景** | 世界地图、词库、个人主页、设置等 | 标准响应式（内容自适应） |
| **游戏场景** | 关卡答题、迷你游戏、Boss 战 | 固定竖屏画布 + 深色背板 |

## 五、游戏场景布局（重点）

### 5.1 核心设计决策

**游戏画布（Phaser 3 Canvas）在 PC 端采用居中竖屏模式**：
- 画布比例固定：**9:16**（手机竖屏比例）
- 画布最大高度：**视口高度的 95%**
- 画布宽度：按比例计算（高度 ÷ 16 × 9）
- 画布居中显示，两侧为**深色背景填充**

### 5.2 H5 游戏画布

```
┌─ 375px × 全屏 ─────┐
│                    │
│   ┌─ Phaser Canvas ─┐ │
│   │ 100% 宽度       │ │ ← 全屏铺满
│   │ 包含游戏 UI      │ │    顶部=关卡信息栏
│   │                 │ │    中间=题目/游戏内容
│   │                 │ │    底部=操作区
│   └─────────────────┘ │
│                    │
└────────────────────┘
```

- 完全全屏，无浏览器地址栏干扰（PWA Standalone 模式自动隐藏）
- 非 PWA 模式：兼容浏览器地址栏，游戏区高度 = `100dvh`

### 5.3 PC 游戏画布

```
┌─────────────────────────────────────────────┐
│  ┌─ 顶部导航栏 ──────────────────────────┐    │
│  │ PlayLingo [Thoát（退出）]               │    │
│  └────────────────────────────────────────┘    │
│                                             │
│  ████████╔══════════════╗████████           │
│  ████████║              ║████████           │
│  ████████║  Phaser      ║████████           │
│  ████████║  Canvas      ║████████           │ ← 深色背景区
│  ████████║  (9:16)      ║████████           │    ████ = 深色填充
│  ████████║  居中显示     ║████████           │    深邃暗色调
│  ████████║              ║████████           │    不放任何功能元素
│  ████████║              ║████████           │
│  ████████╚══════════════╝████████           │
│                                             │
└─────────────────────────────────────────────┘
```

### 5.4 深色背景设计

两侧「深色留白区」的视觉处理：

| 属性 | 值 |
|------|-----|
| 背景色 | `#0a0a0f`（接近纯黑但不刺眼） |
| 装饰 | 可选：极淡的星空粒子或渐变光晕（与"宇宙折射"主题呼应） |
| 交互 | 无任何可交互元素 |
| 中心区域边框 | 可选：极细 1px `rgba(255,255,255,0.05)` 边框 |

### 5.5 画布尺寸计算逻辑

```
设 viewport 宽度 = vw, 高度 = vh

如果 vw < 768（移动端）：
  canvas 宽 = vw
  canvas 高 = 100dvh
  → 全屏显示

如果 vw ≥ 768（PC）：
  可用高度 = vh - 顶栏高度(64px) - 上下间距(各16px)
  canvas 高 = min(可用高度, vh × 0.95)
  canvas 宽 = canvas 高 × 9 / 16
  
  如果 canvas 宽 > vw × 0.6：
    canvas 宽 = vw × 0.6
    canvas 高 = canvas 宽 × 16 / 9
  
  → 居中显示
```

## 八、PWA 相关

### 8.1 PWA 安装体验

| 平台 | 行为 |
|------|------|
| Android Chrome | 浏览器底部提示「添加到主屏幕」 |
| iOS Safari | 引导弹窗教育用户「点击分享 → 添加到主屏幕」 |
| PC Chrome | 地址栏安装图标 |

### 8.2 PWA Standalone 模式

- 安装后打开 = 全屏无地址栏
- `display: standalone`
- 状态栏颜色跟随主题色（Rose #e11d48）
- 横屏锁定：通过 `screen.orientation.lock('portrait')` 锁定竖屏（仅移动端）

### 8.3 离线支持（MVP 最小化）

- 仅缓存 App Shell（HTML/CSS/JS 静态资源）
- 游戏内容需联网加载
- 离线时显示友好提示："Vui lòng kết nối mạng（请连接网络）"
- 会员用户已完成关卡可离线缓存复习（Service Worker + IndexedDB）

## 十一、用户故事与边界条件

### 11.1 核心用户故事

| # | 功能 | 用户故事 | P |
|---|------|---------|---|
| 1 | 移动端自适应 | 作为手机用户，我想不用缩放就能正常操作所有功能，以便获得原生 App 般的体验 | P0 |
| 2 | PC 端游戏 | 作为 PC 用户，我想用键盘和鼠标流畅玩游戏，以便在大屏上也能享受学习乐趣 | P0 |
| 3 | PWA 安装 | 作为用户，我想把 PlayLingo 添加到主屏幕，以便像原生应用一样快速启动 | P0 |

### 11.2 边界条件

| 场景 | 处理方式 |
|------|----------|
| 未知屏幕宽度（折叠屏、平板竖携等） | 375-768px 走 H5 布局，>768px 走 PC 布局，断点切换无闪烁 |
| PC 端横屏玩竖屏游戏 | 游戏画布居中 9:16，两侧深色幕布，不会拉伸变形 |
| 用户突然旋转屏幕（手机端） | 锁定竖屏，横屏时显示“Vui lòng xoay dọc điện thoại”提示 |
| 极小屏幕（<320px） | 最小支持宽度 320px，低于此值显示“屏幕宽度过小”提示页 |
| 极大屏幕（>1920px） | 内容区最大宽度 1440px，居中显示，两侧留白 |
