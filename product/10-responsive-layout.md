## 一、总体布局策略

### 1.1 核心原则

- **Mobile First**：以 H5 移动端（375px 宽）为基准设计非游戏页面
- **PC 渐进增强**：大屏设备通过容器限宽 + 布局调整适配
- **两种屏幕模式**：游戏模式强制横屏（16:9），非游戏模式响应式布局
- **产品形态**：MVP 为 PWA（Progressive Web App），非原生 App
- **不做平板专项优化**：平板自动使用 PC 布局（≥ 768px）

### 1.2 两种核心场景与屏幕方向

| 场景 | 包含页面 | H5 移动端方向 | PC/Web 端方向 | 画布比例 |
|------|---------|-------------|-------------|---------|
| **非游戏模式** | 世界地图、词库、个人主页、设置、订阅、推广中心等 | **竖屏**（portrait） | **横屏**（landscape） | 标准响应式 |
| **游戏模式** | 关卡答题、迷你游戏、Boss 战、过场动画 | **横屏**（landscape） | **横屏**（landscape） | 固定 **16:9** |

**核心区分**：
- 非游戏模式 = 浏览、管理、社交页面，遵循 Web 响应式实践
- 游戏模式 = 沉浸式全屏体验，**所有端统一横屏 16:9**，保证游戏体验一致

### 1.3 屏幕方向切换时机

```
用户在世界地图（竖屏/响应式）
    → 点击关卡节点 → 弹出关卡信息卡片（仍然竖屏/响应式）
    → 点击「开始」→ ⚡ 切换到横屏模式（H5 旋转屏幕 + 全屏）
    → 进入游戏画面（16:9 横屏） 
    → 完成 / 退出
    → ⚡ 切回竖屏/响应式 → 返回世界地图

用户进入小游戏
    → 从小游戏列表（竖屏/响应式）点击「开始」
    → ⚡ 切换到横屏模式
    → 游戏画面（16:9 横屏）
    → 结算 → ⚡ 切回竖屏/响应式

用户触发过场动画
    → 从地图自动触发
    → ⚡ 切换到横屏模式
    → 动画播放（16:9 横屏）
    → 结束 → ⚡ 切回竖屏/响应式
```

### 1.4 横屏切换的技术实现

**H5 移动端**：
```javascript
// 进入游戏模式
async function enterGameMode() {
  // 1. 请求全屏
  await document.documentElement.requestFullscreen();
  // 2. 锁定横屏
  await screen.orientation.lock('landscape');
  // 3. 加载游戏画布
  initPhaserCanvas(16, 9);
}

// 退出游戏模式
async function exitGameMode() {
  // 1. 解除方向锁定
  screen.orientation.unlock();
  // 2. 退出全屏
  if (document.fullscreenElement) {
    await document.exitFullscreen();
  }
  // 3. 返回响应式页面
  navigateBack();
}
```

**降级方案**（不支持 `screen.orientation.lock` 的浏览器，如 iOS Safari）：
- 显示旋转提示覆盖层："Vui lòng xoay ngang điện thoại để chơi game"（请横屏游玩）
- 覆盖层包含一个手机旋转动画图标
- 检测 `window.orientation` 或 `matchMedia('(orientation: landscape)')`，当用户手动旋转后自动隐藏覆盖层

**PC/Web 端**：
- 无需旋转处理，浏览器天然横屏
- 游戏模式：画布居中显示，上下留黑边（letterbox）
- 非游戏模式：标准响应式布局

## 二、断点定义

基于 Tailwind CSS v4 断点系统：

| 断点名 | 宽度范围 | 代表设备 | 设计基准 |
|--------|---------|---------|---------|
| xs | < 375px | 小屏手机 | 缩放适配 |
| sm | 375px — 639px | 标准手机（竖屏） | **非游戏模式主设计稿基准** |
| md | 640px — 767px | 大屏手机竖屏 / 手机横屏 | 拉伸/游戏模式 |
| lg | 768px — 1023px | 平板横屏 | PC 布局启动 |
| xl | 1024px — 1279px | 笔记本 | PC 标准布局 |
| 2xl | ≥ 1280px | 桌面显示器 | PC 宽屏布局 |

> 注意：H5 游戏模式下不使用响应式断点，统一采用 16:9 横屏画布。

## 五、游戏模式布局（重点 — 全端横屏 16:9）

### 5.1 核心设计决策

**游戏画面统一采用 16:9 横屏**——无论 H5 还是 PC，游戏体验完全一致：
- 画布比例固定：**16:9**
- H5 移动端：强制横屏全屏，画布铺满整个屏幕
- PC 端：画布居中，上下留有 letterbox 黑边（如视口宽高比超过 16:9）
- 所有游戏内 UI 元素按 16:9 横屏比例设计

### 5.2 H5 游戏画布（横屏全屏）

```
┌──────────── 手机横屏全屏 ─────────────┐
│                                       │
│  ┌─ Phaser Canvas (16:9 横屏) ──────┐  │
│  │                                   │  │
│  │  ← 暂停   Bài 1-1   ⭐⭐☆ ❤❤❤  │  │ ← 顶栏
│  │  ━━━━━━━━━━━━━━━░░░  68%         │  │ ← 进度条
│  │                                   │  │
│  │       游戏/教学主内容区域            │  │ ← 主区域占大部分
│  │       （根据关卡类型不同）            │  │
│  │                                   │  │
│  │  🪙 +20        [选项A] [选项B]     │  │ ← 底栏操作区
│  │                [选项C] [选项D]     │  │
│  └───────────────────────────────────┘  │
│                                       │
└───────────────────────────────────────┘
```

- 进入游戏时触发全屏 + 横屏锁定
- 画布宽度 = 屏幕物理宽度（横屏后的长边）
- 画布高度 = 屏幕物理高度（横屏后的短边）
- 如果设备比例不是精确 16:9，按短边适配 + 上下或左右微量留黑
- 隐藏浏览器 UI（PWA Standalone 或 Fullscreen API）

### 5.4 深色背景设计（PC 端游戏两侧/上下留白）

| 属性 | 值 |
|------|-----|
| 背景色 | `#0a0a0f`（接近纯黑但不刺眼） |
| 装饰 | 可选：极淡的星空粒子或渐变光晕 |
| 交互 | 无任何可交互元素 |
| 边框 | 可选：极细 1px `rgba(255,255,255,0.05)` 边框 |

### 5.5 画布尺寸计算逻辑

```
设 viewport 宽度 = vw, 高度 = vh
目标比例 = 16:9

═══ H5 移动端（横屏模式下）═══
  canvas 宽 = vw（横屏后的长边）
  canvas 高 = vh（横屏后的短边）
  实际比例 = vw / vh

  如果实际比例 > 16/9（设备更宽）：
    canvas 显示高 = vh
    canvas 显示宽 = vh × 16 / 9
    → 左右留极小黑边

  如果实际比例 < 16/9（设备更窄）：
    canvas 显示宽 = vw
    canvas 显示高 = vw × 9 / 16
    → 上下留极小黑边

  如果实际比例 ≈ 16/9：
    → 完美铺满

═══ PC 端 ═══
  可用高度 = vh - 顶栏高度(64px) - 上下间距(各16px)
  可用宽度 = vw - 左右间距(各32px)

  canvas 宽 = min(可用宽度, 可用高度 × 16 / 9)
  canvas 高 = canvas 宽 × 9 / 16

  如果 canvas 高 > 可用高度：
    canvas 高 = 可用高度
    canvas 宽 = canvas 高 × 16 / 9

  → 居中显示，四周深色背景
```

### 5.6 游戏内 UI 布局框架（横屏 16:9 通用）

所有游戏模式共享的 UI 框架：

```
┌────────────────────────────────────────────────┐
│                                                │
│ ← 暂停    Bài 1-1    ⭐⭐☆        ❤️❤️❤️ 🪙+20 │ ← 顶栏 HUD
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━░░░░ 68%          │ ← 进度条
│                                                │
│ ┌──────────────────────────────────────────┐   │
│ │                                          │   │
│ │              游戏主内容区域                 │   │ ← 占 70% 高度
│ │          （按关卡/游戏类型定制）             │   │
│ │                                          │   │
│ └──────────────────────────────────────────┘   │
│                                                │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐          │ ← 底部操作区
│  │选项 A│ │选项 B│ │选项 C│ │选项 D│          │    占 20% 高度
│  └──────┘ └──────┘ └──────┘ └──────┘          │
│                                                │
└────────────────────────────────────────────────┘
```

横屏的优势：
- **更宽的操作区域**：选项可水平排列 4 个，减少滚动
- **更好的游戏画面**：赛车、射击等游戏天然适合横屏
- **左右布局**：教学内容可左图右文，信息密度更高
- **一致性**：H5 和 PC 游戏体验完全相同

## 六、横竖屏切换过渡动效

### 6.1 进入游戏过渡

```
点击「开始」按钮
  → 按钮 ripple 动效 (200ms)
  → 全屏渐暗 fade-out (300ms)
  → 触发横屏旋转（移动端）
  → 游戏加载骨架屏（横屏 16:9）
  → 游戏画面 fade-in (300ms)
  → 总过渡时间 ≤ 1.5 秒
```

### 6.2 退出游戏过渡

```
关卡结算画面
  → 用户点击「返回地图」
  → 游戏画面 fade-out (300ms)
  → 触发竖屏恢复（移动端）
  → 解除全屏
  → 世界地图 fade-in (300ms)
  → 总过渡时间 ≤ 1 秒
```

### 6.3 异常处理

| 场景 | 处理方式 |
|------|----------|
| 用户在游戏中手动旋转回竖屏 | 显示「请保持横屏游玩」提示覆盖层，暂停游戏 |
| iOS Safari 不支持横屏锁 | 显示旋转提示动画，监听 orientation change 事件 |
| 全屏 API 不可用 | 退化为普通网页横屏，隐藏不必要的 UI 元素 |
| 游戏加载超时（>5s） | 显示进度条 + "加载中" + 取消按钮返回地图 |

## 九、PWA 相关

### 9.1 PWA 安装体验

| 平台 | 行为 |
|------|------|
| Android Chrome | 浏览器底部提示「添加到主屏幕」 |
| iOS Safari | 引导弹窗教育用户「点击分享 → 添加到主屏幕」 |
| PC Chrome | 地址栏安装图标 |

### 9.2 PWA Manifest 配置

```json
{
  "display": "standalone",
  "orientation": "any",
  "theme_color": "#e11d48"
}
```

- `orientation: "any"`：允许应用在竖屏和横屏间切换
- 游戏模式通过 JavaScript `screen.orientation.lock('landscape')` 动态控制
- 非游戏页面不锁定方向，交由系统自适应

### 9.3 离线支持（MVP 最小化）

- 仅缓存 App Shell（HTML/CSS/JS 静态资源）
- 游戏内容需联网加载
- 离线时显示友好提示："Vui lòng kết nối mạng（请连接网络）"
- 会员用户已完成关卡可离线缓存复习（Service Worker + IndexedDB）
