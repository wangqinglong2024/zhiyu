# PlayLingo — MVP 产品需求文档总览

> **版本**: v1.0 | **日期**: 2026-04-15
> **产品形态**: Web 端（PWA）| 游戏模式全端横屏 16:9 + 非游戏模式响应式适配
> **首发市场**: 越南 🇻🇳
> **MVP 目标**: 1 个月完成开发，提供拼音→汉字→词汇→语法的完整 HSK1 学习链路

---

## 一、产品定位

**一句话描述**: 东南亚第一款以精致 2D 游戏为载体的中文学习 Web 应用——边玩游戏，边学中文。

**设计理念**: 面向 Z 世代和千禧一代年轻用户，视觉语言追求**潮流、时尚、活力**——融合 Gen-Z 喜爱的赛博霓虹色彩、流体渐变、弹性微动效和 glassmorphism 质感，打造一款"让人想截图分享到 Instagram/TikTok"的学习产品。

**核心差异化**:
- 不是"课程 + 练习"，而是"策略游戏 + 自然习得"
- 月费 $2.99 vs Duolingo $12.99，极致性价比
- 完整越南语本地化 + 印尼语/英语/中文 UI 支持，辐射整个东南亚
- 分享赚佣金机制，用户即渠道
- 时尚潮流视觉风格，年轻人一看就想下载

## 四、核心用户流程（Happy Path）

### 4.1 新用户首次体验流程

```
访问网站 → 落地页（品牌介绍 + CTA）
  → 点击「开始学习」
  → 注册页（Google / Facebook / 邮箱）
  → 注册成功
  → 选择学习目标（工作/旅游/考试/兴趣，单选）
  → 角色创建（选头像 + 输入昵称）
  → 进入世界地图（拼音群岛自动解锁）
  → 新手引导气泡（指引点击第 1 关）
  → 进入关卡 1-1（声母 b/p/m/f 教学）
  → 完成关卡 → 结算画面（星级 + XP + 金币）
  → 返回世界地图 → 第 2 关解锁
  → 引导尝试小游戏入口
```

### 4.2 日常学习流程（老用户）

```
打开 App/网页 → 世界地图
  → 查看待复习词汇提醒（红点）
  → 选择：
    A) 继续推进关卡 → 进入下一关 → 完成 → 结算
    B) 进入 SRS 复习 → 复习一组词汇 → 完成
    C) 玩小游戏 → 选择一款 → 开始 → 结束排行
  → 完成每日 1 关 → 打卡成功 → streak +1
  → 返回或退出
```

### 4.3 付费转化流程

```
用户推进到汉字谷地（第 2 地图区域）
  → 完成前 50% 免费关卡
  → 尝试进入付费关卡 → 弹出会员引导
  → 展示：免费版 vs 会员版对比表
  → 突出「首月 $0.99」优惠
  → 用户选择套餐（月付/季付/年付/终身）
  → 跳转 Stripe 支付页面
  → 支付成功 → 返回 App → 会员标识激活
  → 全部关卡解锁 + 去广告 + 离线可用
```

### 4.4 推广佣金流程

```
用户进入「推广中心」（个人主页入口）
  → 查看当前等级与佣金比例
  → 点击「复制推广链接」或「生成推广海报」
  → 分享到 Facebook / Zalo / TikTok
  → 好友点击链接 → 跳转注册页（?ref=uid 自动绑定）
  → 好友注册成功 → 推荐关系建立
  → 好友付费 → 推荐人获得佣金（30 天冷却后可提现）
  → 推荐人在推广中心查看收益明细
  → 累计 ≥ $10 → 申请提现
```

## 五、系统状态全局约定

### 5.1 每个页面必须覆盖的 7 种状态

| 状态 | 要求 |
|------|------|
| **空状态** | 插画 + 引导文案 + CTA 按钮 |
| **加载中** | Skeleton 骨架屏，禁止纯 Spinner |
| **首次加载** | 全屏骨架屏 + Logo 呼吸动画 |
| **成功** | 轻量 Toast（顶部滑入，2s 自动消失） |
| **错误** | 说人话的错误描述 + 重试按钮 |
| **部分加载** | 底部加载指示器 |
| **离线** | 顶部 Banner 提示 + 缓存数据可浏览 |

### 5.2 全局交互约定

- 所有操作 100ms 内必须有视觉反馈
- 破坏性操作（退出登录、删除账号）需二次确认弹窗
- 表单即时校验（blur 后），提交失败不清空已填内容
- Toast 通知从顶部滑入，成功绿色 2s/失败红色需手动关闭/信息蓝色 3s
- 页面转场：前进从右滑入，后退向右滑出，Tab 切换淡入淡出

### 5.3 广告规则（免费用户）

- 每完成 3 个关卡，关卡结算画面后插入 1 个插页广告（最长 15 秒，3 秒后可跳过）
- 非游戏页面底部固定 Banner 广告（高度 50px）
- 激励视频广告：用户主动触发（看广告获得金币 / Streak 修复）
- 会员用户：无任何广告

## 七、非功能需求（全局）

### 7.1 性能指标

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 首屏加载时间（LCP） | ≤ 3 秒（越南 4G 网络） | Lighthouse / WebPageTest |
| API 响应时间（P95） | ≤ 500ms | 后端 APM 监控 |
| 游戏画布帧率 | ≥ 30fps（移动端） / ≥ 60fps（PC） | Phaser 内置统计 |
| 页面交互响应 | ≤ 100ms 视觉反馈 | 用户感知测试 |
| Stripe 支付回调 | ≤ 3 秒激活会员 | Webhook 日志 |
| 数据库查询 | 单接口 ≤ 200ms | 慢查询日志 |

### 7.2 可用性与并发

| 指标 | 目标 |
|------|------|
| SLA | 99.9%（全年宕机 < 8.7 小时） |
| MVP 预期峰值 | 500 并发用户 |
| 数据库连接池 | 5-20 连接（asyncpg） |
| CDN 缓存命中率 | ≥ 90%（静态资源） |
| 离线可用 | PWA Service Worker 缓存 App Shell；会员用户缓存已完成关卡数据 |

### 7.3 安全与隐私基线

| 领域 | 要求 |
|------|------|
| **认证** | JWT 令牌（7 天有效期）+ Supabase Auth；密码 ≥ 8 位（含字母+数字+特殊字符） |
| **授权** | Supabase RLS 零信任：所有表启用行级安全策略 |
| **传输** | 全站 HTTPS（TLS 1.2+），HSTS 开启 |
| **支付** | PCI DSS 合规：不存储不处理卡号，全部由 Stripe 托管 |
| **XSS/CSRF** | React 自动转义 + CSP header；API 使用 CORS 白名单 + SameSite Cookie |
| **速率限制** | 登录接口：5 次/分钟（同 IP）；注册接口：3 次/分钟；API 通用：100 次/分钟 |
| **数据存储** | 用户数据存储在新加坡（腾讯云），不回传中国大陆 |
| **隐私合规** | 越南 PDPD 2023 + 新加坡 PDPA；AI 对话数据不用于模型训练 |
| **日志** | 记录关键操作日志（登录/支付/提现），保留 90 天，脱敏处理 PII |

### 7.4 法律文件（MVP 必备）

上线前必须准备以下法律文件（越南语 + 英语双版本）：

| 文件 | 内容要点 | 展示位置 |
|------|---------|---------|
| 服务条款 (Terms of Service) | 用户权利义务、账户管理、内容使用许可 | 注册页底部链接、设置页 |
| 隐私政策 (Privacy Policy) | 数据收集范围、存储地点、第三方共享、用户权利 | 注册页底部链接、设置页 |
| 订阅条款 (Subscription Terms) | 自动续费说明、取消政策、退款规则 | 支付页底部链接 |
| 推广协议 (Referral Agreement) | 佣金规则、反作弊条款、处罚机制 | 推广中心首次进入时弹窗确认 |

### 7.5 无障碍基线（WCAG 2.1 AA）

| 规则 | 要求 |
|------|------|
| 色彩对比度 | 正文 ≥ 4.5:1，大标题 ≥ 3:1 |
| 触控尺寸 | 可点击区域 ≥ 44×44px |
| 键盘导航 | PC 端所有核心操作可通过 Tab + Enter 完成 |
| 屏幕阅读器 | 装饰元素 `aria-hidden="true"`，交互元素有 `aria-label` |
| 动效偏好 | 支持 `prefers-reduced-motion`，可关闭所有动画 |
| 语义 HTML | 使用 `<button>`、`<nav>`、`<main>` 等语义标签 |

### 7.6 国际化（i18n）

| 项目 | 说明 |
|------|------|
| 默认语言 | 越南语（vi-VN） |
| 可切换语言 | 中文简体（zh-CN）、英语（en-US）、印尼语（id-ID） |
| 切换入口 | 落地页底部 + 设置页 → 语言切换 |
| 实现方式 | 前端 i18n JSON 文件（`/locales/vi.json`、`/locales/zh.json`、`/locales/en.json`、`/locales/id.json`） |
| 说明 | 中文 UI 主要用于开发测试和中文运营人员使用 |
| 翻译范围 | 所有 UI 文案、Toast 提示、错误信息、法律文件 |
| 不翻译 | 中文教学内容本身（汉字、拼音固定不变） |
| 日期格式 | 越南语：dd/MM/yyyy；英语：MM/dd/yyyy |
| 货币显示 | 主显 VND（越南盾），辅显 USD |

## 八、异常流与全局错误处理

### 8.1 网络异常

| 场景 | 处理 |
|------|------|
| 完全离线 | 顶部 Banner「Không có kết nối mạng」（无网络连接），缓存数据可浏览 |
| 弱网（超时） | 自动重试 1 次 → 失败后提示「Kết nối chậm，vui lòng thử lại」+ 重试按钮 |
| 恢复在线 | Banner 自动消失，静默刷新当前页数据 |
| 游戏中断线 | 暂停游戏 → 提示重连 → 重连后恢复进度（系统每答一题自动保存） |

### 8.2 支付异常

| 场景 | 处理 |
|------|------|
| 卡被拒绝 | Stripe 返回原因 → 翻译为越南语 → 提示换卡重试 |
| 支付超时 | 提示「请勿重复支付」→ 后台轮询 Stripe 状态 → 确认后激活 |
| Webhook 延迟 | 前端显示「正在确认支付...」→ 最多等 30 秒 → 超时提示联系客服 |
| 重复支付 | 幂等键（idempotency_key）防止重复扣款 |
| 退款 | Stripe Webhook 触发 → 立即降级 → 佣金扣回 |

### 8.3 账号异常

| 场景 | 处理 |
|------|------|
| Token 过期 | 静默刷新 → 失败后跳转登录页 → 登录后回到之前页面 |
| 多设备登录 | 允许（不互踢），数据取服务端最新版本 |
| 账号被封 | 登录时拦截 → 显示封禁原因 + 申诉入口 |
| 忘记密码 | 邮箱链接重置（Supabase Auth 内置） |
