> 路由路径：新建 `/admin/seasons/create` | 编辑 `/admin/seasons/{seasonId}/edit`
> 情绪目标：**严谨、完整** — 赛季配置涉及段位重置和奖励发放，必须逐一配置确认

## 三、赛季操作

### 3.1 手动开始赛季

#### 触发条件

- 赛季状态为"待开始"
- 点击列表操作列的"手动开始"按钮，或当前赛季卡片的操作按钮

#### 确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗标题** | "确认开始赛季" |
| **弹窗内容** | "确认立即开始「{赛季名称}」？" + 赛季起止日期展示 + 提示："开始后，赛季起始日期将更新为今天。" |
| **按钮** | "取消"（灰色描边）+ "确认开始"（成功色 `#22c55e` 实心） |
| **确认后操作** | ① 赛季状态 → "进行中" ② 起始日期更新为当天 ③ Toast"赛季已开始"（绿色）④ 列表刷新 |

### 3.2 手动结束赛季（高风险操作 — 双重确认）

#### 触发条件

- 赛季状态为"进行中"
- 点击列表操作列的"手动结束"按钮，或当前赛季卡片的"手动结束赛季"按钮

#### 第一次确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗标题** | "⚠️ 结束赛季 — 第 1 步确认" |
| **弹窗样式** | 弹窗顶部红色警告条 |
| **弹窗内容** | "您即将结束「{赛季名称}」。赛季结束后将自动执行以下操作，且**不可撤回**：" |
| **操作清单** | 编号列表：① 记录所有用户最终段位 ② 按配置向各段位用户发放赛季奖励（知语币 + 专属皮肤）③ 执行段位重置（所有用户段位按规则下调）④ 赛季限定皮肤全部自动下架 ⑤ 自动生成赛季报告 |
| **按钮** | "取消"（灰色描边）+ "我已了解，继续"（Amber 色实心） |

#### 第二次确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗标题** | "⚠️ 结束赛季 — 最终确认" |
| **弹窗内容** | "请输入赛季名称「{赛季名称}」以确认结束操作：" |
| **输入框** | 文本输入框，Placeholder 显示赛季名称 |
| **校验** | 输入内容必须与赛季名称完全一致（区分大小写），否则"确认结束"按钮保持 Disabled |
| **按钮** | "取消"（灰色描边）+ "确认结束赛季"（错误色 `#ef4444` 实心，默认 Disabled，输入正确后 Enabled） |

#### 确认后执行的操作链（按顺序）

```
管理员点击"确认结束赛季"
       ↓
  ① 记录所有用户最终段位
     └→ 写入 season_records 表（用户ID + 赛季ID + 最终段位 + 最终星数）
       ↓
  ② 按配置发放赛季奖励
     ├→ 遍历所有参与用户
     ├→ 按最终段位查找奖励配置
     ├→ 发放知语币（写入 coin_transactions，事由"赛季{名称}段位奖励"）
     └→ 发放专属皮肤（写入 user_skins，自动解锁）
       ↓
  ③ 执行段位重置
     ├→ 遍历所有用户
     ├→ 按重置规则表更新段位
     └→ 写入段位变更日志
       ↓
  ④ 赛季限定皮肤下架
     ├→ 查找关联当前赛季的所有赛季限定皮肤
     └→ 状态批量改为"已下架"
       ↓
  ⑤ 生成赛季报告
     └→ 自动统计并写入 season_reports 表
       ↓
  操作完成 → Toast"赛季已结束，报告已生成"（绿色）→ 列表刷新
```

#### 执行过程 UI

| 属性 | 说明 |
|------|------|
| **弹窗** | 第二次确认后，弹窗内容替换为进度展示 |
| **进度展示** | 5 个步骤纵向排列，每个步骤前有状态图标：待执行 = 灰色圆圈、执行中 = 蓝色 Loading 旋转、已完成 = 绿色 ✓、失败 = 红色 ✗ |
| **步骤文案** | ① "记录用户最终段位..."② "发放赛季奖励..."③ "执行段位重置..."④ "下架赛季限定皮肤..."⑤ "生成赛季报告..." |
| **完成后** | 全部 ✓ 后显示"赛季结束操作已全部完成"+ "查看赛季报告"按钮 + "关闭"按钮 |
| **失败处理** | 任一步骤失败 → 红色 ✗ + 错误信息 → 全部操作回滚 → 赛季状态恢复为"进行中" → 显示"操作失败，赛季未结束，请联系技术团队排查"+ "关闭"按钮 |

## 四、赛季报告

> 路由路径：`/admin/seasons/{seasonId}/report`
> 优先级：P1
> 条件：仅"已结束"状态的赛季可查看

### 4.1 页面布局

| 属性 | 说明 |
|------|------|
| **页面标题** | "赛季报告 — {赛季名称}"，H1 级别 |
| **返回** | 标题左侧 Lucide `ArrowLeft`，返回赛季列表 |
| **报告区域** | 毛玻璃面板，最大宽度 1000px，水平居中 |

### 4.2 报告内容

#### 4.2.1 基本信息区

| 字段 | 说明 |
|------|------|
| 赛季名称 | 文本 |
| 赛季编号 | 如 S001 |
| 起止日期 | YYYY-MM-DD ~ YYYY-MM-DD |
| 实际持续天数 | X 天 |

#### 4.2.2 核心指标卡片（4 个横排）

| 卡片 | 指标 | 展示 |
|------|------|------|
| 卡片 1 | 参与用户数 | 数字（至少在赛季期间完成 1 场 PK 的用户） |
| 卡片 2 | 赛季期间总对战场次 | 数字 |
| 卡片 3 | 知语币发放总量 | "X 知语币（≈$X.XX）" |
| 卡片 4 | 最活跃游戏 | 游戏名称 + 对战场次 |

卡片样式：毛玻璃面板，圆角 `rounded-2xl`，内边距 24px。

#### 4.2.3 各段位人数分布

| 属性 | 说明 |
|------|------|
| **图表类型** | 水平柱状图 |
| **X 轴** | 人数 |
| **Y 轴** | 7 个大段（青铜 ~ 王者），使用段位图标 + 名称 |
| **柱状颜色** | 青铜 = `#a3a3a3`、白银 = `#c0c0c0`、黄金 = `#fde68a`、铂金 = `#7dd3fc`、钻石 = `#a78bfa`、星耀 = `#fbbf24`、王者 = `#e11d48` |
| **数据标注** | 每根柱状右侧显示人数 + 百分比 |

#### 4.2.4 最活跃游戏排行

| # | 列名 | 说明 |
|---|------|------|
| 1 | 排名 | 按对战场次降序 |
| 2 | 游戏名称 | 如"G1 汉字切切切" |
| 3 | 对战场次 | 数字 |
| 4 | 参与人数 | 数字 |

#### 4.2.5 段位最高 Top 10 用户

| # | 列名 | 说明 |
|---|------|------|
| 1 | 排名 | 1-10 |
| 2 | 用户信息 | 头像 + 昵称（昵称可点击跳转用户详情） |
| 3 | 最终段位 | 段位图标 + 名称 + 星数 |
| 4 | 总对战场次 | 数字 |
| 5 | 总胜率 | 百分比 |

## 五、状态矩阵

### 5.1 赛季列表页

| 状态 | 表现 |
|------|------|
| **空状态 (Empty)** | 表格区域居中显示 Lucide `Trophy` 48px 灰色图标 + "暂无赛季"文案 + "新建赛季"Rose 色 CTA 按钮 |

| 状态 | 表现 |
|------|------|

| **加载中 (Loading)** | 编辑时，表单 Skeleton 骨架屏 |

| **离线 (Offline)** | 顶部 Banner + 保存按钮 Disabled |

### 5.3 赛季报告页

| 状态 | 表现 |
|------|------|
| **空状态 (Empty)** | 不适用（已结束赛季必定有报告数据） |
| **加载中 (Loading)** | 报告区域 Skeleton 骨架屏（模拟卡片 + 图表 + 表格形状） |
| **首次加载 (First Load)** | 同上 |
| **成功 (Success)** | 正常展示报告数据 |
| **错误 (Error)** | "报告加载失败，请重试"+ "重新加载"按钮 |
| **部分加载 (Partial)** | 不适用 |
| **离线 (Offline)** | 顶部 Banner 提示 |
