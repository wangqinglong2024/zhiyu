## 二、数据流向说明

### 2.1 皮肤管理数据流

```
管理后台创建皮肤
       ↓
  皮肤表 skins（写入基础信息+资源文件 URL）
       ↓
  资源文件 → 对象存储（Supabase Storage）
       ↓
  上架操作 → 皮肤状态 = "已上架"
       ↓                        ↓
  用户端皮肤商城（展示可购买皮肤）  管理后台皮肤列表
       ↓
  用户购买（消耗知语币）
       ↓
  知语币流水表 coin_transactions
       ↓
  用户皮肤表 user_skins（写入拥有记录）
       ↓
  游戏加载时读取用户皮肤设置
```

### 2.2 赛季管理数据流

```
管理后台创建赛季
       ↓
  赛季表 seasons（写入赛季信息+段位重置规则+奖励配置）
       ↓
  手动/自动开始赛季
       ↓
  用户端赛季展示（排行榜、段位信息、赛季限定皮肤）
       ↓
  赛季结束（手动触发 5 步链式操作）
  ┌────────────────────────────────────────────┐
  │  ① 记录最终段位 → 赛季段位快照表              │
  │  ② 发放赛季奖励 → 知语币流水 + 用户皮肤表     │
  │  ③ 重置段位 → 用户段位表按规则降段            │
  │  ④ 下架限定皮肤 → 皮肤状态 = "已下架"        │
  │  ⑤ 生成赛季报告 → 赛季报告表                  │
  └────────────────────────────────────────────┘
       ↓
  操作日志 admin_logs（记录每步执行结果）
```

### 2.3 推送管理数据流

```
管理后台创建推送
       ↓
  推送表 push_messages（写入推送内容+目标条件+计划时间）
       ↓
  立即发送 / 定时任务触发
       ↓
  后端根据目标条件查询匹配用户
       ↓
  推送服务（Firebase Cloud Messaging / APNs）
       ↓
  ├── 推送送达 → 更新送达统计
  ├── 用户打开推送 → 更新打开统计
  └── 推送失败 → 记录失败原因
       ↓
  推送统计表 push_stats（实时更新效果数据）
       ↓
  管理后台推送详情页（展示效果数据）
```

### 2.4 多语言管理数据流

```
管理后台编辑/导入翻译
       ↓
  翻译草稿表 i18n_drafts（状态 = "已修改未发布"）
       ↓
  管理员点击"发布"
       ↓
  ┌────────────────────────────────────────────┐
  │  ① 生成新版本号                              │
  │  ② 翻译数据同步到正式表 i18n_published       │
  │  ③ 生成语言包 JSON → CDN 分发               │
  │  ④ 发布记录写入 i18n_history                 │
  │  ⑤ 操作日志写入 admin_logs                   │
  └────────────────────────────────────────────┘
       ↓
  应用端下次打开 → 拉取最新语言包版本 → 按需下载
```

### 2.5 管理员管理数据流

```
超级管理员创建新管理员
       ↓
  管理员表 admins（写入姓名+邮箱+角色+初始密码哈希）
       ↓
  系统发送创建通知邮件（含初始密码）
       ↓
  新管理员首次登录 → 强制修改密码
       ↓
  管理员日常操作 → 所有写操作写入 admin_logs
       ↓
  停用/删除管理员 → 会话立即失效
       ↓
  删除后：管理员记录硬删除，admin_logs 中操作人字段标记为"已删除管理员-{原姓名}"
```

## 三、非功能需求

### 3.1 性能要求

| 页面/操作 | 指标 | 目标值 |
|----------|------|--------|
| 皮肤列表页首屏加载 | 首屏渲染时间 | < 2s |
| 赛季列表页首屏加载 | 首屏渲染时间 | < 1.5s |
| 赛季结束 5 步链式操作 | 全部执行完成 | < 30s（含数据库事务+知语币发放+段位重置） |
| 赛季结束 — 段位重置（10,000 用户） | 批量更新时间 | < 10s |
| 赛季结束 — 奖励发放（10,000 用户） | 知语币批量入账 | < 15s |
| 排行榜列表加载 | 首屏渲染时间 | < 2s |
| 排行榜用户侧边栏加载 | 抽屉内容渲染 | < 1s |
| 推送发送（10,000 目标用户） | 推送队列入队 | < 5s |
| 推送发送（50,000 目标用户） | 推送队列入队 | < 15s |
| 推送目标预估人数 | 实时计算返回 | < 2s |
| 多语言列表页加载 | 首屏渲染时间 | < 2s |
| 多语言发布 | 版本生成+CDN同步 | < 10s |
| 翻译文件导入（5,000 条） | 解析+写入 | < 10s |
| 翻译文件导出 | 文件生成时间 | < 5s |
| 管理员列表页加载 | 首屏渲染时间 | < 1s（管理员数量极少） |
| 操作日志列表加载 | 首屏渲染时间 | < 2s |
| 操作日志搜索/筛选 | 结果响应时间 | < 1.5s |
| 操作日志导出（50,000 条） | 文件生成时间 | < 30s |

### 3.2 并发要求

| 场景 | 预期并发量 |
|------|----------|
| 管理后台同时在线管理员 | 10-20 人 |
| 皮肤管理同时操作 | 5 并发 |
| 赛季结束操作 | 1 并发（同一时间仅允许一个赛季结束操作，加分布式锁） |
| 推送创建和发送 | 3 并发 |
| 翻译发布操作 | 1 并发（加锁防止并发发布导致版本冲突） |
| 日志查询 | 10 并发 |

### 3.3 赛季结束事务一致性

| 规则 | 说明 |
|------|------|
| **原子性** | 5 步链式操作中任一步骤失败，已执行步骤全部回滚 |
| **幂等性** | 赛季结束操作加分布式锁，同一赛季不可同时或重复触发结束；结束按钮点击后立即 Disabled + Loading |
| **进度持久化** | 每步完成后持久化进度状态，异常中断后可从断点恢复（而非从头重试） |
| **超时保护** | 单步操作超时 60s 未完成视为失败，触发全量回滚 |
| **补偿机制** | 回滚顺序为执行的逆序：⑤ → ④ → ③ → ② → ①；回滚失败触发告警通知（邮件 + 日志）给所有超级管理员 |

### 3.4 推送服务可靠性

| 规则 | 说明 |
|------|------|
| **消息队列** | 推送任务入队后异步处理，不阻塞管理后台操作 |
| **重试机制** | 推送失败自动重试 3 次，间隔 1min/5min/15min |
| **降级策略** | 推送服务不可用时，管理员操作正常返回 + Amber 色提示"推送服务暂时不可用，将在恢复后自动发送" |
| **送达率监控** | 送达率低于 80% 时，推送详情页展示 Amber 色警告"送达率异常偏低" |

### 3.5 操作审计安全

| 规则 | 说明 |
|------|------|
| **完整性** | 所有管理员写操作必须记录至 admin_logs，包含操作人+操作类型+操作时间+目标对象+IP+变更前后值 |
| **不可篡改** | 日志一经写入不可修改/删除，数据库层面设置 INSERT-ONLY 权限 |
| **防遗漏** | 后端通过 AOP/中间件统一拦截写操作，确保日志写入不依赖各业务模块单独调用 |
| **日志与业务分离** | 日志写入失败不阻塞业务操作，但需触发告警 |

### 3.6 数据安全

| 规则 | 说明 |
|------|------|
| **权限校验** | 每个 API 请求校验管理员角色权限，越权请求返回 403 |
| **管理员密码安全** | 密码使用 bcrypt 哈希存储（cost factor ≥ 12）；初始密码和重置密码一次性展示，不存明文 |
| **敏感操作二次确认** | 赛季结束（二次确认）、管理员删除（二次确认）、段位重置、游戏数据清除必须二次确认弹窗 |
| **日志导出审计** | 每次导出操作本身也记录到系统日志 |
| **会话安全** | 管理员会话超时 2 小时自动退出；停用/删除管理员后会话立即失效 |

## 五、验收标准汇总

> 覆盖本模块全部 7 个功能页面。

### 5.1 皮肤管理

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 皮肤列表 | 4 个筛选条件任意组合 → 列表实时刷新 → 分页正常（20 条/页） |
| 2 | 创建皮肤 | 16 个字段全部填写 → 校验通过 → 创建成功 → 列表可查 |
| 3 | 三语信息 | 中/英/越三语名称和描述均正确展示，Tooltip 展示完整内容 |
| 4 | 资源上传 | 缩略图/动画预览/精灵图上传正常 → 格式限制 → 大小限制 → 预览正确 |
| 5 | 价格设置 | 知语币价格 ≥ 1 → 原价可选 → 列表正确展示 |
| 6 | 赛季限定 | 勾选赛季限定 → 必须关联赛季 → 赛季结束后自动下架 |
| 7 | 上架/下架 | 上架后用户端可见 → 下架后用户端不可见 → 已购用户保留使用权 |
| 8 | 销售数据 | 销售总量/总收入/退款数 3 项统计准确（P1） |
| 9 | 7 种状态 | 空/加载/首次加载/成功/错误/部分加载/离线 7 种状态表现正确 |

### 5.2 赛季管理

| # | 验收项 | 验收条件 |
|---|--------|---------|
| 1 | 赛季列表 | 当前赛季高亮卡片 + 进度条正确 → 历史赛季按时间倒序 |
| 2 | 创建赛季 | 名称+日期+段位重置规则+奖励配置 → 校验通过 → 创建成功 |
| 3 | 段位重置规则 | 7 个段位各自配置降至目标 → 目标段位 ≤ 当前段位 |
