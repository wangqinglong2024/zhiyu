## 二、异常数据处理

### 2.1 标记可疑用户

#### 触发条件

段位异常快速提升的用户特征（仅供运营人员参考，系统不自动标记）：
- 短时间内连续大量胜场（如 1 小时内 30 场以上且胜率 > 95%）
- 单局答题速度异常快（平均 < 1 秒/题）
- 段位提升速度远超正常水平

#### 标记确认弹窗

| 属性 | 说明 |
|------|------|
| **弹窗标题** | "标记可疑用户" |
| **弹窗内容** | "确认将用户「{昵称}」(ID: {userId}) 标记为可疑？" + 标记原因输入框 |
| **原因输入** | 多行文本框，Placeholder "请描述可疑原因（如：1 小时内胜场 50 场，胜率 100%）"，必填，10-500 字符 |
| **按钮** | "取消"（灰色描边）+ "确认标记"（Amber 色实心） |
| **确认后** | 用户排行榜标记为"⚠ 可疑"→ 写入系统操作日志 → Toast"已标记为可疑"（Amber 色） |

### 2.2 异常处理弹窗

#### 触发条件

- 用户已被标记为"可疑"
- 点击操作列"处理"按钮

#### 弹窗内容

| 属性 | 说明 |
|------|------|
| **弹窗标题** | "处理异常用户" |
| **用户信息** | 头像 + 昵称 + 用户 ID + 当前段位 + 标记原因 |
| **处理方式** | 单选组 |

**处理方式选项**：

| 选项 | 说明 | 二次确认 |
|------|------|---------|
| **重置段位** | 将用户段位重置为青铜 III（最低段位 1 星）；用户的皮肤、知语币不受影响 | 需填写处理原因 |
| **清除游戏数据** | 清除该用户全部对战记录 + 重置段位为青铜 III；等同于游戏数据全部清零 | 需填写处理原因 + 二次确认"此操作不可恢复" |
| **取消标记** | 经核实非异常，取消可疑标记 | 无 |

#### 处理原因输入

| 属性 | 说明 |
|------|------|
| **组件** | 多行文本框，Placeholder "请填写处理原因" |
| **校验** | 选择"重置段位"或"清除游戏数据"时必填，10-500 字符 |

#### 选择"清除游戏数据"时的二次确认

| 属性 | 说明 |
|------|------|
| **确认文案** | "⚠️ 此操作将清除用户「{昵称}」的全部游戏对战记录并重置段位为青铜 III，此操作**不可恢复**。确认执行？" |
| **按钮** | "取消"（灰色描边）+ "确认清除"（错误色 `#ef4444` 实心） |

#### 确认后

| 处理方式 | 执行操作 |
|---------|---------|
| 重置段位 | 用户段位 → 青铜 III（1 星）→ 写入段位变更日志（原因"管理员重置"）→ 写入系统操作日志 → 排行榜标记更新为"已处理" → Toast"段位已重置"（绿色） |
| 清除游戏数据 | 用户段位 → 青铜 III（1 星）→ 清除全部对战记录 → 写入系统操作日志（含处理原因）→ 排行榜标记更新为"已处理" → Toast"游戏数据已清除"（绿色） |
| 取消标记 | 移除可疑标记 → 写入系统操作日志 → Toast"已取消标记"（绿色） |

## 三、状态矩阵

### 3.1 排行榜查看页

| 状态 | 表现 |
|------|------|
| **空状态 (Empty)** | 表格区域居中显示 Lucide `BarChart3` 48px 灰色图标 + "暂无排行数据"文案 + "当前赛季尚未开始或无用户参与"灰色说明 |
| **加载中 (Loading)** | 表格区域显示 Skeleton 骨架屏（15 行），模拟排行榜行形状 |
| **首次加载 (First Load)** | 全表格 Skeleton 骨架屏 + 品牌 Logo 呼吸动画 |
| **成功 (Success)** | 正常展示排行数据；Top 3 高亮；操作成功后顶部 Toast 通知 |
| **错误 (Error)** | "排行榜加载失败，请重试"+ Lucide `AlertCircle` 图标 + "重新加载"按钮 |
| **部分加载 (Partial)** | 翻页时底部加载指示器 |
| **离线 (Offline)** | 页面顶部 Amber 色 Banner："网络连接中断，排行数据可能不是最新" |
