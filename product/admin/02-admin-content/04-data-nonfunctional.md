## 一、数据流向说明

### 1.1 文章数据流

```
[Dify 工作流]                         [管理后台]                          [应用端]
    │                                     │                                 │
    ├── AI 生成初稿 ──────────────→ 草稿（自动写入）                        │
    │                                     │                                 │
    │                               运营编辑/审核/润色                       │
    │                                     │                                 │
    │                               [上架发布] ──────────────────────→ 用户可见
    │                                     │                                 │
    │                               [下架] ──────────────────────────→ 用户不可见
    │                                     │                                 │
    │                                     │                          用户浏览产生数据
    │                                     │                                 │
    │                               浏览量回写 ←────────────────────── 浏览量统计
```

**关键节点**：
- Dify 工作流生成的文章自动写入管理后台，状态为「草稿」，来源标记为「Dify」
- 运营人员编辑、审核后上架发布，应用端实时可见
- 应用端用户浏览数据回写至管理后台，体现在列表页「浏览量」列
- 文章内容存储于 Supabase 数据库，多语言内容按字段分别存储（text_zh / text_pinyin / text_en / text_vi）
- 封面图和音频文件存储于 Supabase Storage

### 1.2 课程数据流

```
[Dify 工作流]                         [管理后台]                          [应用端]
    │                                     │                                 │
    ├── AI 生成课程初稿 ──────────→ Level/Unit/Lesson 结构                  │
    │                                     │                                 │
    ├── AI 生成题目 ──────────────→ 题目库                                  │
    │                                     │                                 │
    │                               运营编辑/审核                            │
    │                                     │                                 │
    │                               [Level 上线] ─────────────────────→ 用户可见
    │                                     │                                 │
    │                                     │                          学习进度数据
    │                                     │                          考核成绩数据
    │                                     │                                 │
    │                               学习数据回写 ←───────────────────  SRS 复习队列
```

**关键节点**：
- 课程结构（Level→Unit→Lesson）由运营在管理后台手动创建或 Dify 批量导入
- 题目数据分两类：课程题目（绑定 Lesson）和评测题目（独立题库）
- Level 上线后，该级别所有课程内容在应用端可见
- 用户学习进度、考核成绩、SRS 复习队列数据回写至 Supabase
- 音频文件由管理员手动上传至 Supabase Storage

### 1.3 金句数据流

```
[管理后台]                                              [应用端]
    │                                                      │
    运营创建金句 + 排期                                      │
    │                                                      │
    [排期日 00:00 UTC+7] ── 定时任务自动发布 ──────────→ 首页金句展示
    │                                                      │
    │                                                 用户点击分享
    │                                                      │
    │                                                 生成分享图片
    │                                              （金句+背景图+品牌水印+二维码）
```

**关键节点**：
- 金句由运营人员在管理后台手动创建和排期
- 定时任务在排期日 00:00 UTC+7 自动将金句状态变为「已发布」
- 应用端首页展示当日已发布金句（若当日无排期，复用上一条已发布金句）
- 用户可将金句生成为分享图片（含背景图 + 品牌水印 + 二维码），背景图来源于管理后台上传或系统默认

## 二、非功能需求

### 2.1 富文本编辑器性能

| 指标 | 要求 | 说明 |
|------|------|------|
| 编辑器初始化时间 | ≤ 1.5s | 包括编辑器框架加载 + 工具栏渲染 |
| 输入响应延迟 | ≤ 50ms | 用户输入文字到屏幕显示的延迟 |
| 拼音标注弹出延迟 | ≤ 300ms | 选中文字→点击「拼」按钮→浮层弹出的时间 |
| 自动拼音生成时间 | ≤ 2s（1000 字以内） | 全文自动标注的响应时间 |
| 编辑器最大支持内容长度 | 5000 字 | 超过限制时提示「内容已超过最大长度」 |
| 自动保存 | 每 30s 自动保存草稿 | 在编辑器内容发生变化后 30s 自动保存一次；页面底部显示「自动保存于 HH:mm:ss」灰色小字 |
| 编辑器离开保护 | 有未保存修改时浏览器 beforeunload 确认 | 防止意外关闭浏览器丢失内容 |

### 2.2 图片上传

| 指标 | 要求 | 说明 |
|------|------|------|
| 支持格式 | JPG / PNG / WebP / GIF（GIF 仅限正文内插图） | — |
| 单张大小限制 | 封面图 ≤ 2MB；正文插图 ≤ 5MB；金句背景图 ≤ 2MB | 超限时提示「图片大小超出限制（最大 XMB），请压缩后重新上传」 |
| 封面图尺寸 | 建议 16:9 比例，最小 800×450px | 低于最小尺寸时提示「图片尺寸过小，建议使用至少 800×450px 的图片」 |
| 上传进度 | 显示进度条 + 百分比 | — |
| 上传超时 | 30s | 超时后提示「上传超时，请检查网络后重试」 |
| 并发上传 | 最多同时上传 3 张图片 | — |
| 图片裁剪 | 封面图支持裁剪（16:9 比例锁定） | 裁剪完成后上传裁剪后的图片 |

### 2.3 音频上传

| 指标 | 要求 | 说明 |
|------|------|------|
| 支持格式 | MP3 / WAV | 其他格式提示「不支持的音频格式，仅支持 MP3 和 WAV」 |
| 单文件大小限制 | 正文内嵌音频 ≤ 20MB；课时朗读音频 ≤ 50MB | 超限时提示「音频文件过大（最大 XMB），请压缩后重新上传」 |
| 上传进度 | 显示进度条 + 百分比 + 已上传大小/总大小 | — |
| 上传超时 | 60s | — |
| 试听功能 | 上传后可在管理后台内播放试听 | 播放/暂停按钮 + 进度条 + 音频时长 |
| 音频来源 | 管理员手动上传 | 非 TTS 生成，所有音频由管理员上传真人录制的音频文件 |

### 2.4 列表页性能

| 指标 | 要求 | 说明 |
|------|------|------|
| 列表首次加载 | ≤ 1s（20 条/页） | 从点击菜单到列表渲染完成 |
| 筛选响应时间 | ≤ 500ms | 从筛选条件变更到列表刷新完成 |
| 翻页响应时间 | ≤ 500ms | 从点击页码到新一页渲染完成 |
| 排序响应时间 | ≤ 500ms | 从点击列头到排序完成 |
| 拖拽排序保存 | ≤ 300ms | 从松手到排序保存成功 |

### 2.5 日历视图性能

| 指标 | 要求 | 说明 |
|------|------|------|
| 月历加载时间 | ≤ 800ms | 从切换月份到日历格子渲染完成 |
| 格子点击响应 | ≤ 200ms | 从点击到浮层弹出或页面跳转 |

### 2.6 数据安全

| 要求 | 说明 |
|------|------|
| 操作日志 | 所有内容的创建、编辑、上架、下架、删除操作记录到系统日志（操作人 + 时间 + 操作类型 + 操作对象） |
| 权限控制 | 仅「超级管理员」和「内容运营」角色可访问内容管理模块；其他角色访问时返回「无权限」提示页 |
| 内容版本 | 文章和课程内容的每次编辑保存生成新的版本快照（至少保留最近 20 个版本），支持版本回溯查看（P1 功能） |
| 批量操作限制 | 单次批量操作最多 100 条，超出时提示「单次批量操作最多 100 条，请分批处理」 |

### 2.7 无障碍设计

| 要求 | 说明 |
|------|------|
| 键盘导航 | 所有核心操作（筛选、翻页、编辑、保存）可通过 Tab/Enter 完成 |
| 色彩对比度 | 状态标签文字与背景对比度 ≥ 4.5:1 |
| 屏幕阅读器 | 表格有 `aria-label`；操作按钮有 `aria-label`；图片上传区域有 `aria-label` |
