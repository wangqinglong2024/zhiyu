# 管理端模块：提现管理

> 依赖全局文件：`product/GLOBAL.md`
> 涉及 API：`GET /admin/withdrawals`, `POST .../approve`, `POST .../reject`, `POST .../confirm-paid`

---

## 一、路由

`/admin/withdrawals`

**这是最重要的管理模块。用户申请提现后如果长时间没有处理，会直接影响口碑和裂变意愿。**

---

## 二、提现申请列表

### 表格字段

| 字段 | 说明 |
|------|------|
| 申请时间 | MM-DD HH:mm |
| 申请人 | 手机号脱敏 |
| 申请金额（税前） | 用户申请的原始金额 |
| 预计打款金额（税后） | 税后实际应打款金额 |
| 收款方式 | 微信 / 支付宝 |
| 收款账号 | 明文展示（用于打款） |
| 收款人真实姓名 | 明文展示 |
| 状态 | 待审核 / 已通过 / 已拒绝 / 已打款 |
| 操作 | 通过 / 拒绝 |

**默认只展示「待审核」，顶部 Tab 切换其他状态。**

---

## 三、税务处理说明（MVP 简化方案）

- 佣金金额 ≤ ¥800/月：按劳务报酬免税处理，打款全额
- 佣金金额 > ¥800/月：管理员手动计算税额，从打款金额中扣除
- 后期用户量大后再对接财税平台（如易代扣）

---

## 四、审核操作

### 通过
1. 点击「通过」
2. 可选填写备注
3. 状态 → `approved`（等待手动打款）

### 拒绝
1. 点击「拒绝」
2. **必填**拒绝原因（下拉选项 + 自定义输入）：
   - 收款账号信息有误，请重新提交
   - 账号存在异常，暂时冻结
   - 其他（手动填写）
3. 用户可提现余额原路返还（`wallets.balance += amount`）
4. 用户收到系统消息通知

### 确认打款
1. 管理员在微信/支付宝后台完成打款后，回到系统点击「确认已打款」
2. `UPDATE wallets SET total_withdrawn += amount`（balance 在申请时已冻结扣除，此处不重复操作）
3. 状态 → `paid`

---

## 五、打款方式（MVP 全部手动）

管理员登录微信商户后台或支付宝商家后台，手动发起企业付款到零钱/银行卡。打款后在系统中点击「确认已打款」。
