## 一、后台概述

### 1.1 产品定位

管理后台是运营人员日常操作的核心工具，职责：
1. 审核/编辑 Dify 工作流自动生成的内容素材
2. 控制内容上架/下架（类似商品管理）
3. 管理用户、订单、知语币
4. 监控运营数据

### 1.2 访问方式

- Web 端桌面优先（1280px 起）
- 登录方式：管理员账号密码 + 二次验证
- UI 语言：中文/英文切换

### 1.3 角色权限

| 角色 | 可访问模块 |
|------|-----------|
| 超级管理员 | 全部模块 + 系统设置 + 角色管理 |
| 内容运营 | 数据看板（只读）+ 内容管理（发现中国、课程、金句） |
| 用户运营 | 数据看板（只读）+ 用户管理 + 订单管理 |
| 游戏运营 | 数据看板（只读）+ 游戏管理（皮肤、赛季） |

## 二、数据看板

### 2.1 核心指标卡片

| 指标 | 说明 | 展示方式 |
|------|------|---------|
| DAU / MAU | 日活/月活用户数 | 数字+趋势折线 |
| 新注册用户 | 当日/本周/本月 | 数字+趋势 |
| 付费转化率 | 免费用户→付费用户 | 百分比+对比 |
| 课程收入 | 当日/本周/本月 | 金额+趋势 |
| 游戏收入 | 皮肤收入 | 金额+趋势 |
| 在线用户数 | 当前实时 | 数字 |

### 2.2 图表

- 用户增长趋势图（日/周/月维度）
- 收入趋势图（课程/游戏分开）
- 各课程级别购买分布（柱状图）
- 各游戏活跃度排行
- 用户留存漏斗（D1/D7/D30）
- 地区分布（越南→印尼→其他）

### 2.3 快捷操作

- 待审核内容数量提醒（点击跳转至内容管理）
- 待处理退款数量提醒（点击跳转至订单管理）
- 被举报用户提醒（点击跳转至用户管理）

### 3.2 课程内容管理

#### 3.2.1 课程结构层级

```
Level（12 级）
  └── Unit（若干单元）
        └── Lesson（课时）
              ├── 教学内容（多语言文本+拼音+音频+图片）
              ├── 课时小测验（3-5 题）
              └── 课时练习
        └── 单元阶段测评（10-15 题，70 分通过）
  └── 级别综合考核（85 分通过，单项不低于 60）
```

#### 3.2.2 Level 管理

| 字段 | 说明 |
|------|------|
| Level 编号 | L1-L12 |
| Level 名称（多语言） | 如"启蒙之旅" |
| 对标体系 | K-12 年级 + HSK/CEFR 等级 |
| 定价 | L1-L3 = 免费 / L4-L12 = $6 |
| 状态 | 开发中 / 已上线 / 已下线 |
| 课程大纲 | 该 Level 包含的单元列表 |

#### 3.2.3 课时内容编辑

| 字段 | 类型 | 说明 |
|------|------|------|
| 课时标题（多语言） | 文本 | 必填 |
| 知识模块 | 标签 | M1-M9 对应 |
| 教学正文 | 富文本（多语言） | 中文+拼音+英文+越南语 |
| 教学音频 | 上传 | 标准发音 |
| 教学图片/插图 | 图片 | 可选 |
| 重点词汇列表 | 结构化数据 | 汉字+拼音+释义+例句 |
| 语法点 | 结构化数据 | 语法规则+例句 |

#### 3.2.4 题目管理

| 题型 | 所属 |
|------|------|
| 选择题（单选/多选） | 课时小测验 / 单元测评 / 综合考核 |
| 听力选择 | 单元测评 / 综合考核 |
| 拼音标注 | 课时小测验 / 单元测评 |
| 排序组句 | 课时小测验 / 单元测评 / 综合考核 |
| 填空题 | 单元测评 / 综合考核 |
| 阅读理解 | 综合考核 |
| 看图说话（简答） | 综合考核 |

每道题字段：
- 题干（多语言）
- 选项/答案区
- 正确答案
- 解析（多语言）
- 难度标签
- 关联知识点（SRS 用）

#### 3.2.5 考核配置

| 考核层级 | 可配置项 |
|---------|---------|
| 课时小测验 | 题目数量（默认 3-5）、出题范围 |
| 单元阶段测评 | 题目数量（默认 10-15）、通过分数（默认 70）、出题范围 |
| 级别综合考核 | 题目数量、通过分数（默认 85）、单项最低分（默认 60）、重考间隔（默认 24h）|

### 3.3 每日金句管理

| 字段 | 说明 |
|------|------|
| 金句内容（多语言） | 中文+拼音+英文+越南语 |
| 出处/作者 | 文本 |
| 展示日期 | 排期日历 |
| 背景图 | 用于分享图片生成 |
| 状态 | 待发布 / 已发布 / 已过期 |

- 查看任意用户知语币余额和完整流水

## 五、订单管理

### 5.1 订单列表

| 列 | 说明 |
|----|------|
| 订单号 | Paddle 订单 ID |
| 用户 | 用户昵称+ID |
| 商品 | 课程级别 / 皮肤名称 |
| 金额 | USD |
| 支付方式 | Paddle (Web) / Apple IAP |
| 状态 | 待支付 / 已支付 / 已退款 / 已取消 |
| 支付时间 | — |
| 操作 | 查看详情 / 退款 |

- 支持按商品类型、支付状态、时间范围筛选

### 5.2 退款处理

- 退款需审核（内容运营无权操作退款）
- 退款后自动：
  - 回收对应课程权限
  - 扣除关联推荐奖励知语币（可至负数）
  - 推送退款通知给用户
  - 推送知语币扣除通知给推荐人
- 退款审计日志完整记录

### 5.3 对账

- Paddle Webhook 自动同步订单状态
- 月度对账报表（Paddle 入账 vs 系统订单）
- 差异告警

## 八、全局交互规范

### 8.1 列表页通用规范

- 每页默认 20 条，支持翻页
- 支持列排序（点击列头切换升序/降序）
- 操作前需二次确认（删除/封禁/退款/下架等不可逆操作）
- 批量操作需显示影响数量确认

### 8.2 表单规范

- 必填字段标红星
- 实时校验 + 提交校验
- 保存成功/失败 Toast 提示
- 离开未保存表单时弹窗提醒

### 8.3 设计规范

- 遵循 Cosmic Refraction 设计系统
- 桌面端优先（1280px 起），侧边导航布局
- 支持明暗模式切换
- 颜色体系：Rose / Sky / Amber
