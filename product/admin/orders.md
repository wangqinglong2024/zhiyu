# 管理端模块：订单管理

> 依赖全局文件：`product/GLOBAL.md`
> 涉及 API：`GET /admin/orders`, `GET /admin/orders/{id}`, `POST /admin/orders/{id}/refund`

---

## 一、路由

`/admin/orders`

---

## 二、订单列表

### 表格字段

| 字段 | 说明 |
|------|------|
| 订单号 | order id 前8位（可点击查看详情） |
| 用户 | 手机号脱敏（138****8888） |
| 类目 | 职场困局 / 情感困局 |
| 金额 | ¥28.8 |
| 状态 | 待支付 / 已支付 / 生成中 / 已完成 / 生成失败 / 已退款 |
| 支付时间 | MM-DD HH:mm |
| 操作 | 查看详情 / 退款（仅已完成状态可退款） |

### 筛选项
- 日期范围（默认今天）
- 订单状态
- 搜索手机号

**分页：** 每页 20 条，后端分页。

---

## 三、订单详情（弹窗或抽屉）

点击订单行展开：
- 用户输入的原始困境（完整文本）
- AI 生成的报告（格式化展示）
- 支付流水号（微信/支付宝返回的）
- 是否有邀请来源（展示邀请人手机号脱敏）

---

## 四、退款操作

### 流程
1. 点击「退款」 → 弹窗确认：「确认对订单 XXXX 执行退款 ¥28.8 吗？」
2. 确认后：
   - 在微信/支付宝商家后台手动原路退款
   - 管理后台将订单状态改为 `refunded`
   - 若该订单产生过佣金，手动在备注中记录，由运营人工判断是否从 wallets 扣除

### 完整数据流（退款）

```
管理员                   管理端                  FastAPI                 Supabase        支付平台
 │                       │                        │                       │               │
 │  点击退款              │                        │                       │               │
 │──────────────────────▶│                        │                       │               │
 │                       │  POST /admin/orders/{id}/refund                 │               │
 │                       │──────────────────────▶│                       │               │
 │                       │                        │  验证 Admin JWT        │               │
 │                       │                        │  查询 order 信息       │               │
 │                       │                        │──────────────────────▶│               │
 │                       │                        │  调用退款 API           │               │
 │                       │                        │──────────────────────────────────────▶│
 │                       │                        │◀──────────────────────────────────────│
 │                       │                        │  UPDATE orders         │               │
 │                       │                        │  SET status='refunded' │               │
 │                       │                        │──────────────────────▶│               │
 │  退款成功              │◀──────────────────────│                       │               │
```

### 注意
- 退款后佣金是否追回由管理员人工判断（MVP 不自动处理）
- 用户端**不开放**退款入口，所有退款由客服在管理后台手动处理
