# 角色权限体系

> 关联文档：[管理后台概述](../00-admin-overview.md) | [全局导航](03-navigation.md)
> 核心规则：四种角色互斥，一个账号只能分配一个角色

---

## 一、四种角色定义

### 1.1 超级管理员

| 属性 | 说明 |
|------|------|
| **角色标识** | `super_admin` |
| **角色标签** | "超级管理员"（顶部栏角色标签使用 Amber 色 `#d97706` 背景） |
| **权限范围** | 全部功能模块的查看和编辑权限 + 系统设置 + 管理员管理 |
| **职责定位** | 系统最高权限者，负责账号管理、权限分配、系统配置、数据总览 |
| **账号创建** | 系统初始化时创建第一个超级管理员，后续超级管理员由现有超级管理员创建 |

### 1.2 内容运营

| 属性 | 说明 |
|------|------|
| **角色标识** | `content_ops` |
| **角色标签** | "内容运营"（顶部栏角色标签使用 Sky 色 `#0284c7` 背景） |
| **权限范围** | 仪表盘（内容相关数据只读）+ 内容管理（发现中国文章、课程、每日金句）完整 CRUD |
| **职责定位** | 负责审核 Dify 自动生成的内容素材、编辑调整、上架/下架、内容排期 |

### 1.3 用户运营

| 属性 | 说明 |
|------|------|
| **角色标识** | `user_ops` |
| **角色标签** | "用户运营"（顶部栏角色标签使用 Sky 色 `#0284c7` 背景） |
| **权限范围** | 仪表盘（用户和订单相关数据只读）+ 用户管理 + 订单管理（含退款审核）+ 知语币管理 |
| **职责定位** | 负责用户查看/封禁/解封、订单查看/退款处理、知语币余额调整 |

### 1.4 游戏运营

| 属性 | 说明 |
|------|------|
| **角色标识** | `game_ops` |
| **角色标签** | "游戏运营"（顶部栏角色标签使用 Sky 色 `#0284c7` 背景） |
| **权限范围** | 仪表盘（游戏相关数据只读）+ 游戏管理（皮肤商城管理、赛季配置、排行榜管理） |
| **职责定位** | 负责皮肤上架/下架、赛季创建和配置、排行榜数据管理 |

## 三、无权限页面的访问表现

### 3.1 菜单层面

| 规则 | 说明 |
|------|------|
| **菜单隐藏** | 无权限的一级/二级菜单项在侧边栏中**完全不显示** |
| **一级菜单折叠** | 如果一级菜单下所有二级菜单均无权限，则该一级菜单也不显示 |
| **示例** | 内容运营登录后，侧边栏只显示"仪表盘"和"内容管理"两个一级菜单 |

### 3.2 URL 直接访问

| 场景 | 表现 |
|------|------|
| **通过 URL 直接访问无权限页面** | 页面内容区显示"无权限"提示页 |
| **无权限提示页内容** | 居中显示：Lucide `ShieldX` 图标（64px，中性灰）+ 标题"无权限访问"（H2）+ 说明文字"您的角色（[角色名称]）无权访问此页面，如需权限请联系超级管理员"（Body）+ "返回仪表盘"按钮（Rose 色主按钮） |
| **面包屑** | 正常显示当前路径 |
| **侧边栏** | 该菜单项不高亮（因为本身不可见） |

### 3.3 API 层面

| 规则 | 说明 |
|------|------|
| **接口鉴权** | 所有 API 请求携带角色信息，服务端二次校验权限 |
| **越权请求** | 返回 403 状态码，前端显示 Toast 错误提示："无权执行此操作" |

## 四、角色互斥规则

| 规则 | 说明 |
|------|------|
| **一人一角色** | 一个管理员账号只能分配一个角色，不可叠加多个角色 |
| **角色变更** | 由超级管理员在"管理员管理"页面修改，立即生效 |
| **变更生效时机** | 角色变更后，该管理员的下一次页面刷新或导航操作时重新加载权限，侧边栏菜单立即更新 |
| **变更通知** | 角色变更后，目标管理员当前会话页面顶部弹出蓝色 Toast："您的角色已变更为 [新角色名称]，页面将刷新"，3 秒后自动刷新页面 |
| **不可自行变更** | 管理员不能修改自己的角色（包括超级管理员） |
| **超管保护** | 系统中至少保留一个超级管理员账号，最后一个超级管理员不能被删除或降级 |
