# 提示词 12：管理后台 — 用户与订单管理 PRD

---

## 你的角色

你是「知语 Zhiyu」的高级产品经理，正在为开发团队编写**开发级产品需求文档（PRD）**。你写的文档必须精确到每个按钮的标签文字、每个操作的触发条件和结果、每个页面的 7 种状态表现，让前端和后端开发人员拿到后**零疑问直接开发**。

---

## 本次任务

生成**管理后台用户管理、订单管理、知语币管理**的详细 PRD。

覆盖范围：
1. **用户列表与搜索**（全量用户检索、筛选、排序）
2. **用户详情页**（完整用户画像：基本信息+学习数据+游戏数据+消费数据+知语币+推荐关系）
3. **用户封禁/解封**（封禁流程、封禁原因、封禁期限、对用户端的影响）
4. **订单列表**（全量订单查看、筛选）
5. **订单详情**（订单完整信息、Paddle 交易关联）
6. **退款管理**（退款申请审核、自动退款规则、对推荐返利的扣回）
7. **知语币管理**（查看用户币余额和明细、管理员手动调整）

## 质量要求

1. 禁止定义任何技术实现细节
2. 退款流程是本模块最复杂的业务逻辑，退款→课程回收→返利扣回→负余额的完整链路必须说清
3. 用户详情页的 5 个 Tab 必须逐一详细描述，这是运营人员最常用的页面
4. 封禁规则必须明确：封禁期间课程有效期是否暂停（不暂停）
5. 知语币管理员调整必须有审计（操作人+原因+时间+金额）

## 特别注意

- 退款通过 Paddle 处理，管理后台发起退款请求后由 Paddle 执行原路退回
- 退款必须同时执行：1）课程权限回收 2）推荐返利扣回——这两步缺一不可
- 知语币负余额是允许的设计（因退款扣回导致），不阻止用户正常使用
- 用户数据导出需符合 GDPR——不导出密码、不导出精确位置
- 管理员的每一个"写操作"（封禁、退款、币调整）都必须记录到系统日志
