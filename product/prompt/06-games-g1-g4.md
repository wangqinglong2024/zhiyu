## 质量要求

1. 禁止定义任何技术实现细节（不写 Phaser 代码、不写物理引擎参数）
2. 每款游戏的画面必须用文字清晰描述**横屏下的区域划分**（顶部 HUD / 中间游戏区 / 底部操作区）
3. 每个操作必须描述**用户动作→视觉反馈→音效反馈→分数变化**的完整链路
4. PK 模式必须说明双方看到的内容是否一致、如何判定胜负
5. 上瘾机制中的 Combo、每日挑战等必须给出具体的规则数值

## 特别注意

- G1 汉字切切切：声调颜色区分（一声蓝、二声绿、三声橙、四声红）是重要的视觉辅助设计
- G2 拼音泡泡龙：泡泡消除的连锁反应（cascade）是核心爽感，需详细描述
- G3 词语消消乐：匹配规则不是简单的颜色匹配，是近义词/反义词/搭配词的语义匹配，交互需额外说明
- G4 语法大厨：是最复杂的一款游戏，涉及"选词→排序→提交"的多步操作，需详细拆解每一步
- 所有游戏的题库来自对应 Level 课程，需说明题库范围但不列出具体题目
