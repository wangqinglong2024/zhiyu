## 二、游戏子模式

### 2.1 子模式总览

| 子模式 | 人数 | 影响段位 | 结束条件 | 特殊规则 |
|--------|------|---------|---------|---------|
| **单人·挑战** | 1 人 | 否 | 累计答错 3 题（3 错出局） | 无尽答题，题目越来越难，挑战最长连击 |
| **单人·限时** | 1 人 | 否 | 2 分钟倒计时归零 | 2 分钟内答对最多题目 |
| **1v1 抢答** | 2 人 | 是 | 10 题结束 | 同题抢答，快且对者得分 |
| **多人大会** | 6-12 人 | 是 | 淘汰至剩最后 1 人 | 模拟诗词大会，每轮淘汰排名最低者 |

### 2.2 单人·挑战模式

- 系统连续出题，每题 10-15 秒倒计时
- 答对得分（基础分 + Streak 加成），答错倒扣 50 分且 Streak 归零
- 累计答错 3 题 → 出局 → 结算
- 题目难度逐步提升：前 10 题为 L11 基础题 → 10-20 题为进阶题 → 20 题后为冷门/高难题
- 记录个人最高分和最长连击数

### 2.3 单人·限时模式

- 2 分钟倒计时，连续出题
- 每题 8 秒（比挑战模式更短），答错不出局但扣分 + Streak 归零
- 2 分钟内答对题数和总分排行

### 2.4 1v1 抢答模式

- 双方同时看到同一道题目
- **抢答机制**：谁先选择正确答案谁得分
  - 先答对者：得基础分 + 速度奖励
  - 后答对者（在对方答对后 2 秒内也答对）：得基础分 × 50%
  - 答错者：-50 分 + Streak 归零
  - 超时：无扣分但不得分
- 共 10 题，总分高者胜
- 胜 +1 星，负 -1 星

### 2.5 多人大会模式（6-12 人淘汰赛）

**流程**：
1. 6-12 人匹配成功后进入「诗词大会」舞台
2. **第一轮**（预赛）：5 题抢答，所有人参与。排名最后 2 人淘汰
3. **第二轮**（半决赛）：5 题抢答，剩余人参与。排名最后 2 人淘汰
4. **第三轮**（决赛）：5 题抢答，剩余人参与。本轮得分最高者为冠军
5. 冠军 +3 星，亚军 +2 星，季军 +1 星，其余不扣星

**淘汰规则**：
- 每轮结束后，按本轮得分从低到高排名
- 排名最后 2 人被淘汰（6 人场）/ 最后 3 人（9 人以上场）
- 被淘汰者进入「观众席」，可继续观看但不能答题
- 如果分数相同，则比较本轮答题速度

## 四、核心交互

### 4.1 题目入场

**触发**：前一题结束 / 游戏开始

**视觉流程**：
1. **题目面板入场**（0ms）：题目面板从下方弹入（或从舞台灯光中「聚光灯照亮」效果），持续 300ms
2. **题型标签显示**（100ms）：题型标签淡入
3. **题目内容展开**（300ms）：题目文字/图片/音频逐渐展现（淡入或打字机效果）
4. **选项入场**（600ms）：4 个选项依次从下方滑入（每个间隔 100ms）
5. **倒计时开始**（800ms）：倒计时环开始运行
6. **如果是听音频题**：音频在选项入场后自动播放

### 4.2 答题（答对）

**触发**：玩家点击正确选项

**视觉反馈**：
1. **选项变绿**（0ms）：选中选项背景变为绿色半透明（`rgba(34,197,94,0.2)`），边框绿色
2. **得分飘字**（100ms）：选项上方飘出得分（如 `+200`，Amber 色 H3 20px），含 Streak 加成（如 `+200 × 1.5`）
3. **连击更新**（100ms）：HUD 连击数 +1，火焰图标脉动一次
4. **舞台效果**（200ms）：舞台灯光短暂亮闪（如观众鼓掌时的灯光）
5. **诗句完整展示**（400ms）：题目区展示完整诗句（填空题填入答案 + 全文），持续 1.5 秒（让玩家有短暂学习时间）
6. **过渡到下一题**（2000ms）：题目面板淡出 → 下一题入场

**音效**：清脆的正确确认音 + 观众鼓掌声（多人模式）

**得分规则**：

| 题型 | 基础分 | 速度奖励（<3 秒答对） | Streak 倍率 |
|------|-------|---------------------|-----------|
| 古诗词填空 | 100 | +50 | 按连击档位 |
| 看图猜诗 | 150 | +75 | 按连击档位 |
| 听音频选诗 | 150 | +75 | 按连击档位 |
| 诗人匹配 | 100 | +50 | 按连击档位 |
| 赏析选择 | 200 | +100 | 按连击档位 |

### 4.3 答题（答错）

**触发**：玩家点击错误选项

**视觉反馈**：
1. **选项变红**（0ms）：选中选项背景变为 Rose 红半透明，边框 Rose 红
2. **正确答案标记**（200ms）：正确选项背景变绿 + ✅
3. **扣分飘字**（100ms）：选项上方飘出 `-50`（Rose 红色）
4. **连击中断**（200ms）：Streak 归零 + 火焰熄灭动画
5. **错误计数更新**（挑战模式）：错误数 +1，若达到 3 错 → 触发出局
6. **舞台灯光闪暗**（200ms）：舞台灯光整体暗淡 300ms → 恢复
7. **解析展示**（400ms）：题目区下方展示正确答案解析（Body S 14px，白色 60%），持续 2 秒
8. **过渡到下一题**（2500ms）

**音效**：错误蜂鸣音 + 观众叹息声（多人模式）

### 4.4 答题超时

**触发**：倒计时归零

**视觉反馈**：
1. **倒计时环消失**（0ms）：环完全消失 + `时间到！` 文字（Rose 红色）
2. **正确答案标记**（200ms）：正确选项自动标绿 + ✅
3. **单人挑战模式**：不计为错误，但不得分且 Streak 归零
4. **PK/多人模式**：视为未答，不得分不扣分
5. **过渡到下一题**（2000ms）

### 4.5 抢答机制（PK/多人模式）

> **公平性处理**：为解决网络延迟导致的不公平，采用「服务端时间戳+容差窗口」机制。

**抢答判定流程**：
1. 服务端向所有玩家同时下发题目
2. 玩家选择答案后，客户端立即发送 `{答案, 客户端时间戳}` 给服务端
3. 服务端以自身时钟为基准，记录收到答案的时间
4. **200ms 容差窗口**：如果两名玩家的答案到达时间差 ≤ 200ms，视为「同时答对」
   - 同时答对：双方各得基础分 × 80%
5. 差异 > 200ms：先到者得全分，后到者（2 秒内也答对）得基础分 × 50%
6. 超过 2 秒才答对：仅得基础分 × 30%

**抢答成功视觉反馈**：
- 先答对者：选项变绿 + `抢答成功！` Amber 色标签弹出（300ms）+ 得分飘字
- 被抢答者：题目区顶部闪现 `对手抢答成功！`（Rose 红，1 秒淡出），自己仍可作答但得分降低

### 4.6 单人挑战模式 — 三错出局

**触发**：第 3 次答错

**视觉流程**：
1. **第 3 个错误标记**（0ms）：错误计数变为 `3/3`，3 个 ❌ 全部 Rose 红
2. **舞台灯光全暗**（300ms）：所有灯光熄灭效果（500ms 过渡到暗色）
3. **出局文字**（500ms）：屏幕中央 `挑战结束！`（H1 32px，Rose 红色淡入）
4. **战绩总结**（1200ms）：
   - `总得分：3680`
   - `答对题数：18`
   - `最高连击：×12`
   - `最高 Streak：🔥🔥🔥`
5. **过渡到通用结算页**（3000ms）

## 五、多人大会模式（6-12 人淘汰赛流程）

### 5.1 入场阶段

**视觉流程**：
1. **舞台搭建动画**（0ms）：舞台背景从暗到亮，灯光依次亮起（500ms）
2. **选手入场**（500ms）：所有选手头像从底部依次弹入选手席，每人间隔 200ms
3. **对手信息展示**（1500ms）：快速闪过每位对手的头像+昵称+段位（每人 500ms）
4. **大会开始**（3000ms）：`诗词大会 · 开始！`（Display 48px，Amber 色），舞台烟花粒子特效

### 5.2 每轮流程

1. **轮次提示**：`第一轮 · 预赛`（H1 32px，白色，持续 1.5 秒）
2. **5 题抢答**：流程同 §4（核心交互），但所有选手同时答题
3. **本轮得分排名**：5 题结束后展示本轮排名面板（所有选手头像 + 分数 + 排名）
4. **淘汰宣布**：
   - 淘汰线以下的选手头像变暗
   - 逐个宣布淘汰：`XXX 淘汰！`（Rose 红色），头像从选手席滑出 + 灰化
   - 被淘汰者看到 `您已被淘汰` + 进入观众模式
5. **下轮准备**：1.5 秒后进入下一轮

### 5.3 淘汰规则详情

| 参赛人数 | 第一轮淘汰 | 第二轮淘汰 | 第三轮（决赛） |
|---------|-----------|-----------|-------------|
| 6 人 | 淘汰 2 人（剩 4 人） | 淘汰 2 人（剩 2 人） | 2 人决赛 |
| 8 人 | 淘汰 3 人（剩 5 人） | 淘汰 2 人（剩 3 人） | 3 人决赛 |
| 10 人 | 淘汰 3 人（剩 7 人） | 淘汰 3 人（剩 4 人） | 4 人决赛 |
| 12 人 | 淘汰 4 人（剩 8 人） | 淘汰 4 人（剩 4 人） | 4 人决赛 |

### 5.4 冠军加冕

**触发**：最后一轮得分最高者

**视觉流程**：
1. **所有灯光聚焦冠军**（0ms）：舞台灯光全部照向冠军头像，其余暗淡
2. **冠军特效**（500ms）：冠军头像放大至 200×200px + 金色光芒 + 皇冠图标
3. **「冠军」文字**（800ms）：`🏆 诗词大会冠军！`（Display 48px，Amber 金色），全屏金色粒子喷射
4. **掌声音效**（0ms）：持续 3 秒的热烈掌声
5. **全体排名**（2000ms）：展示最终排名（1-N 名，各自得分和段位变化）

### 5.5 观众模式

被淘汰后进入观众模式：
- 仍可看到舞台画面和题目
- 选项区变灰不可点击
- 底部显示 `观众模式 · 您已被淘汰`（白色 40%）
- 可看到其他选手的答题结果

## 七、上瘾机制

### 7.1 Streak 连击追求

- 连击特效随档位越来越华丽 → 想维持连击不中断
- 最高连击记录保存 → 想刷新个人纪录
- 连击倍率影响得分 → 高连击 = 高分

### 7.2 诗词图谱可视化

- 个人中心展示「诗词图谱」（按朝代分类：先秦→汉→唐→宋→元→明→清→近现代）
- 每个朝代显示已掌握/总数（如 `唐诗 45/68`）
- 答对某首诗相关的题目 → 该诗标记为「已掌握」
- 全朝代完成率可视化 → 「朝代通关」成就
- 驱动力：看到未掌握的区域 → 想去填满

### 7.3 诗词大师称号

| 称号 | 获取条件 | 图标 |
|------|---------|------|
| 诗童 | 初始称号 | 📖 |
| 秀才 | 累计答对 100 题 | 📝 |
| 举人 | 累计答对 300 题 + 最高连击 ≥10 | 🎓 |
| 进士 | 累计答对 600 题 + 最高连击 ≥20 + 多人大会冠军 1 次 | 🏅 |
| 状元 | 累计答对 1000 题 + 百人大会前 10 名 | ⭐ |
| 诗仙 | 累计答对 2000 题 + 百人大会冠军 1 次 + 诗词图谱完成率 ≥90% | 👑 |

### 7.4 每周百人大会的仪式感

- 固定时间 → 形成每周仪式感
- 百人竞技 → 大规模活动的社交兴奋
- 限定称号 → 稀缺性驱动
- 往期冠军展示 → 荣誉激励

## 八、状态矩阵

### 8.1 游戏进行中状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **题目入场** | 新题目展示 | 面板弹入 + 题型标签 + 选项入场 + 倒计时开始 |
| **等待答题** | 倒计时进行，等待玩家选择 | 选项可点击，倒计时环运行 |
| **答对** | 选择正确选项 | 选项变绿 + 得分飘字 + Streak 更新 + 诗句展示 |
| **答错** | 选择错误选项 | 选项变红 + 正确标记 + 扣分 + Streak 归零 + 解析 |
| **超时** | 倒计时归零 | 正确答案标记 + Streak 归零 |
| **连击里程碑** | 达到 5/10/20 连击 | 全屏闪光 + 大字弹出 + 舞台光效变化 + 音效 |
| **连击中断** | 答错导致 Streak 归零 | 火焰熄灭 + 数字归零 + 灯光暗淡 |
| **三错出局**（挑战模式） | 第 3 次答错 | 灯光全暗 + 出局文字 + 战绩总结 |

### 8.2 PK 模式状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **双方答题** | 各自选择答案 | 双方分数实时可见，对方答题状态不可见 |
| **己方抢答成功** | 己方先答对 | `抢答成功！` 标签 + 得分 |
| **对方抢答成功** | 对方先答对 | `对手抢答成功！` 提示 + 己方仍可答题 |
| **同时答对** | 200ms 容差内双方都答对 | 双方各得基础分 × 80% |
| **10 题结束** | 全部题目完成 | 总分对比 → 胜/负/平局 → 结算 |
| **对手断线** | 对手网络断开 | 等待 15 秒重连，超时判己方获胜 |

### 8.3 多人大会状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
| **等待入场** | 匹配/报名中 | 舞台搭建动画 + 选手陆续入场 |
| **轮次提示** | 宣布本轮信息 | 轮次名称 + 剩余人数 |
| **抢答进行** | 本轮题目进行中 | 所有选手同时答题，选手席分数实时更新排序 |
| **本轮结算** | 本轮 5 题结束 | 排名展示 + 淘汰线标注 |
| **淘汰宣布** | 宣布被淘汰选手 | 逐个宣布 + 头像灰化滑出 |
| **观众模式** | 被淘汰后 | 可观看但不可答题 |
| **冠军加冕** | 最终冠军产生 | 聚光灯 + 冠军特效 + 全体排名 |

### 8.4 空状态 / 异常状态

| 状态 | 描述 | 设计要求 |
|------|------|---------|
