## 三、单元卡片列表

### 3.1 单元卡片内容

每个单元卡片包含以下元素：

```
┌────────────────────────────────────────────────┐
│  📖 单元 3：议论文初步                     🔒    │
│                                                  │
│  5 课时                                          │
│  ████████░░░░░░░░░░░░  0/5                      │
│                                                  │
│  单元测评  🔒 未开放                              │
│                                                  │
│  请先完成上一单元测评                              │
└────────────────────────────────────────────────┘
```

| 元素 | 位置 | 规格 | 说明 |
|------|------|------|------|
| 单元图标 | 卡片左上 | 📖 或自定义图标 | 已完成用 ✅，进行中用 📖（Rose 色），锁定用 🔒（灰色） |
| 单元编号+名称 | 图标右侧 | Body 字号（16px），加粗 | "单元 X：[名称]"，多语言 |
| 状态图标 | 卡片右上 | 16px | 根据状态展示（见 §3.2） |
| 课时数量 | 名称下方 | Body S 字号（14px），灰色 | "X 课时" |
| 课时进度条 | 课时数量下方 | 线性进度条，宽度 100%，高度 6px | Rose 色填充比例 = 已完成课时 / 总课时 |
| 课时进度文字 | 进度条右端 | Caption 字号 | "X/Y"（已完成/总数） |
| 单元测评入口 | 进度条下方 | Body S 字号 | 根据状态展示不同内容（见 §3.3） |
| 锁定提示 | 卡片底部（仅锁定态） | Caption 字号，灰色 | "请先完成上一单元测评" |

### 3.2 单元的 4 种状态

#### 状态 1：已完成 ✅

| 维度 | 描述 |
|------|------|
| 适用条件 | 该单元全部课时已完成且单元测评已通过（≥70 分） |
| 视觉表现 | 卡片正常毛玻璃态；左侧 ✅ 绿色勾选图标（Lucide `CheckCircle`）；右上角无额外图标；进度条满格（绿色 `#22c55e`） |
| 单元测评展示 | "单元测评 ✅ XX 分" — 绿色文字，展示通过分数 |
| 点击行为 | 进入单元课时列表（可回顾已完成课时） |
| 无障碍标签 | "单元 X，已完成，单元测评 XX 分，点击查看" |

#### 状态 2：进行中 🟢

| 维度 | 描述 |
|------|------|
| 适用条件 | 该单元已解锁，至少完成 1 个课时但未全部完成；或全部课时已完成但测评未通过 |
| 视觉表现 | 卡片毛玻璃高亮态（`.glass-elevated`）；左侧 📖 图标（Rose 色）；进度条部分填充（Rose 色） |
| 单元测评展示 | 全部课时未完成时："单元测评 🔒 完成全部课时后开放"；全部课时已完成时："单元测评 — 开始测评"（Rose 色可点击文字）；测评未通过时："单元测评 ❌ XX 分 — 重新测评"（红色分数 + Rose 色链接） |
| 点击行为 | 进入单元课时列表，自动定位到待学习课时 |
| 无障碍标签 | "单元 X，进行中，已完成 X/Y 课时，点击继续学习" |

#### 状态 3：已解锁 ⚪

| 维度 | 描述 |
|------|------|
| 适用条件 | 该单元已解锁（前一单元测评通过）但用户尚未开始任何课时 |
| 视觉表现 | 卡片正常毛玻璃态；左侧 📖 图标（灰色）；进度条空（灰色背景，无填充） |
| 单元测评展示 | "单元测评 🔒 未开放" — 灰色文字 |
| 点击行为 | 进入单元课时列表，从第一个课时开始学习 |
| 无障碍标签 | "单元 X，已解锁，点击开始学习" |

#### 状态 4：锁定 🔒

| 维度 | 描述 |
|------|------|
| 适用条件 | 前一单元测评尚未通过 |
| 视觉表现 | 卡片低透明度毛玻璃态（opacity 0.5）；左侧 🔒 锁图标（Lucide `Lock`，灰色）；进度条灰色；整体灰度滤镜 |
| 单元测评展示 | "单元测评 🔒 未开放" — 灰色文字 |
| 锁定提示 | 卡片底部灰色文字："请先完成上一单元测评"，多语言 |
| 点击行为 | Toast 提示"请先完成「单元 X：[上一单元名称]」的测评"（多语言，2s 自动消失） |
| 无障碍标签 | "单元 X，已锁定，需完成上一单元测评后解锁" |

### 3.3 单元解锁条件规则

```
Level 内第一个单元 → 默认解锁（用户进入 Level 即可开始）

后续单元解锁条件：
  前一单元的全部课时已完成
  AND 前一单元测评已通过（≥70 分）
  → 下一单元解锁

示例：
  单元 1 全部 5 课时完成 + 单元测评 ≥70 分 → 单元 2 解锁
  单元 2 全部 5 课时完成 + 单元测评 ≥70 分 → 单元 3 解锁
  ...
```

### 3.4 单元测评入口行为

| 条件 | 测评入口展示 | 点击行为 |
|------|------------|---------|
| 课时未全部完成 | "单元测评 🔒 完成全部课时后开放" | 不可点击 |
| 课时全部完成 + 未测评 | "单元测评 — 开始测评" | 进入单元测评页面（在课程考核 PRD 中定义） |
| 课时全部完成 + 测评通过 | "单元测评 ✅ XX 分" | 进入测评结果详情页 |
| 课时全部完成 + 测评未通过 | "单元测评 ❌ XX 分 — 重新测评" | 进入单元测评页面（重考） |
| 单元锁定 | "单元测评 🔒 未开放" | 不可点击 |

### 3.5 单元卡片内的课时列表展开（点击后）

点击已解锁或进行中的单元卡片后，进入**单元课时列表页**：

```
┌─────────────────────────────────────────────────┐
│  ← 返回 Level 详情           单元 2：深度阅读技巧  │
│                                                   │
│  课时 1：精读方法论              ✅ 已完成          │
│  课时 2：文章结构分析            ✅ 已完成          │
│  课时 3：推理与判断              🟢 进行中          │
│  课时 4：比较阅读                ⚪ 待学习          │
│  课时 5：综合练习                ⚪ 待学习          │
│                                                   │
│  ─────────────────────────────────────────────── │
│  单元测评                        🔒 完成全部课时后开放│
└─────────────────────────────────────────────────┘
```

课时状态：

| 状态 | 图标 | 条件 | 点击行为 |
|------|------|------|---------|
| 已完成 ✅ | 绿色勾 | 课时学习完毕 + 课时小测验已做 | 进入课时学习页（复习模式） |
| 进行中 🟢 | Rose 色圆点 | 课时已开始但未完成小测验 | 进入课时学习页（恢复上次位置） |
| 待学习 ⚪ | 灰色圆点 | 前一课时已完成 | 进入课时学习页（从头开始） |
| 锁定 🔒 | 灰色锁 | 前一课时未完成（首个课时除外） | Toast "请先完成上一课时" |

> 课时的解锁规则：同一单元内，课时**按顺序解锁**。第一个课时默认可学习，完成后解锁下一个课时。这里的"完成"指课时内容学完且课时小测验已做（无通过门槛）。

## 四、底部综合考核入口

### 4.1 位置

- 单元列表最末尾，与最后一个单元卡片间距 24px
- 独立的特殊卡片样式（带 🏆 图标）

### 4.2 综合考核入口的 3 种状态

#### 状态 A：锁定（未满足条件）

│                                                  │
│  完成全部单元测评后开放                            │
│  已通过 X/Y 个单元测评                            │
│                                                  │
│  ━━━━━━━━━━━━━━░░░░░░░░░░░░  X/Y                │

| 维度 | 描述 |
|------|------|
| 适用条件 | Level 内有单元测评尚未通过 |

| 维度 | 描述 |
|------|------|

## 五、状态矩阵

### 5.1 Level 详情页

| 页面状态 | 描述 | 设计要求 |
|---------|------|---------|
| **空状态 (Empty)** | 不会出现 — 每个 Level 至少有 8 个单元 | — |
| **加载中 (Loading)** | Level 数据和单元列表加载中 | 顶部 Level 信息区 Skeleton（标题行 + 标签 + 进度条占位）+ 单元卡片列表 Skeleton（3-4 个卡片占位形状） |
| **首次加载 (First Load)** | 用户首次进入某 Level | 单元卡片依次淡入 + 微上滑动画（每个延迟 80ms，单个 250ms） |
| **成功 (Success)** | 数据加载完成 | 正常展示所有单元卡片和综合考核入口 |
| **错误 (Error)** | 网络请求失败 | 全屏错误提示"加载失败，请检查网络后重试"（多语言）+ 重试按钮 |
| **部分加载 (Partial)** | 不适用 — 单元列表一次性加载 | — |
| **离线 (Offline)** | 用户无网络 | 顶部 Banner "网络连接已断开"；展示本地缓存的单元进度（只读）；点击单元卡片提示"请连接网络" |

### 5.2 单元课时列表页

| 页面状态 | 描述 | 设计要求 |
|---------|------|---------|
| **空状态** | 不适用 | — |
| **加载中** | 课时列表加载中 | 课时行 Skeleton（5-6 行占位） |
| **成功** | 加载完成 | 课时列表正常展示，自动定位到当前进行中课时 |
| **错误** | 加载失败 | 错误提示 + 重试按钮 |
| **离线** | 无网络 | 展示缓存数据（只读），点击课时提示"请连接网络" |
