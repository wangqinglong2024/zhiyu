## 一、数据流向说明

### 1.1 课程内容数据流

```
Dify 工作流（内容生产工具）
  │  AI 生成课程初稿（文本/题目/多语言翻译）
  ↓
教学专家审核 + 编辑调整
  ↓
管理后台 — 课程内容管理
  │  · 课时内容 CRUD
  │  · 练习题库管理
  │  · 音频上传
  │  · 手动发布 / 上架下架
  ↓
Supabase 数据库（唯一数据主存储）
  │  结构化存储：Level > Unit > Lesson > Content Block
  │  · text_zh（中文正文）
  │  · text_pinyin（拼音）
  │  · text_vi（越南语翻译）
  │  · text_en（英语翻译）
  │  · audio_url（音频链接）
  │  · vocabulary（重点词汇 JSON）
  │  · grammar_points（语法点 JSON）
  │  · metadata（级别、模块、知识点标签）
  ↓
应用端 — 课时学习页
  │  · 前端按用户语言配置渲染对应内容层
  │  · 音频按需加载播放
  │  · 已加载内容缓存到本地（离线可读）
```

### 1.2 学习进度数据流

```
用户在课时学习页操作
  │  · 滚动浏览内容
  │  · 点击词汇卡片
  │  · 播放音频
  │  · 完成课时小测验
  ↓
前端实时记录进度状态
  │  · 当前滚动位置
  │  · 已查看词汇索引
  │  · 已播放音频索引
  │  · 每 30 秒自动保存 + 退出时保存
  ↓
Supabase 数据库 — 用户学习进度表
  │  · user_id + lesson_id → 进度百分比、最后学习时间
  │  · user_id + unit_id → 已完成课时数、测评分数
  │  · user_id + level_id → 总进度、综合考核状态
  ↓
课程首页 / Level 详情页
  │  · 读取进度数据渲染 Level 地图节点状态
  │  · 读取进度数据渲染单元卡片状态
```

### 1.3 SRS 复习队列数据流

```
复习项来源
  │  · 课时小测验答错题 → 自动加入 SRS
  │  · 已学课时重点词汇 → 自动加入 SRS
  │  · 已学课时语法点 → 自动加入 SRS
  │  · 单元测评答错题 → 自动加入 SRS
  ↓
Supabase 数据库 — SRS 队列表
  │  · user_id + item_id + item_type（vocab/grammar/quiz）
  │  · current_interval（当前间隔阶段 1-6）
  │  · next_review_date（下次复习日期）
  │  · consecutive_correct（连续正确次数）
  │  · created_at / last_reviewed_at
  ↓
SRS 复习页面
  │  · 查询 next_review_date ≤ 今日 的项目
  │  · 按优先级排序（逾期 > 低阶段 > 类型穿插）
  │  · 用户操作后更新 interval + next_review_date
  ↓
Supabase 更新
  │  · "记住了" → current_interval + 1，计算新 next_review_date
  │  · "还没记住" → current_interval = 1，next_review_date = 明天
  │  · consecutive_correct ≥ 6 → 标记毕业，不再出现
```

### 1.4 入学测试结果数据流

```
用户完成入学测试
  ↓
前端收集答题数据
  │  · 每题答案 + 答题时间 + 题目难度级别
  ↓
后端计算推荐 Level
  │  · 根据正确率 + 自适应难度 + 各模块表现
  │  · 生成推荐 Level + 各维度能力分数
  ↓
Supabase 数据库 — 测试结果表
  │  · user_id + test_date + recommended_level
  │  · dimension_scores（词汇/阅读/语法/听力百分比）
  │  · raw_answers（原始答题数据）
  │  · first_test_reward_claimed（是否已领取首次奖励）
  ↓
结果页展示 + 课程首页 Level 推荐
```

### 1.5 课程购买数据流

```
用户点击付费墙"购买"按钮
  ↓
前端调用后端创建 Paddle Checkout Session
  ↓
Paddle 托管支付页面（Overlay）
  │  · 用户完成支付
  ↓
Paddle Webhook → 后端
  │  · 验证 Webhook 签名
  │  · 创建订单记录
  │  · 更新用户课程权限（解锁对应 Level）
  │  · 写入知语币流水（推荐人返佣，若有）
  ↓
Supabase 数据库
  │  · orders 表（订单记录）
  │  · user_course_access 表（课程权限：level_id + 有效期）
  │  · coin_transactions 表（知语币流水）
  ↓
前端刷新权限 → Level 状态从 🔒 变为 ⚪/🔵
```

## 二、非功能需求

### 2.1 性能要求

| 指标 | 目标 | 说明 |
|------|------|------|
| 课程首页首屏加载 | < 2 秒 | Level 地图 + 用户进度数据 |
| 课时学习页内容加载 | < 1.5 秒 | 教学文本 + 词汇数据（音频懒加载） |
| Level 详情页加载 | < 1.5 秒 | 单元列表 + 进度数据 |
| 入学测试出题响应 | < 800ms | 自适应算法计算 + 下一题加载 |
| SRS 复习卡片切换 | < 500ms | 含翻转动画 + 数据提交 + 新卡片加载 |
| 付费墙弹窗弹出 | < 300ms | 弹窗 DOM 预加载，仅弹出动画耗时 |
| 音频开始播放 | < 500ms | 音频首次加载后缓存 |
| 学习进度保存 | < 300ms | 异步提交，不阻塞 UI |
| API 响应时间 | < 500ms（P95） | 所有课程相关 API |

### 2.2 并发与容量

| 指标 | 目标 |
|------|------|
| 预期峰值并发用户 | 500（MVP 阶段） |
| 单用户 SRS 队列上限 | 5,000 项 |
| 单用户学习进度记录 | 无上限（12 Level × 12 Unit × 6 Lesson = 864 条） |
| 入学测试并发 | 50 人同时测试 |
| 课程内容缓存 | 已访问课时本地缓存 7 天 |

### 2.3 可用性

| 指标 | 目标 |
|------|------|
| SLA | 99.9%（全年宕机 < 8.7 小时） |
| 数据持久性 | 学习进度数据不可丢失（Supabase 自动备份） |
| 降级策略 | 课程内容加载失败时展示缓存数据（只读）；SRS 复习必须在线 |

### 2.4 安全性

| 需求 | 说明 |
|------|------|
| 课程内容防盗 | 课程文本内容不在前端完整缓存，按课时粒度加载；音频 URL 签名保护 |
| 支付安全 | Paddle Webhook 签名验证；前端不处理任何支付敏感信息 |
| 知语币防刷 | 入学测试奖励去重（`first_test_reward_claimed`）；兑换操作后端原子事务 |
| 进度数据隔离 | 用户只能访问自己的学习进度（Supabase RLS 策略） |

### 2.5 无障碍

| 需求 | 说明 |
|------|------|
| 色彩对比度 | 正文 ≥ 4.5:1、标题 ≥ 3:1 |
| 触控尺寸 | 可点击区域 ≥ 44×44pt |
| 屏幕阅读器 | 所有交互元素有 `aria-label`；Level 地图节点有语义化标签 |
| 键盘导航 | Tab/Enter 可完成核心操作（桌面端） |
| 动效敏感 | 支持 `prefers-reduced-motion`，可关闭动画 |

### 2.6 国际化

| 需求 | 说明 |
|------|------|
| UI 语言 | 汉语 / 英语 / 越南语 三种切换 |
| 内容语言 | 中文（始终展示）+ 拼音（可选）+ 解释语言（可选，跟随 UI 语言） |
| 货币本地化 | 价格以 USD 为基准，付费墙同时展示当地货币等值（Paddle 汇率） |
| 日期格式 | 跟随用户 locale（如有效期展示） |
| RTL 支持 | 不需要（目标语言均为 LTR） |
