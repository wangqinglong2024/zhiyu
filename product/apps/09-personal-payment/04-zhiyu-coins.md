## 一、余额展示

### 1.1 页面顶部区域

```
┌─────────────────────────────────────┐
│  ← 返回          知语币              │
├─────────────────────────────────────┤
│                                     │
│         [知语币图标 48×48]            │
│                                     │
│            320                      │
│         知语币余额                   │
│       ≈ $3.20                       │
│                                     │
├─────────────────────────────────────┤
```

| 元素 | 规格 | 说明 |
|------|------|------|
| 知语币图标 | 48×48px | 品牌定制金币图标，Amber 色 |
| 余额数字 | Display（48px，`font-weight: 800`），`font-variant-numeric: tabular-nums` | 正数使用 Amber 色；零使用中性灰；负数使用语义错误色 `#ef4444` 并前加 `-` 号 |
| "知语币余额"标签 | Body S（14px，中性灰），居中 | — |
| 等值美元 | Caption（12px，中性灰），居中 | 格式：`≈ ${余额 × 0.01}`；固定汇率 1 币 = $0.01 |

### 1.2 负余额特殊展示

当知语币余额为负数时，余额数字下方增加一行提示文案：

```
            -180
         知语币余额
        ≈ -$1.80

  ⚠️ 因您推荐的用户发起了课程退款，
  对应的推荐奖励已扣回。后续获得
  新奖励时将自动抵充。
```

| 元素 | 规格 |
|------|------|
| 负余额数字 | Display（48px），语义错误色 `#ef4444` |
| 提示文案 | Body S（14px），语义警告色 `#f59e0b`，居中，最多 3 行 |
| ⚠️ 图标 | 16px，语义警告色，文案前方 |

## 二、收支明细列表

### 2.1 筛选 Tab

余额展示区下方为筛选 Tab 栏：

```
┌─────────────────────────────────────┐
│  [ 全部 ]  [ 收入 ]  [ 支出 ]        │
├─────────────────────────────────────┤
```

| Tab | 筛选逻辑 | 默认 |
|-----|---------|------|
| 全部 | 显示所有收支记录 | ✅ 默认选中 |
| 收入 | 仅显示金额 > 0 的记录 | — |
| 支出 | 仅显示金额 < 0 的记录 | — |

选中 Tab 使用 Rose 色下划线 + Rose 色文字，未选中使用中性灰文字。

## 六、状态矩阵

| 状态 | 表现 |
|------|------|
| **空状态 (Empty)** | 余额为 0 且无任何收支记录：余额区域显示 `0`（中性灰）；明细列表区域显示"暂无收支记录" + 引导文案"去签到或推荐好友赚取知语币吧" |
| **加载中 (Loading)** | 余额数字区域显示圆形 Skeleton；明细列表显示 4-5 行 Skeleton 条 |
| **首次加载 (First Load)** | 同加载中 |
| **成功 (Success)** | 所有数据正常渲染 |
| **错误 (Error)** | 余额加载失败：余额区域显示 `--` + "加载失败" + \[重试\]；明细列表失败：列表区域显示错误插画 + \[重试\] |
| **部分加载 (Partial)** | 余额先渲染；明细列表异步加载，加载中显示 Skeleton |
| **离线 (Offline)** | 顶部离线 Banner；展示上次缓存的余额和明细；余额旁标注"离线数据" |
