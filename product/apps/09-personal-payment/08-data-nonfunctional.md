# 数据流向与非功能需求

> 关联：[00-index.md](00-index.md) · `plan/05-tech-architecture.md` · `01-global-framework/09-nonfunctional.md`
> 本文档定义个人中心 + 支付 + 推荐返利系统的数据流向、非功能需求及全模块验收标准汇总。

---

## 一、数据流向说明

### 1.1 用户资料数据流

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│   客户端      │ ──────→ │   后端服务    │ ──────→ │   对象存储    │
│  Profile 页   │ ←────── │  User API    │ ←────── │  头像/证书    │
└──────────────┘         └──────────────┘         └──────────────┘
       │                        │
       │ 本地缓存               │ 数据持久化
       ↓                        ↓
┌──────────────┐         ┌──────────────┐
│  IndexedDB   │         │   数据库      │
│  (离线缓存)   │         │  用户表/偏好  │
└──────────────┘         └──────────────┘
```

- 用户编辑资料 → 客户端表单校验 → 提交后端 → 后端校验（昵称敏感词 + 唯一性）→ 写入数据库
- 头像上传 → 客户端裁剪压缩 → 上传对象存储 → 返回 URL → 写入用户表
- 个人中心首页 → 优先读取本地缓存 → 后台静默拉取最新数据 → 有变更则更新 UI

### 1.2 课程与学习数据流

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│   客户端      │ ──────→ │   后端服务    │ ──────→ │   数据库      │
│  我的课程页   │ ←────── │ Course API   │ ←────── │ 课程/进度/权限│
└──────────────┘         └──────────────┘         └──────────────┘
       │
       │ 学习进度
       ↓
┌──────────────┐
│  本地进度缓存 │
│  (离线可查看) │
└──────────────┘
```

- 课程列表 → 拉取用户已购课程 + 免费课程 + 进度 → 合并渲染
- 课程有效期 → 后端计算剩余天数 → 客户端展示（不在客户端自行计算）
- 收藏/证书 → 标准 CRUD 操作

### 1.3 知语币数据流

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│   客户端      │ ──────→ │   后端服务    │ ──────→ │   数据库      │
│  知语币页面   │ ←────── │  Coin API    │ ←────── │ 余额/流水表   │
└──────────────┘         └──────────────┘         └──────────────┘
                                │
                   事件驱动：购课返利 / 退款扣回
                   签到奖励 / 游戏奖励 / 活动奖励
                                │
                                ↓
                         ┌──────────────┐
                         │  事件队列     │
                         │  (异步处理)   │
                         └──────────────┘
```

- 余额读取 → 直接查询余额表 → 返回当前余额（含正/负状态）
- 收支明细 → 分页查询流水表 → 支持按类型筛选
- 余额变动 → 所有加减操作通过后端事务处理，保证原子性

### 1.4 支付数据流

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│   客户端      │ ──────→ │   后端服务    │ ──────→ │   Paddle API  │
│  支付流程     │ ←────── │ Payment API  │ ←────── │  (MoR 支付)   │
└──────────────┘         └──────────────┘         └──────────────┘
       │                        │                        │
       │ Paddle.js              │ Webhook                │
       │ Overlay Checkout       │ 回调通知               │
       ↓                        ↓                        │
┌──────────────┐         ┌──────────────┐               │
│  Paddle 收银台│ ────────│  订单表       │←──────────────┘
│  (Overlay)   │  result │  支付状态     │   webhook
└──────────────┘         └──────────────┘
```

**购买流程数据流**：
1. 客户端 → 后端：创建订单（课程 ID、知语币抵扣额、推荐码）
2. 后端 → Paddle：创建 Checkout Session（商品、价格、客户信息）
3. 后端 → 客户端：返回 Paddle Checkout URL / Client Token
4. 客户端：调起 Paddle.js Overlay Checkout
5. Paddle → 用户：支付页面（Paddle 为法律卖方）
6. 用户完成支付 → Paddle → 后端：Webhook 通知（`transaction.completed`）
7. 后端：更新订单状态 → 解锁课程权限 → 扣减知语币 → 触发推荐返利
8. 后端 → 客户端：轮询订单状态 → 跳转支付成功页

**退款流程数据流**：
1. Paddle → 后端：Webhook（`transaction.refunded`）
2. 后端：更新订单状态 → 回收课程权限 → 退回知语币抵扣部分 → 触发推荐返利扣回

### 1.5 推荐返利数据流

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│  被推荐人      │ ──────→ │   后端服务    │ ──────→ │   数据库      │
│  注册/购买     │         │ Referral API │         │ 推荐关系/返利 │
└──────────────┘         └──────────────┘         └──────────────┘
                                │
                      ┌────────┼────────┐
                      ↓        ↓        ↓
               ┌─────────┐┌────────┐┌────────┐
               │ 冷却期   ││ 发放   ││ 扣回   │
               │ 30 天    ││ 知语币 ││ 知语币 │
               │ 定时任务 ││ 事务   ││ 事务   │
               └─────────┘└────────┘└────────┘
```

- 注册绑定 → 记录推荐关系（推荐人 + 被推荐人）
- 被推荐人购课 → 创建返利记录（冷却中）→ 30 天定时任务 → 检查是否退款 → 发放知语币
- 退款 → 若冷却期内：取消返利记录 → 若已发放：扣回知语币（可能导致负余额）

## 二、非功能需求

### 2.1 支付安全

| 要求 | 说明 |
|------|------|
| 不经手用户支付信息 | 所有支付信息由 Paddle 收集和处理，客户端和后端均不接触银行卡号、CVV 等 |
| Paddle Webhook 验签 | 所有 Paddle Webhook 请求必须验证签名（`Paddle-Signature` Header），防止伪造 |
| 幂等性 | 支付回调处理必须幂等（同一事件多次投递不重复处理），使用 Paddle Event ID 去重 |
| 订单状态机 | 订单状态只能按预定义路径流转（待支付 → 已支付 → 已退款），禁止跳跃或回退 |
| 金额校验 | 后端创建订单时独立计算价格，不信任客户端传入的金额，与 Paddle 返回金额交叉校验 |
| 知语币抵扣原子性 | 抵扣知语币和创建订单在同一事务内完成，支付失败自动回退 |
| 退款限制 | 退款由 Paddle 处理（MoR 模式），后端仅处理 Paddle 退款 Webhook，不自行发起退款 |

### 2.2 数据加密

| 层面 | 要求 |
|------|------|
| 传输加密 | 全站 HTTPS（TLS 1.2+），禁止 HTTP 明文传输 |
| 敏感数据存储 | 用户密码使用 bcrypt/scrypt 哈希存储（不可逆），不存储明文 |
| Token 安全 | JWT Token 使用 RS256 签名，Access Token 有效期 ≤ 15 分钟，Refresh Token ≤ 7 天 |
| 对象存储 | 头像/证书图片使用签名 URL 访问，过期时间 ≤ 1 小时 |
| 日志脱敏 | 日志中不记录用户密码、Token、支付信息；邮箱/手机号部分脱敏 |
| 本地存储 | 客户端不在 localStorage 中存储敏感信息（Token 存储在 httpOnly Cookie 或安全存储中） |

### 2.3 GDPR 合规

| 要求 | 说明 |
|------|------|
| 数据最小化 | 仅收集业务必要的数据（姓名、邮箱、学习数据），不收集无关个人信息 |
| 知情同意 | 注册时展示隐私政策摘要 + 同意勾选框，详细政策可查看全文 |
| 数据导出权 | 用户有权导出个人数据（P2 功能，后续版本实现） |
| 删除权 | 注销账号功能满足"被遗忘权"要求（详见 07-settings.md §九） |
| 冷静期 | 注销 30 天冷静期，期间可撤销 |
| 数据保留 | 支付记录保留 7 年（税务合规要求），其余数据注销后永久删除 |
| 匿名化 | 注销后统计数据匿名化（不可关联到个人），用于产品分析 |
| 第三方告知 | 隐私政策明确列出所有第三方数据处理者（Paddle、云服务商等） |
| Cookie 政策 | 仅使用必要 Cookie + 功能性 Cookie，非必要 Cookie 需用户同意 |

### 2.4 性能要求

| 指标 | 目标 |
|------|------|
| 个人中心首页加载 | ≤ 1.5s（含头像和统计数据） |
| 课程列表加载 | ≤ 1s |
| 知语币余额查询 | ≤ 500ms |
| 知语币流水分页 | ≤ 800ms / 页 |
| 支付订单创建 | ≤ 2s |
| 支付结果轮询 | 2s 间隔，最长 30s 超时 |
| 推荐码验证 | ≤ 500ms（实时校验） |
| 设置项切换 | ≤ 300ms（本地即时生效） |
| 证书图片生成 | ≤ 3s（客户端生成 1080×1440 PNG） |

### 2.5 可用性要求

| 要求 | 说明 |
|------|------|
| 离线可用 | 个人中心首页、课程列表、知语币余额使用本地缓存，离线可查看（标注数据可能不是最新） |
| 降级策略 | Paddle 不可用时：禁用购买按钮 + 提示"支付暂不可用"；不阻塞其他功能 |
| 重试机制 | 网络请求失败自动重试 2 次（间隔 1s、2s），重试后仍失败展示错误态 |
| 最终一致性 | 推荐返利和知语币流水采用最终一致性，允许短暂延迟但保证最终正确 |

### 2.6 国际化要求

| 要求 | 说明 |
|------|------|
| UI 多语言 | 所有界面文案支持中文/英文/越南语三语切换 |
| 金额显示 | 购买金额以 USD 为基准，Paddle 自动展示用户当地货币等价（如 VND） |
| 日期格式 | 中文：`YYYY-MM-DD`，英文：`MMM DD, YYYY`，越南语：`DD/MM/YYYY` |
| 时区 | 所有时间存储为 UTC，展示时转换为用户本地时区 |
