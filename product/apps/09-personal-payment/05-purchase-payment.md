## 一、购买入口

用户可通过以下入口发起课程购买：

| 入口 | 位置 | 按钮文案 | 跳转目标 |
|------|------|---------|---------|
| 课程地图页 Level 节点 | 系统课程 Tab → 点击锁定的 L4-L12 节点 | 弹出购买弹窗 | 课程详情确认页 |
| 课程地图页付费墙 | 系统课程 Tab → L3 学完进入 L4 | "解锁本级课程：$6" | 课程详情确认页 |
| 我的课程页"去购买"按钮 | 个人中心 → 我的课程 → 未购 Level 卡片 | "去购买" | 课程详情确认页 |
| 我的课程页"续费"按钮 | 个人中心 → 我的课程 → 到期/即将到期卡片 | "续费" | 续费确认弹窗（见 `02-my-courses.md`） |
| 知语币足额时的引导 | 多个页面均可出现 | "使用 600 知语币兑换" | 课程详情确认页（知语币全额抵扣模式） |

### 2.4 知语币抵扣逻辑

| 场景 | 行为 |
|------|------|
| 知语币余额 ≥ 600 | 勾选后全额抵扣：应付金额 = $0.00，按钮文案改为"使用 600 知语币兑换" |
| 知语币余额 > 0 但 < 600 | 勾选后部分抵扣：应付金额 = $6.00 - (余额 × $0.01)，按钮文案显示实际应付金额 |
| 知语币余额 ≤ 0 | 复选框不可勾选，灰色禁用态，旁边文字"余额不足" |

**全额抵扣时**：
- 不调用 Paddle 支付
- 点击"使用 600 知语币兑换"→ 弹出确认弹窗："确定使用 600 知语币兑换 L{N} 课程？" + \[确认\] + \[取消\]
- 确认后 → 扣除知语币 → 直接跳转购买成功页

**部分抵扣时**：
- 差额通过 Paddle 支付
- 按钮文案显示实际需付金额，如"确认购买 $2.80"

## 三、Paddle 支付页

### 3.1 触发条件

点击"确认购买"按钮（应付金额 > $0 时）→ 调起 Paddle Checkout。

### 3.2 Paddle Checkout 集成

```
点击"确认购买"
    ↓
按钮变为 Loading 态（"正在准备支付…" + 旋转图标）
    ↓
前端调用后端创建 Paddle Transaction → 获得 Paddle Checkout URL/Client Token
    ↓
使用 Paddle.js Overlay Checkout 打开 Paddle 支付页面（弹窗覆盖式）
    ↓
Paddle Checkout 页面展示：
  ┌──────────────────────────────────┐
  │        Paddle Checkout           │
  │                                  │
  │  知语 Zhiyu — L4 扩展视野        │
  │  价格：$6.00 (≈ ₫150,000)       │
  │                                  │
  │  支付方式：                       │
  │  💳 信用卡/借记卡                 │
  │  🅿️ PayPal                      │
  │  📱 Apple Pay / Google Pay       │
  │                                  │
  │  [信用卡号输入框]                  │
  │  [有效期]  [CVV]                  │
  │                                  │
  │  由 Paddle 处理支付               │
  │  价格含税                         │
  │                                  │
  │  [ 🔵 支付 $6.00 ]               │
  │                                  │
  └──────────────────────────────────┘
```

### 3.3 Paddle MoR 说明

Paddle 作为 Merchant of Record（法律卖方），以下内容在 Paddle Checkout 页面由 Paddle 自动处理：

| 项目 | 说明 |
|------|------|
| 本地货币展示 | Paddle 自动根据用户 IP/浏览器语言展示当地等值货币（越南用户看到 VND） |
| 含税价格 | Paddle 自动计算并展示含 VAT/GST 的最终价格 |
| 支付方式 | Paddle 支持信用卡（Visa/Mastercard/JCB）、PayPal、Apple Pay、Google Pay 等 |
| 支付安全 | Paddle 处理所有卡信息，知语不接触支付敏感数据 |
| 退款 | 通过 Paddle 退款流程处理 |

### 3.4 知语平台不展示 / 不处理的内容

- 不存储用户银行卡号
- 不自行处理税务
- 不展示独立的支付方式选择页（由 Paddle Checkout 统一处理）

## 四、支付中等待页

### 4.1 触发条件

用户在 Paddle Checkout 完成支付操作后，Paddle Checkout 关闭，知语前端显示支付等待页。

### 4.2 页面结构

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│         [旋转加载动画]               │
│                                     │
│       支付处理中，请稍候...           │
│                                     │
│    通常需要几秒钟即可完成             │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

| 元素 | 规格 |
|------|------|
| 加载动画 | 知语品牌 Logo 呼吸动画（缩放 0.95→1.05 循环），居中 |
| 主文案 | Body（16px，`font-weight: 600`），居中 |
| 副文案 | Body S（14px，中性灰），居中 |

### 4.3 等待逻辑

```
Paddle Checkout 关闭
    ↓
前端开始轮询后端订单状态（每 2 秒一次）
    ↓
后端收到 Paddle Webhook（transaction.completed）→ 更新订单状态
    ↓
前端轮询发现订单已完成 → 跳转支付成功页
前端轮询发现订单失败 → 跳转支付失败页
    ↓
超过 30 秒未收到确认 → 展示超时提示：
  "支付结果确认中，请稍后在「我的课程」查看"
  + [返回我的课程] 按钮
```

## 七、购买后自动发放推荐返利

### 7.1 返利触发条件

| 条件 | 说明 |
|------|------|
| 购买者有推荐人 | 注册时通过推荐链接/推荐码绑定了推荐人关系 |
| 购买成功 | Paddle Webhook 确认交易完成 |
| 30 天冷却期 | 自购买日起 30 天内无退款 |

### 7.2 返利计算

```
购买金额 $6.00
    ↓
推荐人返利 = $6.00 × 20% = $1.20 = 120 知语币
被推荐人自得 = $6.00 × 20% = $1.20 = 120 知语币
    ↓
30 天冷却期通过（无退款）
    ↓
系统自动发放 120 知语币到推荐人账户
系统自动发放 120 知语币到被推荐人账户
    ↓
推送通知推荐人："您推荐的好友购买了 L{N} 课程，120 知语币已到账！"
推送通知被推荐人："恭喜购买成功！120 知语币推荐奖励已到账！"
```

### 7.3 冷却期内的展示

在推荐人和被推荐人的知语币收支明细中，30 天冷却期内的返利记录显示为：

```
┌─────────────────────────────────────┐
│  推荐返利（冷却中）          +120 币  │
│  被推荐人购买 L4 课程                │
│  预计到账：2026-04-15                │
│  ⏳ 冷却期剩余 28 天                 │
└─────────────────────────────────────┘
```

- 金额文字使用中性灰（非成功色），表示尚未到账
- "冷却中"标签使用 Amber 色背景标签
- 冷却期结束后变为正常绿色成功色 + 移除标签

### 7.4 退款扣回

若被推荐人在 30 天内退款：
- 取消待发放的返利记录（推荐人和被推荐人的返利均取消）
- 在收支明细中该条记录标注"已取消（被推荐人退款）"

若被推荐人在返利已到账后退款（极端情况，30天内通过但后续 Paddle 仍处理退款）：
- 从推荐人和被推荐人账户扣回 120 知语币
- 若余额不足，扣至负数
- 在收支明细中新增一条"退款扣回"记录
