### 2.2 推荐统计区

| 元素 | 规格 |
|------|------|

## 三、返利记录列表

### 3.1 列表位置

在分享渠道和返利规则说明下方，展示返利记录列表。

### 3.2 每条记录结构

| 元素 | 位置 | 规格 |
|------|------|------|
| 被推荐人昵称 | 左上 | Body（16px，`font-weight: 600`），**脱敏处理** |
| 购买课程 | 左中 | Body S（14px，中性灰），格式：`购买 L{N}` |
| 日期 | 左下 | Caption（12px，中性灰），格式：`YYYY-MM-DD` |
| 返利金额 | 右上 | Body（16px，`font-weight: 600`），格式：`+{N} 币` |
| 返利状态 | 右下 | 标签样式 |

#### 昵称脱敏规则

保留首尾各一个字符，中间替换为 `**`：
- `知语小学生` → `知**生`
- `John` → `J**n`
- `AB` → `A**B`（2 字符也需脱敏）

#### 返利状态标签

| 状态 | 标签文字 | 标签颜色 |
|------|---------|---------|
| 冷却中 | `冷却中` | Amber 色背景 10% + Amber 色文字 |
| 已到账 | `已到账` | 语义成功色背景 10% + 成功色文字 |
| 已取消 | `已取消` | 中性灰背景 10% + 中性灰文字 |
| 已扣回 | `已扣回` | 语义错误色背景 10% + 错误色文字 |

### 3.3 列表排序与分页

- 排序：按日期倒序
- 分页：每次加载 20 条，触底加载更多

## 五、被推荐人端的体验

### 5.1 注册时输入推荐码

**方式一：通过推荐链接注册**
- 用户点击推荐链接 `https://zhiyu.app/r/{推荐码}` → 打开知语注册页
- 推荐码自动填入注册表单的"推荐码"字段（只读显示，不可修改）
- 注册成功后自动绑定推荐关系

**方式二：手动输入推荐码**
- 注册表单底部有可选字段："有推荐码？（选填）"
- 点击展开输入框 → 输入推荐码 → 实时校验：
  - 有效：输入框下方显示"✅ 推荐码有效"（成功色）
  - 无效：输入框下方显示"❌ 推荐码无效，请检查"（错误色）

### 5.2 注册后补填推荐码

已注册但注册时未填推荐码的用户，可在以下入口补填：

| 入口 | 位置 |
|------|------|
| 设置页 → 填写推荐码 | 设置页菜单列表中（仅未绑定推荐人的用户显示） |

**补填规则**：
- 仅限注册后 7 天内可补填
- 超过 7 天后该入口自动隐藏
- 补填成功后 Toast"推荐码绑定成功，购课可享返利！"
- 一旦绑定不可修改、不可解绑

### 5.3 被推荐人购买后的返利

被推荐人购买付费课程后，同样获得购买金额 20% 的知语币返利（与推荐人的返利规则一致）：
- 知语币收支明细中显示：`被推荐返利 — 您通过推荐注册，享受购买返利 — +120 币`
- 同样有 30 天冷却期

## 六、负余额处理

### 6.1 退款扣回机制

```
被推荐人申请退款
    ↓
Paddle 处理退款成功
    ↓
系统检查是否有关联的推荐返利
    ↓
【情况 A：返利在冷却期内（尚未发放）】
  → 直接取消返利记录
  → 推荐人和被推荐人的返利记录标注"已取消（被推荐人退款）"
    ↓
【情况 B：返利已发放（冷却期已过）】
  → 从推荐人账户扣回 120 币
  → 从被推荐人账户扣回 120 币
  → 若余额不足：扣至负数
  → 新增收支明细："退款扣回 — 被推荐人退款 L{N} 课程 — -120 币"
```

### 6.2 负余额展示

当知语币余额为负数时：

| 展示位置 | 表现 |
|---------|------|
| 个人中心主页统计卡片 | 知语币余额数字显示为红色负数（如 `-180`） |
| 知语币页面余额区 | 红色大数字 + ⚠️ 提示文案（见 `04-zhiyu-coins.md`） |
| 知语币收支明细 | "退款扣回"记录显示为红色负数 |

### 6.3 负余额限制

| 操作 | 是否允许 |
|------|---------|
| 购买皮肤 | ❌ 禁止，Toast"知语币余额不足" |
| 用知语币抵扣课程 | ❌ 禁止，抵扣复选框禁用 |
| 获得新的知语币 | ✅ 允许，新获得的知语币自动优先抵充负余额 |
| 查看收支明细 | ✅ 允许 |

### 6.4 负余额恢复提示

当用户获得新知语币并从负余额恢复至 ≥ 0 时：
- Toast"知语币余额已恢复正常"
- 如从 -180 获得 200 币 → 最终余额 20 币 → 收支明细显示"+200 币（其中 180 币用于抵充负余额）"

## 七、状态矩阵

| 状态 | 表现 |
|------|------|
| **空状态 (Empty)** | 无推荐记录时：返利记录区域显示"还没有推荐记录，分享给好友开始赚取知语币吧！"；推荐码/二维码/链接正常展示 |
| **加载中 (Loading)** | 推荐统计卡片 Skeleton；推荐码区域 Skeleton；返利记录 Skeleton 列表 |
| **首次加载 (First Load)** | 同加载中 |
| **成功 (Success)** | 复制推荐码/链接成功 Toast；分享渠道调起成功无额外提示 |
| **错误 (Error)** | 推荐数据加载失败：页面中部错误插画 + \[重试\]；二维码生成失败：二维码区域显示"加载失败" + \[重试\] |
| **部分加载 (Partial)** | 推荐码/链接先渲染（静态数据）；统计和返利记录异步加载 |
| **离线 (Offline)** | 顶部离线 Banner；展示上次缓存的推荐码和统计；分享按钮仍可用（复制链接）；返利记录可能不是最新 |
