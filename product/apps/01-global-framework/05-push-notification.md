## 一、通知权限请求时机和方式

### 1.1 核心原则

> **不在首次打开 App 时请求通知权限。** 在用户感受到产品价值后的合适时机再请求。

### 1.2 触发时机

| # | 触发场景 | 触发条件 | 理由 |
|---|---------|---------|------|
| 1 | 完成首节课 | 用户完成第一个课时的学习 + 课时小测验 | 用户已体验核心价值，此时请求学习提醒最自然 |
| 2 | 完成入学测试 | 用户完成入学测试并获得推荐 Level | 用户刚完成测试，对后续学习有期待 |
| 3 | 首次收藏文章 | 用户第一次收藏发现中国的文章 | 用户表现出内容兴趣，可以推送新内容提醒 |

> 以上三个场景，**取最先触发的那个**。一旦用户做出权限选择（允许或拒绝），不再重复请求。

### 1.3 请求方式

#### 预请求引导弹窗（在系统弹窗之前）

先显示一个应用内的引导弹窗（非系统级），让用户了解通知用途，再触发系统权限请求。

| 元素 | 说明 |
|------|------|
| **弹窗类型** | 应用内模态弹窗（居中） |
| **插图** | 顶部铃铛图标（Lucide `bell`，32px，Amber 色）+ 简短装饰 |
| **标题** | "开启学习提醒" / "Turn on Study Reminders" / "Bật nhắc nhở học tập" |
| **内容** | "开启通知后，我们会在你设定的时间提醒你学习，帮助你保持每日进步。你随时可以在设置中关闭。" / "We'll remind you to study at your preferred time. You can turn this off anytime in Settings." / "Chúng tôi sẽ nhắc bạn học vào thời gian bạn chọn. Bạn có thể tắt bất kỳ lúc nào." |
| **主按钮** | "好的，开启通知" / "Enable Notifications" / "Bật thông báo"（Rose 色背景，白色文字，药丸形） |
| **次按钮** | "暂时不用" / "Not now" / "Để sau"（中性灰色文字按钮） |
| **主按钮点击** | 触发浏览器/系统的通知权限请求弹窗 |
| **次按钮点击** | 关闭弹窗，不请求权限。7 天后如用户再次触发上述场景之一，可再次展示引导弹窗（最多重试 2 次，共 3 次机会） |

#### 权限请求结果处理

| 结果 | 行为 |
|------|------|
| **允许** | 成功 Toast："通知已开启" / "Notifications enabled" / "Đã bật thông báo"；后续按规则推送 |
| **拒绝** | 不再弹出引导弹窗；个人中心设置中"推送通知"开关显示为关闭且不可操作，下方提示："通知已在系统设置中关闭，请前往系统设置开启" / "Notifications are disabled in system settings" / "Thông báo đã bị tắt trong cài đặt hệ thống" |

## 二、通知类型清单

### 2.1 学习提醒

| 属性 | 说明 |
|------|------|
| **类型名称** | 每日学习提醒 |
| **触发条件** | 每日固定时间（默认 20:00 本地时间，用户可在设置中调整） |
| **通知标题** | "该学习啦！" / "Time to Study!" / "Đến giờ học rồi!" |
| **通知内容模板** | "今天的学习还没开始哦，来继续 {当前 Level 名称} 吧！" / "You haven't studied today. Continue {Level Name}!" / "Bạn chưa học hôm nay. Tiếp tục {Level Name}!" |
| **点击跳转** | 打开 App → 系统课程 Tab → 当前学习中的 Level |
| **不触发条件** | 用户当天已有学习记录（已完成至少 1 个课时）→ 不发送 |

### 2.2 课程到期提醒

| 属性 | 说明 |
|------|------|
| **类型名称** | 课程到期提醒 |
| **触发条件** | 已购 Level 到期前 30 天 / 7 天 / 1 天各发一次 |
| **通知标题** | "课程即将到期" / "Course Expiring Soon" / "Khóa học sắp hết hạn" |
| **通知内容模板（30天）** | "你的 {Level 名称} 课程将于 {到期日期} 到期，续购享 8 折优惠" / "Your {Level Name} expires on {date}. Renew with 20% off!" / "{Level Name} hết hạn vào {date}. Gia hạn giảm 20%!" |
| **通知内容模板（7天）** | "你的 {Level 名称} 课程还有 7 天到期，别忘了续购" / "Your {Level Name} expires in 7 days" / "{Level Name} còn 7 ngày nữa hết hạn" |
| **通知内容模板（1天）** | "你的 {Level 名称} 课程明天就要到期了！" / "Your {Level Name} expires tomorrow!" / "{Level Name} hết hạn vào ngày mai!" |
| **点击跳转** | 打开 App → 个人中心 → 课程购买 → 对应 Level 续购页 |

### 2.3 赛季通知

| 属性 | 说明 |
|------|------|
| **类型名称** | 赛季提醒 |
| **触发条件** | 赛季结束倒计时 3 天 / 1 天各一次 |
| **通知标题** | "赛季即将结束" / "Season Ending Soon" / "Mùa giải sắp kết thúc" |
| **通知内容模板（3天）** | "赛季还有 3 天结束，冲刺最后段位！当前段位：{段位名}" / "3 days left! Push for your final rank. Current: {rank}" / "Còn 3 ngày! Cố lên hạng cuối. Hiện tại: {rank}" |
| **通知内容模板（1天）** | "赛季明天结束！最后一天，全力冲刺！" / "Season ends tomorrow! Give it your all!" / "Mùa giải kết thúc ngày mai! Cố gắng lên!" |
| **点击跳转** | 打开 App → 游戏模式 Tab → 游戏大厅 |

### 2.4 活动通知

| 属性 | 说明 |
|------|------|
| **类型名称** | 运营活动通知 |
| **触发条件** | 运营人员在管理后台手动配置并推送 |
| **通知标题** | 运营自定义（如 "春节活动开启！"） |
| **通知内容模板** | 运营自定义（如 "登录领取限定皮肤，活动持续至 X 月 X 日"） |
| **点击跳转** | 运营配置的目标页面（如活动页、游戏皮肤商城等） |

### 2.5 退款通知

| 属性 | 说明 |
|------|------|
| **类型名称** | 退款/知语币变动通知 |
| **触发条件** | 退款完成 / 推荐奖励知语币被扣回 |
| **通知标题** | "退款通知" / "Refund Notice" / "Thông báo hoàn tiền" |
| **通知内容模板（退款完成）** | "退款已到账，{Level 名称} 课程权限已回收" / "Refund processed. Access to {Level Name} has been revoked" / "Đã hoàn tiền. Quyền truy cập {Level Name} đã bị thu hồi" |
| **通知内容模板（知语币扣回）** | "因关联退款，{X} 知语币已扣回，当前余额 {Y} 币" / "{X} Zhiyu Coins deducted due to refund. Balance: {Y}" / "Đã trừ {X} Xu Zhiyu do hoàn tiền. Số dư: {Y}" |
| **点击跳转** | 打开 App → 个人中心 → 知语币 |

## 三、通知开关

### 3.1 设置入口

- 位置：个人中心 → 基础设置 → 推送通知

### 3.2 开关设计

| 元素 | 说明 |
|------|------|
| **设置项标签** | "推送通知" / "Push Notifications" / "Thông báo đẩy" |
| **设置项图标** | Lucide `bell`（20px） |
| **控件** | 开关（Toggle Switch），Rose 色选中态 |
| **开启状态** | 接收所有通知类型 |
| **关闭状态** | 不接收任何推送通知 |

### 3.3 特殊情况

| 情况 | 表现 |
|------|------|
| **系统权限已拒绝** | 开关显示为关闭且不可操作（`opacity: 0.45`），下方提示文字："请在系统设置中开启通知权限" / "Please enable notifications in system settings" / "Vui lòng bật thông báo trong cài đặt hệ thống" |
| **从未请求过权限** | 开关显示为关闭，点击开启时触发权限请求流程（引导弹窗 → 系统请求） |

## 四、应用内通知表现

### 4.1 App 在前台时收到通知

当用户正在使用 App 时收到推送通知，**不弹出系统级通知**，改为应用内 Toast 提示。

| 属性 | 说明 |
|------|------|
| **表现形式** | 应用内 Toast（信息类型，Sky 色左侧色条） |
| **位置** | 屏幕顶部居中 |
| **内容** | 通知标题 + 通知内容摘要（单行截断） |
| **持续时间** | 4 秒自动消失 |
| **点击行为** | 点击 Toast → 跳转到对应通知的目标页面 |
| **可关闭** | 向右滑动关闭 |

### 4.2 App 在后台/未打开时

- 使用系统级推送通知（浏览器 Push Notification / PWA Push）
- 通知图标：知语 Zhiyu App 图标
- 点击通知 → 打开 App → 跳转到对应目标页面

## 五、状态矩阵

| 状态 | 表现 |
|------|------|
| **空状态** | 不适用 |
| **加载中** | 不适用 |
| **首次加载** | 不适用 |
| **成功** | 权限请求成功 → Toast 确认；通知发送成功对用户不可见 |
| **错误** | 推送服务不可用 → 静默失败，不向用户报错（后台自动重试） |
| **部分加载** | 不适用 |
| **离线** | 离线期间的通知在恢复网络后由系统补发（如系统支持）；应用内通知仅在联网时展示 |

## 六、用户故事验收

- 「作为新用户，我不想一打开 App 就被要求开启通知，希望在感受到价值后再决定」
- 「作为学习者，我想要每天收到学习提醒，以便养成学习习惯」
- 「作为付费用户，我想要在课程快到期时收到提醒，以便及时续购」
- 「作为用户，我想要随时关闭推送通知，以便在不需要时不被打扰」
- 「作为用户，我在使用 App 时不希望被系统通知弹窗打断，希望看到不打扰的应用内提示」
