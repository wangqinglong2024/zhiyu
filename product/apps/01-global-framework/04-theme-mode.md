# 深色/浅色模式

> 关联文档：[UI 设计规范 §二.1 / §七](../../../grules/ui-design.md)
> 优先级：P0 | 情绪目标：舒适、体贴、个性化

---

## 一、切换方式

### 1.1 三种模式

| 模式 | 说明 | 图标（Lucide） |
|------|------|--------------|
| **浅色模式** | 固定使用浅色主题，不随系统变化 | `sun` |
| **深色模式** | 固定使用深色主题，不随系统变化 | `moon` |
| **跟随系统** | 自动跟随操作系统的深色/浅色设置 | `monitor`（或 `laptop`） |

### 1.2 默认值

- 首次打开 App → 默认选择**跟随系统**
- 若操作系统无法检测（极少数情况）→ 回退为浅色模式

### 1.3 切换控件

- 使用**三选一分段选择器**（Segmented Control）
- 三个选项并排展示，每个选项包含图标 + 文字标签
- 选中态：背景 Rose 色 10% 透明度 + 文字 Rose 色
- 未选中态：文字中性灰色

| 选项 | 浅色模式标签 | 深色模式标签 | 跟随系统标签 |
|------|-----------|-----------|-----------|
| 汉语 | 浅色 | 深色 | 跟随系统 |
| 英语 | Light | Dark | System |
| 越南语 | Sáng | Tối | Hệ thống |

| 入口 | 位置 | 说明 |

## 三、切换后的页面表现

### 3.1 即时生效

- 选择新模式后，**整个应用界面立即切换**，无需刷新页面或重新加载
- 所有页面的背景色、文字色、组件色、毛玻璃材质参数同步切换

### 3.2 过渡动效

| 属性 | 值 |
|------|-----|
| **过渡时长** | 300ms |
| **缓动曲线** | Standard（`cubic-bezier(0.2, 0, 0, 1)`） |
| **过渡属性** | 背景色、文字色、边框色、阴影（使用 `transition-colors`） |
| **注意** | 图片和图标不参与过渡动效，避免闪烁 |

### 3.3 色彩参数切换

| 参数 | 浅色模式 | 深色模式 |
|------|---------|---------|
| **页面背景** | 白色至浅灰 | 近黑色 |
| **文字主色** | 深色 | 浅色至白色 |
| **毛玻璃背景** | 白色 25% 透明度 | 白色 7% 透明度 |
| **毛玻璃边框** | 白色 45% 透明度 | 白色 12% 透明度 |
| **毛玻璃内高光** | 白色 50% 透明度 | 白色 8% 透明度 |
| **阴影浓度** | 低浓度 | 高浓度 |
| **Rose/Sky/Amber** | 使用各自 Light 变体色值 | 使用各自 Dark 变体色值 |

### 3.4 特殊组件处理

| 组件 | 浅色模式 | 深色模式 |
|------|---------|---------|
| Tab Bar | 白色 95% 透明度背景 | 深色 90% 透明度背景 |
| Toast | 浅色背景 + 深色文字 | 深色背景 + 浅色文字 |
| 模态弹窗遮罩 | 黑色 50% 透明度 | 黑色 70% 透明度 |
| 骨架屏 | 浅灰色渐变 | 深灰色渐变 |
| Apple 登录按钮 | 深色背景 + 白色文字 | 白色背景 + 深色文字 |

## 四、持久化

### 4.1 存储规则

| 规则 | 说明 |
|------|------|
| **本地存储** | 用户选择的模式保存到浏览器本地存储 |
| **下次打开** | App 启动时读取本地存储的模式设置，立即应用（避免白屏闪烁） |
| **跟随系统模式** | 若选择"跟随系统"，App 启动时先读取系统当前模式再渲染（防止先显示浅色再切换为深色的闪烁） |
| **登录同步** | 已登录用户的模式选择同步到服务端；新设备登录时从服务端拉取 |
| **未登录用户** | 仅保存在本地；登录后自动同步至服务端 |

### 4.2 系统模式变化监听

- 选择"跟随系统"时，App 实时监听系统深色模式变化
- 用户在系统设置中切换深色/浅色 → App 自动跟随切换，使用 300ms 过渡动效
- 无需用户手动刷新或重新打开 App

## 五、状态矩阵

| 状态 | 表现 |
|------|------|
| **空状态** | 不适用 |
| **加载中** | 不适用（纯本地操作） |
| **首次加载** | App 启动时根据本地存储/系统设置立即应用正确模式，不出现模式闪烁 |
| **成功** | 切换即时生效，无需 Toast 提示（用户可直接看到视觉变化） |
| **错误** | 不适用（纯本地操作） |
| **部分加载** | 不适用 |
| **离线** | 主题切换正常工作（纯本地操作，不依赖网络） |

## 六、用户故事验收

- 「作为用户，我想要在夜间使用深色模式，以便保护眼睛」
- 「作为用户，我想要 App 自动跟随手机系统的深色模式设置，以便与其他应用保持一致」
- 「作为用户，我想要切换主题后立即看到变化，不需要重新打开应用」
- 「作为用户，我想要下次打开 App 时保持上次的主题选择」
