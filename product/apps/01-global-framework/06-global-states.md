# 全局状态规范

> 关联文档：[UI 设计规范 §三.2](../../../grules/ui-design.md) | [PRD 规范 §一.2](../../../grules/product-design.md)
> 优先级：P0 | 情绪目标：安心、可控、始终有反馈
>
> 本文档定义的 7 种全局状态规范适用于所有页面和模块。各模块 PRD 直接引用本文档，不再重复定义状态表现。

---

## 一、加载状态（骨架屏规范）

### 1.1 核心原则

> **禁止纯 Spinner**。所有加载状态必须使用骨架屏（Skeleton Screen），形状匹配最终内容布局。

### 1.2 骨架屏形状规则

| 内容类型 | 骨架屏形状 | 说明 |
|---------|-----------|------|
| **标题文字** | 圆角矩形条，宽度 60%-80%，高度与标题字号一致 | 圆角 `--radius-sm`（8px） |
| **正文段落** | 3-4 行圆角矩形条，宽度依次递减（100% → 90% → 70%） | 模拟文字排版的参差感 |
| **头像/缩略图** | 圆形（头像）或矩形（封面图），尺寸与实际一致 | 矩形使用 `--radius-md`（12px） |
| **按钮** | 药丸形矩形，尺寸与实际按钮一致 | 圆角 `--radius-full` |
| **卡片** | 完整卡片轮廓，内部含缩略图 + 标题 + 辅助信息骨架 | 圆角 `--radius-2xl`（24px） |
| **列表项** | 左侧圆形/矩形 + 右侧 2 行矩形条 | 单项高度与实际列表项一致 |

### 1.3 骨架屏动画

| 属性 | 值 |
|------|-----|
| **动画类型** | 从左到右的光泽扫描（Shimmer Effect） |
| **动画方向** | 从左到右 |
| **动画时长** | 1.5 秒一个周期 |
| **动画缓动** | 线性循环（`linear`，`infinite`） |
| **浅色模式** | 骨架色块为浅灰色，光泽为白色半透明渐变 |
| **深色模式** | 骨架色块为深灰色，光泽为浅灰色半透明渐变 |

### 1.4 骨架屏持续条件

- 骨架屏持续显示直到数据加载完成
- 数据加载完成 → 骨架屏淡出（200ms），实际内容淡入（200ms）
- 若加载超过 10 秒 → 自动切换到错误状态（详见第四节）

### 1.5 骨架屏布局数量

- 列表页：展示 3-5 个列表项骨架（铺满首屏可见区域）
- 卡片网格页：展示 4-6 张卡片骨架
- 详情页：展示标题 + 封面图 + 3-4 段正文骨架

## 二、首次加载（全屏骨架屏 + 品牌 Logo 呼吸动画）

### 2.1 触发条件

- App 首次打开（冷启动）
- 清除缓存后重新打开

### 2.2 页面结构

```
┌─────────────────────────────────────┐
│                                     │
│                                     │
│        [知语 Zhiyu Logo]             │
│         (呼吸动画)                   │
│                                     │
│        "加载中…"                     │
│                                     │
│                                     │
│   ┌───────────────────────────┐     │
│   │     骨架屏内容区域          │     │
│   │     （模拟首页布局）        │     │
│   └───────────────────────────┘     │
│                                     │
│   ┌─┐  ┌─┐  ┌─┐  ┌─┐              │
│   │ │  │ │  │ │  │ │  ← Tab Bar   │
│   └─┘  └─┘  └─┘  └─┘              │
└─────────────────────────────────────┘
```

### 2.3 品牌 Logo 呼吸动画

| 属性 | 值 |
|------|-----|
| **Logo 位置** | 骨架屏上方居中 |
| **Logo 尺寸** | 宽度 80px |
| **动画效果** | 透明度在 40%-100% 之间循环脉动（呼吸感） |
| **动画时长** | 2 秒一个周期 |
| **缓动曲线** | `ease-in-out` |

### 2.4 加载文案

| 语言 | 文案 |
|------|------|
| 汉语 | "加载中…" |
| 英语 | "Loading…" |
| 越南语 | "Đang tải…" |

- 文字位于 Logo 下方，Body S 级别（14px），中性灰色
- 文字末尾 "…" 使用三点渐现动画（依次出现）

### 2.5 持续时间与退出

- 首次加载骨架屏持续到核心数据（首页内容列表）加载完成
- 加载完成 → 整个首次加载界面淡出（300ms），首页内容以 Stagger 动画依次渐入
- 超过 15 秒未完成 → 显示错误状态页面（详见第四节）

### 3.1 标准空状态结构

```
┌─────────────────────────────────────┐
│                                     │
│          [插画图标]                   │
│          (64px × 64px)              │
│                                     │
│      "这里还没有内容"                 │
│      (引导文案主标题)                 │
│                                     │
│      "去发现中国看看吧"               │
│      (引导文案副标题)                 │
│                                     │
│       ┌──────────────┐              │
│       │   CTA 按钮    │              │
│       └──────────────┘              │
│                                     │
└─────────────────────────────────────┘
```

### 3.2 各场景空状态定义

| 页面/场景 | 插画图标 | 主标题 | 副标题 | CTA 按钮 |

| **课程列表为空（未购买）** | Lucide `book-open`（64px） | "还没有购买课程" / "No courses yet" / "Chưa có khóa học" | "从 Level 1 开始你的学习之旅" / "Start your journey from Level 1" / "Bắt đầu hành trình từ Level 1" | "查看课程" / "View Courses" / "Xem khóa học" |

### 3.3 空状态视觉规范

- 插画图标：使用 Lucide Icons，尺寸 64px，颜色为中性灰 40% 透明度
- 主标题：H3 级别（20px），字重 600，主文字色
- 副标题：Body S 级别（14px），中性灰色
- CTA 按钮：药丸形，Rose 色背景，白色文字，高度 40px
- 整体居中布局，各元素间距 `--space-3`（12px）
- 页面背景可添加 3-4 个 `.glass-decor` 浮动装饰方块（低透明度，增加品牌氛围）

## 四、错误状态

### 4.1 错误文案规范

> **铁律：所有错误提示必须"说人话"。** 禁止技术术语（如 "Error 500"、"timeout"、"null reference"）。

### 4.2 错误类型与文案

| 错误类型 | 主标题 | 副标题 | 重试按钮 |
|---------|--------|--------|---------|
| **网络连接失败** | "网络不给力" / "Connection issue" / "Kết nối có vấn đề" | "请检查网络后重试" / "Please check your connection and try again" / "Vui lòng kiểm tra kết nối và thử lại" | "重试" / "Retry" / "Thử lại" |
| **服务器错误** | "服务开小差了" / "Something went wrong" / "Có lỗi xảy ra" | "请稍后再试，我们正在修复中" / "Please try again later, we're working on it" / "Vui lòng thử lại sau, chúng tôi đang khắc phục" | "重试" / "Retry" / "Thử lại" |
| **页面不存在** | "页面走丢了" / "Page not found" / "Không tìm thấy trang" | "你要找的页面不存在或已被移除" / "The page you're looking for doesn't exist" / "Trang bạn tìm không tồn tại" | "返回首页" / "Go Home" / "Về trang chủ" |
| **内容加载超时** | "加载超时了" / "Loading timed out" / "Tải quá thời gian" | "请检查网络后重试" / "Please check your connection and try again" / "Vui lòng kiểm tra kết nối và thử lại" | "重试" / "Retry" / "Thử lại" |
| **权限不足** | "无法访问" / "Access denied" / "Không thể truy cập" | "请登录后再试" / "Please log in and try again" / "Vui lòng đăng nhập và thử lại" | "去登录" / "Log in" / "Đăng nhập" |

### 4.3 错误页面视觉规范

- 插画图标：Lucide `alert-circle`（64px，错误色 40% 透明度）
- 主标题：H3 级别（20px），字重 600，主文字色
- 副标题：Body S 级别（14px），中性灰色
- 重试按钮：药丸形，Rose 色背景，白色文字，高度 40px
- 整体居中布局

### 4.4 重试按钮行为

- 点击"重试" → 按钮变为加载态（旋转指示器）→ 重新请求数据
- 重试成功 → 错误页面淡出，正常内容淡入
- 重试失败 → 按钮恢复可点击，错误页面保持，Toast 提示："仍然无法加载，请稍后再试" / "Still unable to load, please try later" / "Vẫn không thể tải, vui lòng thử lại sau"

## 五、部分加载

### 5.1 列表底部加载指示器

| 属性 | 说明 |
|------|------|
| **触发** | 列表滚动到底部，自动触发加载下一页数据 |
| **视觉** | 列表底部出现旋转加载指示器（圆形，24px，Rose 色），居中显示 |
| **持续** | 直到新数据加载完成 |
| **成功** | 指示器消失，新数据以 Stagger 动画依次渐入 |
| **失败** | 指示器替换为文字："加载失败，点击重试" / "Failed to load, tap to retry" / "Tải thất bại, nhấn để thử lại"（点击重试） |
| **无更多数据** | 显示文字："已经到底了" / "That's all" / "Đã hết"（Body S 级别，中性灰色，居中） |

### 5.2 下拉刷新

| 属性 | 说明 |
|------|------|
| **触发** | 在页面/列表顶部向下拉动超过 60px |
| **动画** | 下拉时顶部出现自定义加载动画（知语 Logo 缩小版旋转） |
| **释放后** | 加载动画继续旋转，页面内容下移留出动画空间 |
| **成功** | 加载动画消失，页面内容回弹到顶部，数据刷新 |
| **成功反馈** | Toast 提示："已刷新" / "Refreshed" / "Đã làm mới"（成功类型，2 秒消失） |
| **失败** | 加载动画消失，页面回弹，错误 Toast："刷新失败" / "Refresh failed" / "Làm mới thất bại" |

## 六、离线状态

### 6.1 离线检测

- 实时监听网络连接状态
- 网络断开 → 立即触发离线状态

### 6.2 离线 Banner

| 属性 | 说明 |
|------|------|
| **位置** | 页面顶部（导航栏下方），全宽 Banner |
| **高度** | 36px |
| **背景** | Amber 色 10% 透明度 |
| **图标** | Lucide `wifi-off`（16px，Amber 色），居左 |
| **文字** | "当前无网络连接" / "No internet connection" / "Không có kết nối mạng"，Body S 级别（14px），Amber 色 |
| **出现动效** | 从顶部滑入，200ms Decelerate 缓动 |
| **持续** | 一直显示直到网络恢复 |

### 6.3 离线可用内容

| 内容类型 | 离线表现 |
|---------|---------|
| **已浏览过的页面** | 使用本地缓存展示，可正常阅读 |
| **已缓存的文章** | 正常展示全部内容（含图片，如已缓存） |
| **图片未缓存** | 显示占位色块 + 图片图标（Lucide `image`） |
| **需要网络的操作** | 按钮可见但点击后 Toast 提示："此操作需要网络连接" / "This requires an internet connection" / "Thao tác này cần kết nối mạng" |
| **未缓存的页面** | 显示离线专属错误页："当前内容需要联网查看" / "This content requires an internet connection" / "Nội dung này cần kết nối mạng"，附 "设置" 快捷入口引导用户检查 Wi-Fi |

## 七、网络恢复

### 7.1 恢复检测

- 实时监听网络恢复事件
- 网络恢复 → 立即触发恢复流程

### 7.2 恢复行为

| 行为 | 说明 |
|------|------|
| **Banner 消失** | 离线 Banner 向上滑出消失（200ms Accelerate 缓动） |
| **自动刷新** | 当前页面的数据自动静默刷新（不显示骨架屏或 Loading，后台请求） |
| **刷新成功** | 如果数据有更新，内容区域无缝替换为新数据 |
| **刷新失败** | 静默失败，保持缓存数据，不向用户报错 |
| **Toast 提示** | 网络恢复时显示成功 Toast："网络已恢复" / "Back online" / "Đã kết nối lại"（成功类型，2 秒消失） |
| **排队操作** | 离线期间用户触发的操作（如收藏），网络恢复后自动执行并同步 |
