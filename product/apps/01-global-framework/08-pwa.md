# PWA 行为

> 关联文档：[产品总纲 §一](../../00-product-overview.md)
> 优先级：P1（安装提示）/ P2（离线缓存、版本更新）
> 情绪目标：便捷、可靠、期待

---

## 一、安装提示

### 1.1 触发条件

> 不在首次访问时弹出安装提示。在用户表现出一定粘性后再提示。

| # | 触发条件 | 说明 |
|---|---------|------|
| 1 | 用户第 2 次访问且浏览时长 ≥ 3 分钟 | 表明用户对产品有一定兴趣 |
| 2 | 用户完成首节课学习 | 用户已深度体验核心功能 |
| 3 | 用户注册成功后首次回到首页 | 新注册用户的高粘性时刻 |

> 以上三个条件取最先触发的那个。

### 1.2 提示样式

#### 应用内安装引导 Banner

| 属性 | 说明 |
|------|------|
| **位置** | 页面顶部，Tab 内容区域上方，全宽 Banner |
| **高度** | 64px |
| **背景** | 毛玻璃面板，Rose 色 5% 透明度叠加 |
| **左侧** | 知语 App 图标（32px 圆角方形） |
| **中间内容** | 标题："添加知语到主屏幕" / "Add Zhiyu to Home Screen" / "Thêm Zhiyu vào Màn hình chính"（Body S 级别，主文字色）；副标题："更快的访问速度" / "Faster access" / "Truy cập nhanh hơn"（Caption 级别，中性灰色） |
| **右侧按钮** | "安装" / "Install" / "Cài đặt"（小型药丸按钮，Rose 色背景，白色文字，高度 32px） |
| **关闭按钮** | 最右侧 × 图标（Lucide `x`，16px），点击关闭 Banner |
| **出现动效** | 从顶部滑入，300ms Decelerate 缓动 |

#### 点击"安装"后

- 触发浏览器原生的 PWA 安装提示
- 用户确认安装 → Banner 消失 + Toast："安装成功" / "Installed successfully" / "Cài đặt thành công"
- 用户取消安装 → Banner 消失，不显示提示

### 1.3 忽略后的再次提示策略

| 场景 | 策略 |
|------|------|
| **点击 × 关闭** | 7 天内不再显示 Banner |
| **7 天后** | 如用户仍未安装，再次在满足触发条件时显示 Banner（最多 3 次） |
| **3 次关闭后** | 永不再主动提示。用户仍可通过浏览器自带的安装选项安装 |
| **已安装** | 永不显示安装 Banner |

### 1.4 检测 PWA 已安装

- 通过 `display-mode: standalone` 媒体查询检测
- 已安装状态下不显示任何安装提示

## 二、离线缓存

### 2.1 离线可用内容

| 内容类型 | 离线可用 | 说明 |
|---------|---------|------|
| **App Shell（界面框架）** | 是 | Tab Bar、导航栏、页面骨架等 UI 框架始终可用 |
| **静态资源** | 是 | 图标、字体、CSS、JS 等静态资源缓存后可用 |
| **已浏览的文章** | 是 | 用户浏览过的发现中国文章内容和图片 |
| **已学习的课时** | 是 | 用户学习过的课时内容 |
| **个人设置** | 是 | 语言、主题、学习模式等配置 |
| **未浏览的内容** | 否 | 未曾加载过的文章、课时需联网获取 |
| **实时数据** | 否 | 排行榜、段位数据、通知列表需联网 |
| **游戏匹配** | 否 | PK/多人对战必须联网 |
| **支付** | 否 | 课程购买/皮肤购买必须联网 |
| **登录/注册** | 否 | 认证操作必须联网 |

### 2.2 缓存策略

| 资源类型 | 缓存策略 | 说明 |
|---------|---------|------|
| **App Shell + 静态资源** | 优先缓存（Cache First） | 先读缓存，定期后台更新 |
| **内容数据（文章/课时）** | 网络优先（Network First） | 有网络时获取最新数据并更新缓存；无网络回退到缓存 |
| **图片** | 优先缓存（Cache First） | 加载过一次后缓存，减少流量消耗 |
| **用户数据（进度/设置）** | 网络优先 | 确保数据最新；离线时读取本地缓存 |

### 2.3 缓存大小管理

| 规则 | 说明 |
|------|------|
| **缓存上限** | 应用总缓存不超过 100MB |
| **清理策略** | 超出上限时，按 LRU（最近最少使用）策略清理最旧的缓存内容 |
| **手动清理** | 个人中心 → 基础设置 → "清除缓存"按钮 |
| **清除缓存后** | 仅保留 App Shell 和静态资源缓存；所有内容数据需重新联网加载 |

## 三、版本更新提示

### 3.1 检测新版本

- App 每次启动时检查是否有新版本可用
- 使用中时每隔 1 小时后台静默检查
- 检测到新版本后触发更新提示

### 3.2 更新提示方式

#### 非强制更新（默认）

| 属性 | 说明 |
|------|------|
| **提示形式** | 页面顶部 Banner（与安装提示位置相同，不同时出现） |
| **高度** | 56px |
| **背景** | 毛玻璃面板，Sky 色 5% 透明度叠加 |
| **图标** | Lucide `download`（20px，Sky 色），居左 |
| **文字** | "新版本可用，刷新以更新" / "New version available, refresh to update" / "Có phiên bản mới, làm mới để cập nhật"（Body S 级别） |
| **操作按钮** | "刷新" / "Refresh" / "Làm mới"（小型药丸按钮，Sky 色背景，白色文字，高度 32px） |
| **关闭按钮** | × 图标，点击关闭。下次启动或下次检测到时再次提示 |
| **点击"刷新"** | 页面重新加载，应用新版本资源 |

#### 强制更新（重大更新/安全修复时）

| 属性 | 说明 |
|------|------|
| **提示形式** | 全屏模态弹窗，不可关闭 |
| **标题** | "需要更新" / "Update Required" / "Cần cập nhật" |
| **内容** | "此版本需要更新才能继续使用" / "This version requires an update to continue" / "Phiên bản này cần cập nhật để tiếp tục sử dụng" |
| **按钮** | "立即更新" / "Update Now" / "Cập nhật ngay"（Rose 色药丸按钮，全宽） |
| **点击按钮** | 页面重新加载 |
| **无关闭方式** | 不可点击遮罩关闭，不可按 Esc，必须更新 |

### 3.3 更新后反馈

- 更新完成后（页面重新加载）→ 不显示额外提示，用户直接进入最新版本
- 后台自动完成缓存更新，无需用户感知

## 四、状态矩阵

| 状态 | 表现 |
|------|------|
| **空状态** | 不适用 |
| **加载中** | 不适用 |
| **首次加载** | PWA 安装提示不在首次加载时出现 |
| **成功** | 安装成功 → Toast 确认；更新成功 → 静默完成 |
| **错误** | 安装失败 → Toast："安装失败，请稍后重试" / "Install failed, try again later" / "Cài đặt thất bại, thử lại sau" |
| **部分加载** | 不适用 |
| **离线** | 离线时不显示安装/更新提示；离线缓存策略自动生效 |

## 五、用户故事验收

- 「作为常访用户，我想要把知语安装到主屏幕，以便像原生 App 一样快速打开」
- 「作为用户，我不想第一次访问就被打扰安装，希望在适当时机再收到提示」
- 「作为用户，我想要在无网络时仍能阅读之前浏览过的文章」
- 「作为用户，我想要在有新版本时收到温和的提醒，而不是被强制打断」
