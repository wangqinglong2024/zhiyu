## 一、会员体系

### 1.1 免费用户 vs 会员权益对比表

| 功能模块 | 免费用户 | 会员（Premium） |
|---------|---------|----------------|
| **世界地图** | 每个区域部分免费（详见下方） | 全部 60 关解锁 |
| **拼音群岛**（20 关）| 全部 20 关免费 | 全部 20 关 |
| **汉字谷地**（15 关）| 前 8 关免费 | 全部 15 关 |
| **词汇平原**（15 关）| 前 7 关免费 | 全部 15 关 |
| **语法要塞**（10 关）| 前 5 关免费 | 全部 10 关 |
| **免费关卡合计** | 40 关（占 67%） | 60 关（100%） |
| **体力系统** | 3 颗心，耗尽等 30 分钟或看广告恢复 1 颗 | 无限体力 |
| **迷你游戏** | 每日 3 次机会 | 无限次 |
| **词库复习** | ≤ 20 词/日复习 | 无限复习 |
| **排行榜** | 只看 Top 10 | 完整 Top 100 + 排名 |
| **过场动画** | 免费关卡对应的动画（约 2-3 个） | 全部 8 个 |
| **离线学习** | 不支持 | 已完成关卡可离线缓存 |
| **广告** | 每 3 关插页广告，最长 15 秒，3 秒后可跳过 | 无广告 |
| **专属徽章** | 无 | 会员专属头像框 |
| **每月钻石** | 无 | 每月赠送 300 钻石 |

### 1.2 免费内容的边界设计

**核心原则**：
- 免费用户能体验到**每个区域**的内容，感受游戏价值
- 免费到付费的过渡**平滑无差别**——不弹任何"您已达到免费限制"的硬性拦截
- 到达免费边界时，下一关显示为"🔒"状态 → 点击弹出「升级会员」引导

## 二、订阅价格方案

### 2.1 定价

> 基于越南市场购买力和竞品分析（Duolingo Super：$6.99/月）

| 方案 | USD 定价 | VND 参考价 | 折算月费 | 折扣 |
|------|---------|-----------|---------|------|
| 月费 | $2.99/月 | ≈69,000₫/月 | $2.99 | — |
| 季费 | $6.99/季 | ≈165,000₫/季 | $2.33 | -22% |
| 年费 | $19.99/年 | ≈469,000₫/年 | $1.67 | -44% |
| 终身 | $49.99 | ≈1,179,000₫ | — | — |

**MVP 上线促销**：
- 首月 $0.99 促销（适用于月费方案）
- 促销标签：「Ưu đãi đặc biệt（特别优惠）」
- 首月结束后自动按原价续费

### 2.2 Stripe 支付配置

**支持的支付方式**：
- 国际信用卡 / 借记卡（Visa、Mastercard、JCB）
- Stripe 结算币种：USD
- 用户在页面上看到的价格：双显（VND + USD），以 VND 为主

**不在 MVP 范围**：
- 越南本地支付（MoMo、VNPay、ZaloPay）→ V2
- Apple Pay / Google Pay → V2（建 App 后接入）

### 4.4 到期降级逻辑

会员到期时：
- 已完成的付费关卡进度**保留**（不回退）
- 付费关卡重新锁定（不能重新进入）
- 体力恢复为 3 颗心限制模式
- 迷你游戏恢复 3 次/日限制
- 词库复习恢复 20 词/日限制
- 广告重新展示
- 会员专属头像框移除

## 六、广告系统（免费用户）

### 6.1 广告位

| 广告位 | 类型 | 触发条件 | 频率 |
|-------|------|---------|------|
| 关卡结算后 | 插屏广告 (Interstitial) | 关卡完成后，结算页面展示前 | 每 3 关 1 次 |
| 体力恢复 | 激励视频 (Rewarded) | 用户主动点击"看广告恢复体力" | 用户主动触发 |
| 迷你游戏续命 | 激励视频 (Rewarded) | 游戏结束后"看广告继续" | 用户主动触发 |

### 6.2 广告规则

| 插屏广告：最长 15 秒，3 秒后可点击跳过，有倒计时显示
- 激励视频：完整观看（15-30 秒）后发放奖励
- 会员用户：完全跳过所有广告
- 每日广告上限：插屏最多 10 次，激励视频无上限
- 广告加载失败时：静默跳过，不影响用户流程

## 八、数据流与用户故事

### 8.1 支付→激活数据流

```
用户点击订阅按钮
  ↓
前端调用 POST /api/subscriptions/create-checkout
  ↓
FastAPI 创建 Stripe Checkout Session（包含 plan_id + user_id + promotion_code）
  ↓
重定向到 Stripe Checkout 托管页面
  ↓
用户完成支付
  ↓
Stripe 发送 checkout.session.completed Webhook
  ↓
FastAPI Webhook Handler：
  1. 验证 Stripe 签名
  2. 更新 Supabase users.membership_tier = 'premium'
  3. 更新 users.membership_expires_at
  4. 创建 payment_records 记录
  5. 如有 referral_code，创建 commission 记录（冷却 30 天）
  ↓
前端回调页面检测到状态变化 → 显示“激活成功”动效 → 刷新权益
```

**异常分支**：
- Webhook 延迟（>30s）：前端轮询 GET /api/subscriptions/status，最多 3 次，间隔 5s
- Webhook 失败：Stripe 自动重试（8 次，72h 内）；同时记录技术告警
- 支付成功但激活失败：显示“Vui lòng liên hệ hỗ trợ” + 客服可手动激活

### 8.2 核心用户故事

| # | 功能 | 用户故事 | P |
|---|------|---------|---|
| 1 | 订阅 | 作为免费用户，我想知道会员能解锁什么，以便决定是否值得付费 | P0 |
| 2 | 取消订阅 | 作为会员，我想随时取消订阅且到期前仍享受权益，以便无忧无虑地试用 | P0 |
| 3 | 广告 | 作为免费用户，我不希望广告过于影响学习流程，3 秒后可跳过可以接受 | P0 |

### 8.3 支付表单验证

> 支付表单由 Stripe Checkout 托管，无需前端验证。前端仅验证促销码格式：
> - Coupon Code 格式：大写字母+数字，6-20 字符
> - 配合 Stripe Promotions API 实时验证有效性
