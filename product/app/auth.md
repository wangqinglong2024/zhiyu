# 应用端模块：认证（登录/注册）

> 依赖全局文件：`product/GLOBAL.md`
> 涉及表：`users`, `wallets`
> 涉及 API：`/auth/send-sms`, `/auth/login`, `/auth/wechat`

---

## 一、登录方式

### 1.1 手机号 + 短信验证码（MVP 主力）
- 手机号输入框 → 发送验证码按钮 → 验证码输入框 → 登录/注册
- 新手机号：自动创建账号
- 已注册手机号：直接登录

### 1.2 微信 H5 端：微信 OAuth 登录
- H5 页面在微信浏览器内打开时，可选「微信一键登录」
- 后端通过微信 OAuth2 授权码流程换取 openid（需公众号授权）
- 获取手机号需用户点击微信内「授权」弹窗（或引导绑定手机号）
- 与手机号账号自动合并（同一手机号视为同一账号）
- **注意：** 微信 H5 使用 OAuth 2.0（非小程序 `wx.login`），需绑定公众号域名

**用户唯一标识：** Supabase 的 `uuid`，不用手机号做主键。

---

## 二、验证码防刷（上线前必须实现）

| 规则 | 限制 |
|------|------|
| 同一手机号 | 60秒内只能发1次 |
| 同一 IP | 1小时内最多10次 |
| 超限提示 | 「发送过于频繁，请稍后重试」 |

**实现方式：** FastAPI 进程内字典实现内存缓存（MVP 单容器部署足够），不依赖 Redis。

---

## 三、JWT 有效期

登录成功后签发 JWT，有效期 **30 天**。

---

## 四、邀请码绑定（注册时）

- 若 URL 含 `?invite=XXXXXX` → 前端在登录请求中附带 invite_code
- FastAPI 查询 users 表找到邀请人 → 写入 `users.invited_by`
- **仅首次注册时写入，已有邀请人不覆盖**

## 五、完整数据流

```
用户                    前端                    FastAPI                 Supabase        SMS服务商
 │                       │                        │                       │               │
 │  输入手机号            │                        │                       │               │
 │──────────────────────▶│                        │                       │               │
 │                       │  POST /auth/send-sms   │                       │               │
 │                       │──────────────────────▶│                       │               │
 │                       │                        │  检查发送频率（内存缓存）│               │
 │                       │                        │  同一号码60s内限1次    │               │
 │                       │                        │  同一IP 1h内限10次     │               │
 │                       │                        │  生成6位验证码          │               │
 │                       │                        │  存入内存(phone→code, TTL 5min)        │
 │                       │                        │──────────────────────────────────────▶│
 │                       │                        │                       │     发送短信   │
 │   收到验证码            │                        │                       │               │
 │──────────────────────▶│                        │                       │               │
 │                       │  POST /auth/login      │                       │               │
 │                       │  {phone, code}         │                       │               │
 │                       │──────────────────────▶│                       │               │
 │                       │                        │  验证 code 是否匹配    │               │
 │                       │                        │  验证是否过期(5min)    │               │
 │                       │                        │  查询 phone 是否存在   │               │
 │                       │                        │──────────────────────▶│               │
 │                       │                        │  [新用户] INSERT users │               │
 │                       │                        │  生成 invite_code(6位) │               │
 │                       │                        │  INSERT wallets        │               │
 │                       │                        │  [老用户] 直接返回      │               │
 │                       │                        │  签发 JWT (30天有效)   │               │
 │                       │◀──────────────────────│                       │               │
 │  登录成功，进入首页      │                        │                       │               │
 │◀──────────────────────│                        │                       │               │
```

## 六、登录页 UI

- 全屏暗色背景 + 中央毛玻璃面板
- 面板内：Logo（文字版）+ 输入区
- 输入框仅保留底部下划线样式，聚焦时泛起金色光晕
- 注册页必须勾选《用户协议》和《隐私政策》
