# 应用端模块：AI报告生成与展示

> 依赖全局文件：`product/GLOBAL.md`
> 涉及表：`orders`（report 字段）
> 涉及外部：Dify REST API

---

## 一、AI 报告生成流程

**触发时机：** 支付回调成功后，由 BackgroundTask 异步执行。

```
FastAPI                         Dify API                        Supabase
   │                               │                               │
   │  UPDATE orders                │                               │
   │  SET status='generating'      │                               │
   │──────────────────────────────────────────────────────────────▶│
   │                               │                               │
   │  POST /v1/workflows/run       │                               │
   │  {inputs: {                   │                               │
   │    category: "career",        │                               │
   │    content: "用户输入的困境"   │                               │
   │  }}                           │                               │
   │──────────────────────────────▶│                               │
   │  (等待响应，最长60秒超时)       │                               │
   │◀──────────────────────────────│                               │
   │  返回 JSON                     │                               │
   │                               │                               │
   │  UPDATE orders                │                               │
   │  SET status='completed',      │                               │
   │  report={...JSON...}          │                               │
   │──────────────────────────────────────────────────────────────▶│
   │                               │                               │
   │  触发佣金结算（见 commission-wallet 模块）                       │
```

### 失败路径
- Dify 超时/报错 → UPDATE orders SET `status='failed'`
- 用户在「我的报告」页看到「生成失败」
- 客服处理：重新触发生成或退款

### 内容校验
- Dify 返回 JSON 必须包含 `core_issue`、`solutions`、`insight` 字段
- 缺少任意字段 → 视为失败，不存储半成品

## 二、报告返回结构

```json
{
  "core_issue": "一句话点出问题本质",
  "solutions": ["路径1 (50-80字)", "路径2", "路径3"],
  "insight": "一句有深度的总结句"
}
```

## 三、生成中页面 UI

- 全屏暗色背景
- 中央：缓慢旋转的几何图形（CSS animation）
- 文字每隔5秒轮换：「正在解析你的处境...」→「正在寻找破局关键...」→「报告即将呈现...」
- 模拟进度条（30秒走完）

**超时处理：** ＞60秒未返回 → 显示「分析耗时较长，我们会在完成后通知你」→ 后台继续生成 → 用户可在「我的」页面查看

## 五、海报生成与分享

**触发：** 点击「保存为图片」或「分享给朋友」

### 海报内容（9:16 竖图）
- 顶部：产品名称 Logo
- 中部：核心症结 + 认知升维句
- 底部：用户专属邀请码二维码 + 「扫码获取你的专属分析」
- 风格：暗色背景 + 鎏金字体 + 毛玻璃面板感

### 技术实现
- 使用 `html2canvas` 库，将报告内容渲染为 Canvas → 导出 PNG

### 邀请码规则
- 每个注册用户有唯一邀请码（注册时生成，6位字母数字）
- 二维码指向：`https://your-domain.com/invite/{邀请码}`
- 新用户通过该链接注册后，**每次付费** → 邀请人自动得 ¥8.64（持续分润，非仅首次）

### 分享
- 微信：调用 JS-SDK `wx.shareAppMessage`（小程序端）或系统原生分享（H5端）
