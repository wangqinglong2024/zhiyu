# 应用端模块：页面UI详细说明

> 依赖全局文件：`product/GLOBAL.md`
> 本文件包含所有应用端页面的详细 UI 布局与交互说明

---

## 一、页面流程总览

```
[落地页/首页] → [选择困局类目] → [输入困境] → [支付 ¥28.8] → [AI生成中] → [报告展示] → [分享/保存]
                                                                               ↓
                                                              [我的页面：历史记录 + 邀请收益]
```

---

## 二、首页（类目选择）

**路由：** `/`

**核心目标：** 让用户快速选择困境类型，进入付费流程。

### 布局
- 顶部：产品名称 Logo（左上）+ 「我的」入口（右上，头像图标）
- 中部：两张大卡片，垂直排列（移动端）/ 并排（PC端）

```
┌─────────────────────────────┐
│  职场困局                    │
│  向上管理、职场博弈、薪资谈判  │
│  [开始分析 →]               │
└─────────────────────────────┘

┌─────────────────────────────┐
│  情感困局                    │
│  恋爱关系、亲密博弈、破局重建  │
│  [开始分析 →]               │
└─────────────────────────────┘
```

- 卡片样式：毛玻璃面板 + 鎏金图标 + 类目名称 + 一行副标题 + 按钮
- 底部：小字「已有 X 人获得分析报告」（从数据库查询累计付费次数）

**MVP 只做 2 个类目。**

## 三、输入困境页

**路由：** `/analyze/{category}`

### 布局
- 顶部：返回按钮（左）+ 类目名称（居中）
- 中部：大文本输入区（毛玻璃面板，多行文本）
  - 占位文字示例：「描述你的处境，越具体效果越好。例如：我在公司工作三年，上个月被空降的新领导针对，连续两次考核被打C，我该怎么办？」
  - 最少 20 字，最多 500 字
  - 实时字数显示（右下角：XX/500）
- 底部：「投入 ¥28.8，获取深度分析」按钮（鎏金渐变）

**合规文案（按钮下方必须展示）：**
> AI生成内容仅供参考，不构成专业职业/心理建议

## 四、我的页面

**路由：** `/profile`
**入口：** 首页右上角头像图标

### 4.1 用户信息区
- 头像（默认灰色圆形，不做上传）
- 手机号（脱敏：138****8888）

### 4.2 收益中心
```
┌─────────────────────────────┐
│  我的收益                    │
│                             │
│  可提现：¥ 38.52            │
│  历史累计：¥ 112.32          │
│                             │
│  [申请提现]                  │
│  满 ¥50 可提现，工作日3天内到账 │
└─────────────────────────────┘
```

### 4.3 我的邀请
- 显示用户邀请码
- 「复制邀请链接」按钮
- 说明文案：「朋友通过你的链接付费后，你得 ¥8.64」
- 分销合规文案：`本平台采用单级分销机制，邀请人仅对直接邀请的用户产生佣金，与多级传销无关。`

### 4.4 我的报告（历史记录）
- 列表展示：类目 + 日期 + 核心症结前20字
- 点击可重新查看完整报告
- 下拉加载更多（每次10条，后端分页）
- 数据库永久保留（`orders` 表是财务记录，不做物理删除）

## 五、邀请落地页

**路由：** `/invite/{邀请码}`

### 内容
- 产品简介（3行文案）
- 「立即体验」按钮 → 跳转注册页
- 注册时后端自动记录 `inviter_code`，绑定邀请关系

### 绑定规则
- 一个用户只能被一个邀请人绑定（先到先得）
- 已注册用户点击邀请链接不重新绑定
