## 一、系统概述

### 1.1 用户故事

> 作为 PlayLingo 用户，我想邀请朋友注册和付费，从中获得佣金奖励，以便获得收入并有传播动力。

### 1.2 核心机制

- **单层推荐（1 级）**：仅推荐人获得被推荐人付费的佣金
- **佣金来源**：被推荐人的**每次续费**都产生佣金（非仅首次）
- **佣金阶梯**：根据推荐人的累计推荐收入升级佣金比例
- **结算周期**：实时计入，30 天后可提现
- **所有用户均可参与**：免费用户和会员均可生成推荐链接

## 二、推荐链接

### 2.1 链接格式

```
https://playlingo.app/?ref={用户唯一推荐码}
```

- 推荐码：6-8 位字母数字组合，自动生成，不可自定义（MVP）
- 每个用户有且仅有一个推荐码

### 2.2 归属绑定规则

| 规则 | 说明 |
|------|------|
| 绑定时机 | 新用户通过推荐链接打开页面时，ref 参数同时存入 localStorage 和 Cookie（7 天有效，双保险防止清除浏览数据丢失） |
| 注册绑定 | 注册时检查 localStorage 中的 ref 参数，绑定推荐关系 |
| 有效期 | 点击推荐链接后 **7 天内** 注册有效 |
| 唯一性 | 每个用户只能被一个推荐人绑定（首次归属原则） |
| 不可改 | 绑定后推荐关系永久不变 |
| 自推禁止 | 用户不能推荐自己（后端校验） |

### 2.3 分享功能

**分享入口**：推广中心页面 + 个人主页「邀请好友」按钮

**分享方式**：
- 复制推荐链接
- 分享海报图片（带二维码 + 推荐码）
- 直接分享到社交媒体（Facebook、Zalo、Messenger）

## 三、佣金阶梯

### 3.1 四级阶梯

| 等级 | 累计推荐收入 | 佣金比例 | 等级名称 |
|------|------------|---------|---------|
| Lv.1 | $0 — $9.99 | 5% | Người mới（新手） |
| Lv.2 | $10 — $99.99 | 10% | Đại sứ（大使） |
| Lv.3 | $100 — $999.99 | 15% | Đối tác（合作伙伴） |
| Lv.4 | $1,000+ | 20% | Đối tác vàng（金牌合作伙伴） |

**说明**：
- 累计推荐收入 = 历史所有已结算佣金总和
- 升级后**不降级**（即使某月没有新推荐收入）
- 佣金比例基于**推荐人当前等级**计算

### 3.2 佣金计算示例

```
张三推荐李四注册
李四购买月费 $2.99
张三当前等级 Lv.1（5%）

佣金 = $2.99 × 5% = $0.1495 ≈ $0.15（保留 2 位小数，四舍五入）

下个月李四续费 $2.99
张三等级不变
佣金 = $2.99 × 5% = $0.15

假设张三累计佣金达到 $10.00 → 自动升级到 Lv.2（10%）
之后李四续费 $2.99
佣金 = $2.99 × 10% = $0.30
```

## 六、提现系统

### 6.1 提现规则

| 规则 | 说明 |
|------|------|
| 最低提现金额 | $10.00 |
| 每月提现上限 | $500.00（超出部分需人工审核） |
| 提现方式 | PayPal（MVP 仅支持 PayPal） |
| 冷却期 | 佣金产生后 30 天可提现 |
| 处理时间 | 申请后 3-5 个工作日到账 |
| 手续费 | 平台不收手续费；PayPal 自身手续费由用户承担 |
| 退款扣回 | 被推荐人退款 → 对应佣金扣回 |

### 6.2 KYC 验证（反洗钱合规）

用户首次申请提现前，必须完成 KYC 验证：

| 等级 | 触发条件 | 需提供资料 |
|------|---------|----------|
| 基础验证 | 首次提现 | PayPal 邮箱 + 全名（与 PayPal 一致） |
| 增强验证 | 单笔提现 > $100 或累计提现 > $500 | 身份证/护照照片 + 自拍手持证件照 |

- MVP 阶段人工审核 KYC 资料（通过管理后台）
- KYC 审核周期：1-3 个工作日
- 未通过 KYC 时，佣金正常计入但不可提现

### 6.2 提现流程

## 七、反作弊规则

### 7.1 作弊检测

| 风险行为 | 检测方式 | 处理 |
|---------|---------|------|
| 自推自买 | 推荐人和被推荐人同 IP/设备指纹 | 不发放佣金 |
| 批量假注册 | 同一推荐码单日新增 > 10 人触发人工审核 | 冻结推荐功能，人工审核 |
| 刷退款 | 被推荐人频繁购买后退款 | 扣回佣金，标记风险 |
| 多账号刷 | 同一设备/IP 多个账号推荐 | 合并计算，超额部分不发放 |
| 低质量推荐 | 被推荐用户 7 日留存 < 10% | 该批次推荐不计佣金 |
| 支付工具关联 | 同一支付卡关联 > 3 个账号 | 超出部分不计佣金 |

**量化阈值**：
- 单个推广员每日新增上限：**50 人**（超出自动冻结）
- 单日新增 > 10 人：触发人工审核队列
- 被推荐用户 7 日留存必须 > **10%**（否则不计佣金）
- 同 IP 注册的推荐不计佣金

### 7.2 处罚机制

- 轻度违规：佣金冻结 + 警告通知
- 重度违规：永久取消推荐资格，扣回所有未提现佣金
- 申诉渠道：客服邮箱（MVP 不做在线申诉）

## 八、被推荐人视角

### 8.1 注册体验

被推荐人通过推荐链接访问 → Landing Page 与正常一致：
- 不显示任何"您被 XXX 邀请"类文案
- ref 参数静默存入 localStorage
- 注册流程与普通用户完全一致

### 8.2 被推荐人福利

被推荐人通过推荐链接注册后，自动获得以下福利：

| 福利 | 说明 |
|------|------|
| 首月 7 折 | 首次订阅任意方案享 30% 折扣（仅首月/首季/首年） |
| 显示说明 | 订阅页面自动展示「Úu đãi giới thiệu: Giảm 30%」标签 |
| 叠加规则 | 不与首月 $0.99 促销叠加（取价格更低的一个） |
| 技术实现 | ref 参数绑定后生成 Stripe Coupon 自动应用 |

## 十、数据流与输入验证

### 10.1 推荐→佣金→提现数据流

```
推广员分享推荐链接（含 ref=XXXX 参数）
  ↓
被推荐人点击链接 → 进入落地页 → ref 参数存入 localStorage + Cookie（7 天有效）
  ↓
被推荐人注册
  ↓
FastAPI：
  1. 检查 ref 参数有效性（是否存在、是否过期、是否自推）
  2. 创建 referral_relations 记录（referrer_id + referred_id）
  3. 为被推荐人生成 Stripe 30% off Coupon
  ↓
被推荐人订阅付费
  ↓
Stripe Webhook invoice.payment_succeeded：
  1. 查找 referral_relations
  2. 计算佣金金额（订单金额 × 当前阶梯比例）
  3. 创建 commission 记录（status=pending, available_at=now+30d）
  4. 更新推广员累计推荐数，检查是否触发阶梯升级
  ↓
推广员申请提现
  ↓
FastAPI：
  1. 检查 KYC 状态（未验证 → 进入 KYC 流程）
  2. 检查可提现余额（仅 available_at 已到的 pending）
  3. 检查月度提现上限（$500/月）
  4. 发起 PayPal Payout
  5. 更新 commission.status = paid
```

### 10.2 提现表单验证

| 字段 | 规则 |
|------|------|
| PayPal 邮箱 | 合法邮箱格式，必须与 KYC 提交的一致 |
| 提现金额 | 数字，≥$10.00，≤可提现余额，≤$500 月度剩余额度 |
| KYC 姓名 | 2-100 字符，Unicode，必须与 PayPal 注册名一致 |
| KYC 证件照片 | JPEG/PNG，≤ 5MB，分辨率 ≥ 800×600 |
