# 1.6 · 优先级矩阵（RICE + ICE）

## 评分维度

- **R** Reach：每月将影响多少用户（10 万规模）
- **I** Impact：对核心指标（付费率/留存）的影响 0.25 / 0.5 / 1 / 2 / 3
- **C** Confidence：把握度 50%-100%
- **E** Effort：工程人月
- **RICE = R × I × C / E**
- **ICE = (Impact + Confidence + Ease) / 3**（Ease = 10 - Effort 调度）

## 需要"先行"的依赖项（非高 RICE，但是其他事的前置）

- **#050 合规** → 是 Paddle 通过 / 数据存储的前提
- **#021 TTS** → 是发现中国 / 课程 / 小说所有朗读的前提
- **#055 反爬** → 上线前必须，否则上线即被爬
- **#048 Native 审校** → 是付费率的前提（影响口碑）
- **#114 容量测试** → 上线前 1 周必做

## 退出标准

凡是 RICE < 15 且不属于"先行依赖"的，自动进入次发优化或暂缓。
