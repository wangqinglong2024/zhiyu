| Objective | Key Result |

## 输入信息提炼（从用户需求表）

### 项目定位三要素
1. **WHO**：东南亚（越南、泰国、印尼优先）的中文学习者，含 HSK 考生 / 工厂打工者 / 中越中泰贸易者 / 来华日常使用者
2. **WHAT**：基于母语的系统中文教学 + 文化沉浸 + 游戏化 + 阅读闭环
3. **HOW**：AI 一次性生成内容 → 入库 → 多端分发；母语作为讲解第一语言

### 4 大业务模块（已规划）
- M1 发现中国（文化短文，逐句朗读 / 拼音 / 母语对照）
- M2 系统课程（4 轨道 × 12 阶段，知识点级别）
- M3 游戏专区（12 款游戏，强制横屏，与课程进度联动场景与词汇）
- M4 小说专区（章节制阅读，机制类似发现中国但分章）

### 全局功能
- 首页：每日金句 + 温故知新（基于记忆曲线的 SRS）
- 个人中心：知语币、分享奖励（永久二级分销）、隐私协议、设置（语言/拼音显示/朗读模式/语速）、清空进度、IM 入口

### 商业边界（不可妥协）
- 支付：仅 Paddle（全球合规一体化）
- 退款：禁止退款（明确说明）
- 价格锚点：会员 $4/月（vs 竞品 ~$10），$40/年，半年 $12（首发 50%），单门类课段 $4
- 知语币：1 USD = 100 币，新人送 100 币、签到随机、分销 20%、不可提现、不可抵扣现金

### 技术边界（不可妥协）
- 前端：Vite + React + TS（后期 React Native 迁移移动端）
- 后端：Node.js + TS + Express
- 数据：Supabase（DB + Auth + 向量库）
- AI：LangGraph(TS) 复杂工作流；Vercel AI SDK 简单调用
- 反爬：前端数据动态加载

### 不做（明确剔除）
- 实时多人对战 / 直播 / PK
- ASR 语音识别
- 退款 / 提现
- 微信支付 / 支付宝（合规与目标市场不匹配）
- 紫色系视觉（项目规范 grules）

## 自主决策的元规则

设施员在本次会议中将遵循以下决策原则（无需用户介入）：
1. **生成阶段**：不批判任何想法（包括看似荒谬的）
2. **收敛阶段**：以"用户价值 × 与定位一致性 × 可执行性"三维筛选
3. **冲突仲裁**：与项目规范 `grules/` 冲突时，规范优先；与商业边界冲突时，商业边界优先
4. **缺失判断**：用户未明确处，按"东南亚 18-35 岁数字原住民"画像默认推断
5. **争议条目**：进入"实验观察"层，留待数据说话
