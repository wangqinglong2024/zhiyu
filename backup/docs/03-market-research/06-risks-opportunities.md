# 3.6 · 风险与机会

## 一、风险矩阵（Risk Matrix）

### 1.1 评分标准
- 概率（P）：Low (1) / Medium (2) / High (3)
- 影响（I）：Low (1) / Medium (2) / High (3) / Critical (4)
- 风险值 = P × I

## 三、机会与风险的对冲设计

### 3.1 内容审核 vs 内容速度
- 风险：审核失败
- 机会：AI 高速生产
- 对冲：质量优先 + AI 提示词内置红线 + 母语审校并发审

### 3.2 价格战 vs ARPU
- 风险：竞品降价
- 机会：性价比定位
- 对冲：内容深度护城河 + 4 模块复合价值 + 分销裂变降低 CAC

### 3.3 平台依赖 vs 自有渠道
- 风险：TikTok 限流
- 机会：三平台 + SEO
- 对冲：SEO 长期资产化 + 自有 WhatsApp / Telegram 名单

### 3.4 团队小 vs 内容量大
- 风险：执行不足
- 机会：AI 工程师生产力
- 对冲：自动化优先 + 外包审校 + 用户共建

## 五、应急预案（Top 3）

### 应急预案 1：单国内容审核被警告
1. 1h 内：下架相关内容
2. 4h 内：内部审核会议 + PR 草案
3. 24h 内：发出官方道歉 / 解释（用当地语言）
4. 72h 内：法律顾问介入 + 整改报告
5. 1 周内：复盘 + SOP 更新

### 应急预案 2：Paddle 拒绝某市场
1. 立即激活 LemonSqueezy 通道
2. 站内提示用户切换支付方式
3. 修改前端配置（Feature flag）
4. 通知存量用户（影响有限）
5. 申诉 Paddle / 提交合规材料

### 应急预案 3：服务器大故障
1. 自动切换到备用节点（Cloudflare）
2. 静态页 fallback（"系统升级中"）
3. 4h 内修复 + 复盘
4. 影响用户邮件道歉 + 7 天免费会员
