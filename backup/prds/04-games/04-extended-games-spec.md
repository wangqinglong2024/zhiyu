## 二、游戏 1：汉字忍者（Hanzi Ninja · hanzi-ninja）

> **代号**：GM-NINJA | **版本**：v1.5 | **类型**：切水果式动作 | **参考**：`/games/01-hanzi-ninja/`

### 2.1 玩法概述
- 屏幕底部不断抛出"汉字水果"（每个水果含一个汉字 / 拼音 / 部件），玩家用手指（移动）或鼠标拖拽（桌面）划过屏幕切割
- 切对目标字得分，切错（炸弹 / 干扰字）扣命
- 词包模式：屏幕顶部固定显示一个"目标拼音"或"目标义"，需切割匹配的汉字水果

### 2.2 输入与控制
| 平台 | 输入方式 |
|---|---|
| 移动横屏 | 单指或多指滑动（多指最多 3 指同时切） |
| 桌面 | 鼠标按住左键拖拽划线 |
| 触摸板 | 双指模拟滑动 |

### 2.3 关卡参数
- **关卡时长**：60s / 局
- **抛出节奏**：每秒 0.8~1.5 个水果（难度递增）
- **生命**：3 颗心，切错 / 漏掉目标 -1
- **难度递进**：第 0-20s 一次抛 1 个 → 20-40s 一次抛 2 个 → 40-60s 一次抛 3 个 + 偶发炸弹

### 2.4 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | 准确率 ≥ 90% 且切中 ≥ 25 个目标字 |
| ★★ | 准确率 ≥ 75% 且切中 ≥ 18 个目标字 |
| ★ | 准确率 ≥ 60% 且切中 ≥ 10 个目标字 |

### 2.5 错题规则
- 切错的字 → 推入 SRS（标记错答）
- 漏掉超过 2s 的目标字 → 推入 SRS（标记未掌握）

### 2.6 词包映射
- 目标提示语支持 3 种模式（用户在开局前选择）：
  1. **拼音模式**：顶部显示 `xué` → 切所有 `xué` 音节的汉字（学/雪/血...）
  2. **释义模式**：顶部显示母语释义（如越南文 `học`）→ 切表达该意的汉字（学/习/读）
  3. **部件模式**：顶部显示 `氵`（部首）→ 切包含该部件的汉字
- 词包来源遵循 GM-FR-003（当前轨道 / 已学全部 / 错题集 / HSK X）

### 2.7 视觉与音效
- 水果 = 透明毛玻璃球体内嵌汉字（与设计系统一致）
- 切割轨迹：发光剑光（主色调 Rose），暗色模式偏冷蓝光
- 切对：清脆"叮"+绿色粒子，切错炸弹：低沉爆破+红色震屏
- 背景：竹林剪影（亮色）/ 星夜竹林（暗色）

### 2.8 性能与包大小
- PixiJS 8 + Howler.js
- 资源包 ≤ 1.5 MB（角色 / 水果 / 特效 sprite sheet）
- 60fps 中端 Android（红米 Note 11 验证）

### 2.9 技术与防作弊
- HMAC 签名：`score + duration + responses + ts + nonce`
- 服务端校验：60s 局内最多 90 个有效切割（超出判作弊）
- 时间戳异常（< 客户端预估最小耗时 30s）丢弃

### 2.10 验收准则（追加）
- GM-AC-NINJA-01：横屏强制生效，竖屏遮罩显示"请旋转设备"
- GM-AC-NINJA-02：单局结束 < 1.2s 完成提交并显示通关页
- GM-AC-NINJA-03：切错 / 漏字均推入 SRS，可在错题专攻看到
- GM-AC-NINJA-04：日通关币上限 50（与其他游戏共享 GM-FR-004）
- GM-AC-NINJA-05：三星条件命中精度 100%

## 三、游戏 2：汉字俄罗斯方块（Hanzi Tetris · hanzi-tetris）

> **代号**：GM-TETRIS | **版本**：v1.5 | **类型**：益智消除 | **参考**：`/games/04-hanzi-tetris/`

### 3.1 玩法概述
- 标准 10×20 俄罗斯方块网格，每个方块单元格含一个汉字
- 消除条件：满行 OR "拼词"——连续 ≥ 2 个汉字组成有效词语（横/竖/对角，左到右或上到下）
- 拼词消除奖励 = 满行 × 2 分

### 3.2 输入与控制
| 平台 | 操作 |
|---|---|
| 桌面 | ← → 移动 / ↑ 旋转 / ↓ 加速下落 / Space 硬下落 |
| 移动横屏 | 屏幕左右两侧虚拟按键 + 中部滑动 |
| 移动横屏（手势） | 左右滑动 / 双击旋转 / 下滑硬下落 |

### 3.3 关卡参数
- **关卡时长**：3 分钟 / 局，或网格堆顶
- **下落速度**：起始 1 行 / 1.0s，每 30s 加速 10%
- **方块构成**：经典 7 种俄罗斯方块形状（I/O/T/L/J/S/Z），每个单元格随机抽词包汉字

### 3.4 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | 得分 ≥ 8000 且拼词消除 ≥ 8 次 |
| ★★ | 得分 ≥ 5000 且拼词消除 ≥ 4 次 |
| ★ | 得分 ≥ 2500 |

### 3.5 错题规则
- 词典中检测有效词组合后系统弹出 0.5s 词义提示
- 玩家点击"我知道"则不入 SRS；不点 / 跳过 → 入 SRS（标记需学习）
- 游戏结束时未拼出的高频组合也入 SRS（标记未掌握）

### 3.6 词包映射
- 词包内汉字按出现频率加权抽取（高频字出现概率高）
- 系统词典每分钟更新一次"潜在拼词"提示（暗色高亮 2s）

### 3.7 视觉与音效
- 方块 = 透明毛玻璃单元，亮/暗主题色不同
- 拼词成功：金色光柱 + "叮咚"+ 该词分享卡片入收藏
- 满行消除：彩虹波 + 经典俄罗斯方块"咔嚓"音

### 3.8 性能与包大小
- PixiJS 8（无需物理引擎）
- 资源包 ≤ 800 KB
- 60fps 全平台

### 3.9 防作弊
- 服务端记录方块下落序列种子，回放校验得分
- 单局速度 / 操作合理性：每次操作 ts 偏差 ≤ 50ms

### 3.10 验收准则
- GM-AC-TETRIS-01：拼词消除算法正确（双向、含对角）
- GM-AC-TETRIS-02：词典命中后 0.5s 内显示词义
- GM-AC-TETRIS-03：三星精度准确，得分公式可复现
- GM-AC-TETRIS-04：移动端虚拟按键不阻挡视野

## 四、游戏 3：拼音塔防（Pinyin Defense · pinyin-defense）

> **代号**：GM-DEFENSE | **版本**：v1.5 | **类型**：塔防策略 | **参考**：`/games/10-pinyin-defense/`

### 4.1 玩法概述
- 单地图分 5 波敌人，敌人 = 抱着汉字的小怪沿固定路径前进
- 玩家在路径两侧空地建造"拼音塔"，每塔射击一个目标拼音
- 敌人血量 = 该字 HSK 等级；命中需要正确拼音匹配

### 4.2 玩法机制
- **建塔**：拖动塔图标到地图空地，付费"知语点"（局内货币，初始 50，击杀获得）
- **塔升级**：击杀获得点数升级塔（射程 / 射速 / 多目标）
- **塔类型**：
  - 单字塔（默认） → 锁定一个拼音
  - 范围塔 → 射 3 字范围
  - 慢速塔 → 减速敌人 30%
  - 终极塔 → 免费一次扫荡（限 1 次 / 局）

### 4.3 关卡参数
- **关卡时长**：5~7 分钟 / 局（5 波）
- **生命**：基地 20 血，敌人到达 -1
- **波次设计**：W1 慢速 / W2 数量 / W3 装甲（HSK 6+） / W4 飞行（绕过特定塔） / W5 BOSS（150 血连字）

### 4.4 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | 5 波全胜 + 基地剩余 ≥ 18 血 |
| ★★ | 5 波全胜 + 基地剩余 ≥ 10 血 |
| ★ | 至少 3 波胜利且基地未陷落 |

### 4.5 错题规则
- 敌人成功到达基地 → 该字推入 SRS（标记薄弱）
- 玩家手动建错塔（拼音输入错误）→ 不计错题，但塔无效

### 4.6 输入与控制
| 平台 | 操作 |
|---|---|
| 移动横屏 | 拖动建塔 / 点击塔升级 / 双击使用终极技能 |
| 桌面 | 拖动 / 滚轮缩放地图 / 数字键 1-4 选塔 / Space 暂停 |

### 4.7 视觉与音效
- 等距 2.5D 视角，毛玻璃 UI 面板
- 地图：江南水乡（亮）/ 雾峰山影（暗）
- 击杀：金色字粒消散 + 拼音音效

### 4.8 性能
- PixiJS 8 + 简化路径算法（无 A*，使用预定义路径节点）
- 资源包 ≤ 2 MB

### 4.9 防作弊
- 服务端追踪建塔事件 + 时间戳，回放验证总伤害
- 局内货币与击杀绑定，无法在客户端伪造

### 4.10 验收准则
- GM-AC-DEFENSE-01：5 波难度递增可玩，BOSS 战平衡
- GM-AC-DEFENSE-02：塔升级 / 终极技能效果正确
- GM-AC-DEFENSE-03：失败 / 胜利结算页清晰
- GM-AC-DEFENSE-04：音频同步偏差 ≤ 50ms

## 五、游戏 4：汉字贪吃蛇（Hanzi Snake · hanzi-snake）

### 5.1 玩法概述
- 经典贪吃蛇玩法，地图随机生成"汉字食物"
- 蛇身每节显示已吃过的汉字
- 进食机制：屏幕上方提示"目标拼音 / 部件 / 释义"，吃到匹配的字 +2 长度且 +10 分；吃到错的 -1 长度 -5 分

### 5.2 关卡参数

### 5.3 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | 得分 ≥ 600 且存活 ≥ 3 分钟 |
| ★★ | 得分 ≥ 350 |
| ★ | 得分 ≥ 150 |

### 5.4 错题规则
- 吃错的字 → 推入 SRS（标记混淆）

### 5.5 输入与控制
| 平台 | 操作 |
|---|---|
| 移动横屏 | 滑动屏幕（4 方向） / 屏幕左右两侧虚拟方向键 |

### 5.6 视觉与音效
- 蛇身：毛玻璃片段，每节显示汉字

### 5.7 防作弊

### 5.8 验收准则

- GM-AC-SNAKE-02：错字记录完整入 SRS

## 六、游戏 5：汉字节奏（Hanzi Rhythm · hanzi-rhythm）

> **代号**：GM-RHYTHM | **版本**：v2 | **类型**：音游 | **参考**：`/games/08-hanzi-rhythm/`

### 6.1 玩法概述
- 类似《Cytus》/《节奏大师》音游：汉字方块沿轨道下落到判定线，玩家按键 / 触屏配合节奏击打
- 4 条音轨（左 / 中左 / 中右 / 右）
- 每个汉字方块伴随其拼音音节出现，命中时朗读该字（TTS）
- 难度跟随选定歌曲（建议提供 3 首古风 + 3 首流行 + 3 首儿歌，全部 v2 上线）

### 6.2 关卡参数
- **关卡时长**：跟歌曲（90s ~ 150s）
- **判定**：Perfect / Great / Good / Miss（窗口 50ms / 100ms / 150ms / >150ms）
- **难度**：Easy（30 字 / 局）/ Normal（60 字）/ Hard（120 字，含双键 / 长按）

### 6.3 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | Full Combo + Perfect 率 ≥ 95% |
| ★★ | Combo ≥ 80% 且 Perfect 率 ≥ 80% |
| ★ | 通关（不爆条） |

### 6.4 错题规则
- Miss / Good 的字 → 推入 SRS（节奏不准 = 不熟悉）
- 局后展示"我没跟上的字"列表

### 6.5 输入与控制
| 平台 | 操作 |
|---|---|
| 移动横屏 | 4 区域触屏（屏幕宽四等分） |
| 桌面 | D / F / J / K（默认）+ 自定义键位 |

### 6.6 音乐版权
- v2 启动需采购 / 委约 9 首歌曲（中文版权清晰）
- 优先与中国版权方合作（QQ 音乐 / 网易云音乐）
- 备选：自制 BGM（与字幕字时间码同步）

### 6.7 视觉与音效
- 半透明轨道 + 毛玻璃判定环
- 击中粒子爆发（颜色随判定等级）
- 暗模式：星空 / 亮模式：水墨

### 6.8 性能
- PixiJS 8 + Howler.js（精确音轨同步，使用 Web Audio API ctx.currentTime）
- 资源包 ≤ 4 MB（含 1 首示例曲）；其余按需下载

### 6.9 防作弊
- 客户端记录每次按键时间戳（与歌曲 ts 比对），服务端重新评级
- 异常：所有 Perfect = 自动复审

### 6.10 验收准则
- GM-AC-RHYTHM-01：音画同步偏差 ≤ 30ms
- GM-AC-RHYTHM-02：3 首样曲全难度通关测试
- GM-AC-RHYTHM-03：版权信息显示合规

## 七、游戏 6：汉字跑酷（Hanzi Runner · hanzi-runner）

> **代号**：GM-RUNNER | **版本**：v2 | **类型**：跑酷 | **参考**：`/games/09-hanzi-runner/`

### 7.1 玩法概述
- 类似《地铁跑酷》/ 《神庙逃亡》：角色自动向前跑，玩家上滑跳跃 / 下滑滑铲 / 左右切换跑道
- 路上随机出现 3 类要素：
  1. **金币**（汉字硬币）→ 触碰收集，需对应屏顶提示的拼音才计分（错收扣分）
  2. **障碍**（路障 / 列车）→ 跳 / 滑 / 切道避开
  3. **门**（含两个汉字选项）→ 选择正确翻译过门，错门 -1 命

### 7.2 关卡参数
- **关卡时长**：直至 3 命耗尽或主动退出（上限 5 分钟）
- **加速**：每 30s 速度 +5%
- **生命**：3 命，撞障碍 / 错门 -1

### 7.3 三星评级
| 星级 | 条件 |
|:---:|---|
| ★★★ | 距离 ≥ 2000m 且金币 ≥ 200 |
| ★★ | 距离 ≥ 1000m 且金币 ≥ 100 |
| ★ | 距离 ≥ 500m |

### 7.4 错题规则
- 错收 / 错门的字 → 推入 SRS

### 7.5 输入与控制
| 平台 | 操作 |
|---|---|
| 移动横屏 | 滑动（上 / 下 / 左 / 右） |
| 桌面 | 方向键 / WASD / 空格跳 |

### 7.6 视觉与音效
- 3 跑道伪 3D 透视（PixiJS 透视投影 / 简化版本）
- 场景：长安古道（亮）/ 赛博长安（暗）
- 音乐：动感电子鼓点 + 中国笛子混音

### 7.7 性能
- PixiJS 8 渲染（伪 3D，避免使用 Three.js 增加包体）
- 资源包 ≤ 3 MB
