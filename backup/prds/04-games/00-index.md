## MVP 单阶段开发原则（强制）

12 款游戏 **统一只交付 MVP**，**一次性首发**，**不分波次、不切 V1 / V1.5、不使用 `coming_soon` 占位、不发任何奖励**。

### MVP · 最小可玩闭环（每款游戏唯一交付物）
- 统一设置面板 / 按课程权限选择词包（track + stage）/ **固定 60s 单局 · 无限连玩 · 无关卡**
- 基础得分 + 结算页（「再玩一局」 / 「返回大厅」）+ 错题入 SRS（最小子集）
- 横屏强制 + 60fps 底线
- 4 语 i18n key 就位

### MVP **不含**（明确排除 → 全部进 `99-post-mvp-backlog.md`）
- 任何关卡 / 章节 / 多关 / 下一关 / 多波 / 额外难度模式
- 任何形式的奖励 / 知语币 / 道具 / 经验值
- 三星评级 / 关卡星图 / 解锁系统
- 排行榜（日 / 周 / 全时）
- 教学引导 / 30s tutorial
- IAP / 复活 / 解锁
- 自定义词包 / 多模式；但必须支持系统词包按课程权限选择
- 社交分享卡片
- HMAC 防作弊（仅记 raw 分数）
- `coming_soon` 占位卡片

## 关键技术决策
- 强制横屏（force-landscape，桌面也横屏画布）
- PixiJS v8 + Howler.js + Matter.js（物理；MVP 期仅 G12 用）
- 词汇库取自课程知识点；可选主题轨道与阶段范围由用户当前课程权限决定（例如已购电商 Stage 9 则可选择电商 S9）
- 错题入 SRS：游戏中答错的知识点（MVP 必须支持）
- **不**发任何形式的奖励，**不**调用经济 / 知语币 API
- **排行榜 / 三星 / IAP / HMAC 防作弊 / 社交分享 / 奖励 全部 post-MVP**
