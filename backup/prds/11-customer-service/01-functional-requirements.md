# 11.1 · 客服 · 功能需求

## 用户故事
- US-CS-01：用户遇问题随时点客服图标聊
- US-CS-02：客服后台收到消息回复
- US-CS-03：用户上传图片 / 截图说明问题
- US-CS-04：会话历史保留可查
- US-CS-05：会话末用户评分

## 功能需求

### CS-FR-001：客服入口
- 全站固定按钮（右下角浮窗）
- 已登录：点击进会话
- 未登录：弹层"先登录或留邮箱"
- 移动端友好

### CS-FR-002：会话页
- `/me/support`
- 元素：顶部分类 / 客服头像 / 状态（在线 / 离线）
- 消息流（自己 / 客服 / 系统）
- 输入框 + 表情 + 图片上传（≤ 5MB）
- 历史会话 tab

### CS-FR-003：会话路由
- 用户发首条消息 → 系统创建 conversation
- 路由策略：
  - 优先匹配同语种（用户母语 → 客服懂）
  - 在线客服中工单数最少
  - 工作时间外 → 留言 + 邮件通知
- 客服可主动接单（dashboard 中"待接"列表）

### CS-FR-004：消息类型
- text（≤ 2000 字）
- image（≤ 5MB）
- system（系统消息：会话开始 / 客服分配 / 关闭等）
- 链接自动识别 + 卡片预览

### CS-FR-005：实时推送
- Supabase Realtime channel = `conversation:{id}`
- 用户 + 客服订阅
- 新消息推送
- 离线推送（v1.5）

### CS-FR-006：未读 / 已读
- 用户端：客服头像红点 + 数
- 客服端：会话列表未读数

### CS-FR-007：客服后台（详见 AD 模块）
- 工作台 `/admin/cs/workbench`
- 待接单列表
- 当前会话列表（≤ 5 同时）
- 历史搜索

### CS-FR-008：会话状态
- pending（用户发首条等接）
- active（客服接单后）
- waiting（客服等用户回）
- resolved（客服关闭）
- closed_no_response（24h 未回，自动关）

### CS-FR-009：会话评分
- 客服关闭 → 用户收到 1-5 星评分弹窗
- 可选评论（≤ 200 字）
- 评分 ≤ 3 → 转 manager 回访（v1.5）

### CS-FR-010：FAQ（v1.5）
- 常见问题文章库（4 语种）
- 用户进入会话前先看 FAQ
- 命中则不创会话

### CS-FR-011：合规
- 4 语种自动翻译（v1.5）：客服中文回复 → 用户母语展示
- 隐私：用户数据不分享给第三方
- 内容审核：客服消息走红线检测

### CS-FR-012：通知
- 用户收到回复 → 邮件（如不在站内）
- 客服收到新消息 → dashboard 推送 + 桌面通知（浏览器 API）

## 性能
- 消息发送 P95 < 300ms
- 实时推送延迟 < 1s
- 单会话历史加载 < 500ms
- 会话最长 7 天后归档（可查）
