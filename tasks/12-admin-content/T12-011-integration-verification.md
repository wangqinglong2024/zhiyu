### 测试场景详细描述

#### 场景 1: 文章全流程

```
1. 新建文章 → 填写四语言内容 + 拼音标注 + 封面图
2. 保存草稿 → 列表中状态为「草稿」
3. 编辑文章 → 修改内容 → 保存
4. 提交审核 → 列表状态变为「待审核」
5. 进入审核页 → 预览文章内容完整
6. 驳回 → 填写原因 → 文章状态变「已驳回」
7. 修改 → 重新提交 → 审核通过
8. 发布 → 文章状态变「已发布」
9. 下线 → 文章状态变「已下线」
10. 批量操作：选中多篇文章 → 批量发布/下线
```

#### 场景 2: 课程全流程

```
1. Level 总览页 → 12 张卡片展示正确
2. 点击 Level 1 → Unit 列表正确
3. 新建 Unit → 列表更新
4. 拖拽排序 Unit → 编号重排 + Toast
5. 进入 Unit → Lesson 列表正确
6. 新建讲解型 Lesson → 编辑内容（富文本+示例句+词汇+语法）
7. 新建练习型 Lesson → 添加题目（至少 3 种题型）
8. 复制 Lesson → 标题带「(副本)」后缀
9. 拖拽排序 Lesson → 编号重排 + Toast
10. 删除空 Unit → 成功
11. 删除含 Lesson 的 Unit → 按钮禁用
```

#### 场景 3: 7 种题型全覆盖

```
对每种题型逐一验证:
1. single_choice: 创建 → 设置选项 + 正确答案 → 预览 → 保存
2. multi_choice: 创建 → 设置选项 + 多个正确答案 → 预览 → 保存
3. fill_blank: 创建 → 设置空格位置 + 答案 → 预览 → 保存
4. sorting: 创建 → 设置排序项 → 预览 → 保存
5. matching: 创建 → 设置配对项 → 预览 → 保存
6. listening: 创建 → 上传音频 + 设置题目 → 预览 → 保存
7. reading_aloud: 创建 → 设置朗读内容 → 预览 → 保存

每种题型验证:
- 新建 → 编辑 → 保存 → 列表显示正确
- 题型标签颜色正确（无紫色）
- 切换题型确认弹窗
- 预览弹窗模拟交互
```

#### 场景 4: 评测题库全流程

```
1. 评测题库列表 → 筛选 Level=L1, 层级=课时小测验
2. 新建题目 → 选择层级后题型自动过滤
3. 批量导入 → 上传 CSV → 预览校验 → 确认导入
4. 考核配置 → 编辑参数 → 校验(min ≤ passing) → 保存
5. 重置为默认值 → 参数恢复
```

#### 场景 5: 金句全流程

```
1. 列表视图 → 筛选 + 分页正常
2. 切换日历视图 → 月历渲染正确
3. 新建金句 → 四语言编辑 + 拼音自动生成
4. 排期日期选择 → 冲突日期灰色不可选
5. 保存排期 → 日历视图中对应日期更新
6. 点击已排期日期 → 浮层卡片展示
7. 点击未排期的未来日期 → 跳转新建
```

#### 场景 6: 审核全链路

```
1. 文章提交审核 → 审核列表出现
2. Tab 筛选「文章」→ 仅显示文章
3. 审核详情 → 内容预览完整
4. 驳回 → 驳回原因必填(≥10字)
5. 修改后重新提交 → 审核列表重新出现
6. 通过 → 状态更新
7. 批量审核 → 3 条同时通过
8. 审核历史时间线完整
```
