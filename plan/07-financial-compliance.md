## 一、公司架构

### 1.1 建议架构

```
控股公司（新加坡 / 香港）
  ├── 运营主体 — 处理收入、支付、合同
  ├── 开发团队 — 远程团队（中国/越南/其他）
  └── 本地合作 — 越南本地合作伙伴（如需）
```

### 1.2 注册地选择

| 选项 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **新加坡** | 税率低(17%)、国际化、东南亚枢纽、银行开户方便 | 注册维护费用略高 | ★★★★★ |
| **香港** | 税率低(8.25%-16.5%)、免税海外收入、中文支持 | 银行开户审查趋严 | ★★★★☆ |
| **越南** | 本地运营合规、了解市场 | 税率高、外资限制多 | ★★★☆☆ |
| **美国（Delaware）** | 适合未来融资、全球认可 | 税务复杂、初期无必要 | ★★☆☆☆ |

> **推荐**：新加坡注册控股公司。如果初期预算有限，先以个人/团队身份运营（使用 Paddle 作为 MoR），待收入稳定后再注册公司。

### 1.3 分阶段操作

| 阶段 | 操作 |
|------|------|
| **MVP 阶段** | 无需注册公司；Paddle 作为 MoR 处理支付、税务、发票 |
| **收入稳定后** | 注册新加坡公司；签订 Paddle 商户协议；开设公司银行账户 |
| **扩展阶段** | 根据需要在越南/印尼设立本地实体或代表处 |

## 二、税务规划

### 2.1 Paddle MoR 模式下的税务

使用 Paddle 作为 Merchant of Record 的核心优势：

| 维度 | 说明 |
|------|------|
| 销售税 / VAT | **Paddle 负责**计算、收取和缴纳 |
| 增值税注册 | **Paddle 已在全球主要市场注册** VAT |
| 发票开具 | **Paddle 自动开具**符合当地法规的发票 |
| 退款处理 | **Paddle 负责**退款和争议处理 |
| 你收到的 | 扣除 Paddle 费用和税款后的净收入 |

> Paddle MoR 模式大幅降低了跨境税务合规成本和风险。

### 2.2 公司层面税务

| 税种 | 新加坡公司 | 说明 |
|------|----------|------|
| 企业所得税 | 17%（首 10 万新币享受优惠税率） | 年度申报 |
| 预提税 | 对海外支付的特定款项（版税等） | 需规划 |
| GST（消费税） | 9%（年收入超 100 万新币时注册） | 初期大概率不需要 |
| 个人所得税 | 0-22%（渐进税率） | 创始人薪资 |

### 2.3 转移定价

如果有多个实体（如新加坡公司 + 中国开发团队），需注意：
- 中国团队以服务合同方式向新加坡公司提供开发服务
- 服务费定价需合理（arm's length principle）
- 保留完整的服务合同和付款记录

## 三、数据合规

### 3.1 需遵守的法规

| 法规 | 适用范围 | 核心要求 |
|------|---------|---------|
| **越南网络安全法** | 越南用户 | 数据本地化（可能要求）、内容审查 |
| **越南个人数据保护法令** | 越南用户 | 数据收集、处理需合法基础和用户同意 |
| **PDPA（新加坡）** | 公司注册地 | 个人数据保护、跨境传输规则 |
| **GDPR** | 如有欧洲用户 | 最严格的数据保护标准 |
| **Google Play 政策** | Android 用户 | 隐私政策、数据安全表单 |
| **Apple App Store** | iOS 用户 | 隐私标签、ATT 框架 |

### 3.2 合规措施

| 措施 | 说明 |
|------|------|
| **隐私政策** | 多语言版本（中/英/越），详细说明数据收集、使用、存储、共享 |
| **用户协议** | 多语言版本，涵盖服务条款、退款政策、内容使用规则 |
| **Cookie 政策** | Web 端需展示 Cookie 同意横幅 |
| **数据收集最小化** | 只收集必要数据，不过度采集 |
| **用户权利** | 支持数据查看、导出、删除请求 |
| **数据加密** | 传输加密(TLS) + 存储加密(AES-256) |
| **访问控制** | 最小权限原则，员工仅访问必要数据 |
| **数据泄露响应** | 制定数据泄露通知流程（72 小时内通知） |

### 3.3 数据存储位置

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 用户账户数据 | Supabase（新加坡区域） | 贴近用户，满足延迟要求 |
| 教学内容 | Dify + Cloudflare R2 | 全球 CDN 加速 |
| 支付数据 | Paddle 托管 | 不自行存储支付卡信息 |
| 日志数据 | 服务器本地 + 云备份 | 保留 90 天 |

### 3.4 越南特殊要求

- 越南网络安全法可能要求**数据本地存储**，需持续关注执法细则
- 如被要求本地化：在越南部署数据库副本
- 内容方面：避免涉及政治敏感话题，文化内容保持中立客观
- App 上架越南市场可能需要当地代理

## 四、内容合规

### 4.1 版权策略

| 内容类型 | 版权状态 | 使用策略 |
|---------|---------|---------|
| 古诗文（古代作品） | 公共领域 | 直接使用，标注出处 |
| 义务教育课程标准 | 国家文件 | 引用知识体系，不复制原文 |
| 统编教材课文 | 版权保护 | **不使用原文**，同题材改写 |
| 现当代文学作品 | 版权保护 | 引用片段（合理使用） |
| 图片素材 | 需授权 | 购买正版 / 使用 CC0 / 自主制作 |
| 音频素材 | 需授权 | Azure TTS 生成 / 正版采购 |
| 视频素材 | 需授权 | 自主制作 / 正版采购 |

### 4.2 UGC 合规

如果未来开放用户生成内容（如论坛、评论）：
- 建立内容审核机制（AI 自动审核 + 人工复审）
- 屏蔽敏感词、违规内容
- 用户举报通道
- 违规内容 24 小时内下架

### 4.3 商标保护

| 保护项 | 优先级 | 说明 |
|--------|-------|------|
| "LingoBase" 商标 | 高 | 注册新加坡 + 目标市场商标 |
| Logo 设计 | 高 | 原创设计 + 版权登记 |
| "学不可以已" 标语 | 中 | 古文无版权，但品牌使用可注册 |
| 域名 | 高 | lingobase.com / .app / .io |

## 五、应用商店合规

### 5.1 Google Play

| 要求 | 说明 |
|------|------|
| 开发者账户 | $25 一次性注册费 |
| 隐私政策 | 必须提供可访问的隐私政策 URL |
| 数据安全表单 | 声明数据收集和共享情况 |
| 内容分级 | IARC 问卷，获取适用的年龄分级 |
| 支付 | 可使用第三方支付（非强制 Google Play Billing） |

### 5.2 Apple App Store

| 要求 | 说明 |
|------|------|
| 开发者账户 | $99/年 |
| 隐私政策 | 必须提供 |
| 隐私标签 | App Store 页面展示数据使用标签 |
| ATT 框架 | 追踪用户需弹窗获取许可 |
| 支付 | 数字内容**必须使用 Apple IAP**（30% 抽成） |
| 策略 | PWA 优先 → Web 端完成付费 → App 仅作学习工具 |
