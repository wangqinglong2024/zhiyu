## 一、成本结构哲学

### 1.1 为什么我们的成本结构比 AI 套壳更健康

```
传统 AI 语言 App 的成本困境：

  用户增长 → AI API 调用线性增长 → 成本线性增长
  毛利率天花板：40-60%
  用户越多越烧钱

我们的游戏化模式：

  AI 一次性生成内容 → 内容审校打包 → 无限分发
  毛利率：80-90%（边际成本趋零）
  用户越多越赚钱
```

| 成本项 | AI 套壳模式 | 我们的模式 |
|--------|-----------|-----------|
| **内容成本** | 无（靠 AI 实时生成） | 前期投入后趋零（AI 批量生产+人工审校） |
| **AI 运行时成本** | 高（每用户每次调用都花钱） | 极低（AI 是可选增值，非核心流程） |
| **服务器成本** | 中 | 中（游戏资源走 CDN，边际接近零） |
| **美术/内容** | 低 | 中前期投入（像素美术+关卡设计） |
| **整体边际成本** | 线性增长 | 趋近于零 |

### 1.2 单人团队 + AI 的成本优势

```
我们的"团队"：

  创始人（你）：全栈开发 + 产品设计 + 运营
  AI 工具链：代码生成 + 美术辅助 + 内容生产 + 配音
  外包（按需）：像素美术精修 + 本地化校对

对比竞品团队成本：
  Duolingo：700+ 人，年人力成本 >$100M
  Speak：200+ 人，年人力成本 >$30M
  HelloChinese：<20 人，年人力成本 ~$500K
  我们：1 人 + AI，年人力成本 $0（创始人不发工资阶段）
```

## 六、单位经济（Unit Economics）

### 6.1 核心指标

| 指标 | 冷启动期 | 增长期 | 规模期 | 说明 |
|------|---------|--------|--------|------|
| **CAC** | $0.3-0.5 | $0.5-1.0 | $0.8-1.5 | 裂变+自然流量占比高时 CAC 低 |
| **LTV（付费）** | $14 | $20 | $28 | 平均付费 6→8.5→12 个月 |
| **LTV/CAC** | 28-47x | 20-40x | 19-35x | 远超健康线 3x |
| **回收期** | <1 月 | <1 月 | <2 月 | 首月即回收 |
| **毛利率** | 75% | 82% | 88% | 规模效应，边际成本趋零 |

### 6.2 LTV 计算详解

```
LTV = ARPU × 平均生命月数 × 毛利率

冷启动期：
  ARPU = $2.35
  平均生命月数 = 1 / 月流失率 = 1 / 0.12 = 8.3 月
  毛利率 = 75%
  LTV = $2.35 × 8.3 × 0.75 = $14.6

增长期（留存提升）：
  ARPU = $2.35
  平均生命月数 = 1 / 0.08 = 12.5 月
  毛利率 = 82%
  LTV = $2.35 × 12.5 × 0.82 = $24.1

规模期（品牌+社交绑定）：
  ARPU = $2.35
  平均生命月数 = 1 / 0.06 = 16.7 月
  毛利率 = 88%
  LTV = $2.35 × 16.7 × 0.88 = $34.5
```

### 6.3 裂变降低 CAC 模型

```
有机增长占比预测：

冷启动期：自然+裂变 50% / 付费 50%
  → 有效 CAC = 付费 CAC × 50% = $0.5 × 50% = $0.25

增长期：自然+裂变 60% / 付费 40%
  → 有效 CAC = $1.0 × 40% = $0.40

规模期：自然+裂变 70% / 付费 30%
  → 有效 CAC = $1.5 × 30% = $0.45

PvP 对战邀请贡献：
  假设 20% 新用户来自 PvP 邀请
  这些用户 CAC = $0（完全有机）
  且留存率比平均高 30%（有社交绑定）
```

## 九、风险与应对

### 9.1 财务风险矩阵

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| 付费转化率 <5% | 盈亏平衡推迟 | 中 | 调低价格到 $0.99/月攻量；优化付费触发点（关卡锁+PvP限制） |
| 留存率低于预期 | LTV 下降 | 中高 | 加强 Streak 机制+军团社交绑定；每周内容更新 |
| AI API 成本上涨 | 毛利率下降 | 低 | 多供应商（OpenAI/DeepSeek/Claude）+ 中期自建模型 |
| 应用商店抽成 30% | 利润大幅压缩 | 高 | 引导网页订阅；小型开发者 15% 优惠；PWA 绕过商店 |
| 竞品低价跟进 | 价格优势消失 | 低 | 游戏内容+社交关系+荣誉体系 = 不可替代的用户资产 |
| 越南/印尼本地法规变化 | 合规成本增加 | 低中 | 服务器在新加坡；通过应用商店代扣税；预留合规预算 |
| 外汇结汇困难 | 现金流受阻 | 中 | 初期用 Paddle/LemonSqueezy 代收代付；规模化后新加坡公司结算 |

### 9.2 止损线

```
止损判断标准：

Phase 0 止损线（M1 末）：
  · Landing Page 转化率 <2% 且 TikTok 互动率 <1%
  → 行动：转换目标市场或调整产品方向

Phase 2 止损线（M7 末）：
  · 总注册 <2,000 或付费用户 <100
  → 行动：深度用户访谈，判断是产品问题还是推广问题
  → 如果产品问题：大幅调整游戏化设计
  → 如果推广问题：调整渠道策略

持续亏损止损线：
  · 连续 6 个月月亏损 >$1,000 且趋势未改善
  → 行动：缩减推广预算，回到最小运营模式
  → 累计亏损 >$10,000 考虑暂停项目
```
