## 八、决策树

### 8.1 MVP 后关键决策

```
MVP 上线（M2）
  │
  ├── D1 留存 > 30%？
  │     ├── YES → 继续投入，进入增长期
  │     └── NO → 分析原因：
  │           ├── 内容问题？→ 优化课程体验
  │           ├── 技术问题？→ 修 Bug，优化性能
  │           └── 需求问题？→ 深入用户调研
  │
  ├── 付费转化 > 3%？（M3-M4）
  │     ├── YES → 加大投放，扩大规模
  │     └── NO → 优化转化路径：
  │           ├── 调整定价？
  │           ├── 调整付费墙位置？
  │           └── 增加试用内容？
  │
  └── M6 月收入 > ¥15,000？
        ├── YES → 按路线图继续，准备印尼扩展
        └── NO → 控制支出，聚焦越南市场
              ├── 削减投放预算
              ├── 优化变现模型
              └── 考虑是否 pivot
```

### 8.2 扩展决策

```
印尼市场（M7-M9）
  │
  ├── 条件检查：
  │     ├── 越南市场稳定增长？
  │     ├── 月收入覆盖成本？
  │     └── 团队有余力？
  │     全部 YES → 上线印尼
  │     任一 NO → 推迟，聚焦越南
  │
iOS 上架（M10-M12）
  │
  ├── 条件检查：
  │     ├── Android 用户占比 > 30%？（说明移动端需求强）
  │     ├── 月收入 > ¥50,000？（承担苹果抽成）
  │     └── Web 订阅模式验证成功？
  │     全部 YES → 上架 iOS
  │     否则 → 继续 PWA + Android 策略
```

### 8.3 高阶课程决策

```
L10-L12 开发（M13+）
  │
  ├── 条件检查：
  │     ├── L9 完成用户 > 500 人？（有真实需求）
  │     ├── 用户主动反馈要求高阶内容？
  │     └── 收入可覆盖内容制作成本？
  │     全部 YES → 开发 L10-L12
  │     否则 → 暂停，投入到营销/新市场
```

## 九、风险缓冲

| 风险 | 缓冲方案 |
|------|---------|
| MVP 延期 | 预留 2 周 Buffer |
| 用户增长低于预期 | 预留 3 个月运营资金 |
| 内容生产瓶颈 | 提前 2 个 Level 做内容储备 |
| 技术故障 | 自动备份 + 回滚机制 |
| 投放 ROI 低 | 快速切换渠道，控制单日预算上限 |
| 竞品突袭 | 坚持差异化（双轨课程+文化+游戏闭环） |
