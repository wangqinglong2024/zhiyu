# HanYu — 游戏化设计方案

> 把教学变成游戏，把故事变成过场动画，把练习变成战役
> 版本：v1.0 | 日期：2026-04-15

---

## 一、设计哲学

### 1.1 核心原则

1. **学习是游戏推进的燃料**：用户必须掌握中文知识才能推进游戏，但用户的主观体验是"我在玩一个策略游戏"
2. **过场动画代替课文**：传统语言 App 的"故事""对话""语法讲解"全部以游戏过场动画/剧情CG的形式呈现
3. **荣誉感驱动**：段位、军衔、领地版图、勋章体系，让用户有"我在不断变强"的进阶感
4. **即时策略节奏**：不是回合制答题，而是有时间压力、有策略选择、有即时反馈的游戏节奏
5. **2D 像素美术 + 高品质 UI**：像素风格控制美术成本，但 UI/UX 设计必须精致，对标独立游戏而非教育 App

### 1.2 参考游戏

| 参考作品 | 借鉴元素 |
|---------|---------|
| **Kingdom Rush** (塔防) | 策略部署感、关卡地图推进、英雄系统 |
| **Clash Royale** (即时策略) | 实时对战节奏、卡牌收集、段位系统 |
| **文明 (Civilization)** | 领地扩张、科技树（=语言技能树）、文明发展感 |
| **Stardew Valley** | 像素美术风格、经营养成、成就感 |
| **Duolingo** | Streak 打卡、排行榜段位（但我们做得更重度） |
| **刺客信条：探索之旅** | 游戏与教育内容的融合方式 |

## 二、游戏世界观

### 2.1 背景设定

**世界名：汉字大陆（HanZi Continent）**

一个由汉字能量驱动的大陆。大陆被划分为多个领地，每个领地对应一个 HSK 级别或主题领域。玩家扮演一位"语言探索者"（Language Explorer），通过学习中文来解锁力量、征服领地、建设自己的文明。

**核心设定**：
- 汉字是大陆的能量来源，每学会一个汉字就获得对应的"字能"
- 不同领地有不同的文化主题（美食领地、商业领地、历史领地等）
- 每个领地有守护 Boss，击败 Boss 需要掌握该领地的核心词汇和语法
- 过场动画全部使用中文对话（带母语字幕），玩家在观看剧情的同时自然接触中文

### 2.2 领地地图

```
汉字大陆全景地图（由简到难，从左到右展开）：

[拼音群岛]        HSK 1 起步区
  ├─ 声母海湾     声母学习
  ├─ 韵母港       韵母学习
  └─ 声调山       四声调练习

[日常平原]        HSK 1-2 基础区
  ├─ 问候村       打招呼、自我介绍
  ├─ 数字集市     数字、时间、日期
  ├─ 美食街区     饮食相关词汇

[职场要塞]        HSK 2-3 进阶区
  ├─ 工厂前线     制造业场景词汇

  ├─ 办公塔       职场中文
  └─ 面试殿堂     求职面试场景

  ├─ 成语密林     成语故事
```

## 三、核心游戏机制

### 3.1 关卡推进系统

#### 关卡类型一：知识战役（核心学习）

**形式**：即时策略 + 语言知识

```
场景示例 —— "美食街区·第3关：奶茶店攻防战"

过场动画（30秒，全中文配音+母语字幕）：
  → NPC 店员："欢迎光临！请问你要喝什么？"
  → 旁白介绍本关目标：学会用中文点饮料

游戏阶段：
  1. [词汇收集] 屏幕上飘来各种饮料图标，玩家需要快速
     将正确的中文名拖到对应图标上（限时15秒）
     → 正确：获得"字能"，充入战斗能量条
     → 错误：能量条不增加

  2. [句型组装] 出现打散的中文词语卡片，玩家需要在限
     时内拖拽排列成正确句子
     → "我 / 想 / 要 / 一杯 / 冰 / 奶茶"
     → 正确排列后，角色自动说出这句话（TTS播放）

  3. [Boss 对话] 店员 NPC 开始快速提问：
     → "大杯还是小杯？" → 选择正确答案
     → "要加糖吗？" → 选择正确答案
     → "一共15块" → 在虚拟钱包中选出正确金额
     → 全部答对 = 击败 Boss = 获得"奶茶大师"称号

结算画面：
  → 显示本关学会的词汇/句型
  → 获得 XP + 金币 + 称号
  → 星级评分（1-3星，根据用时和正确率）
  → 分享按钮（生成战绩海报）
```

#### 关卡类型二：过场剧情（故事学习）

**形式**：2D 视觉小说 + 自动播放中文对话

```
场景示例 —— "职场要塞·过场剧情：阿明的第一天"

剧情概要：
  越南青年阿明（玩家可自定义名字）来到中国工厂的第一天。
  工厂主管用中文给他布置任务，他必须听懂才能完成工作。

展现方式：
  - 2D 像素角色立绘 + 对话框
  - 全中文语音配音（AI 生成 + 人工校对）
  - 底部显示母语（越南语/印尼语）翻译
  - 关键词汇高亮显示（点击可查看详细解释）
  - 玩家可随时暂停回听

剧情结束后：
  → 弹出"剧情理解测试"（3-5道快速选择题）
  → 通过后解锁下一个战役关卡
  → 剧情中出现的新词汇自动加入玩家的"词库"
```

#### 关卡类型三：即时对战（PvP 竞技）

**形式**：实时 1v1 汉字/词汇竞速

```
对战模式 —— "汉字攻城战"

匹配机制：
  → 按段位匹配（铜/银/金/白金/钻石/大师/王者）
  → 30秒匹配超时则匹配AI对手

对战规则：
  - 双方各有一座城池，城池有 HP 值
  - 屏幕中央出现中文词汇题目
  - 答对一题 = 向对方城池发射一颗"字弹"
  - 答错 = 自己城池受到反弹伤害
  - 连续答对触发"连击"，伤害加倍
  - 30秒内 HP 高的一方获胜

奖励：
  - 胜利获得"战功点" + 金币
  - 段位升降（类似王者荣耀段位机制）
  - 赛季结束按段位发放专属皮肤/边框
```

## 四、教学内容与游戏的融合设计

### 4.1 融合原则

**铁律：学习内容不能脱离游戏情境存在。**

| 传统 App 的做法 | 我们的做法 |
|---------------|-----------|
| "请跟读这个词：苹果" | 在游戏关卡中，玩家需要正确识别水果名称才能给NPC完成任务 |
| "请翻译：我想要一杯水" | Boss 战中，NPC 用中文提问，玩家需要选择正确回答才能发动攻击 |
| "语法课：把字句" | 过场动画中角色使用"把字句"对话，动画结束后有句型重组挑战 |
| "听力练习：听录音选择" | 侦察任务：听取敌方NPC的中文对话，提取关键情报 |
| "阅读理解：读短文答题" | 解密任务：阅读中文线索，找到宝藏位置 |

### 4.2 每种语言技能的游戏化形式

#### 词汇学习

| 游戏机制 | 说明 | 体验感 |
|---------|------|--------|
| **汉字消消乐** | 配对汉字和含义，连续消除触发连击 | 爽快感 |
| **词汇塔防** | 敌人（错误词汇）进攻，放置正确翻译的炮塔防御 | 策略感 |
| **汉字拼装** | 把偏旁部首拖拽组合成完整汉字，限时挑战 | 成就感 |
| **词汇收集** | 关卡中散落的词汇碎片，收集齐全解锁新能力 | 探索感 |

#### 听力训练

| 游戏机制 | 说明 | 体验感 |
|---------|------|--------|
| **情报截听** | 播放中文对话片段，玩家需要选出正确信息 | 紧张感 |
| **声调射击** | 听到词语后射击对应的声调目标（一二三四声） | 即时反馈 |
| **听力迷宫** | 根据中文语音指令（左转、右转、前进）操作角色走迷宫 | 沉浸感 |
| **歌词填空** | 播放中文歌曲，在节奏点填入正确歌词 | 趣味感 |

#### 口语/发音

| 游戏机制 | 说明 | 体验感 |
|---------|------|--------|
| **声调过山车** | 用声调走势控制过山车轨道，声调正确轨道才平稳 | 直觉化 |
| **咒语释放** | 正确朗读中文"咒语"释放技能攻击敌人 | 成就感 |
| **语音解锁** | 特定门/宝箱需要正确朗读中文口令才能打开 | 仪式感 |

#### 阅读理解

| 游戏机制 | 说明 | 体验感 |
|---------|------|--------|
| **密码破译** | 阅读中文提示找到密码/路线 | 解谜感 |
| **剧情选择** | 阅读剧情对话后做出选择，影响故事走向 | 叙事感 |
| **藏宝图** | 阅读中文地图指引找到宝藏 | 探索感 |

#### 汉字书写

| 游戏机制 | 说明 | 体验感 |
|---------|------|--------|
| **书法道场** | 在屏幕上手写汉字，笔画正确得分，连续正确触发连击 | 专注感 |
| **封印解除** | 正确书写汉字可以解除关卡中的"封印" | 仪式感 |
| **书法对决** | 与对手比赛谁写得更准确更快速 | 竞技感 |

### 4.3 过场动画系统

**设计规范**：

```
每段过场动画的标准格式：

时长：30-90 秒
画面：2D 像素角色立绘 + 场景背景 + 对话框
配音：AI 生成中文语音（由 Dify 工作流 + Azure TTS 生产）
字幕：
  - 第一行：中文（汉字）
  - 第二行：拼音
  - 第三行：母语翻译（越南语/印尼语）
交互：
  - 可暂停/回放
  - 关键词汇点击高亮弹出详解
  - 自动添加新词到玩家词库
生产方式：
  - 剧本：Dify 工作流调用 LLM 生成初稿 → 人工审校
  - 配音：Azure Neural TTS 生成 → 质量检查
  - 美术：AI 辅助生成像素背景 → 美术师微调
  - 翻译：LLM 翻译 → 母语者审校
```

**过场动画的叙事主题（贴近东南亚用户）**：

| 系列 | 主角 | 故事线 | 教学场景覆盖 |
|------|------|--------|-------------|
| **阿明闯中国** | 越南青年阿明 | 到中国工厂工作的冒险 | 职场/日常/交通/饮食 |
| **小兰的梦想** | 印尼华裔小兰 | 回中国寻根之旅 | 文化/历史/家庭/情感 |
| **商人李大** | 东南亚商人 | 与中国客户做生意 | 商务/谈判/数字/合同 |
| **留学日记** | 多国留学生 | 在中国大学的日常 | 校园/社交/考试/生活 |

## 五、2D 游戏技术选型

### 5.1 游戏引擎：Phaser 3

**为什么选 Phaser 3**：

| 维度 | Phaser 3 | PixiJS | Cocos Creator | Unity WebGL |
|------|----------|--------|---------------|-------------|
| **定位** | 完整 2D 游戏框架 | 2D 渲染引擎 | 全功能引擎 | 全功能引擎 |
| **TypeScript 支持** | 原生支持 | 原生支持 | 一般 | 不支持（C#） |
| **与 React 集成** | 可嵌入 React 组件 | 可嵌入 | 独立运行 | 独立运行 |
| **包体积** | ~300KB (gzip) | ~200KB | ~2MB+ | ~10MB+ |
| **学习曲线** | 中等 | 低（但需自建游戏逻辑） | 高 | 高 |
| **社区生态** | 极丰富，教程多 | 丰富 | 中等 | 丰富但重 |
| **2D 像素游戏** | 最佳选择 | 需自建 | 可以但重 | 太重 |
| **移动端性能** | 好（Canvas/WebGL） | 好 | 好 | 差（加载慢） |
| **开源/免费** | MIT 开源 | MIT 开源 | 免费 | 收费 |

**结论**：Phaser 3 是最适合的选择——TypeScript 原生支持、可嵌入 React、专为 2D 游戏设计、包体积小、社区成熟。

### 5.3 美术风格规范

**主风格：现代像素风（Modern Pixel Art）**

```
分辨率规范：
  · 角色：32x32 像素基准，4 方向行走动画
  · 场景 Tile：16x16 像素
  · UI 元素：矢量设计（非像素），保证清晰度
  · 过场立绘：像素风格但更精细（128x128 或更高）

配色方案：
  · 主色调：深蓝 #1a1a2e + 金色 #e6b800（东方+高端感）
  · 辅助色：翡翠绿 #00b894 + 珊瑚红 #ff6b6b
  · 背景色：深灰蓝渐变（避免纯白/纯黑，营造游戏氛围）
  · UI 色：半透明深色面板 + 金色描边

美术生产流程：
  1. AI 生成概念图（Midjourney/Stable Diffusion + 像素化 LoRA）
  2. 像素美术师基于概念图绘制正式素材
  3. 动画：使用 Aseprite 制作帧动画
  4. 场景：Tiled Map Editor 编辑地图
  5. UI：Figma 设计 → React 组件实现

成本控制：
  · AI 生成初稿可减少 50-70% 美术时间
  · 像素风格本身比高清手绘成本低 80%+
  · Tile 复用：一组 Tile 可组合出大量场景
  · 角色复用：基础骨架 + 换皮系统
```

### 5.4 音效与音乐

```
音乐风格：中国风 + 电子/Chiptune 融合
  · 世界地图：悠扬的中国风 BGM（古筝+电子节拍）
  · 战役关卡：紧张的策略音乐
  · PvP 对战：激烈的战斗音乐
  · 过场动画：叙事配乐
  · 商城/个人页：轻松休闲音乐

音效：
  · 答对：清脆的"叮"声 + 金币收集声
  · 答错：低沉的"嗡"声
  · 连击：递进的音效（越连击音效越华丽）
  · 升级/升段：史诗级音效 + 视觉特效
  · UI 操作：清脆的点击反馈

生产方式：
  · 音乐：AI 生成（Suno/Udio）+ 人工混音调整
  · 音效：免费素材库（Freesound）+ 简单编辑
  · 中文语音：Azure Neural TTS
```

## 七、内容生产 Pipeline（AI 驱动）

### 7.1 关卡内容生产流程

```
Step 1: 需求定义（人工）
  → 确定关卡主题、目标词汇、目标语法点、HSK 级别

Step 2: 内容生成（Dify 工作流）
  → 输入：主题 + 词汇表 + 语法点 + 难度级别
  → 输出：
    · 关卡剧本（过场动画对话脚本）
    · 题目库（30-50 道，关卡随机抽取 10-15 道）
    · Boss 对话脚本
    · 词汇卡片数据
  → 使用模型：Claude/GPT-4 生成初稿

Step 3: 人工审校（质量关卡）
  → 语言准确性检查（母语中文教师）
  → 教学有效性检查（语言教学专家）
  → 文化敏感性检查（东南亚本地化审核）
  → 游戏趣味性检查（游戏设计师）

Step 4: 素材生产
  → AI 生成配音（Azure TTS）→ 人工校听
  → AI 生成背景概念 → 像素美术师制作
  → 动画制作（Aseprite）
  → 翻译（LLM + 母语者审校）

Step 5: 集成测试
  → 导入 Phaser 关卡引擎
  → 自动化测试（题目逻辑、评分逻辑）
  → 人工试玩测试

Step 6: 上线发布
  → 灰度发布（10% 用户先体验）
  → 收集数据（通关率、用时、满意度）
  → 调整难度/内容 → 全量发布
```

### 7.2 内容产能预估

```
使用 AI 辅助的产能（1人团队）：

每周可产出：
  · 3-5 个战役关卡（含过场动画）
  · 1 个新小游戏关卡
  · 10-15 道 PvP 题目
  · 1 段完整剧情线

每月可产出：
  · 15-20 个战役关卡（约 1 个完整领地分区）
  · 40-60 道 PvP 题目
  · 4 段剧情线

对比传统方式（纯人工）：
  · 传统方式产能约为 AI 辅助的 1/5 到 1/3
  · AI 辅助的核心价值：初稿生成速度，人工只需审校和微调
```
