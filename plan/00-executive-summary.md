## 一、核心理念：三句话说清楚我们做什么

**用 AI 构建产品，而不是把 AI 当产品卖。**

1. **游戏即教学**：所有中文教学内容被设计为 2D 游戏关卡。传统 App 的"故事""对话""语法讲解"全部变成游戏内的过场动画、剧情任务和战役指挥。用户不是在"上课"，而是在"打游戏的过程中自然习得中文"。
2. **AI 构建内容，一次生成、无限分发**：AI 被用于生成高质量的课程脚本、游戏剧情、配音、插画素材。这些内容一旦生成并审校完毕，就成为固定资产，可以分发给无限用户，边际成本趋近于零。
3. **AI 功能透明计费**：平台内的 AI 辅助功能（如 AI 对话练习、作文批改）按实际调用量收费，平台不加价。会员价值体现在更多游戏内容、皮肤、装饰——而不是"AI 调用额度"。

## 二、为什么这样做：与"AI 套壳"模式的本质差异

| 维度 | AI 套壳模式（竞品常见） | 我们的模式 |
|------|----------------------|-----------|
| **核心卖点** | "AI 帮你练口语" | "在游戏中自然学会中文" |
| **成本结构** | 用户越多、AI 调用越多、成本线性增长 | 游戏内容一次制作、无限分发、边际成本趋零 |
| **被替代风险** | 大模型公司随时可以做同样的事 | 游戏内容+荣誉体系+社交关系=不可替代的用户资产 |
| **会员价值** | 更多 Token 额度 | 更多关卡、皮肤、称号、地图 |
| **AI 角色** | 产品核心（没有 AI 就没有产品） | 生产工具 + 可选增值服务 |
| **护城河** | 无（API 调用无壁垒） | 内容库 + 游戏体验 + 用户社区 |

**核心逻辑**：如果一个产品的核心功能就是调用 AI API，那大模型公司自己做一个就能替代你。但如果 AI 只是你的"工厂设备"，你卖的是工厂生产出来的"游戏产品"，那你的价值就不可替代。

## 三、产品定位

**一句话**：东南亚第一款以 2D 策略游戏为载体的中文学习应用——高颜值、有荣誉感、像打游戏一样学中文。

**品牌名**：HanYu（汉语）
- 越南市场：Hoc Han（学汉）
- 印尼市场：HanYu（保持原名，华人多、认知度高）

**Slogan 候选**：
- "征服汉字，征服世界" / "Chinh phuc Han tu, chinh phuc the gioi"
- "用游戏学中文，用中文改变命运"
- "Play to Learn. Learn to Earn."

**核心体验关键词**：
- **荣誉感**：段位、军衔、领地、勋章，让用户有"我在变强"的感觉
- **即时策略**：不是回合制答题，而是有节奏感、有紧迫感的策略操作
- **高颜值**：精致的 2D 像素美术 + 流畅动画，视觉品质对标独立游戏而非教育 App
- **社交竞技**：排行榜、对战、攻城战，学中文变成一场社交竞赛

## 六、竞争格局与我们的差异化

### 6.1 现有竞品的共同问题

所有现有竞品（Duolingo、HelloChinese、Speak、LingoDeer）本质上都是 **"课程 + 练习"模式**——无论 UI 多好看，底层逻辑都是"先学知识点，再做练习题"。用户始终清楚自己在"上课"。

**我们的根本差异**：用户不是在"上课做题"，而是在"玩一个策略游戏"。学习是游戏推进的必要条件，但用户的主观体验是"我在玩游戏"。

### 6.2 差异化矩阵

| 维度 | Duolingo | HelloChinese | Speak | **HanYu（我们）** |
|------|----------|-------------|-------|----------------|
| **产品形态** | 课程闯关 App | 课程闯关 App | AI 对话 App | **2D 策略游戏** |
| **学习动力** | 打卡/排行榜 | 课程进度 | 口语需求 | **游戏荣誉感+领地征服** |
| **视觉品质** | 卡通风 | 简洁风 | 极简风 | **精致像素美术/独立游戏级** |
| **内容载体** | 选择题/翻译 | 选择题/手写 | AI 对话 | **策略关卡/过场动画/Boss战** |
| **AI 角色** | 核心付费功能 | 辅助功能 | 核心产品 | **构建工具+按量付费增值** |
| **会员价值** | 去广告+AI | 解锁课程 | AI 对话额度 | **游戏内容/皮肤/装饰/地图** |
| **月价格** | $12.99-29.99 | $11.99 | $20 | **$1.99-2.99** |
| **东南亚本地化** | 部分 | 无 | 无 | **完整（母语界面+文化场景）** |
| **被大模型替代风险** | 高 | 中 | 极高 | **低** |

## 七、技术栈概览

| 层级 | 技术选择 | 理由 |
|------|---------|------|
| **前端框架** | React + TypeScript | 主站与管理后台 |
| **2D 游戏引擎** | Phaser 3 (TypeScript) | 最成熟的 HTML5 2D 游戏框架，可嵌入 React |
| **2D 渲染增强** | PixiJS | 游戏外的 UI 特效与动画 |
| **后端 API** | FastAPI (Python) | 异步高并发，AI 生态集成 |
| **数据库/Auth** | Supabase (PostgreSQL) | 内置 Auth + RLS + Realtime |
| **AI 工作流** | Dify | 内容生成工作流编排（课程/剧情/配音） |
| **服务器** | 腾讯云新加坡节点 | 东南亚低延迟，中国企业生态 |
| **CDN/安全** | Cloudflare | 全球边缘加速 + DDoS 防护 |
| **支付** | Stripe + 本地支付 | MoMo/GoPay 等本地化支付 |

## 八、商业模式核心逻辑

```
收入结构（成熟期目标）：

会员订阅（游戏内容/皮肤/装饰）    60%    ← 核心收入，边际成本趋零
B2B 企业培训                      20%    ← 第二增长曲线
广告收入（免费用户）                10%    ← 补充收入
AI 按量付费（透传成本）              5%    ← 不赚钱，赚口碑
增值服务（HSK 考试包/证书）          5%    ← 补充收入
```

**会员不卖 AI 额度，卖游戏体验**：
- 免费用户：基础关卡 + 基础皮肤 + 广告
- 会员用户：全部关卡 + 专属皮肤/装饰 + 高级地图 + 去广告 + 排行榜特权

**AI 功能透明计费**：
- AI 对话练习、作文批改等功能，按实际 API 调用成本收费
- 平台不加价，用户充值 HanYu Coins，按实际消耗扣减
- 免费用户每天赠送一定量的 Coins（体验 AI 功能）

## 十、核心风险与应对

| 风险 | 应对 |
|------|------|
| **游戏品质不达标** | 用 AI 生成素材 + 外包像素美术，控制成本的同时保证品质；MVP 先做 1-2 个精品关卡验证 |
| **"游戏"和"学习"体验割裂** | 学习内容必须是游戏推进的必要条件，不是附加品；参考《刺客信条：探索之旅》的融合方式 |
| **2D 游戏技术复杂度** | Phaser 3 生态成熟，社区资源丰富；先简后繁，MVP 用最简单的策略地图 |
| **AI 内容质量不稳定** | AI 生成 → 人工审校 → 质量达标后才上线，建立内容 QA 流程 |
| **东南亚用户对"游戏化学习"的接受度** | Phase 0 用 TikTok 短视频测试游戏化概念的用户反馈 |
| **大模型公司做同类产品** | 我们卖的是游戏体验和内容库，不是 AI 能力；游戏内容是壁垒 |
