## 一、收入结构总览

LingoBase 采用**多元收入结构**，以会员订阅为核心，辅以课程单购、游戏内购和广告收入。

```
收入结构
  ├── 会员订阅（核心收入，占比 50-60%）
  ├── 课程阶段单购（补充收入，占比 15-20%）
  ├── 游戏内购（道具+关卡包，占比 10-15%）
  ├── 广告收入（仅免费用户，占比 5-10%）
  └── 推荐返佣（驱动增长，占比 5-10%）
```

## 五、广告收入

### 5.1 广告策略

| 原则 | 说明 |
|------|------|
| 仅免费用户 | 付费会员/课程购买者无广告 |
| 非侵入式 | 不在课程/游戏过程中弹广告 |
| 激励视频为主 | 用户自愿观看换取奖励 |

### 5.2 广告位

| 广告位 | 类型 | 频率 |
|--------|------|------|
| 游戏关卡间 | 插屏广告 | 每 3 关 1 次 |
| 复活机会 | 激励视频 | 自愿 |
| 双倍积分 | 激励视频 | 自愿 |
| 发现中国信息流 | 原生广告 | 每 5 条内容穿插 1 条 |
| 首页底部 | Banner | 持续展示 |

### 5.3 预估收入

| 场景 | eCPM 估算 | 说明 |
|------|----------|------|
| 插屏广告 | $2-5 | 东南亚市场 |
| 激励视频 | $5-10 | 完播率高 |
| 原生信息流 | $1-3 | 低干扰 |
| Banner | $0.5-1 | 最低但持续 |

## 六、推荐返佣体系

### 6.1 机制

```
用户 A 分享邀请链接 → 用户 B 注册 → 用户 B 首次付费
     ↓
用户 A 获得返佣（B 首次支付金额的 20%）
用户 B 获得优惠（首单 9 折）
```

### 6.2 返佣规则

| 规则 | 说明 |
|------|------|
| 返佣比例 | 被推荐人首次付费的 **20%** |
| 生效条件 | 被推荐人完成首次付费且未退款（7天确认期） |
| 提现门槛 | 累计 ¥50 起提现 |
| 提现方式 | 银行转账 / PayPal / 平台余额（可购买课程/会员） |
| 推荐上限 | 无上限 |
| 有效期 | 推荐关系绑定永久有效 |

### 6.3 推荐人激励

| 累计推荐人数 | 额外奖励 |
|------------|---------|
| 5 人 | 赠送 1 个月会员 |
| 10 人 | 赠送基础道具包 |
| 20 人 | 赠送 3 个月会员 |
| 50 人 | 赠送年度会员 |
| 100 人 | 赠送终身会员 |

## 七、付费转化路径

### 7.1 免费用户转化漏斗

```
下载/访问 → 浏览发现中国（免费）→ 试学 L1 课程（免费）→ 试玩游戏（免费10关）
                                                              ↓
                                                 遇到付费墙 / 体验限制
                                                              ↓
                                         弹出优惠引导 → 选择会员/单购 → 付费
```

### 7.2 付费墙触发点

| 触发场景 | 展示内容 |
|---------|---------|
| L1 课程学完，进入 L2 | "解锁全部 12 级课程，会员仅 ¥22/月" |
| 游戏第 11 关（青铜付费起点） | "解锁更多关卡，或成为会员免费玩" |
| AI 功能达到免费上限 | "升级会员，无限使用 AI 助手" |
| 升级考试（免费用户不可用） | "通过考试获得证书，成为会员解锁" |

### 7.3 促销策略

| 策略 | 时机 | 内容 |
|------|------|------|
| 新用户 7 天优惠 | 注册后 7 天内 | 年度会员 7 折 |
| 节日促销 | 春节/中秋/双11 | 终身会员限时优惠 |
| 学习里程碑 | 完成 L1 / 连续学习 7 天 | 弹出限时折扣 |
| 流失挽回 | 30天未登录 | Push 通知 + 专属优惠 |

## 八、支付实现

### 8.1 Paddle（MoR 模式）

| 维度 | 说明 |
|------|------|
| 角色 | Merchant of Record（商户记录方） |
| 优势 | Paddle 处理全球税务、发票、合规 |
| 费率 | ~5% + $0.50/笔（含税务处理） |
| 支持方式 | 信用卡、PayPal、本地支付 |
| 订阅管理 | Paddle 内置订阅管理（续费/取消/退款） |

### 8.2 支付流程

```
用户选择套餐 → 前端调用 Paddle Checkout → Paddle 处理支付
                                              ↓
                                    Paddle Webhook 通知后端
                                              ↓
                                    后端更新用户会员状态
                                              ↓
                                    前端刷新权限
```

### 8.3 iOS 特殊处理

- iOS App 内的数字内容购买**必须走 Apple IAP**（苹果 30% 抽成）
- 策略：引导用户通过 **Web 端** 完成订阅（Paddle），绕开苹果税
- App 内不展示购买按钮，只展示"已有会员？登录恢复"
- Web 端定价 = 标准价；iOS IAP 定价 = 标准价 + 30%（如果必须上架 IAP）

## 九、定价调研指南

实际上线前需完成以下调研：

| 调研项 | 方法 |
|--------|------|
| 竞品定价 | 下载 Duolingo / HelloChinese / ChineseSkill，记录各套餐价格 |
| 购买力评估 | 调研越南人均月收入、App 付费习惯 |
| 价格敏感度 | A/B 测试不同价位 |
| 本地支付偏好 | 了解越南用户常用支付方式（MoMo、ZaloPay） |
| 汇率影响 | ¥ vs VND 的兑换，确保本地货币标价合理 |

> 以上参考定价以人民币计，实际展示需换算为当地货币。
