# 弹窗与公共组件

> 全部弹窗 / Drawer / Modal 的统一定义；本文件为 [00-index.md §弹窗](./00-index.md) 表格的展开实现细节。

---

## 一、Modal 通用规范

| 维度 | 约定 |
|------|------|
| 宽度 | 二次确认 380px / 编辑表单 480–560px / 复杂表单 720px |
| 关闭 | 右上角 ×、ESC、点击遮罩；危险操作（D-12）禁用 ESC + 遮罩点击 |
| 焦点 | 打开后焦点入第 1 个可输入元素；关闭后归还触发按钮 |
| 动画 | G2 标准 200ms ease-out fade + scale 0.95→1 |
| 堆叠 | 同时最多 1 个 Modal；新 Modal 触发先关旧 Modal |
| 无障碍 | `role="dialog"` + `aria-labelledby` + `aria-describedby` |

---

## 二、Drawer 通用规范

| 维度 | 约定 |
|------|------|
| 位置 | 右侧滑入；移动端全屏 |
| 宽度 | 480 / 560 / 720 三档；超大表单走 720 |
| 关闭 | × 按钮 / ESC / 点击遮罩（脏数据先 D-2） |
| 标题区 | 固定顶部；含主操作按钮（保存 / 发布 / 关闭） |
| 内容区 | 滚动；底部留 64px padding 让按钮悬浮 |
| 多 Drawer | 不允许嵌套；触发新 Drawer 自动关旧 Drawer + 脏检查 |

## 三、各弹窗详细规格

### D-1 · 二次确认 Modal（380px）

通用模板：

```
┌─ 标题 ─────────────────────────[×]┐
│  操作描述（说明影响）                │
│  [ 取消 ]              [ 确认 ]     │
└────────────────────────────────────┘
```

| 触发 | 标题 | 内容 | 危险级别 |
|------|------|------|---------|
| 删除 stage / chapter / lesson / KP / Q / exam | 「确认删除？」 | 「该操作将{软删 / 物理删}，{有学员进度时}不可恢复」 | red |
| 下架 lesson | 「下架后学员将不可见」 | — | yellow |
| 学员举报 dismiss | 「确认 dismiss 该举报？」 | — | normal |
| 切主题 share 未完成 | 「请先完成基础 Stage 0」 | 「主题主题在 share Stage 0 完成后解锁」 | normal（仅一个「我知道了」按钮） |
| AI Job 取消 | 「取消后已生成的子任务保留」 | — | yellow |
| 媒资删除 | 「7 天后真删 CDN」 | 「窗口期可在『清理』入口撤销」 | yellow |
| 重置学员进度（运维专用） | 「该学员所有进度将归零」 | — | red（需输入 yes 才启用） |

```
┌─ 批量导入 KP ─────────────────────[×]┐
│ 拖拽文件到此处或 [选择文件]            │
│   支持：.csv / .json（≤ 5 MB）          │
│ ──────────────────────────             │
│ 模板下载：[CSV 模板] [JSON 模板]        │
│ ──────────────────────────             │
│ 预览（解析 240 行 / 3 错误）             │
│  ✓ 行 1: kp_ec_000999 早安              │
│  ✗ 行 5: 缺少必填字段 title_zh           │
│  ✗ 行 12: i18n.zh 缺失                   │
│  ✓ 行 13: ...                           │
│ ──────────────────────────             │
│ ☑ 跳过错误行                            │
│ [ 取消 ]            [ 导入 237 条 ]      │
└──────────────────────────────────────────┘
```

| 步骤 | 实现 |
|------|------|
| 解析 | 前端 papaparse / JSON.parse；本地校验 [F1-09 §i18nText](../F1-AI-数据模型规范/09-校验规则汇总.md) |

| 进度 | 顶部进度条 + 完成 Toast「导入 235 条 / 跳过 5 条 / 失败 0 条」 |

题目导入入口同上，接口为 `/questions/batch-import`。

## 四、Toast 全局规则

| 类型 | 颜色 | 时长 | 用途 |
|------|------|------|------|
| success | 绿 | 2s | 保存成功 / 发布成功 |
| info | 蓝 | 3s | 一般通知 |
| warning | 黄 | 4s | 数据不完整、限流等 |
| error | 红 | 4s + 折叠面板含 request_id | 接口失败、403、5xx |

- 多 Toast 自动垂直堆叠（最多 5 条），溢出旧的自动消失
- 同 code 错误 1s 内合并

| 场景 | G2 引用 |
|------|--------|
| 列表加载 | [G2-04 §骨架屏](../../../../grules/G2-视觉与交互风格/04-状态与组件.md) |

## 六、键盘快捷键（管理端）

| 快捷键 | 行为 |
|--------|------|
| `⌘/Ctrl + S` | 保存当前 Drawer / 表单 |
| `⌘/Ctrl + K` | 打开全局搜索 |
| `Esc` | 关闭最上层 Modal / Drawer（脏检查） |
| `?` | 显示快捷键帮助层 |
| `g` 然后 `c` | 跳课程目录 |
| `g` 然后 `r` | 跳审核队列 |
| `g` 然后 `j` | 跳 AI 工作台 |
