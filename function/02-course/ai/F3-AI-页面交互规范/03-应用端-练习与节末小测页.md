# P-C-3 应用端 · 练习与节末小测页

**路由**：`/learn/lessons/:lesson_code/quiz`
**面包屑**：学中文 › … › {lesson} › 节末小测
**权限**：登录学员
**前置**：来自 P-C-2 的「完成本节，进入小测」入口或 P-C-1 节列表项点击「补做小测」。

---

## 关键约定

- **节末小测固定 6 题**（[F3-Q8 封板](./13-待确认问题清单.md) 覆盖原 5 题）；不走 `course_exams` 模板。
- **KP 抽题策略**：约 80% 本节新 KP（`is_new_in_lesson=true`，5 题）+ 约 20% 上节复习 KP（1 题），由后端 [C8](../F2-AI-接口规范/02-应用端-答题与SRS.md) 决定。
- **可跳过**（[F1-Q4 封板](../F1-AI-数据模型规范/12-待确认问题清单.md)）：除 `is_quiz_required=true` 节外，学员可中途返回；返回保留草稿（仅本地）。
- **上一题可改答**（[F3-Q2 封板](./13-待确认问题清单.md)）：已答题可回看并修改；改答触发 C9 `overwrite=true` 重判，覆盖原答案与 SRS 副作用。
- **判分**：C9 单题判分（即时反馈）+ C10 提交汇总。

## 元素规格

| 元素 | 组件类型 | 说明 |
|------|---------|------|
| 顶部题号 | `<span>` | `当前 / 6` |
| 进度条 | `Progress` | `当前-1 / 6`（提交后不变） |
| 题目主体 | `QuestionRenderer` | 12 题型路由到对应组件（见下表） |
| 选项 | `Radio` / `Checkbox` / `Sortable` / `Input` 等 | 按 `q_type` 切换 |
| 答完反馈卡 | `FeedbackCard` | 绿色（对）/ 红色（错）；含解释 / 正确答案 / 知识点链接 |
| 上一题 | `Button` | 第 1 题禁用；可重新查看且**可改答**（F3-Q2 封板）|
| 下一题 | `Button(Primary)` | 未答禁用；最后一题文案变「提交并查看结果」 |
| 退出 | 顶部「← 返回」 | 弹 D-1：「答题进度不会保存，确认离开？」 |

### 12 题型渲染（与 [F1-06 §2 题型枚举](../F1-AI-数据模型规范/06-枚举定义.md) 对齐）

| q_type | 渲染要点 |
|--------|---------|
| `mcq_meaning` | 中文 → 4 选 1 翻译（按 `ui_lang`） |
| `mcq_zh` | 翻译 → 4 选 1 中文 |
| `listen_pick` | 🔊 → 4 选 1 中文 |
| `listen_pinyin` | 🔊 → 4 选 1 拼音 |
| `tone_pick` | 拼音（无声调）→ 4 选 1 声调 |
| `match_pairs` | 左右两列连线（中文 ↔ 翻译） |
| `sort_words` | 词块拖拽组成正确顺序 |
| `fill_blank_choice` | 句中缺词，4 选 1 |
| `type_pinyin` | 文本框输入拼音（带声调辅助键盘） |
| `type_zh_ime` | 文本框 IME 输入中文 |
| `image_pick` | 图片 → 4 选 1 中文 |
| `dialog_cloze` | 对话场景 → 4 选 1 缺句 |

## 数据来源

- 进入：调 [C8 GET /api/v1/course/lessons/:lesson_id/quiz](../F2-AI-接口规范/02-应用端-答题与SRS.md) 取 6 题（无草稿）。
- 单题答完：调 [C9 POST /api/v1/course/answers](../F2-AI-接口规范/02-应用端-答题与SRS.md) 即时判分（含写 SRS 副作用）；改答时携带 `overwrite=true`，服务端覆盖原答案 + 重判 + 重算 SRS。
- 全部完成：调 [C10 POST /api/v1/course/lessons/:lesson_id/quiz/submit](../F2-AI-接口规范/02-应用端-答题与SRS.md) 汇总打分 + 写 `course_user_progress`。

## 交互行为

| # | 用户操作 | 技术实现 |
|---|---------|---------|
| C3-01 | 进入页面 | 调 C8；500ms 内显示题目骨架；失败 → 整页错误 + [重试] |
| C3-02 | 选择答案 / 输入 | 本地暂存；按钮「下一题」由 disabled → enabled |
| C3-03 | 点击「下一题」 | 调 C9 提交本题 → 等待 200ms 内返回 → 展开 FeedbackCard → 启用「下一题」；C9 失败 → Toast「提交超时，已重试」+ 自动重试 1 次；最终失败 → Toast「无法提交，可继续答其它题，结束时一并补交」并标记本题 pending |
| C3-04 | 点击「上一题」 | 可回看且可改答；FeedbackCard 仍展示但选项重新可点；选定新答案 → 调 C9 `{overwrite:true}` 重判 → 更新 FeedbackCard |
| C3-05 | 点击最后题「提交并查看结果」 | 弹 D-12 二次确认；确认后调 C10；返回 attempt 总分 + 各题对错 + 是否通过；跳结果子页（同路由 `?view=result`） |
| C3-06 | 中途退出 | 顶部「← 返回」弹 D-1：「未提交将丢失草稿。」选「确认离开」清空本地 → 跳回 P-C-1 节列表 |
| C3-07 | 浏览器刷新 / 关闭 | 触发 `beforeunload` 提示；本地草稿不持久（防作弊） |
| C3-08 | 网络断开 | 顶部离线条；C9 失败按 03 处理（pending） |
| C3-09 | 听力题播放 🔊 | 走 C6（与 P-C-2 一致）；同一 hash 已缓存即返 |
| C3-10 | 题目内点举报 | 弹 D-14，调 C18，提交后 Toast「已收到反馈」 |

|------|------|
| 总分 / 通过状态 | C10 返回 `score / passed` |

## 条件显示规则

| 条件 | 显示 | 隐藏 |
|------|------|------|
| 第 1 题 | 「上一题」disabled | — |
| 当前题 = 6 | 「下一题」文案 → 「提交并查看结果」 | — |
| 当前题 pending（C9 失败） | 标识「⚠ 待补交」+ 提交时一并重试 | — |
| `lesson.is_quiz_required = true` | 「← 返回」弹窗多一段「该节小测必做，跳过将无法解锁下一节」 | — |

---

## 空态 / 异常

| 场景 | 处理 |
|------|------|
| C8 返回少于 6 题（KP 不足） | 顶部黄条「该节题库不足，仅 N 题」+ 仍可作答 |
| C10 服务端 5xx | Toast「提交失败，请稍后重试」+ 按钮可重试；不清空答案 |
| 学员中途切主题 | 阻断 + Toast「请先完成或退出本次小测」 |

---

## 性能 / 可访问性

- C8 返回到首题渲染 < 800ms。
- 听力题音频 `<audio>` 预加载（`preload="auto"`），减少播放延迟。
- 选项可键盘操作：1–4 / A–D 快捷选；Enter 进入下一题。
- 实时反馈卡用 `aria-live="polite"`。
