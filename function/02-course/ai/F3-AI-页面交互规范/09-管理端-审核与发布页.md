## P-A-8 · 内容审核队列页

**路由**：`/admin/course/review`
**权限**：reviewer+

### 关键约定

- 审核对象：`course_knowledge_points` 与 `course_questions` 状态 = `reviewing` 的行。
- 审核员动作：批准（→ approved）/ 驳回（→ draft + 留言）/ 一键编辑（直接改后批准）。
- 审核日志写 `course_content_review_log`。

### 布局

```
┌──────────────────────────────────────────────────────────┐
│  内容审核 (待审 87)        [类别 ▼][主题 ▼][搜索 🔍]    │
├──────────────────────────────────────────────────────────┤
│ ☐ │ 类型│ Code             │ 摘要                  │ 提交人 │ 时间      │ 操作 │
│───┼─────┼─────────────────┼────────────────────────┼───────┼─────────┼─────│
│ ☐ │ KP  │ kp_ec_000999    │ 早安 zǎo ān            │ AI #38289│ 3 min  │ ⋯  │
│ ☐ │ Q   │ q_ec_00012345   │ 听音选词 · 早上          │ AI #38290│ 5 min  │ ⋯  │
│ ☐ │ Q   │ q_ec_00012346   │ 中文选义 · 谢谢          │ admin@x  │ 1 h    │ ⋯  │
│ ...                                                     │
├──────────────────────────────────────────────────────────┤
│  共 87 条      [批量批准] [批量驳回]                      │
└──────────────────────────────────────────────────────────┘
```

### 数据来源

- [A46 GET /admin/v1/course/review/queue](../F2-AI-接口规范/07-管理端-审核与发布.md)：分页 + 类别 + 时间倒序。

### 交互

| # | 操作 | 实现 |
|---|------|------|
| A8-01 | 进入 | 调 A46；侧边显示当前角色每日审核数（KPI） |
| A8-02 | 点击行 | 弹 D-9 审核详情 Drawer |
| A8-03 | 行内「批准」 | [A47 POST /admin/v1/course/review/kps/:id/approve](../F2-AI-接口规范/07-管理端-审核与发布.md) 或 `/questions/:id/approve` |
| A8-04 | 行内「驳回」 | 弹 Modal 输入 reason → [A48 POST .../reject](../F2-AI-接口规范/07-管理端-审核与发布.md) `{reason}` |
| A8-05 | 批量批准 | 选中行 → [A49 POST /review/batch-approve](../F2-AI-接口规范/07-管理端-审核与发布.md) `{type, ids[]}` |
| A8-06 | 批量驳回 | 输入统一 reason → [A50 POST /review/batch-reject](../F2-AI-接口规范/07-管理端-审核与发布.md) |

### D-9 · 审核详情 Drawer（720px）

```
┌─ 审核 KP · kp_ec_000999 早安 ──────[× 关闭][驳回][批准]┐
│ ┌─ 内容预览 ─────────────────────────────────────┐   │
│ │ 中文：早安  拼音：zǎo ān  类型：word              │   │

│ └────────────────────────────────────────────────────┘  │
│ ┌─ 一键编辑（行内可改）─────────────────────────────┐  │
│ │ ... 直接编辑字段 ... [保存并批准] [仅保存草稿]      │  │
│ └────────────────────────────────────────────────────┘  │
│ ┌─ 元信息 ──────────────────────────────────────────┐ │
│ │ 来源 Job：#38289                                   │ │
│ │ 提交时间：2026-04-15 10:20                          │ │
│ │ Prompt: p_gen_kp_word v3                            │ │
│ │ 历史驳回 0 次                                       │ │
│ └─────────────────────────────────────────────────────┘ │
│ 审核备注（可选）                                        │
│ [..............................................]       │
└──────────────────────────────────────────────────────────┘
```

| 操作 | 实现 |
|------|------|
| 批准 | 直接 A47；status → approved |
| 驳回 | 必须填 reason；A48；status → draft + 通知提交人（site message） |
| 一键编辑 → 「保存并批准」 | 先 [A25 PATCH /kps/:id](../F2-AI-接口规范/05-管理端-KP与题目CRUD.md)（修字段），再 A47 批准；事务 |
| 关闭 | 脏检查 D-2 |

### 条件显示

| 条件 | 行为 |
|------|------|
| 题目 `q_type` 为听力 / 图片 | 详情 Drawer 显示音频 / 图片预览（管理端预览**不**触发媒资缓存，仅用已有 url 直读；若无则 Toast「学员尚未触发缓存，无法预览」） |
| KP 关联题目 ≥ 1 | 显示「关联题目：5」可点跳 P-A-5 筛选 |
| 当前用户角色 = editor（非 reviewer） | 「批准 / 驳回」按钮 disabled + Tooltip「需 reviewer 权限」 |

## P-A-9 · 学员举报处理页

**路由**：`/admin/course/reports`
**权限**：reviewer+

### 关键约定

- 学员答错时若发现题目错误，可在 P-C-3 / P-C-7 题目内点举报（D-14）→ 写 `course_content_reports`（[F1-05 表](../F1-AI-数据模型规范/05-表定义-AI生产与媒资.md)）。
- 处理动作：dismiss（无效）/ 转入审核队列（adopt → 把对应 KP/Q 改 status='reviewing'）。

### 布局

```
┌──────────────────────────────────────────────────────────┐
│  学员举报 (待处理 12)     [类型 ▼][主题 ▼][搜索 🔍]      │
├──────────────────────────────────────────────────────────┤
│ ☐ │ 时间      │ 学员      │ 题目 / KP        │ 类型     │ 描述               │ 操作 │
│───┼──────────┼──────────┼──────────────────┼─────────┼───────────────────┼─────│
│ ☐ │ 3 min ago│ user_xxx  │ q_ec_00012345    │ wrong_answer│ "答案应该是早上"   │ ⋯  │
│ ☐ │ 1 h ago  │ user_yyy  │ kp_ec_000124    │ typo     │ "拼音 zǎo 错"        │ ⋯  │
│ ...                                                     │
└──────────────────────────────────────────────────────────┘
```

### 数据来源

- [A51 GET /admin/v1/course/reports](../F2-AI-接口规范/07-管理端-审核与发布.md)：分页 + 状态（默认 pending）。

### 交互

| # | 操作 | 实现 |
|---|------|------|
| A9-01 | 进入 | 调 A51 |
| A9-02 | 点击行 | 弹 Drawer：举报全文 + 题目预览 + 学员答题历史 |
| A9-03 | 「dismiss」 | D-1 + [A52 POST /reports/:id/dismiss](../F2-AI-接口规范/07-管理端-审核与发布.md) `{note}`；status → dismissed |
| A9-04 | 「转审核队列」 | [A53 POST /reports/:id/adopt](../F2-AI-接口规范/07-管理端-审核与发布.md)；副作用：对应 KP/Q `status: approved → reviewing` + 关联本举报；report status → adopted；可选：站内信通知学员 |
| A9-05 | 批量 dismiss | 选中行 + Modal 输入统一 note → [A54 POST /reports/batch-dismiss](../F2-AI-接口规范/07-管理端-审核与发布.md) |

### 条件显示

| 条件 | 行为 |
|------|------|
| 同一题目报错次数 ≥ 3 | 顶部红条「该题被多人举报，建议优先处理」+ 列表行高亮 |
| 已处理（dismissed / adopted） | 灰色行 + 状态标 + 不可再操作（仅查看） |
