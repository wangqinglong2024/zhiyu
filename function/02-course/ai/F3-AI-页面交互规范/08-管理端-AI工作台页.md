## P-A-7 · AI 工作台页

**路由**：`/admin/course/ai-jobs`
**权限**：editor+

### 关键约定

- 不提供「一键全自动 gen_full_stage」（[F2-Q4 封板](../F2-AI-接口规范/12-待确认问题清单.md)）；本页仅做"补漏 / 编辑后重生 / 单点修补"。
- Job 重试 = 新 job_id + idempotency_key 追加 `:retry-N`（[F1-Q12 封板](../F1-AI-数据模型规范/12-待确认问题清单.md)）。

### 布局

```
┌──────────────────────────────────────────────────────────┐
│  AI 工作台    [+ 新建 Job]      [类型 ▼][状态 ▼][搜索]   │
├──────────────────────────────────────────────────────────┤
│ Job ID        │ 类型              │ 状态     │ 进度       │ 创建于    │
│───────────────┼──────────────────┼─────────┼───────────┼─────────│
│ #38291        │ gen_questions_for_kp│ running │ 4/10 ████░░│ 3 min ago│
│ #38290        │ regen_question     │ failed  │ 0/1        │ 12 min …│
│ #38289        │ translate_kp_batch │ done    │ 50/50 ████ │ 1 h ago │
│ ...                                                     │
└──────────────────────────────────────────────────────────┘
```

### 数据来源

- [A41 GET /admin/v1/course/ai-jobs](../F2-AI-接口规范/06-管理端-AI生产.md)：分页 + 筛选；含每个 job 的实时进度（取自 `course_ai_generation_jobs.progress`）。
- 进入详情：[A42 GET /admin/v1/course/ai-jobs/:id](../F2-AI-接口规范/06-管理端-AI生产.md)：含子任务列表（每个 KP 或 Q 一行）+ LLM 原始日志（最近 N 条）。
- 实时刷新：列表页每 5s 轮询一次（仅含 `running` 行），降低带宽。

### 交互

| # | 操作 | 实现 |
|---|------|------|
| A7-01 | 进入 | 调 A41 |
| A7-02 | 「+ 新建 Job」 | 弹 D-7 |
| A7-03 | 点击 Job 行 | 弹 Drawer 详情（720px）：参数 / 子任务 / 输出 / 日志 |
| A7-04 | 「重试」 | 仅 `failed` 状态显示；调 [A43 POST /ai-jobs/:id/retry](../F2-AI-接口规范/06-管理端-AI生产.md)；返回新 job_id；Toast「重试 Job 已创建（#新id）」 |
| A7-05 | 「取消」 | 仅 `pending`/`running` 显示；D-1 + [A44 POST /ai-jobs/:id/cancel](../F2-AI-接口规范/06-管理端-AI生产.md) |
| A7-06 | 实时进度 | running 行每 5s 轮询；列表脱离视口暂停 |
| A7-07 | 完成后跳审核 | Job 状态变 `done` 时 Toast「Job #38289 完成 50 题，前往审核」+ 链接跳 P-A-8 |

### 条件显示

| 条件 | 行为 |
|------|------|
| Job 输出 `0` 子任务（LLM 返回空） | 状态 `failed` + 详情显示 LLM 原文 + 重试按钮 |
| Job `done` 但部分子任务 `failed` | 状态 `done_with_errors`（黄色）+ 详情可单条重试 |
| 学员侧高峰期 | 顶部黄条「服务繁忙，新 Job 可能排队较久」（来自后端 retry-after） |
