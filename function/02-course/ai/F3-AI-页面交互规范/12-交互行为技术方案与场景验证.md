## 二、断网 / 离线策略

| 触发 | 行为 |
|------|------|
| 学员端 navigator.offline | 顶部黄条「网络已断开」；学习地图 / KP 卡片本地缓存可继续浏览；播放、答题按钮 disabled |
| 答题中（C9 / C16）失败 | 标记 pending；下次任意答题成功后自动 flush |
| 管理端断网 | 顶部红条「连接异常」；所有写操作按钮 disabled；Drawer 内容保留 |
| 网络恢复 | 自动消失提示条；管理端 Drawer 自动重发待保存内容 |

## 三、并发与冲突策略

| 场景 | 策略 |
|------|------|
| 同一管理员双浏览器编辑 KP | 后保存 win + Toast「该内容在你编辑期间被你的另一会话修改过」 |
| 不同管理员编辑同 KP | 同上 + Toast 显示 `display_name` |
| 学员两端同时答同一题 | 后端 idempotency_key = `attempt_id + question_id`；重复请求返同一结果 |
| 节同时被发布与编辑 | 编辑保存先于发布 → 发布前再校验；发布先于编辑 → 编辑保存自动转入 reviewing |
| AI Job 重复触发 | idempotency_key + retry-N 后缀；24h 内同 key 命中返同一 job |
