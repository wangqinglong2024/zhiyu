## 1. 表：`course_exams`（试卷模板）

**对应需求中的对象**：节末小测 / 章测 / 阶段考 / HSK 模考的"出卷规则"
**性质**：管理端 CRUD；运行时按 `blueprint` 实时随机抽题，但题源都是 `course_questions` 中已有 approved 题。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `code` | `text` | ✅ | 由 RPC `fn_gen_exam_code()` 生成 | `uq_course_exams_code` (UNIQUE WHERE deleted_at IS NULL) | 试卷编码 | 正则 `^exam_(lq\|ct\|se\|hm)_\d{6}$` |
| `scope_type` | `text` | ✅ | — | `idx_course_exams_scope` | 考试范围 | 枚举 `lesson_quiz\|chapter_test\|stage_exam\|hsk_mock` |
| `scope_ref_id` | `uuid` | ✅ | — | `idx_course_exams_scope_ref` | 关联的 lesson/chapter/stage id | 见 §scope_ref_id 校验 |
| `title_zh` | `text` | ✅ | — | — | 试卷中文标题 | 1–60 |
| `title_i18n` | `jsonb` | ❌ | `null` | — | 标题 4 语言 | 必含 `vi,th,id,en` |
| `pass_score` | `numeric(5,2)` | ✅ | `60.00` | — | 通过分（百分制） | 0–100 |
| `total_score` | `numeric(5,2)` | ✅ | `100.00` | — | 总分 | > 0 且 ≤ 1000 |
| `time_limit_minutes` | `int` | ❌ | `null` | — | 倒计时（[temp/05 §5.7](../../../temp/05-用户端模块设计.md)） | 0–180，`null` 表示不限时 |
| `blueprint` | `jsonb` | ✅ | — | `idx_course_exams_blueprint_gin` (GIN) | 出卷规则 | 见 §blueprint schema |
| `is_published` | `bool` | ✅ | `false` | `idx_course_exams_published` | 是否发布 | — |
| `created_by` | `uuid` | ❌ | `null` | — | — | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `deleted_at` | `timestamptz` | ❌ | `null` | `idx_course_exams_deleted_at` | — | 软删 |

### `blueprint` JSON Schema

> 详细模板见 [temp/03 §3.2.8](../../../temp/03-数据库schema.md) 与 [temp/02 §2.5](../../../temp/02-知识点与题型内容模板.md)。

```jsonc
{
  "items": [
    { "q_type": "mcq_meaning",  "count": 3, "difficulty_range": [1,2] },
    { "q_type": "listen_pick",  "count": 2 },
    { "q_type": "sort_words",   "count": 1 }
  ],
  "kp_filter": {
    "lesson_id":  "<uuid>",       // 节末小测：本节
    "chapter_id": "<uuid>",       // 章测：本章
    "stage_id":   "<uuid>",       // 阶段考 / HSK 模考：本阶段
    "include_review_pct": 20      // 节末小测 80%本节+20%本章前节复习；章测 70%+30%；阶段考均衡抽
  },
  "shuffle": true,                 // 选项顺序是否打乱
  "show_explanation": false,       // 考试不立刻反馈（[temp/05 §5.7]）
  "allow_skip": false              // 阶段考默认不允许跳过
}
```

### `scope_ref_id` 与 `scope_type` 的对应

| `scope_type` | `scope_ref_id` 指向 | 唯一性约束 |
|--------------|--------------------|----|
| `lesson_quiz` | `course_lessons.id` | `(scope_type, scope_ref_id)` 唯一（一节一卷） |
| `chapter_test` | `course_chapters.id` | 同上（一章一卷） |
| `stage_exam` | `course_stages.id` | 同上（一阶段一卷） |
| `hsk_mock` | `course_stages.id`（HSK 主题的 stage） | 不唯一（同一 HSK 等级可有多套模考） |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|

### 三、DDL 草案

```sql
create table zhiyu.course_exams (
  id                  uuid primary key default gen_random_uuid(),
  code                text not null,
  scope_type          text not null,
  scope_ref_id        uuid not null,
  title_zh            text not null,
  title_i18n          jsonb,
  pass_score          numeric(5,2) not null default 60.00,
  total_score         numeric(5,2) not null default 100.00,
  time_limit_minutes  int,
  blueprint           jsonb not null,
  is_published        bool not null default false,
  created_by          uuid,
  created_at          timestamptz not null default now(),
  updated_at          timestamptz not null default now(),
  deleted_at          timestamptz,

  constraint fk_course_exams_creator
    foreign key (created_by) references auth.users(id) on delete set null,
  constraint ck_course_exams_code_format
    check (code ~ '^exam_(lq|ct|se|hm)_\d{6}$'),
  constraint ck_course_exams_scope_type
    check (scope_type in ('lesson_quiz','chapter_test','stage_exam','hsk_mock')),
  constraint ck_course_exams_pass_score
    check (pass_score between 0 and 100 and pass_score <= total_score),
  constraint ck_course_exams_total_score
    check (total_score > 0 and total_score <= 1000),
  constraint ck_course_exams_time_limit
    check (time_limit_minutes is null or time_limit_minutes between 0 and 180),
  constraint ck_course_exams_title_i18n_keys
    check (title_i18n is null or title_i18n ?& array['vi','th','id','en'])
);

create unique index uq_course_exams_code
  on zhiyu.course_exams (code) where deleted_at is null;
create unique index uq_course_exams_unique_per_scope
  on zhiyu.course_exams (scope_type, scope_ref_id)
  where deleted_at is null and scope_type in ('lesson_quiz','chapter_test','stage_exam');
create index idx_course_exams_scope        on zhiyu.course_exams (scope_type)   where deleted_at is null;
create index idx_course_exams_scope_ref    on zhiyu.course_exams (scope_ref_id) where deleted_at is null;
create index idx_course_exams_published    on zhiyu.course_exams (is_published) where deleted_at is null;
create index idx_course_exams_blueprint_gin on zhiyu.course_exams using gin (blueprint);
create index idx_course_exams_deleted_at   on zhiyu.course_exams (deleted_at);

create trigger tg_course_exams_before_update_set_updated_at
  before update on zhiyu.course_exams
  for each row execute function zhiyu.set_updated_at();
```

## 抽题规则与并发安全

> 学员发起考试时，服务端按 `blueprint.items` 抽题：

```sql
-- 伪代码：从 course_questions 中按 q_type + difficulty + kp_filter 随机抽 N 条
select id from zhiyu.course_questions
 where q_type = $1
   and difficulty between $2 and $3
   and status = 'approved'
   and deleted_at is null
   and kp_id in (
     select kp_id from zhiyu.course_lesson_kp
      where lesson_id = $lesson_id
   )
 order by random()
 limit $count;
```

抽到的题 id 列表写入 [`course_user_exam_attempts.snapshot`](./04-表定义-学员进度.md)。**同一 attempt 内不重复抽题**——通过 `not in (already_picked_ids)` 排除。

题目不足时（如某题型 approved 题数 < blueprint 要求），返回业务错误 `COURSE_EXAM_NOT_ENOUGH_QUESTIONS`，前端提示"试卷尚未准备好"，并通知运营补题。
