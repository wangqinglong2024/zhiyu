## 2. 外键级联矩阵

| 子表 | 父表 | 外键列 | ON DELETE | 理由 |
|------|------|-------|-----------|------|
| `course_stages` | `course_tracks` | `track` | RESTRICT | 5 个主题写死，禁止删除 |
| `course_chapters` | `course_stages` | `stage_id` | RESTRICT | 阶段删除前必须先清空章 |
| `course_lessons` | `course_chapters` | `chapter_id` | RESTRICT | 同上 |
| `course_lesson_kp` | `course_lessons` | `lesson_id` | CASCADE | 节删 → 关联清空（KP 仍保留） |
| `course_lesson_kp` | `course_knowledge_points` | `kp_id` | RESTRICT | KP 仍被节引用时禁删 |
| `course_questions` | `course_knowledge_points` | `kp_id` | RESTRICT | 题目挂在 KP 下，KP 删前必须先归档题目 |
| `course_exams` | `course_lessons / chapters / stages` | `scope_ref_id`（弱引用，不建 FK） | — | 多态外键，应用层校验 |
| `course_user_progress` | `auth.users` | `user_id` | CASCADE | 用户注销 → 进度清除 |
| `course_user_progress` | `course_lessons` | `lesson_id` | CASCADE | 节物理删（罕见）→ 进度清除 |
| `course_user_progress` | `course_knowledge_points` | `last_kp_id` | SET NULL | 弱引用 |
| `course_user_answers` | `auth.users` | `user_id` | CASCADE | 用户注销 → 流水清除 |
| `course_user_answers` | `course_questions` | `question_id` | RESTRICT | 题目仍被流水引用时禁物理删；用 `deleted_at` 软删 |
| `course_user_answers` | `course_knowledge_points` | `kp_id` | RESTRICT | 同上 |
| `course_user_srs` | `auth.users` / `course_knowledge_points` | — | CASCADE | SRS 是派生数据，可重建 |
| `course_user_exam_attempts` | `auth.users` | `user_id` | CASCADE | — |
| `course_user_exam_attempts` | `course_exams` | `exam_id` | RESTRICT | 卷模板有学员答卷时禁删 |
| `course_ai_generation_jobs` | `course_ai_prompts` | `prompt_id` | RESTRICT | prompt 有任务引用时禁删（应改 archived） |
| `course_knowledge_points` | `course_ai_generation_jobs` | `ai_job_id` | SET NULL | 弱引用，job 清理后保留 KP |
| `course_questions` | `course_ai_generation_jobs` | `ai_job_id` | SET NULL | 同上 |
| `course_questions` | `course_knowledge_points` | `kp_id` | RESTRICT | 同上 |
| `course_media_assets` | `course_knowledge_points / questions` | `ref_kp_id / ref_q_id` | SET NULL | 弱引用，便于反查 |
| `course_content_review_log` | `auth.users` | `actor_id` | SET NULL | 操作人删除后保留日志 |

> **原则**：内容侧（KP / 题 / 节）一律 **软删 `deleted_at`** + RESTRICT 物理删；用户行为侧（progress / answers / srs / attempts）一律 CASCADE 物理删。

## 3. 跨域引用清单

| 课程域引用 | 跨域目标 | 引用方式 | 备注 |
|----------|---------|---------|------|
| `course_user_*.user_id` | `auth.users.id` | FK CASCADE | 平台公共账号表 |
| `course_*.created_by / created_by` | `auth.users.id` | FK SET NULL | 同上 |
| `auth_user_profiles.current_track` | `course_tracks.code` | 弱引用（应用层） | 账号域字段，不强制 FK |
| `auth_user_profiles.current_stage_id` | `course_stages.id` | 弱引用 | 同上 |

> 关于 `auth_user_profiles` / `auth_admin_profiles` 是否真的存在，详见 [12-待确认问题清单 Q3 / Q5](./12-待确认问题清单.md)。

## 4. 视图与物化视图（应用层只读）

|------|------|------|---------|

## 5. 与 china 域的隔离

- 物理上同一 `zhiyu` schema，逻辑上靠 `course_` 前缀隔离
- **零跨域 FK**——china 与 course 之间不允许互引
- 共享只在账号域（`auth.users`）和媒资域（部分 stock_library 可共享）
