# 待确认问题清单（F1 数据模型）

> 全部已与 PM 拍板封板。本文件留作决议追溯，不再用于"开放问题清单"。
> 后续如发现新问题，直接在对应表/枚举/校验文件内修订并在本文件追加 Qn。

## Q1 主键策略 ✅ 封板

- 决议：**全表统一 `uuid`，`gen_random_uuid()` 自动生成**。
- 业务唯一键（`code`、`kp_code`、`q_code`、`lesson_code` 等）独立列 + UNIQUE 索引。
- 与 [grules/G1-03 §一](../../../grules/G1-架构与技术规范/03-数据库规范.md) + china 域一致。
- 影响：所有表 PK 类型统一，外键 `*_id` 全为 `uuid`。

## Q2 schema 与表前缀 ✅ 封板

- schema = `zhiyu`（与 china 等域共物理库 / 共 schema，靠表前缀隔离）。
- 表前缀 `course_`。
- RLS 全表开启（学员侧 PostgREST 直读必需）。

## Q3 软删 vs 物理删 ✅ 封板

- 用户可见的高价值业务表（KP / Q / Lesson / Chapter / Stage）走 **软删 `deleted_at`**。
- 流水表（answers / progress / job 日志）30 天 cron 物理清理（按 `created_at` < now()-30d）。
- 媒资走「DB 软删 + 7 天 cron 真删 CDN」（详见 Q9）。

## Q4 节末小测是否必做 ✅ 封板

- 默认**可跳过**：跳过后该节进度 `status='in_progress'`（不是 `passed`）。
- 节级开关 `course_lessons.is_quiz_required boolean default false`，true 时强制做。
- 跳过通过节末小测**不**触发"通关 → 下节解锁"，但章/节本身仍可见可学。
- 详见 [04 §course_user_progress](./04-表定义-学员进度.md) 与 [F3-03](../F3-AI-页面交互规范/03-应用端-练习与节末小测页.md)。

## Q5 admins / auth 表归属 ✅ 封板

- 管理员账号复用平台公共表 `auth.users`；本域**不**自建管理员表。
- 域内仅扩展 `auth_admin_profiles`（共用 china 已有定义）。
- 课程域内学员侧设置存独立表 `course_user_settings`（详见 Q11）。

## Q6 5 类赛道是否动态可扩展 ✅ 封板

- **不可动态新增**。固定 `ec / fc / hk / dl / share` 5 个，作为 enum 写死。
- 未来新增赛道走"代码发版 + 数据库 migration"双发；不开运行时入口。

## Q7 KP 类型 7 类是否枚举封闭 ✅ 封板

- 封闭枚举 `kp_type` = `char / word / phrase / sentence / dialog / grammar / culture`。
- 后续扩需 migration。

## Q8 流水表大小 ✅ 封板

- `course_user_answers` 和 `course_ai_generation_jobs` 月分区（`partition by range(created_at)`）。
- 每月分区表自动 cron 创建；旧分区按 30 天删除策略 detach + drop。
- 详见 [03 §分区策略](./03-表定义-考试与作答.md) 与 [05 §AI Job 表](./05-表定义-AI生产与媒资.md)。

## Q9 媒资 CDN 删除策略 ✅ 封板

- 管理端 DELETE → DB 软删 + 入 `course_media_assets_pending_purge` 队列；
- 7 天后 cron 调 `MediaProvider.deleteRemote(hash)` 真删 CDN 文件；
- 7 天窗口期内可在管理端「清理」入口撤销。

## Q10 i18n 字段 key 数量 ✅ 封板

- **强制 5 key**：`{zh, vi, th, id, en}`（与 china 域一致 + temp/00 §3.5）。
- jsonb 列 + Zod 校验规则 `i18nText`：5 key 必须存在，`zh` 与 `en` 必填非空字符串，其它语言可空字符串。
- 详见 [09-校验规则汇总.md §i18nText](./09-校验规则汇总.md)。

## Q11 课程域内学员设置表 ✅ 封板

- 新增表 `course_user_settings`：
  - `user_id uuid pk` (FK → auth.users)
  - `current_track text`（默认 `share`）
  - `shared_stage_done boolean default false`（share Stage 0 是否完成）
  - `native_lang text default 'zh'`（学员母语，影响题目翻译展示）
  - `ui_lang text default 'zh'`（界面语言）
  - 标准 `created_at / updated_at`
- 账号通用资料字段（昵称 / 头像）走 china 已有的 `auth_user_profiles` 视图读取，不复制。

## Q12 AI Job 重试与幂等 ✅ 封板

- `idempotency_key` 必填（管理端创建时自动生成 + 可手改）。
- 重试 = 新 job_id + idempotency_key 追加 `:retry-N` 后缀。
- 24h 内同 idempotency_key 命中返同一 job_id（保护 LLM 成本）。

## Q13 voice_profile 与媒资供应商抽象 ✅ 封板

- DB 字段 `voice_profile text` 是抽象标识符（如 `xiaoxiao_zh`、`yunyang_zh`），**不**绑定供应商品牌。
- 应用层 `MediaProvider` adapter 负责映射到具体 SDK；切换供应商不动 DB。
- 适配器接口：`tts(text, voice_profile) → {url, hash}`、`deleteRemote(hash) → void`。

## Q14 题目去重 ✅ 封板

- 增加 `course_questions.payload_hash text`（SHA256，64 字符）。
- 部分唯一索引 `uq_course_q_payload_dedup UNIQUE(kp_id, q_type, payload_hash)`，`status != 'archived'`。
- 后端写入时自动算 hash；命中重复返错误码 `40402 COURSE_DUPLICATE_PAYLOAD_HASH`。
- 详见 [02 §course_questions](./02-表定义-知识点与题目.md)。
