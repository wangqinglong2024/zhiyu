# 一 · 表定义 · 课程目录（4 表）

> 课程目录的四级树：**主题 → 阶段 → 章 → 节**。
> 对应需求：[temp/01 §1.7 数量小结](../../../temp/01-课程目录骨架.md) —— 5 主题 + 25 阶段 + ~148 章 + ~888 节。

---

## 1. 表：`course_tracks`（主题）

**对应需求中的对象**：[temp/00 §0.4 术语表「Track」](../../../temp/00-总览与设计原则.md) + [temp/01 §1.4 主题码](../../../temp/01-课程目录骨架.md)
**性质**：系统内置数据字典，**5 条固定记录**（`ec / fc / hk / dl / share`），由迁移种子写入；不允许在前端 CRUD。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `code` | `text` | ✅ | — | `uq_course_tracks_code` (UNIQUE) | 主题码 | 枚举 `ec\|fc\|hk\|dl\|share` |
| `name_zh` | `text` | ✅ | — | — | 主题中文名 | 1–20 |
| `name_i18n` | `jsonb` | ✅ | — | — | 主题名（4 语言） | 必含 `vi,th,id,en` 全部 4 个 key |
| `icon_url` | `text` | ❌ | `null` | — | 图标 | URL |
| `sort_order` | `int` | ✅ | `0` | `idx_course_tracks_sort_order` | 列表展示顺序 | — |
| `is_enabled` | `bool` | ✅ | `true` | — | 是否启用 | — |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | 触发器维护 |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `id (uuid)` | `uuid` | 全表主键统一 uuid（G1-数据库规范 §一）；`code` 仍作为业务唯一键 |
| `is_enabled` | `bool` | 临时下线主题时不删数据，仅置 false；前端按此过滤 |

### 三、DDL 草案

```sql
create table zhiyu.course_tracks (
  id          uuid primary key default gen_random_uuid(),
  code        text not null,
  name_zh     text not null,
  name_i18n   jsonb not null,
  icon_url    text,
  sort_order  int  not null default 0,
  is_enabled  bool not null default true,
  created_at  timestamptz not null default now(),
  updated_at  timestamptz not null default now(),

  constraint uq_course_tracks_code unique (code),
  constraint ck_course_tracks_code_enum
    check (code in ('ec','fc','hk','dl','share')),
  constraint ck_course_tracks_name_zh_len
    check (char_length(name_zh) between 1 and 20),
  constraint ck_course_tracks_name_i18n_keys
    check (name_i18n ?& array['vi','th','id','en'])
);

create index idx_course_tracks_sort_order on zhiyu.course_tracks (sort_order);

create trigger tg_course_tracks_before_update_set_updated_at
  before update on zhiyu.course_tracks
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `anon`、`authenticated` | 仅放行 `is_enabled = true` |
| `insert / update / delete` | `service_role` | 仅 service-role 可写；前端禁止 |

策略命名：`course_tracks_select_enabled`、`course_tracks_write_service`。

## 2. 表：`course_stages`（阶段）

**对应需求中的对象**：[temp/00 §0.4 「Stage」](../../../temp/00-总览与设计原则.md) + [temp/01 §1.3 四大主题阶段表](../../../temp/01-课程目录骨架.md)
**性质**：管理端 CRUD；初始化阶段由运营按 [temp/01](../../../temp/01-课程目录骨架.md) 录入 25 条（共享 1 + 4 × 6）。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `track_code` | `text` | ✅ | — | `idx_course_stages_track` | 所属主题 | 外键 → `course_tracks.code`，`ON DELETE RESTRICT` |
| `stage_no` | `int` | ✅ | — | `uq_course_stages_track_stage` | 阶段序号 | `0..6`（0 = 共享预备 stage，仅 `track_code='share'` 可用） |
| `title_zh` | `text` | ✅ | — | — | 阶段中文标题 | 1–40 |
| `title_i18n` | `jsonb` | ✅ | — | — | 阶段标题 4 语言 | 必含 `vi,th,id,en` |
| `desc_i18n` | `jsonb` | ❌ | `null` | — | 阶段简介 4 语言 | 每语言 ≤ 200 |
| `hsk_mapping` | `text` | ❌ | `null` | — | HSK 等级映射 | `HSK1..HSK6`，仅 `track_code='hk'` 时使用 |
| `unlock_rule` | `jsonb` | ✅ | `'{"prev_stage_pass":true}'` | — | 解锁规则 | 见 §unlock_rule schema |
| `vocab_increment` | `int` | ❌ | `null` | — | 词汇量增量（运营展示） | 0–5000 |
| `sort_order` | `int` | ✅ | `0` | — | — | — |
| `is_published` | `bool` | ✅ | `false` | `idx_course_stages_published` | 是否发布 | — |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |

### `unlock_rule` JSON Schema（jsonb 字段约束）

```jsonc
{
  "prev_stage_pass": true,            // 必填：前一阶段是否需通过
  "require_shared_stage_done": false, // 主题 stage 1 自动 true，其它默认 false
  "min_score": 70                      // 阶段考最低通过分（默认 70）
}
```

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `vocab_increment` | `int` | [temp/01 §1.3](../../../temp/01-课程目录骨架.md) 表头有"词汇量增量"列，运营在课程地图上展示 |
| `is_published` | `bool` | 阶段级总开关，便于灰度上线 |

### 三、DDL 草案

```sql
create table zhiyu.course_stages (
  id              uuid primary key default gen_random_uuid(),
  track_code      text not null,
  stage_no        int  not null,
  title_zh        text not null,
  title_i18n      jsonb not null,
  desc_i18n       jsonb,
  hsk_mapping     text,
  unlock_rule     jsonb not null default '{"prev_stage_pass":true}'::jsonb,
  vocab_increment int,
  sort_order      int  not null default 0,
  is_published    bool not null default false,
  created_at      timestamptz not null default now(),
  updated_at      timestamptz not null default now(),

  constraint fk_course_stages_track
    foreign key (track_code) references zhiyu.course_tracks(code) on delete restrict,
  constraint uq_course_stages_track_stage
    unique (track_code, stage_no),
  constraint ck_course_stages_stage_no
    check (stage_no between 0 and 6),
  constraint ck_course_stages_stage0_only_share
    check ((stage_no = 0 and track_code = 'share') or stage_no >= 1),
  constraint ck_course_stages_title_zh_len
    check (char_length(title_zh) between 1 and 40),
  constraint ck_course_stages_title_i18n_keys
    check (title_i18n ?& array['vi','th','id','en']),
  constraint ck_course_stages_hsk_mapping
    check (hsk_mapping is null or hsk_mapping ~ '^HSK[1-6]$')
);

create index idx_course_stages_track     on zhiyu.course_stages (track_code);
create index idx_course_stages_published on zhiyu.course_stages (is_published);

create trigger tg_course_stages_before_update_set_updated_at
  before update on zhiyu.course_stages
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `anon`、`authenticated` | 仅 `is_published=true` |
| `select` | `service_role` | 全量 |
| 写 | `service_role` | 全量；管理端通过 service-role 调用 |

## 3. 表：`course_chapters`（章）

**对应需求中的对象**：[temp/00 §0.4 「Chapter」](../../../temp/00-总览与设计原则.md)
**性质**：管理端 CRUD，每阶段固定 6 章（[temp/01 §1.7](../../../temp/01-课程目录骨架.md) 定量基线；share Stage 0 例外为 4 章）。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `stage_id` | `uuid` | ✅ | — | `idx_course_chapters_stage` | 所属阶段 | FK → `course_stages.id`，`ON DELETE RESTRICT` |
| `chapter_no` | `int` | ✅ | — | `uq_course_chapters_stage_no` | 章序号 | 1–20 |
| `title_zh` | `text` | ✅ | — | — | 章中文标题 | 1–40 |
| `title_i18n` | `jsonb` | ✅ | — | — | 章标题 4 语言 | 必含 `vi,th,id,en` |
| `cover_url` | `text` | ❌ | `null` | — | 章封面 | URL |
| `sort_order` | `int` | ✅ | `0` | — | — | — |
| `is_published` | `bool` | ✅ | `false` | `idx_course_chapters_published` | — | — |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `deleted_at` | `timestamptz` | ❌ | `null` | `idx_course_chapters_deleted_at` | — | 软删 |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `cover_url` | `text` | 用户端学习地图章卡片需要封面图 |
| `deleted_at` | `timestamptz` | 章可能整体废弃；按 G1 规范软删 |

### 三、DDL 草案

```sql
create table zhiyu.course_chapters (
  id           uuid primary key default gen_random_uuid(),
  stage_id     uuid not null,
  chapter_no   int  not null,
  title_zh     text not null,
  title_i18n   jsonb not null,
  cover_url    text,
  sort_order   int  not null default 0,
  is_published bool not null default false,
  created_at   timestamptz not null default now(),
  updated_at   timestamptz not null default now(),
  deleted_at   timestamptz,

  constraint fk_course_chapters_stage
    foreign key (stage_id) references zhiyu.course_stages(id) on delete restrict,
  constraint ck_course_chapters_no
    check (chapter_no between 1 and 20),
  constraint ck_course_chapters_title_zh_len
    check (char_length(title_zh) between 1 and 40),
  constraint ck_course_chapters_title_i18n_keys
    check (title_i18n ?& array['vi','th','id','en'])
);

create unique index uq_course_chapters_stage_no
  on zhiyu.course_chapters (stage_id, chapter_no) where deleted_at is null;
create index idx_course_chapters_stage     on zhiyu.course_chapters (stage_id) where deleted_at is null;
create index idx_course_chapters_published on zhiyu.course_chapters (is_published) where deleted_at is null;
create index idx_course_chapters_deleted_at on zhiyu.course_chapters (deleted_at);

create trigger tg_course_chapters_before_update_set_updated_at
  before update on zhiyu.course_chapters
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

同 `course_stages`：用户端只读 `is_published=true and deleted_at is null`。

## 4. 表：`course_lessons`（节）

**对应需求中的对象**：[temp/00 §0.4 「Lesson」](../../../temp/00-总览与设计原则.md) + [temp/01 §1.5 节标准结构](../../../temp/01-课程目录骨架.md)
**性质**：**最关键的内容容器**。每章固定 6 节，每节固定 12 KP（[temp/01 §1.7](../../../temp/01-课程目录骨架.md) 定量基线）。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `chapter_id` | `uuid` | ✅ | — | `idx_course_lessons_chapter` | 所属章 | FK → `course_chapters.id`，`ON DELETE RESTRICT` |
| `lesson_no` | `int` | ✅ | — | `uq_course_lessons_chapter_no` | 节序号 | 1–30 |
| `code` | `text` | ✅ | — | `uq_course_lessons_code` (UNIQUE) | 节级编码 | 见 [10-编号生成规则.md](./10-编号生成规则.md)，正则 `^(ec\|fc\|hk\|dl\|share)-\d{1,2}-\d{1,2}-\d{1,2}$` |
| `title_zh` | `text` | ✅ | — | — | 节中文标题 | 1–40 |
| `title_i18n` | `jsonb` | ✅ | — | — | 节标题 4 语言 | 必含 `vi,th,id,en` |
| `goal_i18n` | `jsonb` | ❌ | `null` | — | 学习目标（一句话）4 语言 | 每语言 ≤ 80 |
| `est_minutes` | `int` | ✅ | `12` | — | 预计时长（分） | 5–30 |
| `sort_order` | `int` | ✅ | `0` | — | — | — |
| `has_quiz` | `bool` | ✅ | `true` | — | 是否有节末小测 | — |
| `is_published` | `bool` | ✅ | `false` | `idx_course_lessons_published` | 是否发布（学习地图可见） | — |
| `published_at` | `timestamptz` | ❌ | `null` | `idx_course_lessons_published_at` | — | `is_published=true` 时必须非空 |
| `created_by` | `uuid` | ❌ | — | — | — | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `updated_by` | `uuid` | ❌ | — | — | — | 同上 |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `deleted_at` | `timestamptz` | ❌ | `null` | `idx_course_lessons_deleted_at` | — | 软删 |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `code` | `text` | 全局唯一短码（如 `ec-2-3-1`），便于学员、管理端、URL 路由统一引用；见 [10-编号生成规则.md](./10-编号生成规则.md) |
| `published_at` | `timestamptz` | 节级发布是核心动作，列表与缓存失效都依赖此字段 |
| `created_by` / `updated_by` | `uuid` | 管理端操作审计 |
| `deleted_at` | `timestamptz` | 节可能下线；保留进度兼容性 |
| `has_quiz` | `bool` | 个别"概览节 / 复习节"可关闭节末小测 |

### 三、DDL 草案

```sql
create table zhiyu.course_lessons (
  id           uuid primary key default gen_random_uuid(),
  chapter_id   uuid not null,
  lesson_no    int  not null,
  code         text not null,
  title_zh     text not null,
  title_i18n   jsonb not null,
  goal_i18n    jsonb,
  est_minutes  int  not null default 12,
  sort_order   int  not null default 0,
  has_quiz     bool not null default true,
  is_published bool not null default false,
  published_at timestamptz,
  created_by   uuid,
  updated_by   uuid,
  created_at   timestamptz not null default now(),
  updated_at   timestamptz not null default now(),
  deleted_at   timestamptz,

  constraint fk_course_lessons_chapter
    foreign key (chapter_id) references zhiyu.course_chapters(id) on delete restrict,
  constraint fk_course_lessons_created_by
    foreign key (created_by) references auth.users(id) on delete set null,
  constraint fk_course_lessons_updated_by
    foreign key (updated_by) references auth.users(id) on delete set null,

  constraint ck_course_lessons_no check (lesson_no between 1 and 30),
  constraint ck_course_lessons_code_format
    check (code ~ '^(ec|fc|hk|dl|share)-\d{1,2}-\d{1,2}-\d{1,2}$'),
  constraint ck_course_lessons_title_zh_len
    check (char_length(title_zh) between 1 and 40),
  constraint ck_course_lessons_title_i18n_keys
    check (title_i18n ?& array['vi','th','id','en']),
  constraint ck_course_lessons_est_minutes
    check (est_minutes between 5 and 30),
  constraint ck_course_lessons_published_consistency
    check (
      (is_published = true and published_at is not null) or
      (is_published = false and published_at is null)
    )
);

create unique index uq_course_lessons_code
  on zhiyu.course_lessons (code) where deleted_at is null;
create unique index uq_course_lessons_chapter_no
  on zhiyu.course_lessons (chapter_id, lesson_no) where deleted_at is null;
create index idx_course_lessons_chapter      on zhiyu.course_lessons (chapter_id) where deleted_at is null;
create index idx_course_lessons_published    on zhiyu.course_lessons (is_published) where deleted_at is null;
create index idx_course_lessons_published_at on zhiyu.course_lessons (published_at desc) where is_published = true and deleted_at is null;
create index idx_course_lessons_deleted_at   on zhiyu.course_lessons (deleted_at);

create trigger tg_course_lessons_before_update_set_updated_at
  before update on zhiyu.course_lessons
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `anon`、`authenticated` | 仅 `is_published=true and deleted_at is null` |
| `select` | `service_role` | 全量 |
| 写 | `service_role` | 全量 |

策略命名：`course_lessons_select_published`、`course_lessons_write_service`。
