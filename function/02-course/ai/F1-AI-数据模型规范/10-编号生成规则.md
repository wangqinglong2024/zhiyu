# 七 · 编号生成规则

> 课程域共有 5 类业务编码，统一在 `zhiyu` schema 内定义生成规则。
> 所有编码均独立列（不复用 `id` UUID），全局唯一。

---

## 1. `course_lessons.code`（节编码）

格式：`<track>-<stage_no>-<chapter_no>-<lesson_no>`

| 段 | 取值 | 来源 |
|----|------|------|
| `track` | `share / ec / fc / dl / hk` | `course_tracks.code` |
| `stage_no` | 0–6 | `course_stages.stage_no` |
| `chapter_no` | 1–N | `course_chapters.chapter_no` |
| `lesson_no` | 1–N | `course_lessons.lesson_no` |

例：
- `share-0-1-1`：共享拼音入门，第 1 章，第 1 节
- `ec-2-3-4`：电商客服，Stage 2，第 3 章，第 4 节
- `dl-5-7-2`：日常生活/HSK，Stage 5，第 7 章，第 2 节

唯一约束：`UNIQUE (code)`。

生成位置：管理端创建节时由前端拼接，提交后 RPC `fn_create_lesson` 二次校验。

## 2. `course_knowledge_points.kp_code`（KP 编码）

格式：`kp_<track>_<kp_type>_<6位序号>`

| 段 | 取值 |
|----|------|
| `track` | `share / ec / fc / dl / hk` |
| `kp_type` | 7 类 KP 类型 |
| 序号 | 6 位零填充递增整数（按 (track, kp_type) 维度独立） |

例：
- `kp_share_pinyin_000001`
- `kp_ec_word_000123`
- `kp_dl_grammar_000045`

生成方式（PostgreSQL 序列 + 触发器）：

```sql
-- 每个 (track, kp_type) 一个 sequence（共 5 × 7 = 35 个）
do $$
declare
  t text; k text;
begin
  for t in select unnest(array['share','ec','fc','dl','hk'])
  loop
    for k in select unnest(array['pinyin','hanzi','word','phrase','grammar','sentence','dialog'])
    loop
      execute format(
        'create sequence if not exists zhiyu.seq_course_kp_%s_%s start 1 increment 1',
        t, k
      );
    end loop;
  end loop;
end $$;

create or replace function zhiyu.fn_gen_course_kp_code(p_track text, p_kp_type text)
returns text language plpgsql as $$
declare
  v_seq_name text;
  v_no       bigint;
begin
  v_seq_name := format('zhiyu.seq_course_kp_%s_%s', p_track, p_kp_type);
  execute format('select nextval(%L)', v_seq_name) into v_no;
  return format('kp_%s_%s_%s', p_track, p_kp_type, lpad(v_no::text, 6, '0'));
end $$;

create or replace function zhiyu.tg_course_kp_set_code()
returns trigger language plpgsql as $$
begin
  if new.kp_code is null or new.kp_code = '' then
    new.kp_code := zhiyu.fn_gen_course_kp_code(new.track, new.kp_type);
  end if;
  return new;
end $$;

create trigger tg_before_insert_course_kp_set_code
  before insert on zhiyu.course_knowledge_points
  for each row execute function zhiyu.tg_course_kp_set_code();
```

约束：`UNIQUE (kp_code)` + `CHECK (kp_code ~ '^kp_(share|ec|fc|dl|hk)_(pinyin|hanzi|word|phrase|grammar|sentence|dialog)_\d{6}$')`。

> 容量：单 (track, kp_type) 上限 999 999，远超预计 ~3 200/类。

## 3. `course_questions.q_code`（题目编码）

格式：`q_<track>_<8位序号>`

| 段 | 取值 |
|----|------|
| `track` | `share / ec / fc / dl / hk` |
| 序号 | 8 位零填充递增整数（按 track 维度独立） |

例：
- `q_share_00000001`
- `q_ec_00012345`

生成方式（5 个 sequence）：

```sql
do $$
declare t text;
begin
  for t in select unnest(array['share','ec','fc','dl','hk'])
  loop
    execute format(
      'create sequence if not exists zhiyu.seq_course_q_%s start 1 increment 1', t
    );
  end loop;
end $$;

create or replace function zhiyu.fn_gen_course_q_code(p_track text)
returns text language plpgsql as $$
declare v_no bigint;
begin
  execute format('select nextval(%L)', format('zhiyu.seq_course_q_%s', p_track)) into v_no;
  return format('q_%s_%s', p_track, lpad(v_no::text, 8, '0'));
end $$;

create or replace function zhiyu.tg_course_q_set_code()
returns trigger language plpgsql as $$
begin
  if new.q_code is null or new.q_code = '' then
    new.q_code := zhiyu.fn_gen_course_q_code(new.track);
  end if;
  return new;
end $$;

create trigger tg_before_insert_course_q_set_code
  before insert on zhiyu.course_questions
  for each row execute function zhiyu.tg_course_q_set_code();
```

容量：单 track 上限 99 999 999，远超预计 ~1.5 万/主题。

## 4. `course_ai_prompts.prompt_code`（Prompt 编码）

格式：`prompt_<scope>_v<版本>`

| 段 | 取值 | 例 |
|----|------|----|
| `scope` | `kp_word / q_mcq_meaning / tts / image / translate / review_check / ...` | `kp_word` |
| 版本 | `v1 / v2 / v3 / ...` | `v3` |

例：
- `prompt_kp_word_v1`
- `prompt_q_listen_pick_v2`
- `prompt_tts_v1`

约束：
- `UNIQUE (prompt_code)`
- `CHECK (prompt_code ~ '^prompt_[a-z0-9_]+_v\d+$')`
- 同 `scope` 仅 1 条 `is_active=true`（部分唯一索引 `uq_course_prompts_one_active_per_scope`）

生成方式：管理端手动填写（不自增）。

---

## 5. `course_media_assets.hash`（媒资去重 key）

格式：64 字符 hex（sha256）

| 算法 | 输入 |
|------|------|
| sha256 | 文件原始字节 |

例：`3c5e8b9d2f1a4c6e7b8d9f0a1c2e3b4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b`

约束：
- `UNIQUE (hash)`（命中即复用）
- `CHECK (hash ~ '^[a-f0-9]{64}$')`

生成位置：管理端上传 / TTS 合成 / AI 生图后，由后端在写入前计算 hash；命中已有则直接返回旧 `id`。

---

## 6. 序号回收策略

- **不回收**。删除 KP / Q 时（无论软删硬删），其 `kp_code / q_code` 不被复用。
- 序列 gap 可接受，业务无依赖。

---

## 7. 跨环境一致性

| 环境 | 策略 |
|------|------|
| dev | 序列从 1 开始，可重置 |
| staging | 序列从 1 开始，导入 dev seed 数据 |
| prod | 序列独立递增，**禁止重置** |

环境切换不需要保持 code 一致；prod code 是唯一权威。
