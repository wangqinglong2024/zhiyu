## 1. 表：`course_knowledge_points`（KP 总表，跨赛道可复用）

**对应需求中的对象**：[temp/00 §0.5 KP](../../../temp/00-总览与设计原则.md) + [temp/02 §2.1 通用字段 + §2.2 七类差异](../../../temp/02-知识点与题型内容模板.md)
**性质**：内容主表，**跨赛道可复用**（同一汉字"价"在 ec/dl 都用，只存 1 条）。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `kp_code` | `text` | ✅ | 由 RPC `fn_gen_kp_code()` 生成 | `uq_course_kp_code` (UNIQUE WHERE deleted_at IS NULL) | KP 编码 | 正则 `^kp_(ec\|fc\|hk\|dl\|share)_(p\|h\|w\|ph\|g\|s\|d)_\d{6}$`，见 [10-编号生成规则.md](./10-编号生成规则.md) |
| `kp_type` | `text` | ✅ | — | `idx_course_kp_type` | 知识点类型 | 枚举 `pinyin\|hanzi\|word\|phrase\|grammar\|sentence\|dialog`（详见 [06-枚举定义.md §1](./06-枚举定义.md)） |
| `primary_track` | `text` | ✅ | — | `idx_course_kp_track` | 出资生成的赛道（统计用） | FK → `course_tracks.code`，`ON DELETE RESTRICT` |
| `title_zh` | `text` | ✅ | — | `idx_course_kp_title_zh` | 主标题（中文） | 1–60 |
| `pinyin` | `text` | ❌ | `null` | — | 带声调拼音 | 1–200 |
| `difficulty` | `int` | ✅ | `1` | `idx_course_kp_difficulty` | 难度等级（对齐 stage） | 1–6 |
| `hsk_level` | `int` | ❌ | `null` | — | HSK 等级（可选） | 1–6 |
| `content` | `jsonb` | ✅ | — | `idx_course_kp_content_gin` (GIN) | 类型差异内容 | 见 §content schema |
| `translations` | `jsonb` | ❌ | `null` | — | 主标题 4 语言翻译 | 必含 `vi,th,id,en`，每语言 ≤ 80 |
| `audio_url` | `text` | ❌ | `null` | — | 标题 TTS 音频 | URL |
| `image_url` | `text` | ❌ | `null` | — | 配图 | URL |
| `tags` | `text[]` | ✅ | `'{}'` | `idx_course_kp_tags_gin` (GIN) | 标签 | 元素 ≤ 20 字 |
| `status` | `text` | ✅ | `'draft'` | `idx_course_kp_status` | 内容状态 | 枚举 `draft\|reviewing\|approved\|archived` |
| `version` | `int` | ✅ | `1` | — | 版本号 | ≥ 1 |
| `ai_job_id` | `uuid` | ❌ | `null` | `idx_course_kp_ai_job` | 来自哪个 AI Job | FK → `course_ai_generation_jobs.id`，`ON DELETE SET NULL` |
| `reviewed_by` | `uuid` | ❌ | `null` | — | 审核人 | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `reviewed_at` | `timestamptz` | ❌ | `null` | — | 审核时间 | — |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `deleted_at` | `timestamptz` | ❌ | `null` | `idx_course_kp_deleted_at` | — | 软删 |

### `content` JSON Schema（按 `kp_type` 不同结构不同）

> 完整模板见 [temp/02 §2.2](../../../temp/02-知识点与题型内容模板.md)；下面是约束概要。

```jsonc
// kp_type='pinyin'
{
  "components": { "initial": "zh", "final": "ōng", "tone": 1 },
  "is_whole_syllable": false,
  "common_chars": ["中","钟","终"],
  "tongue_diagram_url": "/cdn/img/zh.svg"      // 可选
}

// kp_type='hanzi'
{
  "stroke_count": 6,
  "structure": "left-right",                    // single | left-right | top-bottom | enclosure | ...
  "radicals": ["女","子"],
  "stroke_order_svg": "...",
  "stroke_order_anim_url": "...",
  "common_words": ["你好","好朋友","好看"]
}

// kp_type='word'
{
  "pos": "noun",                                // noun|verb|adj|adv|...
  "examples": [ { "zh":"...", "pinyin":"...", "audio_url":"...", "tr":{"vi":"..."} } ]
}

// kp_type='phrase'
{
  "pattern": "下 + 订单",
  "co_occurrences": ["在网上下订单","帮我下个订单"],
  "examples": [ /* 同上 */ ]
}

// kp_type='grammar'
{
  "pattern": "S + 把 + O + V + 其他",
  "rule_md": { "vi":"...", "th":"...", "id":"...", "en":"..." },
  "positive_examples": ["我把书放在桌子上。"],
  "negative_examples": ["我把吃饭。"],
  "contrast_with": ["被字句"]
}

// kp_type='sentence'
{
  "scenario": "购物",
  "function_tag": "询问价格",
  "reply_examples": ["五十块。","已经卖完了。"]
}

// kp_type='dialog'
{
  "turns": [
    { "role":"buyer",  "zh":"...", "pinyin":"...", "audio_url":"..." },
    { "role":"seller", "zh":"...", "pinyin":"...", "audio_url":"..." }
  ],
  "key_kp_refs": ["kp_ec_w_000231"]            // 引用其它 KP 的 kp_code
}
```

> 全部 7 类的 `content` 都额外可包含 `explanation_md: { vi, th, id, en }`（讲解 ≤ 80 字）和 `examples` 数组。

### `ai_meta` 字段单独处理

> [temp/02 §2.1 ai_meta](../../../temp/02-知识点与题型内容模板.md) 描述了 model/prompt_id/generated_at/reviewer 等字段——
> 在本表中**不存到 `content` 里**，而是通过 `ai_job_id`、`reviewed_by`、`reviewed_at` 字段反查 `course_ai_generation_jobs`，避免冗余。

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `kp_code` | `text` | 业务编码独立列；前端、缓存键、AI 重生引用都用 code 而非 uuid |
| `tags` | `text[]` | 运营按"电商/HSK2/价格"等多维度筛选 |
| `version` | `int` | 改 1 次 +1，老版本归档（status=archived），不影响历史成绩 |
| `deleted_at` | `timestamptz` | 内容下架（软删）；30 天后 cron 物理清理 |
| `idx_course_kp_content_gin` | GIN | 运营 "按 hsk_level 找词" 等动态查询（[temp/03 §3.7](../../../temp/03-数据库schema.md)） |

### 三、DDL 草案

```sql
create table zhiyu.course_knowledge_points (
  id            uuid primary key default gen_random_uuid(),
  kp_code       text not null,
  kp_type       text not null,
  primary_track text not null,
  title_zh      text not null,
  pinyin        text,
  difficulty    int  not null default 1,
  hsk_level     int,
  content       jsonb not null,
  translations  jsonb,
  audio_url     text,
  image_url     text,
  tags          text[] not null default '{}',
  status        text not null default 'draft',
  version       int  not null default 1,
  ai_job_id     uuid,
  reviewed_by   uuid,
  reviewed_at   timestamptz,
  created_at    timestamptz not null default now(),
  updated_at    timestamptz not null default now(),
  deleted_at    timestamptz,

  constraint fk_course_kp_track
    foreign key (primary_track) references zhiyu.course_tracks(code) on delete restrict,
  constraint fk_course_kp_reviewer
    foreign key (reviewed_by) references auth.users(id) on delete set null,
  -- ai_job_id 的 FK 在 course_ai_generation_jobs 建表后通过 alter table 补加（避免循环）

  constraint ck_course_kp_code_format
    check (kp_code ~ '^kp_(ec|fc|hk|dl|share)_(p|h|w|ph|g|s|d)_\d{6}$'),
  constraint ck_course_kp_type
    check (kp_type in ('pinyin','hanzi','word','phrase','grammar','sentence','dialog')),
  constraint ck_course_kp_difficulty
    check (difficulty between 1 and 6),
  constraint ck_course_kp_hsk_level
    check (hsk_level is null or hsk_level between 1 and 6),
  constraint ck_course_kp_status
    check (status in ('draft','reviewing','approved','archived')),
  constraint ck_course_kp_translations_keys
    check (translations is null or translations ?& array['vi','th','id','en']),
  constraint ck_course_kp_title_zh_len
    check (char_length(title_zh) between 1 and 60),
  constraint ck_course_kp_version check (version >= 1)
);

create unique index uq_course_kp_code
  on zhiyu.course_knowledge_points (kp_code) where deleted_at is null;
create index idx_course_kp_type        on zhiyu.course_knowledge_points (kp_type)        where deleted_at is null;
create index idx_course_kp_track       on zhiyu.course_knowledge_points (primary_track)  where deleted_at is null;
create index idx_course_kp_status      on zhiyu.course_knowledge_points (status)         where deleted_at is null;
create index idx_course_kp_difficulty  on zhiyu.course_knowledge_points (difficulty)     where deleted_at is null;
create index idx_course_kp_title_zh    on zhiyu.course_knowledge_points (title_zh)       where deleted_at is null;
create index idx_course_kp_ai_job      on zhiyu.course_knowledge_points (ai_job_id);
create index idx_course_kp_tags_gin    on zhiyu.course_knowledge_points using gin (tags);
create index idx_course_kp_content_gin on zhiyu.course_knowledge_points using gin (content);
create index idx_course_kp_deleted_at  on zhiyu.course_knowledge_points (deleted_at);

create trigger tg_course_kp_before_update_set_updated_at
  before update on zhiyu.course_knowledge_points
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| 操作 | 角色 | 策略 |
|------|------|------|
| `select` | `anon`、`authenticated` | 仅 `status='approved' and deleted_at is null` |
| `select` | `service_role` | 全量 |
| 写 | `service_role` | 全量；管理端通过 service-role |

策略命名：`course_kp_select_approved`、`course_kp_write_service`。

## 2. 表：`course_lesson_kp`（节-KP 多对多关联）

**对应需求中的对象**：[temp/01 §1.6 共享 KP 复用机制](../../../temp/01-课程目录骨架.md) + [temp/03 §3.2.6](../../../temp/03-数据库schema.md)
**性质**：纯关联表，同一 KP 可关联到多节（共享层 KP 被 4 赛道节复用）。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `lesson_id` | `uuid` | ✅ | — | PK 一部分 | — | FK → `course_lessons.id`，`ON DELETE CASCADE` |
| `kp_id` | `uuid` | ✅ | — | PK 一部分 | — | FK → `course_knowledge_points.id`，`ON DELETE RESTRICT` |
| `position` | `int` | ✅ | — | `idx_course_lesson_kp_lesson_pos` | 节内顺序 | 1–50 |
| `is_new_in_lesson` | `bool` | ✅ | `true` | — | 本节首学（false=仅复习） | — |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `is_new_in_lesson` | `bool` | 区分"本节新学的 KP"和"上节带过来巩固的 KP"，[temp/02 §2.5](../../../temp/02-知识点与题型内容模板.md) 节末小测的"80% 本节 + 20% 复习"按此过滤 |

### 三、DDL 草案

```sql
create table zhiyu.course_lesson_kp (
  lesson_id        uuid not null,
  kp_id            uuid not null,
  position         int  not null,
  is_new_in_lesson bool not null default true,
  created_at       timestamptz not null default now(),

  primary key (lesson_id, kp_id),
  constraint fk_course_lesson_kp_lesson
    foreign key (lesson_id) references zhiyu.course_lessons(id) on delete cascade,
  constraint fk_course_lesson_kp_kp
    foreign key (kp_id) references zhiyu.course_knowledge_points(id) on delete restrict,
  constraint ck_course_lesson_kp_position check (position between 1 and 50)
);

create index idx_course_lesson_kp_lesson_pos
  on zhiyu.course_lesson_kp (lesson_id, position);
create index idx_course_lesson_kp_kp
  on zhiyu.course_lesson_kp (kp_id);
```

### 四、RLS 策略

由父表 `course_lessons` + `course_knowledge_points` 的 RLS 间接控制；本表直接放行 `select` 给 `authenticated/anon`，但应用层 join 时父表的 `is_published / status='approved'` 会自动过滤。

## 3. 表：`course_questions`（题库）

**对应需求中的对象**：[temp/00 §0.4 「Question」](../../../temp/00-总览与设计原则.md) + [temp/02 §2.3–2.4](../../../temp/02-知识点与题型内容模板.md)
**性质**：内容主表，**12 类题型**统一存储；`payload` jsonb 按题型不同。

### 一、字段定义

| 字段名 | 类型 | 必填 | 默认值 | 索引 | 对应需求中的信息 | 校验规则 |
|--------|------|------|--------|------|----------------|---------|
| `id` | `uuid` | ✅ | `gen_random_uuid()` | PK | — | — |
| `q_code` | `text` | ✅ | 由 RPC `fn_gen_q_code()` 生成 | `uq_course_q_code` (UNIQUE WHERE deleted_at IS NULL) | 题目编码 | 正则 `^q_(ec\|fc\|hk\|dl\|share)_\d{8}$`，见 [10-编号生成规则.md](./10-编号生成规则.md) |
| `kp_id` | `uuid` | ✅ | — | `idx_course_q_kp` | 关联的 KP | FK → `course_knowledge_points.id`，`ON DELETE RESTRICT` |
| `q_type` | `text` | ✅ | — | `idx_course_q_type` | 题型 | 枚举 12 选 1（[06-枚举定义.md §2](./06-枚举定义.md)） |
| `difficulty` | `int` | ✅ | `1` | `idx_course_q_difficulty` | 难度 | 1–6 |
| `payload` | `jsonb` | ✅ | — | — | 题干/选项/答案/解析 | 见 §payload schema |
| `audio_url` | `text` | ❌ | `null` | — | 题干音频（如听力题） | URL |
| `image_url` | `text` | ❌ | `null` | — | 题干图片（如看图题） | URL |
| `status` | `text` | ✅ | `'draft'` | `idx_course_q_status` | 内容状态 | `draft\|reviewing\|approved\|archived` |
| `version` | `int` | ✅ | `1` | — | 版本号 | ≥ 1 |
| `ai_job_id` | `uuid` | ❌ | `null` | `idx_course_q_ai_job` | 来源 AI Job | FK → `course_ai_generation_jobs.id`，`ON DELETE SET NULL` |
| `reviewed_by` | `uuid` | ❌ | `null` | — | 审核人 | FK → `auth.users.id`，`ON DELETE SET NULL` |
| `reviewed_at` | `timestamptz` | ❌ | `null` | — | — | — |
| `report_count` | `int` | ✅ | `0` | `idx_course_q_report_count` | 学员举报次数（[temp/06 §6.10](../../../temp/06-关键业务流程.md)） | ≥ 0 |
| `created_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `updated_at` | `timestamptz` | ✅ | `now()` | — | — | — |
| `deleted_at` | `timestamptz` | ❌ | `null` | `idx_course_q_deleted_at` | — | 软删 |

### `payload` JSON Schema（12 题型，详见 [temp/02 §2.4](../../../temp/02-知识点与题型内容模板.md)）

```jsonc
// 通用结构
{
  "stem": { /* 题型相关 */ },
  "options": [ /* 题型相关 */ ],
  "answer": "...",                              // 题型相关：string | string[] | int | nested
  "explanation": { "vi":"...","th":"...","id":"...","en":"..." },
  "scoring": { "type":"single", "score":1 }    // single | partial（连线 0.25/对）| binary
}
```

| q_type | stem 关键字段 | answer 类型 |
|--------|---------------|------------|
| `mcq_meaning` | `{zh, pinyin}` | `'A'\|'B'\|'C'\|'D'` |
| `mcq_zh` | `{translation_<lang>}` | 同上 |
| `listen_pick` | `{audio_url, play_limit}` | 同上 |
| `listen_pinyin` | `{audio_url, play_limit}` | 同上 |
| `tone_pick` | `{zh, pinyin_no_tone, audio_url}` | `int[]`（每字声调，如 `[4,2]`） |
| `match_pairs` | `{left:[], right:[]}` | `Array<[leftId, rightId]>` |
| `sort_words` | `{translation_<lang>, tokens:[]}` | `string[]`（正确顺序） |
| `fill_blank_choice` | `{stem_zh, blank_index}` | `string` |
| `type_pinyin` | `{audio_url, hint_zh, play_limit}` | `answer_regex` + `case_insensitive` |
| `type_zh_ime` | `{translation_<lang>, hint_pinyin}` | `string` + `accept_traditional` |
| `image_pick` | `{zh, pinyin}` | `'A'\|'B'\|'C'\|'D'` |
| `dialog_cloze` | `{dialog: turns[]}` | `int`（选项 index） |

### 二、AI 补充字段

| 字段名 | 类型 | 补充理由 |
|--------|------|---------|
| `q_code` | `text` | 业务编码，便于学员"举报错题"、缓存键、URL |
| `report_count` | `int` | 学员反馈聚合（≥3 次自动置顶审核），见 [temp/06 §6.10](../../../temp/06-关键业务流程.md) |
| `version` | `int` | 修题不影响历史成绩（[temp/07 §H4](../../../temp/07-待确认问题清单.md)） |

### 三、DDL 草案

```sql
create table zhiyu.course_questions (
  id           uuid primary key default gen_random_uuid(),
  q_code       text not null,
  kp_id        uuid not null,
  q_type       text not null,
  difficulty   int  not null default 1,
  payload      jsonb not null,
  payload_hash text  not null,                   -- ✅ [12-Q9](./12-待确认问题清单.md)：encode(sha256(payload::text::bytea), 'hex')，题目重生去重
  audio_url    text,
  image_url    text,
  status       text not null default 'draft',
  version      int  not null default 1,
  ai_job_id    uuid,
  reviewed_by  uuid,
  reviewed_at  timestamptz,
  report_count int  not null default 0,
  created_at   timestamptz not null default now(),
  updated_at   timestamptz not null default now(),
  deleted_at   timestamptz,

  constraint fk_course_q_kp
    foreign key (kp_id) references zhiyu.course_knowledge_points(id) on delete restrict,
  constraint fk_course_q_reviewer
    foreign key (reviewed_by) references auth.users(id) on delete set null,
  constraint ck_course_q_code_format
    check (q_code ~ '^q_(ec|fc|hk|dl|share)_\d{8}$'),
  constraint ck_course_q_type
    check (q_type in (
      'mcq_meaning','mcq_zh','listen_pick','listen_pinyin','tone_pick',
      'match_pairs','sort_words','fill_blank_choice','type_pinyin',
      'type_zh_ime','image_pick','dialog_cloze'
    )),
  constraint ck_course_q_difficulty check (difficulty between 1 and 6),
  constraint ck_course_q_status
    check (status in ('draft','reviewing','approved','archived')),
  constraint ck_course_q_report_count check (report_count >= 0),
  constraint ck_course_q_version check (version >= 1),
  constraint ck_course_q_payload_hash_format check (payload_hash ~ '^[0-9a-f]{64}$')
);

create unique index uq_course_q_code
  on zhiyu.course_questions (q_code) where deleted_at is null;
create unique index uq_course_q_kp_qtype_hash
  on zhiyu.course_questions (kp_id, q_type, payload_hash) where deleted_at is null;  -- ✅ [12-Q9](./12-待确认问题清单.md) dedup
create index idx_course_q_kp           on zhiyu.course_questions (kp_id)        where deleted_at is null;
create index idx_course_q_type         on zhiyu.course_questions (q_type)       where deleted_at is null;
create index idx_course_q_status       on zhiyu.course_questions (status)       where deleted_at is null;
create index idx_course_q_difficulty   on zhiyu.course_questions (difficulty)   where deleted_at is null;
create index idx_course_q_ai_job       on zhiyu.course_questions (ai_job_id);
create index idx_course_q_report_count on zhiyu.course_questions (report_count desc) where status = 'approved' and deleted_at is null;
create index idx_course_q_deleted_at   on zhiyu.course_questions (deleted_at);

create trigger tg_course_q_before_update_set_updated_at
  before update on zhiyu.course_questions
  for each row execute function zhiyu.set_updated_at();
```

### 四、RLS 策略

| `select` | `anon` | **拒绝**（题目仅登录用户可见，避免被爬） |
| `select` | `service_role` | 全量 |
| 写 | `service_role` | 全量 |
