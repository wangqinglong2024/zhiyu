# 待确认问题清单（F2 接口规范）

> 全部已与 PM 拍板封板。本文件留作决议追溯。

---

## Q1 URL 风格 ✅ 封板

- **动词形式 `/{resources}/{id}/{verb}`**（路径分隔，无冒号）；批量 `/{resources}:batch-<verb>`。
- 例：发布单节 `POST /admin/v1/course/lessons/:id/publish`；批量发布章 `POST /admin/v1/course/chapters:batch-publish`。
- `/v1` 前缀强制；版本切换=全路径替换。
- 与 [grules/G1-04 §URL 风格](../../../grules/G1-架构与技术规范/04-API接口规范.md) 一致。

## Q2 节末小测题数 ✅ 封板

- **固定 5 题**（[temp/01 §1.5](../../../temp/01-课程目录骨架.md)），后端按 `course_lesson_kp` 抽样。
- 不再支持节级 `quiz_count` 覆盖（除非走 [F3-13 Q8](../F3-AI-页面交互规范/13-待确认问题清单.md) 后续放开）。

## Q3 学员侧读路径 ✅ 封板

- 学员"读"走 PostgREST + RLS（直连 supabase-js）；
- 学员"写"（答题、进度、报告）走 Hono `/api/v1/...`；
- 管理端"读+写"全走 Hono `/admin/v1/...`，**不**用 PostgREST。

## Q4 AI 「一键全自动 gen_full_stage」 ✅ 封板

- **不提供**该 task type。AI 工作台仅做"补漏 / 编辑后重生 / 单点修补"6 个细粒度 task：
  `gen_questions_for_kp / regen_question / gen_kp_for_lesson / translate_kp_batch / translate_q_batch / tts_for_kp_batch`
- 详见 [F3-08](../F3-AI-页面交互规范/08-管理端-AI工作台页.md) 与 [06-管理端-AI生产.md](./06-管理端-AI生产.md)。

## Q5 限流 ✅ 封板

- 默认 IP 60/分、用户 120/分。
- 学员侧 C9（答题）+ C16（考试自动保存）：用户 600/分。
- A17 / A24（举报、搜索）：用户 30/分。
- 管理端**不限流**。
- 详见 [00-index §限流策略](./00-index.md)。

## Q6 错误码体系 ✅ 封板

- 业务命名 `COURSE_*` 作 enum + 5 位数字映射，统一登记到 `packages/shared-config/src/error-codes.js`。
- 5 位数字范围（4xx 段）：40000–42900。
- 5 语言文案表见 [11-错误码登记.md](./11-错误码登记.md)。

## Q7 全文搜索 ✅ 封板

- 与 china 一致：5 语言全字段 ILIKE。
- **不**支持拼音搜索（避免歧义 + 后端简化）。
- 管理端 KP/Q 搜索接口 [A24/A26](./05-管理端-KP与题目CRUD.md)。

## Q8 idempotency_key ✅ 封板

- 凡是写副作用接口（答题 / 提交考试 / 创建 AI Job / 节发布），客户端必传 `Idempotency-Key` Header。
- 服务端 24h TTL 缓存请求-响应映射，重复请求返同结果。

## Q9 媒资缓存触发端 ✅ 封板

- 媒资缓存只在**学员端** C6 `/media/tts` 触发（懒生成）。
- 管理端不能"主动生成"，只能查看 / 软删 / 替换。
- 删除走 7 天延迟（详见 [F1-Q9](../F1-AI-数据模型规范/12-待确认问题清单.md)）。

## Q10 内部 OpenAPI ✅ 封板

- 不向外公开。仅在 `/admin/v1/openapi.json` 由 `super_admin` 角色可下载（仅供内部前后端联调）。
- 不生成 SDK，不发布到 npm。

## Q11 lesson_quiz 单独表？ ✅ 封板

- **不单独建表**。节末小测**不持久化题集**，每次按 `course_lesson_kp` 临时抽样 5 题（同节同学员每次不同顺序，但题目集合相同）。
- 学员答题写入复用 `course_user_answers`（带 `context='lesson_quiz'`）。
- 详见 [03-应用端-考试与举报.md](./03-应用端-考试与举报.md)。

## Q12 考试是否快照题目版本 ✅ 封板

- **不快照**。考试中题目变更仅影响后续 attempt，进行中 attempt 仍读题目当前版本。
- 学员体感影响极小（管理员发布期不会大改 approved 题）；用快照换取了简化 schema + 减少存储。

## Q13 管理端导出 / 导入 ✅ 封板

- 管理端 KP / Q 列表支持「导出 JSON」（一键，全部命中行）。
- 导入支持 CSV / JSON（[D-15](../F3-AI-页面交互规范/11-弹窗与公共组件.md)），用 NDJSON 流式入库 + 部分回滚策略。
- 导出考试 attempt 详细数据走单独 `/admin/v1/course/exams/:id/attempts/export`（仅 reviewer+）。
