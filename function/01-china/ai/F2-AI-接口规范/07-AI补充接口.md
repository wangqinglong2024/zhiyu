# AI 补充的接口

> 需求中没有明确提到，但技术上必要的内部接口。

| 接口名称 | 用途 | 补充理由 |
|---------|------|---------|
| I1 · TTS Mock 内部回调 | 在未配置真实 TTS 密钥的 dev / Docker 验证环境，模拟 TTS 完成回调 | 满足 zhiyu-docker-policy「缺密钥用 mock，不阻塞 Docker 测试」 |
| I2 · 健康检查 | 容器探活、监控 | 标准实践 |
| AUX · 文章 code 校验 | 后台编辑器实时检查输入 code 是否合法（如未来支持搜索） | 仅当列表 q 包含 12 位编码时本地预校验 |
| AUX · TTS 状态轮询 | 当 C4 返回 202 时供前端短轮询 | 简化前端实现 |

---

## I1 · TTS Mock 回调

**方法**：POST
**路径**：`/internal/v1/china/tts/callback`
**权限**：仅 `service-role`（容器内网；外部不可达，nginx 不暴露）

**业务用途**：当 `apps/api-app` 在 `mock` 模式下调用本地 fake-tts adapter，adapter 异步回调本接口写回 `audio_url_zh`、`audio_status='ready'`。生产环境真实 TTS 直接同步返回，无需此接口。

**请求体**：

```json
{
  "sentence_id": "5b7c...",
  "audio_url":   "https://mock-storage/.../0001.mp3",
  "duration_ms": 4820,
  "provider":    "mock",
  "voice":       "mock-zh-female-1"
}
```

**成功响应 200**：

```json
{ "code":0, "message":"ok", "data": { "sentence_id":"5b7c...", "status":"ready" } }
```

**错误场景**：

| HTTP | code | 触发条件 |
|------|------|---------|
| 401 | 40100 | 非 service-role 调用 |
| 404 | 40400 | sentence 不存在 |
| 409 | 40900 | sentence 当前状态不允许（如已 ready 且 url 不同） |

## AUX · TTS 状态轮询

**方法**：GET
**路径**：`/api/v1/china/sentences/:id/audio`
**权限**：公开（与 C4 同侧）

**用途**：C4 返回 202 后供前端轮询；不发起任何写动作，仅读 `audio_status`。

**成功响应 200**：

```json
{
  "code": 0, "message":"ok",
  "data": {
    "sentence_id": "5b7c...",
    "audio": { "status":"processing", "url":null }
  }
}
```

**限流**：单 IP 30 次/分钟。

---

## I2 · 健康检查

**方法**：GET
**路径**：`/api/v1/china/health`
**权限**：公开

**成功响应 200**：

```json
{ "code":0, "message":"ok", "data": { "service":"china", "uptime_s": 12345, "tts_adapter":"mock" } }
```

`tts_adapter` 取值：`mock` / `azure` / `aliyun` / 其他。
