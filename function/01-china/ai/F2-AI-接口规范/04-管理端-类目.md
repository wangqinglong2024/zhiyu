# 管理端：类目

> 类目为系统内置 12 项，**只读**（PM 答 Q6：固定不增加，扩展走代码层）。
> 不提供 POST/PUT/DELETE。

---

## A1 · 列出 12 类目（卡片）

**对应需求中的操作**：F2-用户 §一 · 管理端「发现中国菜单进入后，是 12 个类目卡片」

**方法**：GET
**路径**：`/admin/v1/china/categories`
**权限**：管理员（`role = admin`）

**业务校验**：无。

**请求参数**：无。

**成功响应 200**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": "0a8b...",
        "code": "01",
        "name_i18n":        { "zh":"中国历史","en":"...","vi":"...","th":"...","id":"..." },
        "description_i18n": { "zh":"朝代更替、历史事件、传奇人物","en":"...","vi":"...","th":"...","id":"..." },
        "sort_order": 1,
        "article_count_total": 42,
        "article_count_published": 38
      }
      /* ... 共 12 项 */
    ]
  }
}
```

**字段说明**：
- `article_count_total`：含 `draft + published`，**不含**软删（满足 F2 §一·管理端「文章列表页要显示当前类目的文章总数」的卡片视图）。
- `article_count_published`：仅 `published`，方便管理员看应用端实际可见量。

**错误场景**：

| HTTP | code | 触发条件 | 错误信息 |
|------|------|---------|---------|
| 401 | 40100 | 未登录 | 请先登录 |
| 403 | 40300 | 非管理员 | 无权访问后台 |
