## C6 · 上报阅读进度

**对应需求中的操作**：F2-用户 §一 · 应用端「逐句查看文章」隐含的进度记忆

**方法**：PUT
**路径**：`/api/v1/china/articles/:code/progress`
**权限**：登录用户（任何已认证 supabase 用户）

**业务校验**：
- 文章必须 `published`；否则 404。
- `last_seq_no` ∈ `[1, max_seq_no_of_article]`。
- 同一用户同一文章 PUT 幂等覆盖。

**请求参数**：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| `code` | path | string | ✅ | 文章编码 |
| `last_seq_no` | body | int | ✅ | 最后阅读到的句子号 |
| `completed` | body | bool | ❌ | 是否完成（最后一句通读完） |

```json
{ "last_seq_no": 2, "completed": false }
```

**成功响应 200**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "article_code": "A7K2P9X3M4QR",
    "last_seq_no": 2,
    "completed": false,
    "updated_at": "2026-04-28T10:05:00+08:00"
  }
}
```

**错误场景**：

| HTTP | code | 触发条件 | 错误信息 |
|------|------|---------|---------|
| 400 | 40001 | `last_seq_no` 越界 | 句子序号超出范围 |
| 401 | 40100 | 未登录 | 请先登录 |
| 404 | 40400 | 文章不存在/未发布 | 文章不存在 |

## C7 · 读取阅读进度

**方法**：GET
**路径**：`/api/v1/china/articles/:code/progress`
**权限**：登录用户

**成功响应 200**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "article_code": "A7K2P9X3M4QR",
    "last_seq_no": 2,
    "completed": false,
    "updated_at": "2026-04-28T10:05:00+08:00"
  }
}
```

> 若该用户未读过此文章，返回 `last_seq_no=0, completed=false`。

**错误场景**：

| HTTP | code | 触发条件 | 错误信息 |
|------|------|---------|---------|
| 401 | 40100 | 未登录 | 请先登录 |
| 404 | 40400 | 文章不存在/未发布 | 文章不存在 |
