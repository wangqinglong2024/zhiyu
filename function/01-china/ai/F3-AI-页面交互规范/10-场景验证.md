# 场景验证

> 端到端走查，验证页面/接口/数据闭环无断裂。

## 场景 1：未登录用户浏览公开类目（中国历史 → 文章 → 朗读）

```
1. 用户匿名打开 /china
   → 路由公开
   → 调 C1 → 渲染 12 张卡片（01-03 正常，04-12 显示 🔒）
   → ✅

2. 点击「01 中国历史」卡片
   → 类目 code='01' ≤ '03'，无锁
   → router.push('/china/categories/01')
   → 调 C2 → 渲染文章列表（含「秦始皇统一六国」）
   → ✅

3. 点击「秦始皇统一六国」
   → router.push('/china/articles/A7K2P9X3M4QR')
   → 调 C3 → 渲染 3 句卡片 + prev/next 导航
   → 未登录用户 → 不调 C7；不渲染进度数字
   → ✅

4. 点击第 2 句的 [🔊]（status=pending）
   → 调 C4
   → 上游 mock 同步返回 → 200 + audio.url
   → 立即 <audio> 播放
   → ✅

5. 点击「全文朗读」
   → 顺序播放 seq=1（已 ready，复用缓存）→ seq=2（ready，刚生成）→ seq=3（pending → C4 → 播放）
   → 每句间停顿 300ms（前端控制）
   → ✅

6. 点击「下一篇 → 商鞅变法」
   → router.push('/china/articles/B6J1N8W2L3PQ')
   → 重复 3-5
   → ✅

验证结果：✅ 逻辑连贯
```

## 场景 3：管理员新建并发布一篇文章

```
1. 管理员登录 → /admin/china
   → 调 A1 → 12 卡片
   → ✅

2. 点击「02 中国美食」
   → 调 A2 → 列表
   → ✅

3. 点击「+ 新建文章」
   → 弹 D-1
   → 录入拼音 + 5 语言标题 → 提交
   → 调 A4 → 201 → 跳 /admin/china/articles/{newId}
   → ✅

4. 在 P-A-3 点击「+ 添加句子 → 结尾」
   → 弹 D-4，position=end
   → 录入拼音 + 5 语言中文/翻译 → 提交
   → 调 A11 → 重拉 A9
   → 重复 2-3 次，添加共 3 句
   → ✅

5. 点击底部「发布」
   → 弹 D-5 → 确定
   → 调 A6 → 校验通过 → 200
   → toast「已发布」；状态变 published
   → ✅

6. 应用端用户刷新对应类目页
   → C2 列表已包含新文章
   → C3 详情可看到 3 句 + 音频按钮全 pending
   → ✅

验证结果：✅ 全流程通畅
```

## 场景 4：管理员重排已发布文章的句子（核心副作用验证）

```
1. 管理员进入 P-A-3 编辑「秦始皇统一六国」
   → 状态 published
   → A9 显示 3 句，全部 ready
   → ✅

2. 点击「🔀 重排」 → 弹 D-7
   → 拖拽：seq2 ↔ seq3
   → 提交 → 调 A14 全量 ordered_ids
   → 后端 fn_reorder_sentences：rewrite seq_no = 1..3 + 全部 audio_status='pending', audio_url=null
   → ✅

3. 重拉 A9
   → 3 句仍在，但 seq_no 变化；全部 audio_status=pending
   → toast「已重排，所有句子的音频已失效，且所有用户的阅读进度已重置」
   → ✅

4. 应用端用户访问 /china/articles/A7K2P9X3M4QR
   → C3 返回新顺序，audio.status 全 pending
   → C7 返回 `last_seq_no=0`（A14 副作用调 `fn_clear_progress_by_article` 已清，PM 答 F3-Q4/Q12）
   → 前端检测到本地缓存 last_seq_no>0 → toast「文章已更新，阅读进度已重置」（C3-14）
   → ✅

5. 用户点击「全文朗读」 → 逐句调 C4 → 重新生成 mp3
   → ✅

验证结果：✅ 逻辑连贯（PM 答 F3-Q4/Q12 已覆盖：重排清进度 + 前端提示）
```
