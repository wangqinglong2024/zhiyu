## 元素规格

| 元素 | 组件类型 | 规格 |
|------|---------|------|
| 搜索框 | `Input` | 1–60 字符；空提交无效（按钮 disabled） |
| 范围筛选 | `Select` | `all` / `articles` / `sentences` |
| 类目筛选 | `Select` | 12 类目 + 全部 |
| 状态筛选 | `Select` | `all` / `draft` / `published` |
| 文章命中卡片 | `Card.Article` | 标题（含 `<em>` 高亮）+ 类目 + code + 状态 + 句数 + 更新人/时 + 「打开编辑」按钮 |
| 句子命中卡片 | `Card.Sentence` | 文章 code + 中文标题 + seq_label + 中文片段（含 `<em>` 高亮）+ 「跳到该句」按钮 |
| 高亮 | 内联 `<em>` | CSS：`background:#fff3a8;color:inherit;font-style:normal;font-weight:600`；**前端只信任 `<em>` 标签**，其余 HTML 全部 escape（XSS 防御） |

## 数据来源

- 主：[A15 GET /admin/v1/china/search](../F2-AI-接口规范/11-管理端-全局搜索.md#a15--全局搜索聚合文章--句子) → 含 `articles[]` / `sentences[]` / `summary` / `pagination`。
- 跳转：
  - 文章「打开编辑」→ `/admin/china/articles/{article.id}`（P-A-3）。
  - 句子「跳到该句」→ `/admin/china/articles/{sentence.article.id}?focus_seq=${sentence.seq_no}`（P-A-3 自动滚动到该 seq 卡片并加亮 1.5s）。

## 交互行为

| # | 用户操作 | 技术实现 |
|---|---------|---------|
| A4-01 | 输入 `q` 后回车 / 点击 🔍 | `q` trim 后长度 0 → toast「请输入关键字」；否则更新 URL query → useQuery 触发 A15 |
| A4-02 | 切换 scope/类目/状态 | 同步到 URL query → 重新 A15 |
| A4-03 | 翻页 | URL `page` 同步 → A15 |
| A4-04 | 点击「打开编辑」 | `router.push('/admin/china/articles/' + article.id)` |
| A4-05 | 点击「跳到该句」 | `router.push('/admin/china/articles/' + sentence.article.id + '?focus_seq=' + sentence.seq_no)` |
| A4-06 | 接口 504（搜索超时） | toast「搜索超时，请缩小关键字范围」+ 保留输入框 |
| A4-07 | 接口 400 `CHINA_SEARCH_QUERY_TOO_SHORT/LONG` | 输入框下红字错误，按钮变 disabled |

## 性能与缓存

- A15 响应**不进任何客户端缓存**（搜索结果时效性强）。
- 输入防抖 300ms（仅手动回车提交，不做"输入即搜"以减负）。
- A15 默认 `page_size=20`，上限 50；翻页用 URL state 而非滚动加载。

---

## 安全

- `<em>` 高亮：使用 `dangerouslySetInnerHTML`（React）/ `v-html`（Vue）**仅渲染服务端返回的 snippet**；进入前用一个白名单 sanitize（仅保留 `<em>`），其它一律 escape。
- 搜索关键字 `q` 不进 URL fragment、不打 GA 事件含 PII。
