## D-1 · 新建文章 Modal

**触发**：P-A-2「+ 新建文章」
**类型**：Modal 居中
**尺寸**：宽 640px，高自适应

### 布局

```
┌────────────────────────────────────────────────┐
│ 新建文章                                  [✕]  │
├────────────────────────────────────────────────┤
│ *类目：       [当前类目（不可改）]              │
│ *标题拼音：   [_____________________________]  │
│ *标题（5 语言 Tab）：                          │
│   [zh|en|vi|th|id]                            │
│   [____________________________________]      │
├────────────────────────────────────────────────┤
│  说明：新建后状态为「草稿」，需添加至少 1 句并   │
│        完成 5 语言翻译后方可发布。               │
├────────────────────────────────────────────────┤
│                       [取消]   [保存为草稿]    │
└────────────────────────────────────────────────┘
```

### 表单字段

| 字段 | 组件 | F1 字段 | 必填 | 校验 |
|------|------|--------|------|------|
| 类目 | `Select`（默认锁定为当前类目） | `category_id` | ✅ | 必须存在 |
| 标题拼音 | `Input` | `title_pinyin` | ✅ | 1–200 |
| 标题 zh/en/vi/th/id | `Input` × 5 Tab | `title_i18n.*` | ✅ | 1–40 每项 |

### 交互

| 操作 | 实现 |
|------|------|
| 提交 | 前端校验 → 调 [A4 POST /admin/v1/china/articles](../F2-AI-接口规范/05-管理端-文章CRUD.md#a4--新建文章) → 成功关闭 + 跳转 `/admin/china/articles/{newId}` |
| 取消 | 若有任何字段被改 → 弹 D-6；否则直接关闭 |

## D-4 · 句子新建 Drawer（开头/结尾/中间）

**触发**：P-A-3「+ 添加句子」下拉
**类型**：右侧 Drawer
**尺寸**：宽 600px

布局与 D-3 类似，顶部多一行「插入位置：开头 / 结尾 / 在 0002 之后」（由触发时的菜单决定，不在 Drawer 内再选）。

### 接口

[A11 POST /admin/v1/china/articles/:id/sentences](../F2-AI-接口规范/06-管理端-句子CRUD.md#a11--新建句子开头结尾中间)，根据触发选项传入 `position` + `after_seq_no`。

### 交互

成功后用响应 `affected_sentence_ids` 局部刷新或整页 A9 重拉；toast「已新增；受影响的 N 句音频已失效」。

## D-5 · 二次确认 Modal（通用危险操作）

**触发**：删除/发布/下架/批量重排等
**类型**：Modal 居中
**尺寸**：宽 380px

### 布局

```
┌────────────────────────────────┐
│ ⚠ {标题}                       │
├────────────────────────────────┤
│ {正文：说明操作影响}             │
├────────────────────────────────┤
│           [取消]   [确定]       │
└────────────────────────────────┘
```

### 各场景文案模板

| 场景 | 标题 | 正文 |
|------|------|------|
| 删除文章 | 确认删除？ | 删除后 30 天内不可恢复，到期物理清除。 |
| 下架文章 | 确认下架？ | 下架后所有用户对该文章的阅读进度将被清空（不可恢复）。 |
| 发布文章 | 确认发布？ | 发布后用户即可看到该文章。 |
| 删除句子 | 确认删除该句？ | 剩余句子会自动重排、所有句子音频缓存失效，且所有用户的阅读进度会被重置（不可恢复）。 |
| 批量重排 | 确认重排？ | 重排后所有句子的音频缓存会失效，且所有用户的阅读进度会被重置（不可恢复）。 |
| 应用端「重新开始」 | 确定重新开始？ | 阅读进度将重置（不可恢复）。 |

### 交互

| 操作 | 实现 |
|------|------|
| 取消 | 关闭，无副作用 |
| 确定 | 按钮变 loading；调用对应接口；成功关闭 + toast；失败保留 + Modal 内显示错误 |

## D-6 · 未保存离开提示 Modal

**触发**：路由切换前 / 关闭 Drawer 前 / 关闭浏览器标签前（`beforeunload`），且当前表单 `dirty=true`
**类型**：Modal 居中
**尺寸**：宽 380px

### 文案

```
你有未保存的修改，确定离开？
未保存的内容会丢失。

[继续编辑]   [放弃修改并离开]
```

### 实现

- 全局 `dirty` 状态由各表单 hook 注册到 `useUnsavedChangesGuard()`。
- 路由切换通过 `router.beforeEach` 拦截；关闭浏览器通过 `window.addEventListener('beforeunload', ...)`。
- 「继续编辑」取消导航；「放弃」清空 dirty + 完成导航。

## 公共反馈组件

| 组件 | 用途 | 配置 |
|------|------|------|
| `Toast` | 操作成功/失败反馈 | 右上角；成功 2s，失败 4s |
| `Skeleton` | 列表/卡片加载占位 | > 200ms 才显示 |
| `EmptyState` | 列表为空 | 居中插画 + 文案 + 可选行动 |
| `ErrorState` | 接口失败 | 居中 + 错误说明 + [重试] 按钮 |
| `OfflineBar` | 网络断开 | 顶部横条 |
