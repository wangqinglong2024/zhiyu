# 枚举定义

> 按 G1-数据库规范 §四，统一使用 **`text` + `CHECK` 约束**，不使用 PG enum 类型。
> 标签颜色映射遵循 G2-AI 色彩系统（`status.*` 语义色）。

---

## 1. `china_articles.status`（文章状态）

| 枚举值 | 对应需求中的状态名 | 显示文本（zh / en） | 标签颜色 |
|--------|------------------|--------------------|---------|
| `draft` | 待发布 | 待发布 / Draft | `status.warning`（琥珀） |
| `published` | 已发布 | 已发布 / Published | `status.success`（绿） |

- DDL：`check (status in ('draft','published'))`
- 默认值：`draft`
- 备注：需求中未提及"已下架"独立状态，按"下架 = 回到 draft"处理（见 [06-状态机.md](./06-状态机.md)）。

---

## 2. `china_sentences.audio_status`（中文 TTS 音频状态）

| 枚举值 | 含义 | 显示文本（zh / en） | 标签颜色 |
|--------|------|--------------------|---------|
| `pending` | 句子入库后初始状态，等待首位用户触发 | 待生成 / Pending | `status.neutral`（灰） |
| `processing` | 已发起 TTS 调用，等待返回 | 生成中 / Processing | `status.info`（蓝） |
| `ready` | 调用成功，`audio_url_zh` 已写入永久缓存 | 就绪 / Ready | `status.success`（绿） |
| `failed` | 上一次调用失败，`audio_error` 记录原因；下次用户点击会再次触发，**不限次数**（PM 答 Q10） | 失败 / Failed | `status.danger`（红） |

- DDL：`check (audio_status in ('pending','processing','ready','failed'))`
- 默认值：`pending`
- 触发模型：**用户驱动**——任意用户首次点击播放按钮才发起 TTS；全平台共享缓存，第二位用户直接命中 `ready`。
- `failed` 不阻塞用户：前端按钮仍可点，再次进入 `processing`；后端不做自动重试。

## 3. 多语言 locale 取值（与 i18n 规范一致）

> 不是数据库枚举，而是应用层 Zod 常量；多处复用，集中在此声明。

```ts
export const CHINA_LOCALES = ['zh','en','vi','th','id'] as const;
export type ChinaLocale = (typeof CHINA_LOCALES)[number];
```

- 用于 `name_i18n` / `description_i18n` / `title_i18n` 的键校验
- 缺省回退顺序：`<用户 locale>` → `en` → `zh`（与 G1-数据库规范 §八一致）
