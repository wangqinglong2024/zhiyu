# 状态机：`china_articles.status`

## 状态转换图

```
                 ┌──── 创建（默认） ────┐
                 ▼                     │
              ┌──────┐  发布  ┌──────────┐
   (创建)──▶ │draft │ ─────▶ │published │
              │待发布│ ◀───── │ 已发布   │
              └──────┘  下架  └──────────┘
```

---

## 状态转换规则

| 从 | 到 | 对应需求 | 校验条件 | 字段变更 |
|----|---|---------|---------|---------|
| （无） | `draft` | F1-用户 §二「新建时默认状态：待发布」 | 创建文章时 | `status='draft'`，`published_at=null` |
| `draft` | `published` | F1-用户 §二「点击发布」 | ① 5 种语言标题非空且 ≤40<br>② 至少 1 条句子<br>③ 每条句子 5 种语言内容非空且 ≤400<br>④ 操作者为管理员（G3） | `status='published'`，`published_at=now()`，`updated_by=<admin>` |
| `published` | `draft` | F1-用户 §二「点击下架」 | 操作者为管理员 | `status='draft'`，`published_at=null`，`updated_by=<admin>` |

---

## 禁止的转换

```
× draft     → draft       （无意义，前端按钮置灰）
× published → published   （无意义；若需"重新发布"应先下架）
× 任何状态  → 删除         （文章软删独立于状态字段，不计为状态转换）
× 普通用户  → 任何转换     （仅 admin 角色可调用 fn_publish_article / fn_unpublish_article）
```

---

## 实现位置

- RPC：
  - `fn_publish_article(p_article_id uuid) returns china_articles`
  - `fn_unpublish_article(p_article_id uuid) returns china_articles`
- 校验由 RPC 内部完成，前端仅触发；后端绝不信任前端的 `status` 直接写入。
- 前端按钮根据当前 `status` 切换显示文本（"发布" / "下架"）。

---

## 触发副作用

| 转换 | 副作用 |
|------|--------|
| `draft → published` | 1) 写入审计日志；2) 触发应用端列表缓存失效（按 `category_id`）；3) 不预生成音频，等用户点击时再触发 TTS（PM 答 Q10） |
| `published → draft`（下架） | 1) 写入审计日志；2) 触发缓存失效；3) **清空所有用户对该文章的学习进度**（`fn_clear_progress_by_article`，PM 答 Q5）；4) 已下发到客户端的内容由下一次拉取自然过期 |
| 软删（任意状态 → `deleted_at` 非空） | 1) 同步把状态置回 `draft`；2) 同样清空学习进度；3) 30 天后由 cron 物理清理（PM 答 Q9） |
