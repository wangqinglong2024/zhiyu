# 1.2 · 应用的创意技法

为了对抗 LLM 自然的语义聚类偏差（sequential bias），本次会议**强制每 10-15 条创意切换一次技法 + 视角象限**。共应用 7 种技法，覆盖 8 个象限。

## 七种技法清单

### T1 · 5W1H 拆解（Who/What/When/Where/Why/How）
- 用途：穷举每个模块的"使用场景 × 用户角色"组合
- 应用区段：创意 #001-#020（覆盖 4 大模块的场景细分）

### T2 · SCAMPER（Substitute / Combine / Adapt / Modify / Put to other use / Eliminate / Reverse）
- 用途：对现有 4 模块和 7 个全局功能做变体生成
- 应用区段：创意 #021-#040

### T3 · 第一性原理（First Principles）
- 用途：拆解"中文学习为什么难"到原子层（音/形/义/用），围绕原子重组解决方案
- 应用区段：创意 #041-#055

### T4 · Crazy 8s（疯狂 8 分钟，强制 8 个截然不同的方向）
- 用途：对"差异化壁垒"和"冷启动获客"做高密度发散
- 应用区段：创意 #056-#080

### T5 · 逆向思维（Reverse / How to fail）
- 用途：列出所有让产品失败的方式 → 反向得到必须做的事
- 应用区段：创意 #081-#095

### T6 · 类比迁移（Analogous Inspiration）
- 用途：从 Duolingo / 多邻国 / Anki / Migii / TikTok / 拼多多 / Pinduoduo 砍价 / Steam 创意工坊借鉴机制
- 应用区段：创意 #096-#110

### T7 · Pre-Mortem（项目尸检）
- 用途：假设"产品上线 12 个月后失败"，反推失败原因 → 转化为风险对冲项
- 应用区段：创意 #111-#120

## 反偏差检查表

每完成 10 条创意，自检：
- [ ] 最近 10 条是否覆盖至少 3 个不同象限？
- [ ] 是否出现了一个"第一眼看上去荒谬"的想法？（若没有，强制再生成 1 条）
- [ ] 是否有跨象限组合（如"合规 × 游戏"、"内容 × 商业"）？
