## 二、风险对策详细方案（高优先级）

### 2.1 S01 内容审核

#### 三道筛
1. **AI 提示词内置红线**：每次生成都拒绝触敏话题
2. **规则字典**：30+ 敏感词 / 实体（越南红线 / 泰国王室 / 印尼宗教）
3. **LLM 二审**：DeepSeek 判断"是否触及政治宗教民族敏感"

#### 应急 PR
- 1h 内：下架相关内容
- 4h 内：内部审核会议 + PR 草案
- 24h 内：官方道歉 / 解释（当地语言）
- 72h 内：法律顾问介入 + 整改报告

#### 多市场分散
- 三市场同步推进，单一市场封禁不致命
- 月度审核报告 + 季度演练

### 2.2 S02 Paddle 准入

#### 时间表
- W-6：Paddle 申请 + LemonSqueezy 备账号同步申请
- W-3：Paddle 沙盒集成 + Webhook 测试
- W-2：Paddle 审核通过（如延迟则用 LemonSqueezy）

#### 双账号架构
- 数据库 schema 用通用 `provider` 字段
- 前端 Feature flag 控制 checkout 入口
- 切换零代码（仅配置）

### 2.3 S05 团队执行

#### AI 工程师
- LangGraph 内容工厂 = 5 倍生产力
- 自动化测试 = 2 倍工程效率
- LLM 辅助代码 = 1.5 倍开发效率

#### 外包
- 母语审校：每市场 1-2 人 $300/月
- 设计：按项目 $500-2000
- 法务：按需 $100-200/h

#### 自动化
- 全 CI/CD（不依赖人工部署）
- 监控自动告警
- 内容审校工作台高效

### 2.4 B01 CAC 高于预期

#### 三渠道并行
- TikTok（高频内容）/ FB（社群）/ YT（深度）
- 一渠道 CPM 上涨 → 倾斜其他

#### SEO 长期资产
- 4800 个母语长尾页 = 6-12 月后稳定免费流量

#### 分销裂变
- K-factor ≥ 1.0 后 → 自然增长 + CAC 下降

### 2.5 B03 D30 留存

#### 钩子
- D1 / D3 / D7 push（智能内容）
- streak 激励
- 学习日报邮件

#### 游戏化
- 12 款游戏 + 知语币 + 皮肤
- 排行榜 + 异步好友挑战

#### 知语币
- 签到 / 任务持续返币
- 兑换出口（皮肤 / 会员）

### 2.6 T05 数据格式漂移

#### 单 schema
- AI 生成 → 入库 → 管理后台 → 应用端 全链路统一 schema
- TypeScript 强类型 + zod 运行时校验

#### CI 校验
- PR 必须运行 schema 校验测试
- 任何字段变化必须有 migration

#### 数据治理
- drizzle introspect → 自动生成文档
- 季度 schema review

### 2.7 C02 泰国 / 宗教 / 民族

#### 红线字典（细化）
- 泰国王室名单（皇室成员 + 历代王）
- 宗教评论禁词
- 民族冲突历史（1979 中越战争 / 1998 印尼排华）
- 边境争议地名

#### 双层 LLM 审
- 字典扫描（毫秒级）
- LLM 判断（5 秒级）
- 双层都通过才发布

#### 用户举报
- 应用内"举报"按钮
- 24h 内响应

## 四、应急预案 SOP

### 4.1 内容下架 SOP
1. 收到投诉 / 检测触发 → 立即标记
2. 1h 内：下架 + 缓存清空
3. 4h 内：内部 review + PR 草案
4. 24h 内：发布回应（当地语言）
5. 72h 内：根本原因分析 + 制度更新
6. 1 周内：全量复盘报告

### 4.2 Paddle 故障 SOP
1. Webhook 失败率 > 5% 持续 10 分钟 → 自动切 LemonSqueezy
2. 用户支付失败 → 站内提示 + IM 接管
3. 1h 内：分析根因（接口变化 / 风控）
4. 4h 内：恢复 / 申诉

### 4.3 服务器故障 SOP
1. Uptime Robot 告警 → 自动重启
2. 重启失败 → 切换备用节点
3. 静态页 fallback（"系统升级中"）
4. 4h 内：修复 + 复盘
5. 影响用户邮件道歉 + 7 天免费
