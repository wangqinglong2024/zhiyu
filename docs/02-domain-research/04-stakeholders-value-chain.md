## 二、价值链分析（波特价值链改造）

### 2.1 主价值活动

```
原料(数据/选题)
   │
   ├── 内容工厂(AI 生成 + 母语审校)
   │
   ├── 平台技术(前后端/数据/AI 服务)
   │
   ├── 分发(Web / 后期 App / SEO / TikTok 嵌入)
   │
   ├── 用户运营(签到/复习/Push/IM)
   │
   └── 商业化(订阅/单课/B2B/分销)
```

### 2.2 支持活动
- 法务合规
- HR / 团队

- 战略 / 数据分析（AI 自助仪表盘）

| 环节 | 投入占比（成本） | 价值占比（用户感知） |
|---|:---:|:---:|

→ **内容工厂是知语的"利润中心 + 价值中心"，必须先于一切构建。**

## 三、关键合作伙伴决策

### 3.1 LLM 选型
- **首选**：Claude Opus 系列（中文 + 多语种综合能力强）+ DeepSeek V4（成本最优做大批量）
- **次选**：通义千问（中文最强，且多模态）
- **策略**：双供应商，按任务路由（高质量用 Claude，大批量用 DeepSeek）

### 3.2 TTS 选型
- **首选**：Azure Speech（覆盖中越泰印尼，质量稳定，价格中等）
- **次选**：ElevenLabs（情感更丰富但贵）+ Edge TTS（免费但商用限制）
- **策略**：中文 / 越南语 / 泰语 / 印尼语 4 个语言池分别 A/B 测试，选最自然 voice

### 3.3 支付
- **首选**：Paddle（MoR + 税务一站式）
- **备选**：LemonSqueezy（同样 MoR）
- **不选**：Stripe（虽然便宜，但要自行处理税务，对小团队负担太大）
- **本地支付**：v2 评估接入印尼 GoPay、越南 MoMo（通过 Paddle Local Payments 拓展）

### 3.4 基础设施
- **托管**：Vercel（前端）+ 自有 Docker（后端，遵循项目 grules）
- **DB / Auth**：Supabase Pro 新加坡区
- **CDN**：Cloudflare（含 Turnstile 反爬）
- **监控**：Sentry + Plausible

## 四、生态位与可持续优势

### 4.1 知语的独特生态位
- 与 Duolingo 错位：Duolingo 中文模块只到 HSK 3，且讲解英语；知语全 12 阶段且母语
- 与 HelloChinese 错位：HelloChinese 讲解英语，付费高；知语母语 + $4
- 与 Migii 错位：Migii 只做题库，知语做完整学习闭环
- 与孔子学院错位：孔院线下，知语线上；可未来合作而非竞争

### 4.2 可持续优势来源
1. **网络效应（弱）**：分销机制带来用户增长
2. **数据飞轮（中）**：用户答题数据反哺 AI 改进内容
3. **内容资产（强）**：12 阶段 × 4 轨道 × 12 章 × 12 节 × 12 知识点的体系化内容是壁垒
4. **本地化深度（强）**：母语原生讲解 + 母语审校团队
5. **品牌（中长期）**：东南亚 Z 世代心智

### 4.3 不可持续 / 短期优势
- AI 内容生成成本优势：18-24 个月内被赶上
- → 必须在窗口期建立网络 + 内容资产 + 品牌

## 五、价值链中潜在脱节点（风险）

| 脱节点 | 风险 | 缓解 |
|---|---|---|
| AI 生成 → 审校 | 母语审校速度跟不上 AI 生成速度 | 招聘多语 freelancer；建立质量评分 |
| 审校 → 入库 | 数据格式漂移 | 严格的 schema + CI 校验 |
| 入库 → 应用端 | 多端展示不一致 | 单一数据源 + 组件库 |
| 应用端 → 用户 | 学习路径混乱 | 课程体系预先定义、用户体验路径设计 |
| 用户 → 付费 | 付费墙不科学 | A/B 测试 |
| 付费 → 续费 | 续费提醒不到位 | 自动 renewal email + 提前 7 天预告 |
