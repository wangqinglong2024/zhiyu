产品整体说明 (Overview) V1.0
一、 架构与技术基座 (Infrastructure)
定位： 一款打破传统 UI 范式的多端响应式沉浸式 AI 应用。
部署端口： Web 端（PC 浏览器）、原生 App（iOS/Android）、微信 H5/小程序。
核心技术栈 (Tech Stack)：
- 前端 (Frontend)： React / TypeScript (保障多端组件的高阶复用与类型安全)。
- 后端 (Backend)： FastAPI (Python，完美契合 AI 算力调度与高并发业务流)。
- 数据与认证 (BaaS)： Supabase (PostgreSQL 驱动，接管 JWT 鉴权与用户记忆沙盒存储)。
- AI 调度中枢 (LLM Ops)： Dify (可视化的 AI 工作流编排、RAG 知识库管理与模型路由)。
- 运维部署 (DevOps)： Docker (全容器化隔离部署，保障各微服务高可用)。
二、 全局美学与沉浸交互 (Aesthetics & Interaction)
视觉基调： 极简、高奢、绝对沉浸。全局摒弃传统的按钮样式、底部 Tab 导航栏和死板的列表。
- 动态网格背景 (Mesh Gradient)： 全局背景采用流体的网格渐变。完美兼容 Light / Dark 双色模式（亮色纯白底，暗色纯黑底）。色彩由系统状态驱动，同时支持用户自定义任意两种色彩进行氤氲交融。
- 全局毛玻璃引擎 (Glassmorphism)： 所有的信息面板、可操作入口，全部采用高透的毛玻璃材质（附带物理级别的光学折射与高斯模糊）。
- 无界水滴 UI： 页面中的可操作元素（如功能入口）具象化为气泡或水滴形态。
- 全景画廊式漫游： 放弃“点击跳转新页面”的传统生硬逻辑。
  - App / 小程序： 通过左右/上下丝滑“滑动”切换功能模块。
  - Web 端： 如同 3D 艺术画廊，通过滚轮或点击左右边缘进行平滑的镜头推拉切换。
- 高维视觉输出： 应用为用户生成的所有内容（如解答、打卡、日记），必须具备顶级杂志排版美学，支持一键生成 9:16 毛玻璃海报，直接化作用户发朋友圈或短视频的“社交货币”。
- 全局中所有的水滴，都是方形的或者长方形的带圆角的，其实本质就是按钮，只不过其样式是水滴的哪种液态感觉。
三、 核心新增：前端优雅降级策略 (Graceful Degradation)
为了保住绝佳的沉浸感，同时防止在微信 H5 或中低端机型上出现发热卡顿的性能灾难，首页及下方所有的引擎必须严格执行“硬件嗅探与分级渲染”。

四、 分享裂变闭环 (Social & Virality)
以下每个模块都支持分享，并且分享时秉承统一的规则：
- 沉浸式定格： 用户在生成的内容上单击（或移动端手动点击），界面中多余的切换按钮瞬间隐去。再次单机会显示操作按钮：包括分享和收藏两个按钮；
- 点击分享：用户点击分享按钮后， 唤起原生分享面板，支持保存高清长图或直接拉起微信分享。分享到微信时，添加微信的小程序码。并且每个用户有一个唯一的小程序码，新用户扫码时，自动绑定为当前分享者的子用户。保存为图片时，不添加任何的水印或者码，避免因为链接而被其他短视频平台风控。下文所有涉及分享的地方，都遵循此规则。

注册登录 PRD：缔结契约 (Authentication & Identity) V1.0
一、 沉浸式 UI/UX 与视觉表现
摒弃传统 App 冰冷刺眼的白色输入框和硕大的“登录/注册”实心按钮，全流程需遵循“无界水域”的流体力学美学。
- 环境背景： 随系统 Light/Dark 模式自动切换的动态流体网格渐变（Mesh Gradient）。新用户首次打开时，水面极其平静。
- 交互介质（毛玻璃面板）： 屏幕中央悬浮一块高透、边缘带有细腻高光的毛玻璃面板（Glassmorphism）。
- 呼吸感输入框： 输入框仅保留极细的底边线或采用更深一层的内嵌毛玻璃质感。当光标聚焦（Focus）时，输入框周围会泛起微弱的同色系光晕（Glow），伴随极轻微的触觉马达震动。
- 无感切换动效： 登录、注册、找回密码的形态切换，面板不进行生硬的页面跳转，而是通过极其丝滑的面板高度拉伸、元素淡入淡出（Fade-in/out）来实现。
二、 跨端身份缔结链路 (Auth Workflows)
底层的身份验证与 Token 分发由 Supabase Auth 全权接管，确保多端状态同步与极速响应。
1. Web 端 / App 原生端 (手机号主导)
- 账号注册：
  - 用户输入手机号码 + 动态短信验证码。
  - 必须强制设置登录密码（作为双通道凭证）。
  - 点击“凝聚身份”（注册按钮）后，面板化作星光消散，瞬间下潜至首页【无界水域】。
- 常规登录：
  - 提供双通道切换：【手机号 + 验证码】 或 【手机号 + 密码】。
  - 沉浸式保持： 登录成功后，下发长效 JWT Token。在用户不主动点击“退出/切断连接”的情况下，会话有效期设定为 1 年，期间无论杀后台还是重启设备，均无感秒进应用。
- 找回密码（遗忘场景）：
  - 在登录态外，输入当前手机号获取验证码，校验通过后直接进入“重塑密码”界面，输入新密码即完成找回并自动登录。
2. 小程序 / 微信 H5 端 (微信生态主导)
- 快捷授权登录：
  - 利用微信 OpenID 机制，用户点击“微信快捷登录”。
  - 调用微信获取手机号组件（需用户确认授权）。
  - 静默注册闭环： 获取手机号后，系统后台自动校验。若该手机号未注册，则自动为其创建 Supabase 主账户并绑定微信体系；若已注册，则直接打通登录态，无缝进入。
3. 微信或手机号互转
如果用户起初通过微信注册，后来改为手机号在Web登录，则登录时，先进入密码设置页，强制其设置密码；如果用户先通过手机号，再用微信登录，系统发现该手机号已注册后，则直接进入。

三、 账号生命周期与资产继承 (Account Lifecycle)
用户在应用内积累了大量 AI 记忆、情绪标签和真实的财富（推手佣金），账号的流转必须绝对严谨。
1. 密码重塑 (修改密码)
- 前置条件： 用户处于已登录状态。
- 安全校验： 必须先输入【当前密码】。
- 双重确认： 连续输入 2 次【新密码】，前端需进行实时的一致性校验。
- 断开重连： 密码修改成功后，系统强制作废当前设备的 Token，面板坠落，退回登录首页要求重新缔结连接。
2. 物理锚点转移 (更换手机号)
现代人换手机号是刚需，决不能让用户因为换号丢失“数字灵魂”。
- 权限校验： 必须输入当前账户的【登录密码】以自证身份。
- 新锚点绑定： 输入新手机号，并填入新手机收到的短信验证码。
- 资产无损继承： 更换成功后，原账号下的所有【时光之珀】（记录）、【AI 心智标签】（记忆画像）、【流金之池】（分销佣金网络）百分之百归属于新手机号。
- 旧锚点释放： 原被解绑的旧手机号在数据库中物理剥离，化为“空白态”，随时可用于注册全新的独立账号。
- 所以手机号不应该作为用户标识，而应该以uuid为用户的唯一标识。
四、 铁血风控与防黑产网关 (Anti-Fraud Shield)
作为包含【流金之池】分销裂变与 AI 算力消耗的应用，验证码接口是黑客和羊毛党的首要攻击目标。必须在网关层（FastAPI 结合 Redis）部署绝对死守的风控策略。
1. 动态人机校验 (CAPTCHA)
- 触发时机： 每次点击“获取验证码”之前，必须前置触发。
- 交互形式： 采用高质量、开源的真人校验组件（如优雅的滑块拼接、空间逻辑点击）。UI 需深度定制，融入毛玻璃质感，拒绝廉价感。拦截掉 99% 的自动化机器脚本。
2. 阈值熔断机制 (Rate Limiting)
所有短信下发请求，必须经过以下三层漏斗校验。一旦触碰红线，立即熔断并返回极其克制的标准提示语：“您获取验证码过于频繁，请明日重试。”

首页 PRD：无界水域 (Boundless Waters) V1.0
一、 模块概述与核心视觉基调
定位： 应用的精神入口。摒弃一切传统导航，打造一片随机的缓慢流淌的“数字水面”。
核心表现： 12 个功能入口化作 12 颗大小不一、晶莹剔透的“毛玻璃长方形水滴”，在流体网格背景上随呼吸感上下浮动。屏幕正中央留白处透出底层 Slogan：“一念起，万物生.”。
二、 核心新增：启航仪式与“隐形引导” (Invisible Onboarding)
极简不是让用户去猜。我们要在初次打开应用时，用极其优雅的动效完成“无感教学”。
1. 启航破冰（首次打开应用）：
- 视觉： 屏幕初始为纯粹的平静水面（纯白或纯黑渐变）。中央落下一滴无形的水滴，伴随“叮——咚”的空灵回声。
- 涟漪绽放： 一圈巨大的涟漪荡开，12 颗毛玻璃水滴随之浮出水面，底层亮起 Slogan。全过程 1.5 秒。
2. 隐形交互教学（仅新用户首次触发）：
- 触碰教学（进入模块）： 画面静止 1 秒后，最上方的一颗水滴（如【回音之泉】）会自动产生极其轻微的“高频抖动”，并在水滴下方缓缓浮现一行透明度仅为 40% 的极细文字：“轻触水滴，潜入回音”。当用户第一次成功点击并触发放大下潜动效后，该提示文字永久消失。
- 滑动教学（探索水域）： 如果在 App 端，水面边缘会持续有极微弱的波纹向屏幕内涌动。并在屏幕最右侧边缘隐约闪烁：“划动水面，探索未知”。用户完成一次横向滑动后，提示永久隐藏。
三、 涟漪拓扑：12 颗水滴的物理集群分布
12 颗水滴按照功能属性，在全景水面上形成 5 组引力群落（支持镜头漫游滑动）：
- 第一群落：涌动之泉
  - 包含： 回音之泉[签到抽卡]、觉醒之境[深度阅读]、万象之澜[前沿资讯]。
  - 形态： 呈不规则“品”字形，水波向外涌动最活跃，代表思想源泉。
- 第二群落：静水流深
  - 包含： 时光之珀[生活日记]、止水之渊[灵性冥想]、破晓之径[习惯管理]。
  - 形态： 散落于左下偏暗水域。质感厚重、雾化度高，如琥珀般静止，代表心境沉淀。
- 第三群落：晨露折射
  - 包含： 先知之殿[智能问答]、微光之洲[社区互助]。
  - 形态： 悬浮于右上方。体积稍大，内部折射的背景高光最为明亮耀眼，暗示“高维启示”与价值。
- 第四群落：交错波纹
  - 包含： 忘忧之谷[解压游戏]、洞见之迷[智益游戏]。
  - 形态： 右侧中央。边缘流体张力最强，看起来最 Q 弹。彼此靠近时涟漪会产生干涉网纹。
- 第五群落：渊底流金
  - 包含： 流金之池[分享收益]、归心之屿[个人中心]。
  - 形态： 稳固沉淀在屏幕最底端。透出暗金色质感，作为整个商业和身份的物理托底。
四、 跨端沉浸式交互设计 (Interaction Engine)
- 移动端（重力视差与下潜）： 调用陀螺仪，倾斜手机时，水滴高光随之偏移，产生 Z 轴景深。用户点击某水滴，伴随清脆融合音效，该水滴如涟漪般瞬间炸开，磨砂质感顺势吞噬全屏，极其丝滑地过渡到下一级页面。
- Web 端（Hover 扰流引擎）： 鼠标在水面上移动会划出尾迹波纹。光标靠近水滴时，水滴产生物理吸附感（微靠拢），内部折射光晕变亮，并播放轻微 ASMR 频率。

回音之泉 PRD：每日一签与内心共鸣 (Echo Springs) V1.0
一、 模块概述与核心规则 (Business Logic)
模块定位： 应用的高频留存抓手。为用户提供每日一次、极具仪式感的沉浸式体验，获取 AI 赋予的高质量情绪与认知价值。
1. 极致克制的输入/输出约束：
- 输入限制： 支持用户输入任意文本（倾诉、提问、碎碎念），字数限制：最少 1 个字，最多 200 字。
- AI 输出强制格式（由 Dify Prompt 严格控制）：
  - 一句箴言： 必须在 20 个字以内（作为卡片主视觉核心）。
  - 一段解读： 必须在 240 个字以内（提供深度情绪抚慰或认知降维打击）。
2. 宿命感防刷机制（绝对频次限制）：
- 每日一签： 每个账户（User ID）每天（00:00 刷新）仅有 1 次 呼唤回音的权限。
- 绝对禁止重试： 如果用户对生成的结果不满意，系统绝对不提供“重新生成”、“撤销”或“再抽一次”的按钮。这种“所见即宿命”的严苛设定，将极大地提升这张卡片在用户心中的珍贵感和神圣感。
二、 沉浸式全链路交互 (Audio-Visual Workflow)
整个抽签过程分为三个阶段，前端必须实现视觉与空间音频（Spatial Audio）的帧级同步配合。
阶段一：起笔（输入期）
- 视觉表现： 全屏依然是极其美观、缓慢流动的 Mesh Gradient 渐变网格。背景中央隐约浮现极具呼吸感的提示文案：“落笔随心，召唤未知的回音。”
- 听觉表现： 用户进入页面并呼出键盘开始输入时，自动淡入播放一段舒缓、带有包裹感的沉浸式背景音乐。
阶段二：冥想（等待期 - AI 算力调度阶段）
- 视觉表现： 用户点击提交。输入框平滑消散，背景继续流动，文字提示平滑渐变切换为：“静心冥想，等待属于你的答案。”
- 听觉表现： 音乐不中断，但旋律平滑过渡（Crossfade）到具有悬念感、空灵的等待音效，掩盖后端 Dify 调用大模型 API 的 2-5 秒延迟。
阶段三：回音（结果呈现期）
- 听觉表现： AI 生成完毕的瞬间，必须伴随一声清脆、直击人心的“滴水击磬”提示音，随后背景音乐转为开阔、治愈的旋律。
- 视觉表现： 从水面深处（Z 轴推入）缓缓浮现出最终的 9:16 实体卡片。卡片材质与背景完美融合，达到可以直接发朋友圈的顶级杂志排版水准。
三、 跨端布局与 9:16 黄金比例 (Responsive Layout)
由于卡片承载着核心的“分享裂变”使命，无论在哪一端，最终生成结果的视觉比例必须锁死在 9:16。

五、 全局战场画布：动态领地推演背景 (The Dynamic Canvas)
整个战场没有任何可见的 UI 边界，屏幕本身就是一汪深潭。这汪深潭的底层颜色，是双方战局最直观的“隐形计分板”。
- 双极色彩引擎 (Bipolar Mesh Gradient)：
  - 亮色模式（空明之战）： 双方主色调分别为“破晓橘（光）”与“晨曦青（影）”。底层是纯白与极浅灰的交融。
  - 暗色模式（深渊之战）： 双方主色调分别为“极昼白（光）”与“深海墨（影）”。底层是纯黑与极深紫的交融。
- 领地光学推移 (Territory Shift Animation)：
  - 背景不是静止的渐变，而是受后端 territory_ratio 数据实时驱动的流体。
  - 开局时，双方色彩在屏幕中线（Y轴中心）形成极其缓慢的波浪交界。
  - 当“破晓橘”一方占据了 70% 的矩阵面积时，背景底层的破晓橘色光芒会像潮水一样，向上/下压迫“晨曦青”的深色水域。这种大面积的底层色彩吞噬，能在不出现任何数字的情况下，给劣势方带来极其强烈的心理压迫感。
六、 核心元素材质与形态设定 (Materials & Forms)
在这个 20x12（共计 240 个点位）的矩阵中，没有任何方形格子，只有光影和水滴。
1. 基础矩阵点位 (The Dormant Nodes)：
- 非激活态： 屏幕上均匀分布着 240 个极其微弱的、带有 10% 透明度的白色/暗色微光小圆点（类似深海里极小的浮游生物）。
- 呼吸律动： 这些圆点会随着背景的网格渐变产生极轻微的高斯模糊（Blur）和缩放（Scale: 0.95 ~ 1.05）的全局呼吸效。
2. 玩家落子：琉璃水滴 (The Glass Drops)：
- 材质表现： 当玩家落子后，该点位瞬间膨胀为一颗高透的“毛玻璃水滴”。水滴边缘必须带有高光描边（Inner Glow），中心透明度极高，能够折射出底层背景的颜色。
- 团战异色核 (Micro-Signatures)： 在 3v3 中，为了区分同队玩家（比如都是橘色阵营），每颗水滴的正中心封装着一个极其微小的“异色发光粒子”（如：玩家A是橘水滴+白星芒内核，玩家B是橘水滴+金粉内核）。
3. 流体力学牵引带 (Fluid Connections)：
- 触发条件： 当同色水滴在 8 个方向上相邻时生成。
- 视觉张力 (Metaballs Effect)： 这绝不能是一条死板的直线！相连的两个水滴，其边缘必须产生一种类似水银融合、或者细胞分裂时的“粘滞感”（视觉上称为 Metaball 融合特效）。光带中间细、两头粗，内部有极其快速的光影流动，表现出能量的传输。
4. 绝对结界：领地冻结 (The Absolute Territory)：
- 闭环瞬间： 当几个点位连成闭环的瞬间，环线内部的所有空位水滴会被瞬间点亮。
- 材质变异： 整个多边形闭环区域在 0.3 秒内，从液态瞬间“凝固”成一块巨大的、整面的高透磨砂玻璃（Frosted Glass Panel）。
- 视觉阻断： 这块巨大的玻璃板会对其覆盖的底层背景产生强烈的折射和高斯模糊。同时，结界的边界线会固化为一条不再流动的、极其明亮的纯色光带，宣告主权神圣不可侵犯。
七、 核心交互引擎：防误触与物理反馈 (Interaction & Haptics)
在手机上盲点 240 个密集点位是反人类的。我们通过“滑行磁吸”彻底重塑了 RTS 的微操体验。
1. 悬浮磁吸落子 (Hold, Drag & Snap)：
- 起手（按下）： 玩家手指按住屏幕的任意安全区域（不需要精准按在某个点上），指尖上方（Y轴向上偏移约 30 像素，防止手指遮挡视线）会瞬间悬浮起一颗代表玩家颜色的“虚拟高亮水滴”。
- 滑行（拖拽）： 手指在屏幕上滑动，虚拟水滴跟随移动。当水滴靠近矩阵上的某个合法空位时，会产生极强的视觉磁吸（Magnetic Snap）——虚拟水滴会被强制吸附到该网格点上，且该网格点会泛起一圈高频涟漪。
- 落锤（松开）： 玩家松开手指，伴随一声清脆的“滴水”音效，虚拟水滴瞬间砸入网格，化为实体琉璃水滴。能量池同时扣除 1 点。
2. 攻击性交互：强拆与替换 (Attack Gestures)：
- 强拆（耗 2 点）： 对准敌方未闭环的水滴，连续快速点击 2 次（双击）。
  - 动效： 敌方水滴瞬间布满玻璃裂纹，随后“砰”地一声炸裂成无数微小的玻璃渣，向四周溅射并淡出消失。伴随清脆的碎玻璃音效和短促重震。
- 替换（耗 3 点）： 对准敌方水滴，连续快速点击 3 次（三连击）。
  - 动效： 第一、二下出现裂纹，第三下点击的瞬间，敌方水滴不仅炸裂，并在炸裂的核心瞬间涌出你自己的颜色，直接完成材质的“夺舍”。伴随极其沉闷的低频重击音效和最高级别的马达长震。
八、 流体能量经济大盘 (Energy Economy Dashboard)
彻底抛弃顶部的倒计时和进度条，将经济系统无缝融入环境。
- 布局与材质： 屏幕最底部，沿极轻微的弧度排列着 12 个空心的“玻璃胶囊”（能量槽）。它们半沉在水底，透明度极低，绝不抢戏。
- 流体充能动效 (Fluid Fill)：
  - 随着每 2 秒 1 点的恢复设定，空槽底部会缓缓涌出代表玩家颜色的发光流体。
  - 流体向上攀升的过程带有轻微的水波纹物理摇晃效果。
  - 当一滴能量彻底充满时，整个胶囊会闪烁一次高光，并发出极微弱的“叮”声，提示弹药就绪。
- 耗能反馈： 落子瞬间，充满的胶囊不是瞬间变空，而是像水气球被扎破一样，内部的流体瞬间向下抽干，视觉上极具“消耗感”。
九、 终局视觉与结算海报 (End Game & Poster)
- 色彩海啸 (Color Tsunami)： 游戏结束瞬间，所有未闭环的散落点位全部消散。获胜方的颜色从其占领的最大结界中心爆发，像海啸一样在 1 秒内席卷全屏，将敌方的底色彻底吞噬。

二、 商业逻辑与合规底线 (The Engine & Compliance)
这是后端的绝对生命线，架构师必须在数据库层面焊死以下规则，绝不能有一丝越权漏洞。
1. 阳光单级分销（锁客与反哺）：
- 绝对单级： 为规避国内多级分销（传销）的法律红线，系统仅支持一级分润。
- 终身/保护期绑定： 用户 A 分享了绝美海报，新用户 B 扫码注册登录的瞬间，后端在 User 表中将 B 的 inviter_id 强绑定为 A。
- 实时分账： 此后，B 在【先知之殿】产生任何一笔付费（如 28.8 元解锁心智星图），支付回调成功的 0.1 秒内，后端按照配置的分润比例（如 30%），实时将佣金写入 A 的【流金之池】待结算账户。
2. 铁血税务与提现合规 (Tax & Payout)：
- 依法纳税： 平台绝不走私账。用户发起的每一笔提现，后端必须接入国家个税阶梯算法，按“劳务报酬”自动代扣 20%（或对应阶梯）的个税。
- 通道打通： 前端展示税后净额，后端对接微信/支付宝的“企业付款到零钱/银行卡”接口，实现秒级合规打款。
三、 暗黑鎏金美学与前端降级 (Dark Gold Aesthetics & Rendering)
前端页面分为三大核心展示区域，全局摒弃传统列表，采用悬浮在深空背景上的多维毛玻璃控制台。
1. 全局视觉基调：
- 强制深色模式： 背景是极度深邃的深空黑（Space Black）与暗金色（Dark Gold）缓慢交融的动态 Mesh Gradient。
- 材质规范： 所有数据面板采用最高透的黑色/暗金色毛玻璃（Glassmorphism），边缘带有极细的高光描边。
2. 帧率保卫战 (S/A/B 级降级策略)：
- S 级（旗舰机/PC Web）： 开启星轨图谱的 WebGL 实时粒子渲染，背景动态网格全开。
- A/B 级（中低端机/小程序）： 星轨图谱降级为基于 Canvas 2D 或 SVG 的连线动画；背景降级为静态暗金渐变色 CSS 铺底，确保资产页面的绝对稳定和丝滑滑动。

3. 核心交互与生命周期
- 分页加载 (Pagination)： 考虑到性能，列表采用无限滚动（Infinite Scroll）。单次 API 请求 limit=20。滑动至底部触发加载动效。
- 下钻详情 (Tap to View)： 点击任意卡片，触发视图放大过渡（Hero Animation），展示该内容的完整细节。详情页 UI 原样复刻该内容在其所属原模块的 UI。
- 取消收藏 (Unfavorite)：
  - 操作途径： 列表页支持卡片左滑呼出“抹除”按钮；详情页底部支持点击已高亮的“收藏”图标取消。
  - 反馈逻辑： 无二次确认弹窗。执行后，该卡片在列表中以“化作光尘”的动效淡出卸载（DOM 移除），调用后端 DELETE 接口解除关联。
- 快捷分享 (Quick Share)： 列表卡片右下角常驻“分享”图标。点击直接拉起底层海报生成引擎，生成 9:16 专属海报。
- 空状态兜底 (Empty State)： 当某分类下数据为 0 时，居中展示极简占位图（如空心的玻璃气泡），配引导文案（如：“星海暂无波澜，前往获取共鸣”），点击文案整页路由跳转至对应业务模块。

五、 面板四：合规与安全边界 (底层契约区)
业务目标： 满足应用市场上架与网信办/工信部的硬性合规要求。
- 互联网契约 (Legal Protocols)：
  - 列表： 包含《用户协议》、《隐私政策》、《充值与付费服务协议》。
  - 交互： 点击后不跳转新页面，在当前页面自下而上弹出全屏高透毛玻璃阅读层（Webview 加载富文本）。
- 抹除数字痕迹 (注销账号 - Account Deletion)：
  - 触发： 极简暗红色文本。
  - 阻断流程（极高危）： 点击 -> 弹出风险提示窗 -> 强制要求获取并输入当前手机号的短信验证码 -> 最终确认。
  - 后端执行： 一旦验证通过，系统立将该账号置为“预注销状态”（强制下线）。7天反悔期后，底层定时任务物理粉碎/匿名化该 User ID 关联的所有 UGC、收藏与记忆标签数据。
- 切断连接 (退出登录 - Logout)：
  - 触发： 悬浮在屏幕最底部的光环按钮。
  - 交互与清理： 点击后，前端清除本地所有的 Token 和敏感持久化缓存。页面所有面板向 Z 轴深处缩放坠落消失，极度丝滑地路由过渡回【注册/登录】欢迎页。
