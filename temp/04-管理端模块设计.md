# 04 · 管理端模块设计（web-admin）

> ⚠️ **2025-11 变更**：§4.6 AI 工作台 整节作废（题目/KP 由运营线下生成后导入）；§4.7 审核 仅保留学员举报子模块；考试中心管理改为主题→阶段→章→节层级，按 exam_scope 统计。
> 详见 [function/02-course/ai/CHANGELOG-2025-11.md](../function/02-course/ai/CHANGELOG-2025-11.md)。本文档历史段落若与变更冲突，以 CHANGELOG 为准。

> 这是给"内容运营 + 审核员 + AI 操作员 + 老板"用的后台。
> 设计目标：**让一个 6 人小团队能把 ~11 200 个 KP / ~5 万道题在 3–4 个月内出完并发布**（数量对齐 00/01）。

---

## 4.1 角色与权限

| 角色 | 看 | 能做 |
|---|---|---|
| `super` | 全部 | 全部，含发布、改 prompt、改用户、改账号 |
| `content_admin` | 自己 `tracks_scope` 内 | 课程目录 CRUD、组卷、发布预览（不能终发布） |
| `reviewer` | 自己 `tracks_scope` 内的草稿/待审 | 审核 KP / Question（通过/驳回/改） |
| `ai_operator` | AI 工作台 | 跑 AI Job、看用量、调 prompt（草稿态） |
| `readonly` | 全部数据 | 只看不动（数据/财务） |

行级权限通过 `tracks_scope` 数组控制：列表页 SQL 自带 `WHERE track_code = ANY($scope)`。

## 4.3 模块 1：仪表盘

一屏看清楚"还差多少没出 / 还差多少没审 / 昨天 AI 烧了多少钱"。

| 卡片 | 指标 |
|---|---|
| 今日新增 KP / Q | 草稿 / 待审 / 通过 |
| 内容缺口 | 按主题分阶段，未达标节数 |
| AI 用量 | 昨日 / 本月，token & USD |
| 审核 SLA | 平均待审天数、超 3 天数量 |
| 学员活跃 | DAU / WAU / MAU、付费转化（来自 user_*） |
| 题目质量 | 错题率最高 Top20、争议题（多人反馈） |

---

## 4.4 模块 2：课程内容（树形结构）

左侧是"主题 → 阶段 → 章 → 节"四级树，右侧是节详情。

### 4.4.1 树视图操作
- 拖拽排序（更新 `sort_order`）
- 右键：新增子节点 / 复制 / 移动到 / 改发布状态
- 节点上显示徽章：`草稿 12` / `待审 3` / `已发布`

### 4.4.2 节详情页（最常用页面，一屏要塞下）

```
─────────────────────────────────────────────────────────────
[节信息]  ec-2-3-1  优惠券与折扣        [编辑] [预览] [发布]
学习目标 | 预计时长 12 分钟 | 状态：草稿
─────────────────────────────────────────────────────────────
[KP 列表]  ▢全选            [+ 添加现有 KP] [+ AI 批量生成]
┌─排序─类型──标题──拼音──难度──题数──状态──操作─────────┐
│ 1   word  价格  jià gé   2     5     ✅审核通过 [编辑][换][删]│
│ 2   word  优惠券 yōu...   2     4     🟡待审   [审核]         │
│ 3   gram  把字句         3     6     📝草稿   [生成题]        │
│ ...                                                          │
└──────────────────────────────────────────────────────────────┘

[节末小测]  blueprint：5 题（3 单选 + 1 听音 + 1 排序）
            [生成 / 重生] [试做]
─────────────────────────────────────────────────────────────
```

### 4.4.3 KP 编辑抽屉（点 KP 任意行打开）

- 顶部：KP code / 类型 / 版本 / 来自 AI Job 链接
- 中部 Tab：
  - **基础**：标题/拼音/难度/标签
  - **内容**：按 KP 类型动态表单（word/grammar/dialog 各异）
  - **翻译**：4 种语言并排，支持"AI 重译此字段"
  - **媒资**：音频试听 + "重新 TTS"按钮、图片上传/换 AI 生图
  - **题目**：本 KP 关联的所有 Question 列表（4–6 条）
- 右上角：通过 / 驳回 / 申请重生（带备注）

## 4.5 模块 3：题库（横向视角）

> 课程内容是"按章节看"，题库是"按题看"——审题专用。

筛选器：主题 / 题型 / 难度 / 状态 / 关联 KP 类型 / 来源 AI Job。

每行：q_code / 题干预览 / 答案 / KP / 错题率 / 状态 / 操作（审、改、归档、AI 重出）。

**审题双开布局**：左半屏列表，右半屏题目预览（与用户端 1:1 渲染）+ 解析编辑框。

批量操作：批量通过 / 批量驳回 / 批量打标签 / 批量重出。

---

## 4.6 模块 4：AI 工作台（系统的引擎）

这是和别家后台**最不一样**的地方。三个子页：

### 4.6.1 Prompt 管理
- 列表：scope（kp_word / q_mcq_meaning / tts / image …）/ 版本 / 是否启用
- 编辑器：模板 + 占位符提示 + "用样例输入跑一次"按钮
- 灰度切换：v3 启用，v2 保留只读，v1 归档

### 4.6.2 生成任务（Jobs）
- 新建 Job 表单：
  - **类型**：批量生成 KP / 批量生成题 / 配音 TTS / 配图
  - **范围**：选主题 → 阶段 → 章 → 节（多选）
  - **数量**：自动算缺口 or 手填
  - **prompt 版本** + **模型** + **温度/参数**
  - **预算上限**：USD（超出自动停）
  - **是否同时生成翻译/音频**
- 提交 → 写 `ai_generation_jobs` queued → 后台 worker 拉
- 任务详情页：进度条、生成预览（每 100 条采样 5 条）、失败列表、重试按钮、cost
- "**一键导入草稿池**"：成功的内容写到 `knowledge_points / questions` status=draft

### 4.6.3 用量与成本
- 按模型 / 按月 / 按 prompt 拆账
- 异常告警：单 Job 超预算、出错率 > 5%

## 4.7 模块 5：考试中心

| 子页 | 干嘛 |
|---|---|
| 节末小测模板 | 默认按 lesson 自动生成 blueprint，可手改 |
| 章测 / 阶段考 | 每章 / 每阶段一份，可拖题、可改难度分布 |
| HSK 模考（HSK 主题专属） | 按官方等级题型比例配置：听 35 / 阅 30 / 写 5 等 |
| 试做 | 管理员以"匿名学员"身份完整答完，看反馈 |
| 学员答卷 | 抽看具体某次 attempt 的快照，用于争议判定 |

---

## 4.8 模块 6：媒资库

- 音频：按声线分桶（女声标准 / 男声标准 / 慢速版），支持试听、批量重生
- 图片：场景图分类（电商场景、工厂车间、街景、菜市场…），AI 生图也走这里
- 去重：上传时按 `hash` 命中已有就直接复用
- 关联：每个媒资能反查"被哪些 KP / Q 引用"，删除前必拦截

| 子页 | 干嘛 |

## 4.11 关键工作面板：「内容缺口看板」

```
│ 1     1        ec-1-1-1   10        10     10    10       0   ✅

```

## 4.12 发布流程（每个内容必经）

```
draft → reviewing → approved → (chapter/stage published=true) → CDN 同步
                ↘ rejected → 回到 draft（带审核备注）
                ↘ regen   → 触发 AI Job 重出，旧版本归档
```

发布是"**节级**"开关：
- `lessons.is_published = true` 才会出现在用户端学习地图。
- 发布前自动校验：节内 KP 全部 `approved` ？每个 KP 至少 N 道题？翻译 4 语言齐 ？音频齐 ？
