# 05 · 用户端模块设计（web-app / 移动端）

> 学员侧。**纯静默学习**：选、连、点、拖、键盘输入。无麦克风、无手写。
> UI 默认越/泰/印/英 4 语言，中文是学习目标。

---

## 5.1 信息架构（5 个一级 Tab）

```
[首页] [学习] [复习] [考试] [我的]
```

| Tab | 解决什么 |
|---|---|
| 首页 | 一眼看到"今天学什么"，3 秒内能开始 |
| 学习 | 完整课程地图，按赛道/阶段/章/节浏览 |
| 复习 | SRS 到期队列 + 错题本 |
| 考试 | 节末小测/章测/阶段考/HSK 模考入口 |
| 我的 | 进度统计、订阅、设置、语言切换 |

---

## 5.2 首次进入流程（onboarding）

```
启动 → 选 UI 语言（vi/th/id/en）
     → 注册/登录（手机号或邮箱 + OTP）
     → 选目标主题（电商/工厂/HSK/日常，可多选但只主修一个）
     → 自评水平（零基础 / 认得拼音 / HSK1 等）
     → 听写小测（5 题，决定起始 stage）
     → 进入主页，第一节高亮"开始学"
```

零基础默认进入**共享 Stage 0**；自评有基础的可跳过。

## 5.4 学习地图（学习 Tab）

横向 PC、纵向移动端，类似多邻国"路径树"。

- 顶部赛道切换（已订阅赛道才能切）
- 阶段卡片（Stage）→ 展开看章 → 展开看节
- 节状态：🔒 锁定 / ⚪ 未开始 / 🟡 进行中 / 🟢 已通过（百分比）
- 解锁规则：`unlock_rule`（默认前一节通过且共享 Stage 0 完成）

---

## 5.5 节内学习页（核心页面，最复杂）

整个节是一个"卡片流"，类似 Tinder 滑动 / 多邻国答题序列。

### 5.5.1 顺序

```
进度条 ▮▮▯▯▯▯▯▯▯▯ 2/12

┌─ KP 讲解卡 ─────────────────┐
│  价格  jià gé               │
│  🔊 [自动播放 + 重复按钮]   │
│  giá cả                     │
│  例句：这个价格太贵了。     │
│  [ 我懂了 ➜ ]               │
└────────────────────────────┘

→ 题 1（mcq_meaning）→ 反馈 → 下一题
→ 题 2（listen_pick）→ 反馈 → 下一题
→ 下一个 KP 讲解卡...
→ ...
→ 节末小测 5 题
→ 完成结算页：得分、连击、+10 经验、错题加入复习
```

### 5.5.2 答题统一交互
- 选择题：点选项即提交（不要二次确认）
- 连线：拖线 / 点点配对（移动端最佳）
- 排序：拖词块到下方答题区，"提交"按钮变亮才可点
- 输入：原生键盘 + 系统拼音输入法（不内嵌 IME）
- 反馈：
  - ✅ 绿色 + 正确音 + 解析（默认折叠，点开看）
  - ❌ 红色 + 提示音 + 显示正解 + 解析展开 + "再做一次"按钮（错题入 SRS box1）

### 5.5.3 顶部右上角操作
- 退出（弹窗确认，已答题进度自动保存到 `user_progress.last_kp_id`）
- 设置：自动播放、显示拼音、字体大小

### 5.5.4 离线模式
- 进入节时**整节预下载**（KP 内容 + 题 + 音频）
- 答题先写本地，回网后批量同步 `user_answers`

---

## 5.6 复习页（SRS）

```
今日到期 18    本周累计 92    错题本 23
────────────────────────────────────────
[开始今日复习]   每次 10 题为一组

策略：
  1. 优先盒 1（昨日错题）
  2. 然后到期日 ≤ 今天，按 box 升序
  3. 同一 KP 抽题轮换不同题型，避免死记答案
```

复习页**不是教学页**——直接进题，做完反馈，调整 SRS 盒位。

---

## 5.7 考试页

| 入口 | 触发 |
|---|---|
| 节末小测 | 节内学完 12 个 KP 后自动跳转 |
| 章测 | 章内全部节通过后解锁 |
| 阶段考 | 阶段全部章测通过后解锁，**未过不能进下阶段** |
| HSK 模考 | HSK 赛道独立菜单，可随时刷 |

考试体验区别于练习：
- **不立刻反馈**（交卷后统一看）
- **倒计时**（章测 15 min / 阶段考 45 min / HSK 模考按官方时长）
- **不可中途退出**（退出 = 计零分，给二次机会的话写在 attempt 里）
- 交卷页：得分、是否通过、错题列表（点进去看解析、加入复习本）

## 5.8 我的 Tab

- 学习画像：累计学习天数、KP 量、词汇量曲线
- 成就徽章（连续打卡、章节通关、考试满分）
- 订阅与账单
- 语言设置 / 母语设置（影响题目翻译）
- 帮助与反馈（举报错题入口）

---

## 5.9 学员"举报错题"流程

每道题反馈页有"🚩 这题有问题"按钮：
- 类型：答案错 / 翻译错 / 音频错 / 图错 / 其他
- 备注 + 截图（可选）
- 提交后写 `content_review_log` + 关联 question_id
- 管理端 4.9 反馈中心收到 → 二审 → 修正后版本号 +1，老答案不影响历史成绩

---

## 5.10 跨设备 / 跨语言切换

- 一个账号多端登录，进度以服务端为准（`user_progress / user_srs`）
- 切换 UI 语言：立即生效（i18n 字典前端缓存）
- 切换母语：影响**未来题目的翻译**展示，不重置历史

## 5.11 性能与体验底线

| 指标 | 目标 |
|---|---|
| 进入"继续学习"到第一张卡可交互 | < 1.5 s（地区 P95） |
| 节内切换下一张卡 | < 200 ms（本地预加载） |
| 整节音频预下载 | 12 KP × ~30 KB ≈ 0.4 MB / 节 |
| 离线可用 | 已下载节、已下载复习包 |
| 弱网重试 | 答题先写本地队列，回网批量 sync |

---

## 5.12 本地化注意（东南亚）

- 字体：思源黑体 + Noto Sans Vietnamese / Thai / 印尼 Latin（拒绝系统字体一致性差异）
- 数字 / 货币：默认显示越南盾 / 泰铢 / 印尼盾的本地化金额
- 网络：印尼/泰国部分地区 4G 不稳，**必做离线包**
- 安卓占比 80%+，移动端优先安卓适配

---

## 5.13 与管理端的"内容契约"

用户端**永远不查草稿**。统一通过两个聚合接口拿数据：

```
GET /app/lesson/{code}     → 节完整包（KP + 题 + 媒资 URL）
GET /app/srs/today         → 今日 SRS 队列（包含题快照）
```

服务端在生成响应时强制 `WHERE status='approved' AND is_published=true`。
