## A · 业务范围类（最先拍）

### A1. 首发市场顺序？
- **默认**：3 国（vi / th / id）UI 同时上，但**首发主推越南**（汉字接受度最高、跨境电商需求最强）。
- 备选：① 只先做越南，单语言上线；② 同时印尼+越南，先放弃泰国（泰语字体最特殊）。
- 影响：i18n 工作量、TTS 声线选型、本地支付接入。

### A2. 4 主题是否同步开发？
- **默认**：先「日常 + HSK」MVP 上线（共享 Stage 0 复用度最高），3 个月后再上电商和工厂。
- 备选：① 先「电商」单主题（B 端付费意愿高）；② 4 主题并行（成本×4，工期失控风险）。
- 影响：AI 出稿排期、目录骨架优先级。

### A3. HSK 主题 1–6 全做还是只做 1–4？
- **默认**：一期只做 HSK 1–4（覆盖 90% 学员），HSK 5–6 二期。
- 备选：① 全做 1–6；② 只做 1–3 + 模考，5/6 引第三方题库。
- 影响：题库量、教研难度。

## B · 课程颗粒度类

### B1. 阶段数 6 个够不够？
- **默认**：6 个阶段（对齐 HSK 1–6），每主题 ~2 500 KP（精算 6×6×6×12 ≈ 2 592；与 00/01 数字保持一致）。
- 备选：① 3 阶段（粗）；② 9 阶段（对齐 HSK 3.0 新版）。
- 影响：节数和总 KP 量直接 ±50%。

### B2. 节末小测是否强制通过才能进下节？
- **默认**：**不强制**，过 60% 算通过，未过可重做但允许跳；阶段考强制通过。
- 备选：① 节末强制（更严但学员易流失）；② 全无门槛（学习率高但完课率虚假）。
- 影响：解锁逻辑和数据指标定义。

### B3. 共享 Stage 0 是否在 4 主题首页都展示？
- **默认**：首登选主题后**强制先上完 Stage 0** 才能学主题。
- 备选：① Stage 0 设为可跳过；② 让学员先进主题，遇到拼音再回炉。
- 影响：onboarding 流程、解锁规则。

## C · 题型与教学法类

### C1. 12 类题型是否全做？
- **默认**：MVP 上 8 类（Q1/Q2/Q3/Q6/Q7/Q8/Q11/Q12），二期补 Q4/Q5/Q9/Q10。
- 备选：① 一上就 12 类；② 只上 5 类（最简）。
- 影响：前端组件数量、AI prompt 数。

### C2. 是否需要"汉字笔顺动画 + 跟写"？
- **默认**：**只看动画不跟写**（手写识别要算力，违反"零个性化算力"原则）。
- 备选：① 接入开源笔顺校验（hanzi-writer JS 库，纯前端 SVG 比对）；② 完全不做笔顺。
- 影响：KP 媒资字段、前端组件、是否要图形识别。

### C3. 是否使用系统拼音输入法（IME）？
- **默认**：Q10 题让用户用**手机系统自带拼音输入法**打字，不内嵌 IME。
- 备选：① 内嵌一个简化 IME（前端体积+200KB，但可控）；② 完全不出 IME 题。
- 影响：用户端依赖、东南亚学员是否要先教他们装中文输入法。

### C4. SRS 算法选哪个？
- **默认**：**Leitner 5 盒**（实现简单、可解释）。
- 备选：① SM-2（动态间隔，更精准但参数难调）；② FSRS（最新，效果最好但实现复杂）。
- 影响：`user_srs` 字段、调度逻辑。

## D · AI 与内容生产类

### D1. 主用哪个 LLM？
- **默认**：内容用 GPT-5 / Claude Sonnet，TTS 用 Azure TTS（普通话女声 Xiaoxiao + 男声 Yunyang），生图用 GPT-Image / Imagen。
- 备选：① 全 OpenAI 一站式；② 用阿里通义（中文质量好+合规）。
- 影响：成本、合规（数据出境）、声线一致性。

### D2. 翻译是 AI 一次出 4 语言还是分别走母语审核？
- **默认**：AI 一次出 4 语言 → 各语言**各自找一名母语审核员**点检。
- 备选：① 只 AI 不审（成本最低，质量风险）；② 全部母语人工出（成本高 5×）。
- 影响：审核人员配置、上线节奏。

### D3. AI 生图 vs 真人摄影 vs 图库？
- **默认**：场景图（电商/工厂/街景）AI 生图 + 关键场景人工挑图库。
- 备选：① 全 AI 生图（便宜但同质化）；② 全图库（贵+版权风险）；③ 只用 emoji + 简笔画（最省）。
- 影响：媒资库工作量、视觉一致性。

### D4. 是否给每个 KP 配两个声线（标准 + 慢速）？
- **默认**：**只标准声**，慢速通过前端 `playbackRate=0.75` 实现。
- 备选：单独录慢速版（音频量×2）。
- 影响：媒资量、TTS 成本。

## E · 技术栈类（你说接口不要，但栈还是得定）

### E1. 后端框架？
- **默认**：NestJS（与你 `system/apps/api-*` 已有结构一致）+ PostgreSQL + Redis（SRS 队列缓存）。
- 备选：① Go + Gin；② Python FastAPI（AI 调用最顺手）。
- 影响：后端团队招聘、复用现有代码。

### E2. 前端？
- **默认**：用户端 web 用 Next.js + 移动端 React Native（共享组件）；管理端 Next.js + Ant Design Pro。
- 备选：① 移动端用 Flutter；② 移动端先做 PWA 不出 App。
- 影响：人力配置和上线时间。

### E3. CDN / 文件存储？
- **默认**：Cloudflare R2 / AWS S3 + CloudFront；东南亚就近节点。
- 备选：① 阿里云 OSS（国内）；② Bunny CDN（性价比）。
- 影响：合规、成本。

## F · 商业与合规类

### F1. 订阅模型？
- **默认**：免费试学每主题前 1 章；月度订阅 ~$5 / 全主题 $9。
- 备选：① 一次性买断；② 完全免费 + 广告；③ B2B 卖给工厂/电商公司。
- 影响：账户体系、`subscription` 字段、支付接入。

### F2. 数据合规？
- **默认**：用户数据存在新加坡区，不出境到中国大陆。
- 备选：① 主区放越南/印尼本地；② 走中国云。
- 影响：合规审计、网络架构。

### F3. 内容审查？

- 备选：再加一层人工敏感内容专审。
- 影响：审核成本和上线速度。
