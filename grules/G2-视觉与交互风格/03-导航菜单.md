### 1.1 头像下拉（右上角，登录后）

```
我的资料        → /me
我的钱包        → /me/wallet
订单与会员      → /me/orders
邀请好友        → /me/referral
设置（语言/主题）→ /settings
─────────────
退出登录
```

### 1.2 未登录态

- 一级菜单**全部可见**（保证品牌呈现与发现路径完整）；但仅「发现中国」可部分浏览，其他菜单项点击后**直接跳转** `/auth/login?redirect=...`（不弹提示，减少打断）。
- 「发现中国」未登录可浏览**前 3 个主题卡片**（按 PRD 02 主题顺序取前三）：
  - 首页 / 发现页仅渲染前 3 个主题卡片 + 主题详情页；
  - 第 4 个及以后的卡片显示但点击 / 滚动到后跳登录；
  - 主题详情内部的「快闪卡 / 游戏跳转 / 评论 / 点赞」等交互一律请跳登录。
- 右上角头像位置替换为「登录 / 注册」两个文字按钮（`text-button`，红色文字）。
- **路由守卫实现**：`_auth` 路由组未登录访问 → 重定向 `/auth/login?redirect=...`；「发现中国」路由 loader 检测未登录时在请求参数里拼 `?guest=1&limit=3`，后端仅返回前 3 个主题。

## 二、管理端（`web-admin`，路由前缀 `/admin`）

| 菜单名称 | 路由 | 图标 | 可见角色 | 父级 | 备注 |
|---------|------|------|---------|------|------|
| 用户管理 | `/admin/users` | `users` | `admin` | — | 对齐 PRD 12 |
| 发现中国 | `/admin/discover` | `compass` | `admin` | — | 内容审核 / 上架 |
| 系统课程 | `/admin/courses` | `graduation-cap` | `admin` | — | 课程管理 / 上下架 |
| 小说专区 | `/admin/novels` | `book-open` | `admin` | — | 小说审核 / 章节管理 |

> **当前管理端不暴露的模块**：游戏配置、经济、邀请、客服、安全、内容工厂、i18n 校对、支付配置等后台功能。
> PM 明确本期**不需要**这些菜单（Q1 已关闭）。待后续迫切诉求出现时再补，届时另起 G2 变更单。

### 2.1 管理端头像下拉

```
账号信息    → /admin/me
切换到应用端 → /
─────────────
退出登录
```

### 2.2 角色守卫

- 路由 `/admin/*` 由 `_admin` 路由组守卫：
  - 未登录 → 重定向 `/auth/login?redirect=/admin/...`；
  - 已登录但非 `admin` 角色 → 重定向 `/403`（毛玻璃 403 页）。
- 顶栏菜单渲染时，对未匹配角色的菜单项**不渲染**（不灰显），减少诱导。

## 三、菜单项数据源

- 单一真相源：`system/packages/shared-config/src/menus.ts`，导出 `appMenu` / `adminMenu`，结构：
  ```ts
  type MenuItem = {
    key: string;
    i18nKey: string;          // 'nav.discover'
    path: string;
    icon: LucideIconName;
    roles?: ('guest'|'user'|'admin')[];  // 缺省=登录用户
    children?: MenuItem[];
  };
  ```
- 名称必须走 i18n（`packages/shared-i18n/<locale>/nav.json`），禁止硬编码中文。
- 新增菜单：先改 `menus.ts` + 5 语言文案 + 路由 + 本文件。
