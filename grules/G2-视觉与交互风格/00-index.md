## 核心原则（速记）

1. **风格基调**：大气、克制、专业，参照 **Stitch / Figma** 的现代质感；内容区参照 **Notion** 的简洁。
2. **配色双模**：仅 **黑白 + 红**。亮模式（默认）= 白底 + 红强调；暗模式 = 黑底 + 红强调。品牌红采用**小红书红 `#FF2442`**（亮）/ `#FF3B5C`（暗）；**不引入第二种品牌色**，但提供「主题色自定义」（详见 [06 §七](./06-响应式与暗黑模式.md)）。
3. **全局毛玻璃**：导航、卡片、弹窗、按钮、Tab、Toast 全部采用半透明毛玻璃面板（`backdrop-filter: blur`）。详见 [01-设计Token.md §四](./01-设计Token.md)。
4. **顶部导航全局统一**：应用端与管理端**均使用顶部导航**，**不使用侧边栏**。
5. **全屏布局**：Web 端内容区跟随视口宽度自适应，**不固定中心列宽**；手机 / 平板 / 宽屏自动响应（详见 [06-响应式.md](./06-响应式与暗黑模式.md)）。
6. **组件库基线**：以 **shadcn/ui** 规范为准；项目级毛玻璃主题在 `system/packages/ui-kit/` 中通过 token + variant 覆盖。
7. **主题切换无闪烁**：首屏在 `index.html` 内联读取 `localStorage` + `prefers-color-scheme` 后立即设置 `<html data-theme>` 与 `--brand`，避免 FOUC。默认主题 = `light`（PM 决策）。
8. **可访问性**：所有毛玻璃面板必须叠加足够对比度的文字层；WCAG AA 起步。
9. **角色权限**：管理端菜单仅对 `admin` 角色可见；应用端菜单对所有登录用户可见（未登录限"发现中国 / 小说专区"等只读浏览，详见 03 §三）。
10. **单文件 ≤ 1200 行**：超出立刻拆分。
