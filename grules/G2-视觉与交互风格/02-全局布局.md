# 02 · 全局布局

> 引用：[00-index.md](./00-index.md) · [01-设计Token.md](./01-设计Token.md)
> 适用：应用端（`web-app`）+ 管理端（`web-admin`）所有页面。

---

## 一、布局总览

```
┌──────────── 顶部导航栏 GlassNav (h: 64 desktop / 56 mobile, sticky top:0) ─────────────┐
│  Logo + 产品名 ┊ 一级菜单（横向）        ┊ 语言切换 │ 主题切换 │ 通知 │ 头像下拉      │
└──────────────────────────────────────────────────────────────────────────────────────┘
┌───────────── 远景层（页面底色 + 红色径向光斑 + 微噪）—— 全局固定 ─────────────────────┐
│                                                                                       │
│   ┌─────────── 内容区 .page (max-width: none, 全屏自适应宽度) ─────────────┐          │
│   │  page padding: 32 桌面 / 16 移动；顶部留白 24px                       │          │
│   │  内部任意组合：GlassCard / 表格 / 表单 / Tabs / 列表                  │          │
│   └──────────────────────────────────────────────────────────────────────┘          │
│                                                                                       │
└──────────────────────────────────────────────────────────────────────────────────────┘
                                  无 Footer（除营销 / 落地页外）
```

- **无侧边栏**：应用端 / 管理端均为顶部导航（用户决策）。
- **全屏宽度**：内容区**不固定中心列宽**；只在阅读型长文（小说、课程详文）内部用 `max-w-prose` 限制段落宽度，外层容器仍占满视口。
- **Sticky 顶栏**：始终 `position: sticky; top: 0; z-index: 50`，毛玻璃透出下方滚动内容。

## 二、顶部导航 GlassNav

| 属性 | 桌面 | 移动 |
|------|------|------|
| 高度 | 64px | 56px |
| 内边距 | 左右 32px | 左右 16px |
| 背景 | `glass-nav`（`--glass-blur-md`，无圆角，底部 1px `--border-subtle`）| 同左 |
| z-index | 50 | 50 |
| 阴影 | 滚动 > 4px 时追加 `shadow-md`（监听 `window.scrollY`，加 class `is-scrolled`）| 同左 |
| 安全区 | `padding-top: env(safe-area-inset-top)` | 同左 |

### 2.1 桌面布局（≥ md）

```
[Logo + 产品名]   [菜单项 横向间距 24px]                [Locale] [Theme] [Bell] [Avatar▾]
```

- 一级菜单选中态：底部 2px 红色短线（`--brand`，长度 = 文字宽度）+ 文字 `--brand`；
- hover：文字变 `--brand`，无下划线；
- 菜单项间距 24px，字号 `text-button`。

### 2.2 移动布局（< md）

```
[Logo]                              [Theme] [Avatar▾] [Hamburger ☰]
```

- 一级菜单收纳到右侧 Drawer（自右向左滑入，`glass-modal` 风格）；
- Drawer 内菜单项纵向排列，行高 48px，选中态左侧 2px 红色竖线 + 浅底；
- Drawer 顶部含语言切换 + 通知入口；
- Drawer 关闭：点遮罩 / ESC / 点 X。

### 2.3 Logo

- 文字版：`知语 · Zhiyu`，`text-h2`，前缀色块 24×24（`--brand` 圆角 6px，内含白色"语"字简化图形）。
- 点击 Logo 总是返回各端首页（应用端 `/`，管理端 `/admin`）。
- v1 暂用文字 + 色块；最终图形 Logo 待定（见 99 Q4）。

## 三、远景层（毛玻璃的关键背景）

固定在 `<body>` 之下、所有内容之上的零交互层：

```css
body::before {
  content: "";
  position: fixed; inset: 0; z-index: -1;
  background:
    radial-gradient(60% 50% at 15% 10%, rgba(255,36,66,0.10), transparent 60%),
    radial-gradient(45% 40% at 85% 90%, rgba(255,36,66,0.08), transparent 65%),
    var(--bg-app-tint);
}
body::after {
  /* 1px 微噪 PNG 平铺，opacity = var(--glass-noise-opacity) */
}
```

- 暗模式下 `--bg-app-tint` 使用 `#111114`，红色径向光斑提升到 `0.14 / 0.10` 透明度。
- 远景层禁止承载业务内容；仅做"光感"。

## 四、内容区 `.page`

| 属性 | 桌面（≥ md）| 移动（< md）|
|------|------------|------------|
| 宽度 | `width: 100%`，`max-width: none` | 同 |
| 左右内边距 | 32px | 16px |
| 顶部内边距 | 24px | 16px |
| 容器栅格 | 12 列，`gap: 24px`（推荐 CSS Grid）| 1 列 |
| 阅读型长文容器 | 内部 `<article class="prose max-w-[72ch] mx-auto">` | 同左，但移动端不限宽 |

### 4.1 通用页面骨架

```tsx
<Page>
  <PageHeader title="..." subtitle="..." actions={<Button>新建</Button>} />
  <PageContent>
    <GlassCard>...</GlassCard>
  </PageContent>
</Page>
```

- `PageHeader`：`text-h1` 标题 + `text-caption` 副标题 + 右侧操作槽；下方 `space-5` 间距。
- `PageContent`：仅承载业务，**禁止**直接写 `bg-white` / 实心背景。
- 所有"卡片型"展示一律走 `<GlassCard>` 组件（`.glass-card`）。

## 五、空状态与加载

### 5.1 空状态

| 元素 | 规格 |
|------|------|
| 容器 | `<GlassCard class="text-center py-12">` |
| 插图 | 96×96 SVG（线条风格，单色 `--text-tertiary`，与红色辅助高光）|
| 标题 | `text-h3 text-primary` |
| 描述 | `text-body text-secondary` |
| 操作 | 主按钮（如适用），居中 |

### 5.2 加载

- 默认：骨架屏（`@/components/Skeleton`），毛玻璃风格，行 / 卡片轮廓与最终内容一致；
- 一次请求 < 200ms 时不展示骨架，避免闪烁；
- 全屏路由切换：顶部 2px 红色进度条（fork TanStack Router progress）。

---

## 六、不规范用法（禁止）

1. **侧边栏**：v1 不允许任何侧边栏布局（除非 PM 单点放行）。
2. **固定中心列宽**：`max-w-7xl mx-auto` 等中心容器禁止用于普通页面（仅长文内部段落允许）。
3. **不透明卡片**：禁止 `bg-white` / `bg-zinc-900` 直接当卡片底；必须使用毛玻璃组件。
4. **第二种品牌色**：禁止使用蓝、紫、绿等作为强调色。
5. **底部 Footer 全局存在**：首页/落地页可有营销 Footer，业务页面（应用端登录后 / 管理端）一律无 Footer。
