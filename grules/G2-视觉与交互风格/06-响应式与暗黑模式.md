## 一、断点矩阵

| 名称 | min-width | 设备 | 内容区内边距 | 顶栏高度 | 一级菜单 |
|------|-----------|------|------------|----------|---------|
| `xs` | 0 | 小屏手机 | 16px | 56px | 收纳到右侧 Drawer |
| `sm` | 640px | 大屏手机 / 小平板 | 16px | 56px | 收纳到右侧 Drawer |
| `md` | 768px | 平板 | 24px | 64px | 横向显示（如不挤压再考虑收纳）|
| `lg` | 1024px | 笔记本 | 32px | 64px | 横向显示 |
| `xl` | 1280px | 桌面 | 32px | 64px | 横向显示 |
| `2xl` | 1536px | 大屏 | 48px | 64px | 横向显示 |

- **设计基线**：移动端 360×780；桌面端 1440×900。
- **应用端**：mobile-first，所有页面 `xs` 必须完整可用。
- **管理端**：desktop-first，`< md` 仅保证只读查看，关键编辑提示"建议使用更大屏幕"。
- **全屏宽度**：所有页面在 `2xl` 下仍占满视口宽度，**不在中间居中**（用户硬性要求）。

## 二、移动端适配细则

1. 触控热区 ≥ 44×44 CSS 像素，按钮默认 `lg` 尺寸（高 36 → 移动端高 44）。
2. 关键操作按钮固定底部安全区：`padding-bottom: env(safe-area-inset-bottom)`。
3. 输入框聚焦时禁止页面缩放：`<meta name="viewport" content="width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no">`。
4. 长列表必须虚拟滚动（TanStack Virtual）。
5. 横竖屏切换：游戏页（PixiJS）需要监听 `resize` + `devicePixelRatio` 适配；横竖屏切换提示见游戏模块 PRD。

## 三、暗黑模式

### 3.1 切换逻辑

- 优先级：用户显式选择（写入 Supabase user metadata + `localStorage.theme`）→ `prefers-color-scheme` → **默认 `light`**（PM 决策）。
- 顶栏右侧「主题切换」图标按钮（`sun` / `moon`），点击循环 `light ↔ dark`，自动持久化。
- `index.html` 内联脚本（< 1KB）首屏执行：

  ```html
  <script>
    (function () {
      var t = localStorage.getItem('theme');
      if (!t) t = matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
      document.documentElement.setAttribute('data-theme', t);
      var c = localStorage.getItem('brand-color');
      if (c) document.documentElement.style.setProperty('--brand', c);
    })();
  </script>
  ```

  避免 FOUC（首屏白闪 / 黑闪）。`brand-color` 同时在此处提前写入，避免主题色闪变。

### 3.2 设计准则

- 亮模式（默认）：白底 + 黑字 + 红强调；远景层带极低饱和的红色径向光斑。
- 暗模式：纯黑底（`#0A0A0B`，避免纯 `#000` 与红色对比过冲）+ 白字 + 红强调；同样有红色径向光斑（透明度略高）。
- 红色（`--brand`）亮模式使用小红书红 `#FF2442`；暗模式略提亮为 `#FF3B5C` 以保证对比度。
- 所有插画 / 图标必须提供"亮 / 暗"两套或使用 `currentColor`。
- 截图 / 图片缩略图在暗模式下叠加 `1px solid var(--border-subtle)` 防止边缘融化。

### 3.3 测试

- Storybook 双主题快照；CI 检查关键页面在 `light/dark` 下的对比度（`axe-core` 自动跑）。
- 不允许任何 `bg-white` / `text-black` 等硬编码颜色 class。

## 四、毛玻璃在低端设备的降级

> **策略（PM 决策）**：偏向**宽松**——默认保持毛玻璃启用，仅在浏览器明确不支持或运行时实际揉帧时才降级，**不再仅凭 `deviceMemory` / `hardwareConcurrency` 静态阈值推断**。

| 条件 | 行为 |
|------|------|
| `CSS.supports('backdrop-filter: blur(1px)')` 为 `false` | 使用「伪毛玻璃」：`--bg-surface-strong`（高不透明度）+ 噪点 + `shadow-md`；视觉接近，但无模糊 |
| 运行时帧率采样：滚动连续 1s 内平均 FPS < 40 | 进入「动态降级」：仅保留顶栏与当前能见区卡片的毛玻璃，不可见区卡片 `backdrop-filter: none`；连续 5s 低于阈值则表层进入「伪毛玻璃」|
| `navigator.deviceMemory <= 1` **且** `hardwareConcurrency <= 2`（极低端）| 启动时直接「伪毛玻璃」；设置内可手动开回 |
| 用户在「设置 → 显示」主动选择「省电模式」| 全局「伪毛玻璃」|
| 滚动中（`is-scrolling` class） | 顶栏 `backdrop-filter` **保持开启**；列表卡片可短暂禁用模糊以稳定 60fps（仅滚动 200ms 内）|

## 五、字体响应式

- 桌面 base = 14px，移动 base = 14px；标题在 `< md` 时降一级（`text-h1` 28→24，`text-h2` 22→20）。
- 中文教学卡片字号优先按"卡片宽度 / 7 字"自适应（CSS `clamp()`），保证一行不溢出。

---

## 六、可访问性总结

- 颜色对比度 ≥ WCAG AA（正文 4.5:1 / 大字 3:1）；红底白字、白底红字均达标。
- 键盘焦点 ring 必须在毛玻璃面板上仍清晰可见（`--ring` 红色半透明 2px 解决）。
- 减弱动效尊重 `prefers-reduced-motion: reduce`：所有 `--motion-*` 退化为 `0ms`，毛玻璃保留（视觉非动效）。

## 七、主题色自定义（皮肤系统，PM 决策）

> 本期**仅提供「主题色自定义」**，节日皮肤（圣诞 / 春节等）本期不实现。

### 7.1 能力范围

- 用户可在「设置 → 显示 → 主题色」选择：
  - 预设调色板（默认）：小红书红 `#FF2442` / Reddit 橙红 `#FF4500` / 珊瑚 `#FF6B6B` / 薰衣草 `#A855F7` / 深海蓝 `#3B82F6` / 森林绿 `#22C55E` …6 色；
  - 自定义 HEX：弹出调色盘（`react-colorful`），仅限 6 位 HEX，不提供 alpha；
  - 选定后预览顶栏 / 主按钮 / Tag 实时变色。
- 仅覆盖下列 token，其他黑白中性色均不变：
  - `--brand` = 用户选色；
  - `--brand-hover` = 选色亮度 × 0.92（HSL `L × 0.92`）；
  - `--brand-active` = 选色亮度 × 0.84；
  - `--brand-soft` = 选色 alpha 0.10 / 暗模式 0.14；
  - `--brand-soft-strong` = 选色 alpha 0.18 / 0.22；
  - `--ring` = 选色 alpha 0.45 / 0.55；
  - `--brand-on` = 依据选色及背景反反推（`relative-luminance > 0.5` → 黑字，否则白字）。

### 7.2 存储与同步

- 本地：`localStorage.brand-color`（仅 HEX 7 字节，防注入）；
- 帐号：同步到 Supabase user metadata `display.brand_color`，多端一致；
- FOUC：首屏内联脚本（见 §3.1）同时读取并设置。

### 7.3 约束

- 主题色变化后必须重新计算 `--brand-on` 以保证 WCAG AA；不达标时在调色盘提示「对比度偏低」但不阻止保存。
- 状态色（`--success / --warning`）**不跟随**主题色变化；`--danger` 始终复用 `--brand`（保持语义一致）。
- 在 Storybook 内提供「主题色预览」装饰器，打开后全面板随选色动态重染。
