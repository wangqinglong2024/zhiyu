# 04 · 数据结构

> 引用：[00-index.md](./00-index.md) · 与 G1 [03-数据库规范.md](../G1-架构与技术规范/03-数据库规范.md) 保持一致。
> 所有表位于 `public` schema（除特殊注明 `auth.*`）；所有 `id` 默认 UUID v7（按 G1 §2.3）；所有时间戳 `timestamptz`；列命名 `snake_case`。

---

## 一、`auth.users`（Supabase 内置，仅约定字段）

> 该表由 Supabase Auth 维护，**不允许**业务侧直接 INSERT / UPDATE / DELETE，统一通过 supabase-js Auth API 操作。

| 字段 | 类型 | 业务约定 |
|------|------|---------|
| `id` | uuid | 主键；其他业务表 `user_id` 全部引用此列 |
| `email` | text | 唯一；Google 登录时来自 Google 账号 |
| `encrypted_password` | text | bcrypt；Google 账号此列为 `null` |
| `email_confirmed_at` | timestamptz | Google 注册自动填；邮箱注册需点验证邮件后才填 |
| `raw_app_meta_data` | jsonb | **业务约定**：`{ "role": "user" \| "super_admin", "provider": "email" \| "google" }` |
| `raw_user_meta_data` | jsonb | 用户可改的元数据（display.brand_color 等，由 G2 皮肤系统使用）|
| `last_sign_in_at` | timestamptz | Supabase 自动维护 |

> 角色读取统一从 `raw_app_meta_data->>'role'`；JWT 中映射为 `app_metadata.role`。

## 二、`profiles`（业务用户档案）

```sql
create table public.profiles (
  id              uuid primary key references auth.users(id) on delete cascade,
  email           text not null,
  display_name    text,
  avatar_url      text,
  locale          text not null default 'zh',           -- zh / en / vi / th / id
  is_disabled     boolean not null default false,
  disabled_reason text,
  disabled_at     timestamptz,
  disabled_by     uuid references auth.users(id),
  created_at      timestamptz not null default now(),
  updated_at      timestamptz not null default now()
);

create index profiles_email_idx on public.profiles (email);
create index profiles_is_disabled_idx on public.profiles (is_disabled) where is_disabled = true;
```

### 2.1 自动创建 Trigger

```sql
-- auth.users 新增 → 自动 insert profiles
create function public.handle_new_user()
returns trigger language plpgsql security definer as $$
begin
  insert into public.profiles (id, email, display_name, avatar_url, locale)
  values (
    new.id,
    new.email,
    coalesce(new.raw_user_meta_data->>'name', split_part(new.email, '@', 1)),
    new.raw_user_meta_data->>'avatar_url',
    coalesce(new.raw_user_meta_data->>'locale', 'zh')
  );
  return new;
end $$;

create trigger on_auth_user_created
after insert on auth.users
for each row execute function public.handle_new_user();
```

### 2.2 RLS

```sql
alter table public.profiles enable row level security;

create policy profiles_self_read on public.profiles
  for select using (auth.uid() = id);

create policy profiles_self_update on public.profiles
  for update using (auth.uid() = id)
  with check (auth.uid() = id and is_disabled = (select is_disabled from public.profiles where id = auth.uid()));
  -- 防止用户自改 is_disabled

-- service_role（Hono 后端）绕 RLS，无需额外 policy
```

## 三、`user_sessions`（多设备会话追踪）

```sql
create table public.user_sessions (
  id                  uuid primary key default gen_random_uuid(),
  user_id             uuid not null references auth.users(id) on delete cascade,
  refresh_token_hash  text not null,             -- sha256(refresh_token) 16进制；不存明文
  device              text not null,              -- 'web' / 'mobile-web'
  user_agent          text,
  ip                  inet,
  created_at          timestamptz not null default now(),
  last_seen_at        timestamptz not null default now()
);

create index user_sessions_user_idx on public.user_sessions (user_id, created_at);
create unique index user_sessions_refresh_idx on public.user_sessions (refresh_token_hash);
```

### 3.1 业务规则

- 登录写 + 检查 count > 3 → 撤销最早的 1 条（详见 [02-认证流程.md §四](./02-认证流程.md)）。
- 登出删除自己那条；过期清理由 cron job 每日清 `last_seen_at < now() - interval '40 days'` 的死行。
- RLS：用户只能读自己的会话；`service_role` 可全权操作。

```sql
alter table public.user_sessions enable row level security;
create policy sessions_self_read on public.user_sessions for select using (auth.uid() = user_id);
create policy sessions_self_delete on public.user_sessions for delete using (auth.uid() = user_id);
```

## 四、`auth_login_attempts`（登录失败计数）

```sql
create table public.auth_login_attempts (
  id           bigserial primary key,
  email        text not null,
  ip           inet not null,
  succeeded    boolean not null,
  attempted_at timestamptz not null default now()
);

create index login_attempts_email_time_idx on public.auth_login_attempts (email, attempted_at desc);
create index login_attempts_ip_time_idx on public.auth_login_attempts (ip, attempted_at desc);
```

- 计数：当前 15 分钟内 `email + ip` 失败次数 ≥ 5 → 锁定到该窗口结束。
- cron job 每日清 `attempted_at < now() - interval '7 days'` 的旧行。
- RLS：仅 `service_role` 可访问。

## 五、`audit_logs`（管理端关键操作审计）

```sql
create table public.audit_logs (
  id          bigserial primary key,
  actor_id    uuid references auth.users(id),
  actor_role  text not null,                       -- super_admin / system
  action      text not null,                       -- 'user.disable' / 'user.enable' / 'user.delete' / ...
  target_type text,                                -- 'user' / 'course' / ...
  target_id   text,
  payload     jsonb,                               -- 变更前后 diff 或参数
  ip          inet,
  user_agent  text,
  created_at  timestamptz not null default now()
);

create index audit_logs_actor_idx on public.audit_logs (actor_id, created_at desc);
create index audit_logs_target_idx on public.audit_logs (target_type, target_id, created_at desc);
```

> 与 G3 直接相关的最小事件集：`auth.signin_success` / `auth.signin_failed` / `auth.signout` / `auth.session_kicked` / `user.disable` / `user.enable` / `user.password_reset` / `super_admin.self_delete_blocked`。
