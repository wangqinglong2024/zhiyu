## 一、整体原则

1. **全栈 TypeScript strict**：前端、后端、共享包、脚本、Edge Functions 一律 `.ts`。**禁止 `.js` / `.mjs` / JSDoc 替代方案**（用户硬性要求：本期代码必须人工可维护，强类型不打折）。
2. **Docker-only**：所有依赖、构建、测试、运行均在容器内完成，本地不直装 Node/Postgres/Supabase。
3. **Supabase first 且本地化**：DB / Auth / Storage / Realtime / Edge Functions 全走**自托管 Supabase 开源栈**（复用主机已部署的 `supabase_default` 网络），**不使用 supabase.com 云服务**。**pgvector 由该镜像内置但默认不启用**，仅在 AI 向量检索场景按需 `create extension vector`，详见第六节。
4. **数据访问统一走 supabase-js + Supabase MCP**：禁止引入 Drizzle / Prisma / Kysely 等 ORM；schema 与迁移由 Supabase CLI 管理。
5. **前端纯 CSR / SPA**：HTML 仅空壳，业务数据全部异步拉取，**禁止 SSR / SSG / 数据预注入**（反爬虫硬性要求）。
6. **缺 Key 不阻塞**：第三方服务统一通过 Adapter 接口注入，缺 Key 自动 fallback 到 mock 实现。
7. **AI 工作流二分**：复杂状态机/多智能体 → LangGraph（TS，接受 beta）；单次 prompt / 流式 → Vercel AI SDK。

> - 入口 `index.html` 不允许通过 `<script>window.__INITIAL_STATE__=...` 等方式注入业务数据；

> - 详细约束见 [05-编码规范.md §二·渲染与反爬](./05-编码规范.md)。

## 四、Edge Functions（鉴权 / 回调 / 轻量编排）

> Hono 容器与 Supabase Edge Functions **互补，职责清晰不重叠**。

| 用途 | 落地位置 | 说明 |
|------|---------|------|
| 用户鉴权（注册/登录/OAuth/找回密码） | **Edge Functions** | 与 Supabase Auth 同进程，调用最短；Google 登录、邮箱注册 mock 也放此处 |
| 第三方回调（Paddle webhook、Google OAuth callback） | **Edge Functions** | 边缘转发后落 BullMQ；缺 Key 走 mock 验签 |
| 轻量、无状态、低 QPS 接口（健康、版本、配置）| Edge Functions | — |
| 业务 API（课程、游戏、小说、积分、IM 等核心 CRUD）| **Hono 容器** | 需要队列、长连接、复杂事务 |
| AI 工作流编排（LangGraph） | **Hono 容器** + BullMQ worker | Edge 不适合长任务 |

> Edge Functions 与 Hono 通过 `packages/shared-schemas` 共享 Zod 类型与错误码。

| 用途 | 技术 | 备注 |

## 六、AI 工作流

| 用途 | 选型 | 备注 |
|------|------|------|
| 复杂工作流编排 | **LangGraph (TS, beta)** | 用户接受 beta；封装在 `packages/ai-adapters/workflow/` |
| 简单 LLM 调用 / 流式 | **Vercel AI SDK** | 纯 prompt → 单次/流式响应 |
| 向量数据库 | Supabase pgvector（**仅 AI 向量场景按需启用**） | 用户指定；非 AI 功能严禁使用向量列 |
| Adapter 层 | `LLMAdapter` / `WorkflowAdapter` / `EmbeddingAdapter` / `TTSAdapter` / `ASRAdapter` | 缺 Key 自动 fallback 到 fixture |
| 主创作模型 | Anthropic Claude Sonnet（待 Key）| `env.md` 缺 → mock |
| 副创作模型 | DeepSeek V3（待 Key）| 缺 → mock |
| 评估 | LangSmith（可选）| v1 不强制 |

> **禁用**：Dify、NocoBase、OpenClaw（参考 `env.md`，已标注停用）。

## 九、版本锁定与升级策略

1. 全部依赖在 `package.json` 中**精确版本**（不写 `^` / `~`），由 `pnpm-lock.yaml` 锁定。
2. 主版本升级须先在 PR 描述中列出 breaking change，并经过 PM 单点放行。
3. 每月一次安全补丁批量升级，CI 跑全量测试通过后合入。
4. LangGraph 处于 beta，每次升级前先在 `packages/ai-adapters/workflow/__bench__/` 跑回归用例。
