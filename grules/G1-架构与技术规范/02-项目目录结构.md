# 02 · 项目目录结构

> 引用：[00-index.md](./00-index.md)
> 适用：知语 monorepo 全仓库
> 包管理：**pnpm workspaces**（用户指定）
> 语言：**全栈 TypeScript**
>
> **代码根目录硬约束**：所有可运行/可构建的开发代码（apps、packages、supabase、docker、scripts、配置）**全部位于 `/opt/projects/zhiyu/system/`** 之下。`/opt/projects/zhiyu/` 顶层只放文档与素材（`grules/`、`prds/`、`content/`、`prompt/`、`template/`、`backup/`、`env.md`）。AI 在生成任何代码、命令、Docker 配置、CI 路径时，工作目录与相对路径**必须以 `system/` 为根**。

---

## 一、顶层布局

```
/opt/projects/zhiyu/
├── grules/                     # G 系列规则文档（本目录）
├── prds/                       # 产品需求文档
├── content/                    # 静态业务内容（中国文化、课程素材、小说大纲）
├── prompt/                     # 用户原始提示词
├── template/                   # 文档模板
├── backup/                     # 历史/备份文档
├── env.md                      # API Key 与环境变量说明
│
└── system/                     # ★ 唯一开发根目录（pnpm workspace 根）
    ├── apps/                   # 可独立部署的应用
    │   ├── web-app/            # 应用端前端（C 端用户，端口 3100，CSR/SPA）
    │   ├── web-admin/          # 管理后台前端（端口 4100，CSR/SPA）
    │   ├── api-app/            # 应用端后端（Hono，端口 8100）
    │   └── api-admin/          # 管理后台后端（Hono，端口 9100）
    │
    ├── supabase/               # 自托管 Supabase 工程（CLI 管理）
    │   ├── config.toml         # 本地 Supabase 配置
    │   ├── migrations/         # SQL 迁移（supabase migration new ...）
    │   ├── seed.sql            # 初始化数据
    │   ├── tests/rls/          # pgTAP / 自定义 RLS 测试
    │   └── functions/          # Edge Functions（Deno + TS）
    │       ├── _shared/        # 由脚本从 packages/shared-schemas 同步
    │       ├── auth-signup/    # 邮箱注册（mock）
    │       ├── auth-google/    # Google OAuth 回调（mock）
    │       ├── webhook-paddle/ # Paddle 支付回调（mock 验签）
    │       └── health/         # 健康检查
    │
    ├── packages/               # 跨应用共享代码（全部 TS）
    │   ├── shared-schemas/     # Zod schema → z.infer 类型，前后端共用
    │   ├── shared-config/      # 常量、错误码、枚举
    │   ├── shared-utils/       # 纯函数工具（日期、字符串、汉字处理）
    │   ├── shared-i18n/        # 5 语言文案（zh/en/vi/th/id）
    │   ├── ui-kit/             # 基于 shadcn/ui 的项目级组件 + 毛玻璃主题（详见 G2）
    │   ├── supabase-client/    # 封装 supabase-js（browser / server / admin）+ MCP 调用工具
    │   ├── ai-adapters/        # LLM / Workflow(LangGraph) / TTS / ASR 适配器 + mock fixture
    │   └── game-engine/        # PixiJS 游戏共享内核
    │
    ├── docker/                 # Docker 配置（唯一 dev 环境）
    │   ├── compose.yaml        # 唯一 compose 文件；复用主机 supabase_default 外部网络
    │   ├── Dockerfile.web      # 前端镜像（多阶段）
    │   ├── Dockerfile.api      # 后端 Hono 镜像
    │   ├── postgres/           # 仅占位说明：DB 复用主机 supabase-db，不再自建初始化
    │   ├── redis/              # redis.conf
    │   └── env/
    │       ├── .env.example    # 全部变量模板（必交付）
    │       └── README.md       # 各 API Key 来源、申请方式
    │
    ├── scripts/                # 一次性脚本（数据迁移/导入），全部 TS（tsx 执行）
    │
    ├── pnpm-workspace.yaml
    ├── package.json            # workspace 根
    ├── tsconfig.base.json      # 全栈共享 TS 配置（strict）
    ├── .editorconfig
    ├── .gitignore
    └── README.md
```

> **注意**：
> - **代码根 = `/opt/projects/zhiyu/system/`**。所有 `pnpm`、`docker compose`、`supabase` 等命令的 cwd 都是 `system/`。
> - 顶层 `/opt/projects/zhiyu/` 不放任何源码，只放文档/规则/素材，便于 Docker 构建上下文裁剪与 `.dockerignore` 过滤。
> - 本仓库**不存在 Drizzle、`packages/db`、`.mjs` 后缀**。所有数据库 schema 与迁移在 `system/supabase/migrations/` 中以 SQL 维护，由 Supabase CLI 执行。

## 二、`system/apps/web-app`（应用端前端，C 端，CSR/SPA）

```
system/apps/web-app/
├── src/
│   ├── main.tsx                # 唯一入口；不允许在此处注入业务数据
│   ├── app/                    # 应用根、provider、错误边界
│   ├── routes/                 # TanStack Router 文件路由
│   │   ├── _public/            # 未登录可见
│   │   ├── _auth/              # 登录后
│   │   └── games/              # 游戏专区
│   ├── features/               # 业务模块（按 PRD 编号对齐）
│   │   ├── 02-discover-china/
│   │   ├── 03-courses/
│   │   ├── 04-games/
│   │   ├── 05-novels/
│   │   ├── 06-user-account/
│   │   ├── 08-economy/
│   │   ├── 09-referral/
│   │   ├── 10-payment/
│   │   ├── 11-customer-service/
│   │   └── 15-i18n/
│   ├── components/             # 跨 feature 的展示组件
│   ├── hooks/
│   ├── lib/                    # supabase client、http、analytics
│   ├── styles/
│   └── assets/
├── public/
├── index.html                  # 仅空壳；禁止业务数据预注入（反爬虫）
```

> **feature 内部统一三层**：`api/`（请求 hooks）+ `ui/`（组件）+ `model/`（store/derive）。
> **反爬虫强制**：`index.html` 中禁止出现 `<script>window.__*` 形式的数据注入；所有列表/详情数据必须通过 API 在客户端运行时拉取。

## 六、`system/supabase/`（自托管 Supabase 工程）

```
system/supabase/
├── config.toml                 # 本地端口、auth provider、storage 桶等
├── migrations/                 # SQL 迁移（按时间戳命名）
│   ├── 20260428_120000_init_schema.sql
│   ├── 20260428_130000_add_users.sql
│   └── ...
├── seed.sql                    # 字典/初始管理员/演示数据
├── tests/rls/                  # RLS 正反例
└── functions/                  # Edge Functions（Deno 运行时，TS）
    ├── _shared/
    │   ├── cors.ts
    │   ├── errors.ts
    │   └── schemas.ts          # 与 packages/shared-schemas 同步（脚本同步，不软链）
    ├── auth-signup/index.ts
    ├── auth-google/index.ts
    ├── webhook-paddle/index.ts
    └── health/index.ts
```

> **schema 与迁移规则**：
> - 新建表/字段 → `pnpm supabase migration new <name>` → 写 SQL → `pnpm supabase db reset`（dev）→ 提交。
> - **禁止**手改已合入迁移；变更只能新增。
> - 复杂查询封装为 SQL 函数（RPC），由后端 `supabase.rpc('fn_name', args)` 调用。

## 九、命名规则汇总

| 对象 | 规则 | 示例 |
|------|------|------|
| 目录 | kebab-case | `web-admin/`、`game-engine/` |
| TS 普通文件 | kebab-case | `course-list.handler.ts` |
| React 组件文件 | PascalCase | `CourseCard.tsx` |
| Hook 文件 | camelCase 加 `use` 前缀 | `useCourseProgress.ts` |
| Zustand store | camelCase + `.store.ts` | `gameSession.store.ts` |
| 测试文件 | 与被测同名 + `.test.ts(x)` | `course-list.handler.test.ts` |
| Edge Function 目录 | kebab-case，与路径同名 | `supabase/functions/webhook-paddle/` |
| SQL 迁移 | `YYYYMMDD_HHMMSS_<verb>_<object>.sql` | `20260428_120000_add_user_referral_code.sql` |

## 十、禁止事项

1. **跨 app 直接 import**：必须通过 `packages/*`。
2. **绕过 `packages/supabase-client` 直接 `createClient`**：禁止；保证连接、超时、日志统一。
3. **绕过 adapter 直接 fetch 第三方 LLM**：必须经 `ai-adapters`。
4. **新增 ORM**（Drizzle/Prisma/Kysely 等）：禁止。schema 一律 SQL + Supabase CLI。
5. **使用 `.js` / `.mjs` 写源码**：禁止；脚本用 `tsx` 执行 `.ts`。
6. **新增顶层目录**：除非更新本规范，否则一律拒绝。
7. **前端 SSR / SSG / 业务数据预渲染**：禁止（反爬虫）。
