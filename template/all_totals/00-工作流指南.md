# 工作流指南：AI开发模板体系

## 体系总览

本体系将需求→开发的过程拆成最小单元，每一步AI只处理一小块内容，杜绝上下文遗忘。

所有模板都是**成对**出现的：你写一个，AI根据你写的生成一个。你审核通过后才进下一步。

---

## 全局规范（项目启动时做一次）

做完后形成3份技术文档，后续所有功能开发都引用它们。

```
步骤1：技术架构
你填写 → [G1-用户] 技术与架构偏好
发给AI → AI生成 [G1-AI] 架构与技术规范
你审核 → 确认后存档

步骤2：UI/UX设计
你填写 → [G2-用户] 视觉与交互风格
发给AI → AI生成 [G2-AI] UI设计系统
你审核 → 确认后存档

步骤3：角色权限
你填写 → [G3-用户] 角色与权限描述
发给AI → AI生成 [G3-AI] 权限与认证规范
你审核 → 确认后存档
```

每一步都是独立的对话，AI一次只处理一份小文档，不会遗忘。

---

## 功能开发（每个功能做一轮）

每个功能按"数据→接口→页面"的顺序，分三步完成。

```
步骤1：数据模型
你填写 → [F1-用户] 数据与业务规则
发给AI → 连同 [G1-AI] 一起发 → AI生成 [F1-AI] 数据模型规范
你审核 → 确认后存档

步骤2：接口设计
你填写 → [F2-用户] 操作与业务逻辑
发给AI → 连同 [G1-AI] + [F1-AI] 一起发 → AI生成 [F2-AI] 接口规范
你审核 → 确认后存档

步骤3：页面交互
你填写 → [F3-用户] 页面与交互描述
发给AI → 连同 [G2-AI] + [F1-AI] 一起发 → AI生成 [F3-AI] 页面交互规范
你审核 → 确认后存档
```

注意每一步AI只接收2-3份小文档，不会出现"一次塞7000字然后丢细节"的问题。

---

## 编码阶段（审核全部通过后）

把确认好的技术文档按以下顺序喂给编码AI，每一步只喂最小必要文档：

```
编码第1步：建数据库/类型定义
输入：[G1-AI 中的数据库规范部分] + [F1-AI]
产出：数据库迁移文件 + 类型定义文件
验收：逐字段对照 F1-AI

编码第2步：逐个开发接口
输入：[G1-AI 中的API规范部分] + [F1-AI 中相关的表] + [F2-AI 中的单个接口]
产出：该接口的完整代码
验收：对照 F2-AI 中该接口的定义 + 校验规则 + 错误场景
（每做完一个接口验收一个，再做下一个）

编码第3步：逐个开发页面
输入：[G2-AI] + [F2-AI 中相关接口] + [F3-AI 中的单个页面]
产出：该页面的完整代码
验收：对照 F3-AI 中该页面的交互行为表
（每做完一个页面验收一个，再做下一个）

编码第4步：联调验证
输入：[F3-AI 中的场景验证]
产出：完整流程测试结果
```

---

## 关键原则

1. **每一步只处理一小块**——不要贪快合并步骤
2. **每一步都审核**——在AI记忆最清楚的时候抓住偏差
3. **AI不确定就问你**——每份AI文档末尾都有"待确认问题清单"
4. **编码时喂最小文档**——只给当前步骤需要的内容，不给多余的

## 文件清单

| 文件名 | 谁写 | 内容 |
|--------|------|------|
| G1-用户-技术与架构偏好 | 你 | 产品简介、技术偏好、约束条件 |
| G1-AI-架构与技术规范 | AI | 技术栈、目录结构、命名规范、DB规范、API规范 |
| G2-用户-视觉与交互风格 | 你 | 风格描述、参照物、布局偏好、导航结构、交互偏好 |
| G2-AI-UI设计系统 | AI | 设计Token、布局规格、组件规范、交互模式 |
| G3-用户-角色与权限描述 | 你 | 角色定义、登录方式、权限规则 |
| G3-AI-权限与认证规范 | AI | 认证流程、Token策略、角色模型、权限校验方案 |
| F1-用户-数据与业务规则 | 你 | 数据对象、属性、关系、状态、计算规则 |
| F1-AI-数据模型规范 | AI | 表结构、字段定义、枚举、状态机、校验规则 |
| F2-用户-操作与业务逻辑 | 你 | 需要哪些操作、权限、边界情况 |
| F2-AI-接口规范 | AI | 接口定义、参数、响应、错误场景 |
| F3-用户-页面与交互描述 | 你 | 页面布局、元素、用户操作和系统响应 |
| F3-AI-页面交互规范 | AI | 页面规格、元素规格、交互行为技术方案、场景验证 |
