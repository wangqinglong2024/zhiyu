## 一、这个功能需要哪些操作

> 用大白话列出用户在这个功能里能做哪些事情。不用写接口路径，只说"做什么"。

```
[例如：]

1. 查看列表：能看到所有订单，支持搜索（按订单号、客户名）、筛选（按状态、日期范围）、分页、排序
2. 查看详情：点击某个订单看它的完整信息
3. 新建：手动录入一个新订单
4. 编辑：修改订单的客户信息、备注等（已发货的不能改）
5. 确认订单：把"待确认"变成"待发货"
6. 确认发货：填入快递单号，变成"已发货"
7. 确认收货：变成"已完成"
8. 取消订单：需要填取消原因
9. 重新打开：把已取消的订单恢复（仅老板能操作）
10. 导出：把当前列表导出为Excel
11. 添加备注：随时给订单添加备注
```

## 三、操作的特殊规则

> 某些操作有条件限制或特殊逻辑时写在这里。

```
[例如：]

编辑订单：
- 只有"待确认"和"待发货"状态的订单可以编辑
- "已发货"之后不能再编辑

确认发货：
- 必须填入快递单号才能确认
- 快递单号只允许字母和数字

取消订单：
- "已发货"的订单不能直接取消
- 取消时必须填写原因

导出：
- 导出的是当前筛选条件下的数据，不是全部数据
- 导出格式为Excel
```
